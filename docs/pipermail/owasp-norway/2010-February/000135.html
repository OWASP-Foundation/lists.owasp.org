Vil vel si at stateless pages ikke kan utsettes for CSRF. CSRF er ute etter
å tvinge en bruker som allerede har en state til å utføre en handling med
den staten. Til vanlig gjelder dette inloggings-state. Så - by nature - en
REST page er ikke utnyttbar for en CSRF.

Ref OWASP:
'CSRF is an attack which forces an end user to execute unwanted actions on a
web application in which he/she is currently authenticated.'

/Roy
