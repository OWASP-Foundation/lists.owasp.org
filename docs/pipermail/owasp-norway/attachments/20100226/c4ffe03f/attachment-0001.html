Vil vel si at stateless pages ikke kan utsettes for CSRF. CSRF er ute 
etter å tvinge en bruker som allerede har en state til å utføre en 
handling med den staten. Til vanlig gjelder dette inloggings-state. Så -
 by nature - en REST page er ikke utnyttbar for en CSRF.<br>
<br>Ref OWASP:<br><div style="margin-left: 40px;">&#39;CSRF is an attack 
which forces an end user to execute unwanted actions 
on a web application in which he/she is currently authenticated.&#39;<br><br></div>/Roy<br><br><div class="gmail_quote">2010/2/26 Knut Vidar Siem <span dir="ltr">&lt;<a href="mailto:knutsiem@gmail.com">knutsiem@gmail.com</a>&gt;</span><br>
<blockquote class="gmail_quote" style="margin: 0pt 0pt 0pt 0.8ex; border-left: 1px solid rgb(204, 204, 204); padding-left: 1ex;"><div>Hei</div><div><br></div><div>Hvordan beskytter man en tilstandsløs serverapplikasjon mot CSRF? Tilstandsløsheten gjør det vel vanskelig å benytte tokenløsningen beskrevet som tiltak nr 1 på A5 i OWASP Top 10 2010 og detaljert på [1].</div>

<div><br></div><div>Fra Roy Fieldings &quot;Architectural Styles and the Design of Network-based Software Architectures&quot;, kapittel 5, &quot;Representational State Transfer (REST)&quot; [2]: &quot;All REST interactions are stateless. That is, each request contains all of the information necessary for a connector to understand the request, independent of any requests that may have preceded it.&quot; Argumentene for tilstandsløsheten er mange og gode, men hvordan beskytter man seg mot CSRF?</div>
