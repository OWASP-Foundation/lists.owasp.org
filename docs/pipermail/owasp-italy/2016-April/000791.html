Rispondo a tutta la lista.

Quindi, se ho capito bene, tu hai un backend J2EE che hai protetto con
mod_security. Ti suggerisco di aggiungere anche il core ruleset di
Owasp (https://www.owasp.org/index.php/Category:OWASP_ModSecurity_Core_Rule_Set_Project).

Lato flex, visto che è eseguito lato client, la strada che puoi fare è
quella di fare una code review di quel codice.
Per le "chiamate anomale", in realtà il tuo script o un attaccante
posto in mezzo tra lo script ed il tuo BE, se ne occupa mod_security.

Cmq un penetration test applicativo, io per scrupolo lo farei anche
sulla parte esposta e protetta da mod_security.
