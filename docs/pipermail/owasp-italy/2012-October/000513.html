di solito noi in azienda usiamo ibm appscan source code edition oppure
checkmarx cx suite, due prodotti commerciali, però abbastanza costosi.

anche se di solito facciamo anche attività manuale che è più precisa e
puntuale sia lato revisione di codice che penetration test.

ti consiglio di leggere l'asvs di owasp (se vuoi la sto traducendo in
italiano) per fare un idea della copertura delle varie modalità di
test:
 - scansione sorgente automatica
 - scansione dinamica automatica
 - penetration test
 - analisi del codice manuele
 - analisi interna
 - ....

> Già che ci sono, approfitto ancora della vostra pazienza.
> Sapreste indicarmi qualche corso e/o certificazione adatta ad un ruolo come
> il mio?

come certificazioni di "alto livello" quindi non tecniche ci sono la
cissp, cisa, cism o la opsa (quest'ultima è comunque più tecnica)
anche se, al netto delle certificazioni, le competenze conviene
acquisirle sul campo con risorse su internet (a parte l'asvs c'è anche
la testing guide e la code review guide), eventi (linux day, owasp
day).

in generale se posso darti un consiglio - che do spesso ai clienti - è
implementare un ciclo sicuro di sviluppo del software - in cui
integrare le varie attività.

tendenzialmente internamente si procede con strumenti automatici per
analisi dinamica (tipo appscan standard, acunetix, ecc...) e statica
(ibm asspcan source code e ci si affida ad aziende esterne per
penetration test o analisi statica manuale (per il quale servono
competenze che di solito non sono interne alle aziende di sviluppo e
che potrebbero comunque creare problemi all'interno dei team).

secondo la quantità di applicazioni/attività può convenire l'acquisto
(che però hanno comunque una coverage limitata, vedi asvs) o affidarti
a security tester.
