<div dir="ltr">Ciao a tutti,<div>cercando in giro ho capito che il tema è sentito, ma sinceramente non ho trovato una risposta valida che sia una.</div><div>Parto dall'inizio.</div><div><br></div><div>Per lo più la mia azienda si occupa dello sviluppo di servizi web. Quindi web application Java che esponiamo in internet a cui i clienti accedono, si autenticano, lavorano, etc etc etc.</div>
<div>I nostri servizi hanno spesso bisogno di utilizzare altri servizi: alcuni interni altri esterni. Ma è normale, per accedere ad un servizio, che sia necessario utilizzare una password o una chiave privata (nel senso più generico del termine).</div>
<div>Il problema che stiamo cercando di risolvere è: dove è sicuro scrivere tali password/chiavi?</div><div><br></div><div>Alcune risposte sono state:</div><div><br></div><div>- in un file di configurazione, ovvero in chiaro sulle macchine su cui gira l'applicazione -> non è accettabile, perché se un attaccante prende il controllo della macchina o riesce ad accedere al file può leggere il file di configurazione e leggere pertanto la password;</div>
<div><br></div><div>- annegate nel codice dell'applicazione -> non è accettabile perché verrebbero scritte nei repository dei sorgenti e quindi tutti gli sviluppatori potrebbero leggere tali password; inoltre essendo le nostre applicazioni scritte in Java (e non potendo offuscare tutti i binari) è come scrivere le stesse password in chiaro su un file di configurazione;</div>
<div><br></div><div>- in un file di configurazione ma cifrate -> non è accettabile: per cifrare serve una chiave e dove scrivo la chiave? il problema è solo stato spostato;</div><div><br></div><div>- in un archivio (database relazionale, directory service, ...) accessibile solo da pochi: ma per accedere all'archivio avrei bisogno di una password, e siamo di nuovo al punto di prima.</div>
<div><br></div><div>Immagino che questo problema non sia solo della nostra azienda. E, come dicevo, ho trovato altri che hanno posto domande simili. Ma non ho trovato risposte valide.</div><div>Voi cosa ne dite? Come avete risolto voi questo problema?</div>
