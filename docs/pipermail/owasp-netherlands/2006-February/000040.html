Hello Eelco, all,

Actually, the questions asked when outsourcing software are:

- How safe is my software?
- How does the software developer respond to security incidents?
- etc.

To really know something about the security of a software product, we have to evaluate the full development cycle and the product itself. This has a 
big overlap with SSE-CMM (ISO/IEC 21827) and the metrics project, as Eelco stated earlier.

Maybe we could combine 'safe outsourcing', 'metrics' and a 'capability maturity model' into one project? Something like:

- Guideline for developing software with verifiable security objectives
or
- Security Maturity Level of the development life cycle (-> CMMI extension?)

Please let me know your thoughts...

Best Regards,

Bert
