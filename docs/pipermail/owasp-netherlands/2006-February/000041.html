Hello all,

This book deals with a lot of topics, recently discussed on this list.

http://www.awprofessional.com/title/0321356705

Software Security is about putting the touchpoints to work for you. Because you can apply these touchpoints to the software artifacts you already 
produce as you develop software, you can adopt this book's methods without radically changing the way you work. Inside you'll find detailed 
explanations of

* Risk management frameworks and processes
* Code review using static analysis tools
* Architectural risk analysis
* Penetration testing
* Security testing
* Abuse case development

Could be interesting for some people here...
