<p class=MsoNormal><font size=2 face=Verdana><span lang=NL style='font-size:
10.0pt;font-family:Verdana'>Net terug van een bijeenkomst in Amerika m.b.t. software
security. Tijdens die bijeenkomst werd door Mary-Ann Davidson (CISO van Oracle)
uit de doeken gedaan hoe Oracle haar ontwikkelproces mbt security de afgelopen
tijd ingrijpend heeft aangepast om hun uitdagingen op dit gebied te adresseren.
Ipv het uitvoeren van handmatige code reviews op sommige delen van de software (kostenoverwegingen)
wordt nu alle code op security geverifieerd (en daarna handmatig gemodificeerd)
mbv Static Code Analysis technieken van Fortify Software. Meer info op:<o:p></o:p></span></font></p>

<p class=MsoNormal><font size=2 face=Verdana><span lang=NL style='font-size:
10.0pt;font-family:Verdana'>Het (laten) ontwikkelen van maatwerk blijft nog
steeds een uitdaging vwb het security aspect van de software aangezien het security
bewustzijn van ontwikkelende organisaties nog in de kinderschoenen staat, laat
staan het inzichtelijk krijgen van dit aspect op een manier die effectief en
efficient is zodat dit een meetbaar en herhaalbaar proces wordt waaruit
management informatie afgeleid kan worden waarop men vervolgens kan bijsturen. Een
combinatie van technologie welke vele aspecten hiervan automatiseert,
gecombineerd met kennis zou het antwoord kunnen zijn. Deze trend is in 2005
duidelijk in de VS opgepakt door vele hoogwaardige bedrijven in oa de financiele
sector.<o:p></o:p></span></font></p>
