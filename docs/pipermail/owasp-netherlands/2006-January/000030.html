Dennis, Eelco,

 

Net terug van een bijeenkomst in Amerika m.b.t. software security. Tijdens die bijeenkomst werd door Mary-Ann Davidson (CISO van Oracle) uit de doeken gedaan hoe Oracle haar ontwikkelproces mbt security de afgelopen tijd ingrijpend heeft aangepast om hun uitdagingen op dit gebied te adresseren. Ipv het uitvoeren van handmatige code reviews op sommige delen van de software (kostenoverwegingen) wordt nu alle code op security geverifieerd (en daarna handmatig gemodificeerd) mbv Static Code Analysis technieken van Fortify Software. Meer info op:

http://www.eweek.com/article2/0,1895,1903960,00.asp <http://www.eweek.com/article2/0,1895,1903960,00.asp> 

 

Het (laten) ontwikkelen van maatwerk blijft nog steeds een uitdaging vwb het security aspect van de software aangezien het security bewustzijn van ontwikkelende organisaties nog in de kinderschoenen staat, laat staan het inzichtelijk krijgen van dit aspect op een manier die effectief en efficient is zodat dit een meetbaar en herhaalbaar proces wordt waaruit management informatie afgeleid kan worden waarop men vervolgens kan bijsturen. Een combinatie van technologie welke vele aspecten hiervan automatiseert, gecombineerd met kennis zou het antwoord kunnen zijn. Deze trend is in 2005 duidelijk in de VS opgepakt door vele hoogwaardige bedrijven in oa de financiele sector.

Goedendag,

Wellicht dat het onderwerp "beveiligen en pakketten" een onderwerp van discussie en uitdieping kan zijn.
Het inzetten van standaard pakketten als SAP, Siebel (Oracle) etc. i.p.v. maatwerk applicaties kan behoorlijk wat hoofdbrekers met betrekking tot beveiliging opleveren.
Maatwerk kun je nog (laten) ontwikkelen volgens de OWASP / Secure coding richtlijnen, maar wat kun je hierover roepen t.o.v. die "standaard" pakketten.
Ik denk dat dit onderwerp een behoorlijke relatie heeft met het "safe outsourcen" van informatiesystemen.
