>> slow, tedious or time-consuming actions.
>>
>> *Description*
>>
>> Using speed to violate explicit or implicit assumptions about the
>> application’s normal use to achieve unfair individual gain, often
>> associated with deceit and loss to some other party.

>> related to faster progression through a series of application processes.

>> to add information, and may not involve the concept of process progression.

>>>>> *New OAT?*
>>>>>
>>>>> As you might notice on that flowchart, we have added a possible 21st
>>>>> OAT, with the draft name of "Inventory Exhaustion". This suggestion for a
>>>>> new OAT came up during the work for v1.1 but Tin and I wanted longer to
>>>>> discuss it with other people on this list. The idea is making goods or
>>>>> services unavailable to legitimate users, without actually purchasing. In
>>>>> e-commerce this could be creating adding limited stock to one or more
>>>>> shopping baskets, but never checking out. Therefore, it is not exhaustion
>>>>> of application system resources (OAT-015 Denial of service), and not
>>>>> Scalping (OAT-005 Scalping) since in that the intention is to resell them.
>>>>>
>>>>> As people interested in this project:
>>>>>
>>>>> * Do you think this threat event is different enough to any existing
>>>>> OAT and therefore be added as a new OAT?
>>>>> * Or is it really an example of an existing OAT?
>>>>> * Any other thoughts?
>>>>> * If added as new, is "Inventory Exhaustion" or "Inventory Exhausting"
>>>>> okay as a short name, or would you suggest something else, or is there a
>>>>> name already in use which we should copy?
