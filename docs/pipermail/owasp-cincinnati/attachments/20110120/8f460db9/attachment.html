<p>Is anyone on the list deploying HTML 5 applications yet?<br></p><h1 class="title"><a href="http://threatpost.com/en_us/blogs/security-concern-html5-gains-traction-091610">Security a Concern as HTML5 Gains Traction</a></h1>

  <div class="content">
    <span class="print-link"></span><p>From animated logos to Web videos
 for hip, independent bands, HTML5 is getting buzz and gaining traction.
 But concerns about the security of features in the new version of the 
Web&#39;s <em>lingua franca</em> persist. </p><div>Every technology 
innovation has its coming out party, and Google Inc.&#39;s recent &quot;dancing 
balls&quot; logo experiment was widely interpreted as a high-impact debut for
 the next version of HTML, dubbed HTML5. But web security experts are 
warning that the sprawling new Web standard may favor functionality over
 security, enabling a new generation of powerful Web based attacks.  </div><p>Currently under development by the World Wide Web Consortium (W3C), the Web&#39;s standards-setting organization, the <a href="http://dev.w3.org/html5/spec/Overview.html" target="_blank">HTML 5 specification </a>is an effort to revamp the Web&#39;s <em>lingua</em><em> </em><em>franca</em><em> </em>after
 its last major makeover, HTML4, in the late 1990s. The new 
specification adds features that support the kind of rich interactions 
and application functions that have become common on the Web, mostly 
through the addition of third party tools like Adobe&#39;s Flash and 
Microsoft&#39;s Silverlight platforms. Among the additions to HTML5 are new 
elements and APIs that will enable two dimensional drawing, native 
rendering of video and audio files, drag and drop capabilities and 
offline capabilities for Web applications using local storage and 
databases.</p><p>HTML5 has some major backers, among them: search giant 
Google. In addition to its eye catching logo animations, that company 
has lent development talent to the development of HTML5 with projects 
like the Arcade Fire Web video &quot;the Wilderness Downtown,&quot; which 
highlight the capabilities of Google&#39;s Chrome Browser, and which <a href="http://googleblog.blogspot.com/2010/08/arcade-fire-meets-html5.html" target="_blank">Google promoted on its blog</a>. The company has also <a href="http://www.html5rocks.com/" target="_blank">created a Web site, HTML5rocks.com</a>, to extol the virtues of the new version of HTML.</p>

              <p>The new standard does make strides in plugging up some 
gaping security issues inherited from earlier versions of HTML, said 
Adam Barth of the University of California, Berkeley, who has been 
involved with the development of the new specification. </p><p>For one 
thing, HTML5 actually specifies how HTML code should be parsed by Web 
browsers. Previous iterations of the language left it up to individual 
platform vendors to develop their own interpretations of how the code 
should be parsed, which led to differences in how the language was 
rendered on different browsers. That also created opportunities for 
attackers to exploit quirks in the HTML parsing of specific browsers in 
Cross Site Scripting and other Web based attacks, Barth wrote in an 
e-mail response to questions from Threatpost. </p><p>New features are 
also intended to make HTML5 more secure than its predecessors. Among 
them is a new sandbox attribute that allows Web sites that use iFrames 
to aggregate untrusted content from external sources to run it in a 
secured environment. The postMessage() feature, for cross document 
messaging, allows secure communication between different web sites in 
the browser, Barth said.  </p><p>Web security experts contacted by 
Threatpost agree that there are security enhancements in HTML5, but all 
expressed the same concern: that the new specification will greatly 
increase the &quot;attack surface&quot; of HTML - providing more avenues by which 
malicious code can be delivered through the Web.</p><p>&quot;HTML5 has an 
enormous amount of functionality. The (specification) is just huge,&quot; 
said Jeremiah Grossman of Web security firm WhiteHat. The breadth of the
 new specification gives him concern. &quot;I know that we&#39;re still finding 
vulnerabilities in HTML4,&quot; Grossman said. </p><p>Others echoed that concern. </p><p>&quot;With
 any new functionality you&#39;re going to have new security concerns. HTML5
 is going to increase the attack surface considerably,&quot; said Neil 
Daswani of Web security firm Dasient. </p><p>Though no specific 
vulnerabilities or attacks against HTML5-specific features are 
documented, experts point to a few areas of concern: </p><p>Local, 
database and session storage: More options for storing user data locally
 will make it easier to develop Web applications that can also work in 
offline environments. They&#39;ll also reduce the reliance on session 
cookies and improve the performance of Web based applications, which can
 retrieve data locally instead of having to make requests out to a Web 
server and retrieve the results. </p><p>The conventional wisdom has been
 that these sensitive data stores will be a rich target for malicious 
hackers who might leverage cross site scripting attacks or other means 
to siphon sensitive data from them. Security experts we talked to played
 down that threat, noting that local storage has been around for a long 
time with technologies like Flash. However, engineers at Veracode in May
 <a href="http://www.veracode.com/blog/2010/05/html5-security-in-a-nutshell/" target="_blank">raised warnings about an implementation issue with the sessionStorage</a> feature that could make it vulnerable to manipulation from untrusted Web sites. </p>
<p>The
 new sandboxing and postMessage() features are examples of tools that, 
if not used properly, could fail to provide protection against hacks, or
 even enable new types of attacks. Veracode, in its analysis of HTML5, 
 raised red flags about the security of the postMessage() feature, as 
well, noting that Web applications that use cross-document messaging 
could be vulnerable to attack from malicious Web sites, which could 
spoof rogue messages. </p><p>Even features, like postMessage() designed 
to improve security could actually make Web sites less secure if they&#39;re
 implemented incorrectly, said Isaac Dawson, a security researcher at 
Veracode. </p><p style="margin: 1em 0px;">&quot;As with any security functionality, over reliance or simple misuse of it could unknowingly make a site less secure,&quot; he wrote. </p><div>In
 addition, HTML5&#39;s ability to support native rendering of audio and 
video files could allow attackers to take advantage of security 
vulnerabilities in supported audio and video file formats for Web based 
attacks. </div><p>&quot;I imagine we&#39;ll see a lot of people building custom 
fuzzers for the supported video codecs,&quot; said Dawson. That will put the 
onus on the Web browser makers themselves - Microsoft, Google, Mozilla 
and Apple - to review the code used for video parsing and rendering 
heavily to prevent them from being exploited, he said. </p><p>Grossman, 
of WhiteHat, also worries about the ripple effects of HTML5 adoption. In
 particular, the way that new HTML features will break or disable Web 
security filters that are designed to block malicious activity. In just 
one example, he notes that HTML5&#39;s broader support of event handlers on 
HTML elements could defeat blacklist filters designed to allow HTML but 
block scripting. </p><p>Still, with the HTML5 not finalized and adoption
 still in its infancy, the full impact of the specification - for good 
and bad - won&#39;t be felt for years, experts agreed. </p><p>Grossman said 
he sees support for HTML5 in around 10% to 12% of the 2,000 Web sites 
his firm monitors. It could be three years or more before the security 
implications of the new capabilities in HTML5 are fully grasped, he 
said. </p><p>However, HTML5 features could start working their way into 
the fabric of the Web long before wholesale adoption of the new 
standard, as Web sites look to take advantage of features, like local 
storage, on browsers that can support it, Grossman said. </p><p>&quot;Honestly,
 I think it will make some things better,&quot; said Dawson of Veracode. &quot;But
 as with all technologies it&#39;s going to take a while for 
the repercussions of certain designs to be fully known.&quot; <br></p></div>
