Security is becoming more and more a part of our mainstream in software development.  It's getting hard to develop without a good, solid nod towards security, but often times, security closes holes that we need in testing to verify our software quality and produce better product.
<br>

<br>
What's more, how do we test the security itself?  Please join us at the Security SIG to discuss the issues of how security lines up with testing.<br>
