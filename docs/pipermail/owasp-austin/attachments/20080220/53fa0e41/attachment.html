<br>&lt;/p&gt;&lt;p&gt;&lt;b&gt;Who:&lt;/b&gt; Michael Howard, Author of Writing Secure Code
<br>&lt;/p&gt;&lt;p&gt;MICHAEL HOWARD, PRINCIPAL SECURITY PROGRAM MANAGER, MICROSOFT&rsquo;S SECURITY ENGINEERING TEAM
<br>&lt;/p&gt;&lt;p&gt;Michael Howard is a principal security program manager on the
<br>Trustworthy Computing (TwC) Group&rsquo;s Security Engineering team at
<br>Microsoft, where he is responsible for managing secure design,
<br>programming, and testing techniques across the company. Howard is an
<br>architect of the Security Development Lifecycle (SDL), a process for
<br>improving the security of Microsoft&rsquo;s software. &lt;/p&gt;&lt;p&gt;Howard began his career with Microsoft in 1992 at the
<br>company&rsquo;s New Zealand office, working for the first two years with
<br>Windows and compilers on the Product Support Services team, and then
<br>with Microsoft Consulting Services, where he provided security
<br>infrastructure support to customers and assisted in the design of
<br>custom solutions and development of software. In 1997, Howard moved to
<br>the United States to work for the Windows division on Internet
<br>Information Services, Microsoft&rsquo;s next-generation web server, before
<br>moving to his current role in 2000. &lt;/p&gt;&lt;p&gt;Howard is a Certified Information Systems Security
<br>Professional (CISSP) and a frequent speaker at security-related
<br>conferences. He regularly publishes articles on security design and is
<br>the co-author of six security books, including the award-winning
<br>Writing Secure Code, 19 Deadly Sins of Software Security, The Security
<br>Development Lifecycle and his most recent release, Writing Secure Code
<br>for Windows Vista.
