<br>&lt;b&gt;Topic: &lt;/b&gt; Securely Handling Sensitive Configuration Data.
<br>&lt;/p&gt;&lt;p&gt;One of the age old problems with web applications was keeping sensitive
<br>data available on a need to know basis.  The classic case of this is
<br>database credentials.  The application needs them to connect to the
<br>database but developers shouldn&#39;t have direct access to the DB -
<br>particularly the production DB.  The presentation will discuss how we
<br>took on this specific problem, our determination that this was a
<br>specific case of a more general problem and how we solved that general
<br>problem.  In our solution, sensitive data is only available to the
<br>application and trusted 3rd parties (e.g. DBAs).  We will then cover our
<br>implementation of that solution in a .Net 2.0 environment and discuss
<br>some options for J2EE environments.  So far, we used our .Net solution
<br>successfully for database credentials and private encryption keys used
<br>in XML-DSig.  Sensitive data is only available to the application and
<br>trusted 3rd parties (e.g. DBAs).
