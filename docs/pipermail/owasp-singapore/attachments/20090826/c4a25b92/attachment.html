vulnerabilities. There are even automated script or a bot exploiting

I am sure we can see more interesting ways to inject malicious code on
websites in the days to come.<br>
    <br>
  For prevention, besides the server-side protection mention, on the
client side, I think the "no-script" firefox plugin might mitigate the
risk to a certain extend. However, such protective technologies are
usually for people "in the know",  everyday users will find such
protective technologies a hassle!    <br>

      <div>I forgot to mention one cause - Password Stealing Trojan.<br>
      <br>
Stolen FTP passwords are sent along with FTP url to the attacker
controlled server.<br>
The server then passes them to Injector which does:<br>
         - login to ftp server<br>
         - get index pages in each directory<br>
         - append malicious codes in places ( after &lt;body&gt;, above
&lt;/body&gt;, at end of file when it finds ?&gt;  %&gt;)<br>
         - upload infected pages<br>
      <br>
If you analyze the FTP log for such cases, you will see a continuous
long list of get/upload actions only on index pages in a few seconds.
