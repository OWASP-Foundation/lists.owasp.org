    <br>
    The exploit involved the bypass of a digital token system that Sony
    used when users reset their PSN password. Attackers could carry out
    the attack by visiting <a
href="https://store.playstation.com/accounts/reset/resetPassword.action?token"
      target="_blank">https://store.playstation.com/accounts/reset/resetPassword.action?token</a>
    and then, in a separate browser tab, opening a different page on <a
      href="http://us.playstation.com/" target="_blank">us.playstation.com</a>
    and following Sony's reset procedure, which required only the date
    of birth and email address associated with the account.
    <p>The attacker would then return to the original tab and, armed
      with the browser cookie just issued by Sony's servers, complete an
      image verification on the page. The attacker would then proceed to
      a scree allowing him to change the victim's password.</p>
    <p>&#8220;The page <a
href="https://store.playstation.com/accounts/reset/resetPassword.action?token"
        target="_blank">https://store.playstation.com/accounts/reset/resetPassword.action?token</a>,
      acts as though you had clicked the unique link sent to you via
      Sony for completing the second page's password reset,&#8221; Pilkington
      said during a discussion over instant message. He said it's
      &#8220;highly unlikely&#8221; the exploit technique was discovered until
      Tuesday evening.</p>
