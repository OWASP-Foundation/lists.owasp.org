    Solution: Install EMET and protect the iexplore.exe process.<br>
    <br>
    <a moz-do-not-send="true" class="moz-txt-link-freetext"
      href="http://www.theregister.co.uk/2010/12/23/ms_zero_day/">http://www.theregister.co.uk/2010/12/23/ms_zero_day/</a><br>
    <br>
    Microsoft warned on Wednesday of a new zero-day vulnerability in
    Internet Explorer.<br>
    <br>
    The flaw creates a means for hackers to inject malware onto
    vulnerable systems, providing surfers are first tricked into
    visiting booby-trapped websites. As such the flaw poses a severe
    drive-by download risk.<br>
    <br>
    All established version of IE (from 6 to 8) are affected. It's
    unclear whether or not the IE 9 beta is similarly vulnerable. The
    flaw <a moz-do-not-send="true" target="_blank"
      href="http://www.vupen.com/english/advisories/2010/3156">reportedly</a>
    involves the handling of Cascading Style Sheets by Microsoft's
    browser software. The bug first came to light on the <a
      moz-do-not-send="true" href="http://seclists.org/">seclists.org</a>
    full disclosure mailing list earlier this month<br>
    <br>
    A module exploiting the bug &#8211; which is noteworthy because it defeats
    Data Execution Prevention (DEP) and Address Space Layout
    Randomisation (ASLR) security defences in Microsoft products &#8211; has
    been added by the Metasploit project<br>
    <br>
    No patch is available but Redmond has published an <a
      moz-do-not-send="true" target="_blank"
href="http://blogs.technet.com/b/srd/archive/2010/12/22/new-internet-explorer-vulnerability-affecting-all-versions-of-ie.aspx">advisory</a>
    explaining how to mitigate against possible attack.<br>
    <br>
    A more detailed discussion of the flaw can be found in a blog post
    by Paul Duckin of Sophos <a moz-do-not-send="true" target="_blank"
href="http://nakedsecurity.sophos.com/2010/12/23/internet-explorer-zero-day-exploit-explanation-and-mitigation">here</a>.
