have been very prevalent. NoScript and RequestPolicy prevents requests to internal LAN IPs address from probing via CSS, JavaScript, HTML image tag and so on. <br><br>
<br>RequestPolicy bydefault allows requests to : <br><br>&nbsp;&nbsp; -&nbsp; <a href="http://example.com"><a href="http://example.com">example.com</a></a><br>&nbsp;&nbsp; - *.<a href="http://example.com"><a href="http://example.com">example.com</a></a> (like <a href="http://a.b.c.example.com">a.b.c.example.com</a>)<br>

<br>Otherwise, it denies requests according to its default-deny policy. <br><br>RequestPolicy monitors HTTP requests from the following objects:<br><br>HTML(script, iframe) CSS JS Image Flash<br><br>NoScript is not designed to monitor HTTP requests. Instead it's first designed to block:<br>

<br>&lt;script&gt; tag <br>&lt;embed&gt; tag<br>&lt;object&gt; tag<br><br>Later, NoScript is being enhanced to protect latest web threats like clickjacking.<br><br>To illustrate, consider the following iframe tag in a <a href="http://victim.com"><a href="http://victim.com">victim.com</a></a> :<br>

<br>&lt;iframe src="<a href="http://attacker.com/"><a href="http://attacker.com/">http://attacker.com/</a></a>" border="0" width="0" height="0"&gt;&lt;/iframe&gt;<br><br>RequestPolicy&nbsp; will block (untrusted/cross-request) <a href="http://attacker.com"><a href="http://attacker.com">http://attacker.com</a></a> unless you have allowed it or <a href="http://attacker.com"><a href="http://attacker.com">attacker.com</a></a> in white-lists.<br>

NoScript will not block <a href="http://attacker.com"><a href="http://attacker.com">http://attacker.com</a></a> from request. Instead, it will block malicious script tag, embeded objects and the like. <br><br>RequestPolicy has the ability to block HTTP requests from the flash object while NoScript doesn't have. <br>

Once you let NoScript to allow flash object, then the flash object can do anything it wants.<br>Even if you allow NoScript to allow flash object, RequestPolicy will block any cross-site HTTP requests of the flash object. <br>
