Back to my constant pitch that most, if not all, security issues
originate from software.

Hence, we need the next generation of IS pros to be software experts in
being about to *prevent*, not just mitigate, security vulnerabilities.

The "a-peh" or older generation of IS pros had been playing a reactive
game, mainly restricting ourselves to the OS and network.
We need to change how we play to better survive in the new era.

I hope this is a wake-up call for IS pros to engage more in the software
development circle.

Just my $0.02.
