Hi All,

Just wanted to send a message out there to try and gauge how many
people are already part of this list, and I guess what you're current
level of use of OWASP materials or tools is.

I'll start off with myself.

My name's Christian and I currently work within the BankWest Security
Consulting Services (sounds a bit like the kick off to a round-robin
intro at an AA meeting!?). As part of my role we currently undertake
and help coordinate business as usual and project based Vulnerability
Assessments (VA), Penetration Testing (PT) and Application
Assessments.

1. Our current methodology used for app assessments is based, and
draws heavily, from the OWASP Testing Guide.
(http://www.owasp.org/index.php/Category:OWASP_Testing_Project)

2. We *try* and ensure that our app architects and developers are
aware of the OWASP Top Ten.
(http://www.owasp.org/index.php/Category:OWASP_Top_Ten_Project)

3. One of our multitude of local proxies used in testing is the great
open source, WebScarab.
(http://www.owasp.org/index.php/Category:OWASP_WebScarab_Project)

Some of the other projects that I'm personally interested in and
hopefully might make their way into work:
- OWASP .NET Project
(http://www.owasp.org/index.php/Category:OWASP_.NET_Project)
- OWASP Application Security Verification Standard Project
(http://www.owasp.org/index.php/Category:OWASP_Application_Security_Verification_Standard_Project)
- OWASP Code Review Project
(http://www.owasp.org/index.php/Category:OWASP_Code_Review_Project)
- OWASP Code Crawler
(http://www.owasp.org/index.php/Category:OWASP_Code_Crawler)
- OWASP AntiSamy Project .NET
(http://www.owasp.org/index.php/Category:OWASP_AntiSamy_Project_.NET)
- GTK+GUI for w3af Project
(http://www.owasp.org/index.php/Category:GTK_plus_GUI_for_w3af_Project).

The last one is interesting, because w3af is another tool that we
occasionally use in our suite of testing tools, and one of the
problems with it is its unintuitive interface (not to say that a GUI
will make it better, I believe some of the command interfaces need to
be a bit more intuitive).

Regards,

Christian
