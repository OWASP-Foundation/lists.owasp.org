<div><a href="http://msdn.microsoft.com/en-us/library/system.net.mime.contentdisposition.filename.aspx">http://msdn.microsoft.com/en-us/library/system.net.mime.contentdisposition.filename.aspx</a></div>
<div> </div>
<div>Yukaridaki linkte soyle bir ibare geciyor (okunmasi gerekeni yer kalin olarak isaretledim);</div>
<div> </div>
<div><em>&quot;The <span><span class="selflink">FileName</span></span> property allows the sender to suggest the name to be used to store an e-mail attachment on the recipient&#39;s computer. This name is a suggestion only; the receiving system can ignore it. <strong>The name must not include path information</strong>; any such information is ignored by the receiving computer.</em>&quot;</div>

<div> </div>
<div>Yani, ozelde <a href="http://ASP.NET">ASP.NET</a> icin konusursak, myC<em>ontentDisposition.FileName</em> field&#39;ini set ederken path bilgisini cikararak kullanmak guvenli olan yoldur. Mesela bu is icin </div>
<div><span class="selflink"><em>System.IO<span>.</span>FileInfo.FullName</em> yerine <span class="selflink"><em>System.IO<span>.</span>FileInfo.Name </em>field&#39;i kullanilmali... </span></span>veya <em>Path.GetFileName(myFileName) </em>kullanilmali.<br>
</div>
<div>Simdi senin test ettigin duruma donersek, developer anladigim kadari ile kullanicidan (saldirgandan) gelen dosya ismini direk olarak Content-Disposition&#39;a set etmemis. Once TextReader gibi bir wrapper ile okumus sonra bu streamden dosya ismini cikarirken fullname&#39;i almis. Content-Disposition meselesini bir kenara birakirsak, direk download&#39;a izin verilmemeli, verilecekse de dosya isimleri kontrol edilmeli.</div>
