<blockquote class="gmail_quote" style="margin:0 0 0 .8ex;border-left:1px #ccc solid;padding-left:1ex;">pasif dinleyici durumu, arp poisining yada sniffing alanýna girer. birde, hangi os uzerinden de yapildigi onemli. Bu tur attact lar, yapabilmek icin, pasif dinleyici diye tabir ettigin kisi yada sistemlerin sistemini dinlemesi ve network agini zehirlemesi (fake mac adresleri ile sistemde gezinme yada bagli bulundugu network ortaminda dns dagitan makineyi bulup onu sniff etmesi)  gerekir. <div>


<br></div><div>Bana gore alternatiflerin su sekilde olmali </div><div>-- Windows ortaminda, mac adreslerinden yola çýkarak bulabilirsin (fake adres varmi, sacma ama final kisminda isine yarayacak)</div><div><div>-- dns uzerinden giden paketleri kontrol etmen gerekebilir</div>


<div>-- dns uzerinde bir yavaslama, baglanti kaybi varmi, bunlari anlayabilmen,</div><div>-- yada, alakasiz gordugun portlari kapaman gerekebilir,</div><div>-- network adminisin, ve giden gelen paketlerin haddi hesabi yok, dersen de, iste burada ilk bastaki secenek olan mac adres kontrolunu yapip, ilgili pc&#39;yi bulacak ve admin olarak o pc&#39;yi kurcalaman gerekecek. Belki pc sahibi, gizli biseyler yapiyor olabilir.</div>
