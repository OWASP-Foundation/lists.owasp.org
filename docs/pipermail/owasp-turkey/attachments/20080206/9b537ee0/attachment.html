<p>It allows batch download, offline debugging and debug under 2005 for MS Permissive Licence Code.</p>
<p>NET Mass Downloader <br>Welcome to the .NET Mass Downloader project. While it&#39;s great that Microsoft has released the .NET Reference Source Code, you can only get it one file at a time while you&#39;re debugging. If you&#39;d like to batch download it for reading or to populate the cache, you&#39;d have to write a program that instantiated and called each method in the Framework Class Library. Fortunately, .NET Mass Downloader comes to the rescue!<br>

&nbsp;</p>
<p>Using .NET Mass Downloader <br>Open a command or PowerShell prompt and navigate to where you extracted the current release. The tool itself is NetMassDownloader.exe and when run without parameters shows the following help screen:</p>


<p>.Net Mass Downloader <a href="http://1.0.0.0" target="_blank" onclick="return top.js.OpenExtLink(window,event,this)">1.0.0.0</a> - (c) 2008 by Kerem Kusmezer, John Robbins<br>&nbsp;<br>Batch download the Microsoft .NET Reference Source code.<br>
&nbsp;<br>Usage: NetMassDownloader [-file &lt;file&gt;]<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; [-directory &lt;directory]<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; [-output &lt;directory&gt;]<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; [-vsver &lt;version&gt;]<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; [-force] [-nologo] [-verbose] [-?]<br>

&nbsp;<br>&nbsp;&nbsp;&nbsp; -file&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; - Download an individual file&#39;s PDB and source code. You can<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; specify multiple file parameters. (Short -f).<br>&nbsp;&nbsp;&nbsp; -directory - Download all the found PDB and source code for all files in<br>

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; the specified directory. You can specify multiple<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; directory parameters (Short -d).<br>&nbsp;&nbsp;&nbsp; -output&nbsp;&nbsp;&nbsp; - The output directory for PDB and source files. The default<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; directory is the cache directory set in Visual Studio 2008.<br>

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; By using the cache directory, you&#39;ll have the PDB and source<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; files available to Visual Studio 2008. However, to use the<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; .NET Reference Source Code with VS 2005, use the -output<br>

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; switch and in the Options dialog, Debugging, Symbols property<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; page, add the specified output directory to the &quot;Symbol file<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; (.pdb) locations.&quot; Also, add the directory to the Solution<br>

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Properties, Common Properties, Debug Source Files, Directories<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; containing source code location. The Visual Studio 2005<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; debugger will automatically load the source code. (Short -o)<br>

&nbsp;&nbsp;&nbsp; -vsver&nbsp;&nbsp;&nbsp;&nbsp; - The Visual Studio version number to use for finding the cache<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; directory. The default is Visual Studio 2008,<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; but if you want to use the cache directory for Visual Studio<br>

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; 2005, you would pass &#39;-vsver 8.0&#39; (without quotes) (Short -vs)<br>&nbsp;&nbsp;&nbsp; -force&nbsp;&nbsp;&nbsp;&nbsp; - If specified, forces the downloading the PDB files into the<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; symbol server. When downloading to a symbol server if the PDB<br>

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; exists, it&#39;s not downloaded. Using the -output switch will<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; always download and process the PDB. (Short -fo)<br>&nbsp;&nbsp;&nbsp; -nologo&nbsp;&nbsp;&nbsp; - Don&#39;t show the logo information. (Short -n)<br>
&nbsp;&nbsp;&nbsp; -verbose&nbsp;&nbsp; - Do verbose output. May be worth turning on as the downloading<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; source code can take a long time. (Short -v)<br>&nbsp;&nbsp;&nbsp; -?&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; - This help message.<br>The only required arguments are –file or –directory, both of which can be specified as many times as you&#39;d like. When you specify a directory, only the .DLL and .EXE files from that directory will be processed. If you wanted to download all the source code from binaries in the .NET 2.0 32-bit and 64-bit directories, the command line you&#39;d pass is: -d C:\Windows\Microsoft.NET\Framework\v2.0.50727 –d C:\Windows\Microsoft.NET\Framework64\v2.0.50727.</p>


<p>The main purpose of Net Mass Downloader is to populate the source code download cache for debugging, the default download location is the cache you specified to Visual Studio 2008. The –vsver switch to account for future Visual Studio versions so Mass Downloader could work with future CTPs and versions. </p>


<p>While it&#39;s great to see the .NET Reference Source Code in Visual Studio 2008, there are a lot of developers out there who can&#39;t upgrade yet, but would love to be able to debug into the .NET Reference Source Code. If you specify the -output parameter, the PDB and .NET Reference Source Code will be written to the specified directory. In Visual Studio 2005, place that directory in the Options dialog, Debugging, Symbols property page. In the &quot;Symbol file (.pdb) locations&quot; list box as the first item. Also in the Options dialog, Debugging, General property page, uncheck &quot;Require source files to exactly match the original version.&quot; Finally, in each Visual Studio 2005 project go into the solution property pages, Common Properties, Debugging Source Files, and in the &quot;Directories containing source code&quot; add the output directory to the top of the list. That&#39;s enough for Visual Studio 2005 to debug into the .NET Reference Source Code.</p>


<p>When you first run Net Mass Downloader, you will be prompted with the current EULA for accessing the source code. If you don&#39;t agree with the Microsoft EULA, clicking the Decline button will not download the source code. <br>
