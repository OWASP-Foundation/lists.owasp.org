<div>HTTPS, yani SSL uzerinden HTTP, uctan uca sifreleme/butunluk koruma/anahtar degistirme/.. standardi. Eger soruyu yanlis anlamadiysam;</div>
<div>Sunucu tarafinda sifrelenen kullanici adi/sifre ikilisi (veya kredi karti, yer bilgisi, mesaj icerigi, v.b.) tarayicinin bulundugu bilgisayara ulastiginda otomatik olarak cozuluyor.Yani tarayiciya sifresi cozulmus (decrypt) olarak geliyor. Bu nedenle de sifre size (web browser&#39;a) sifresi cozulmus halde gosteriliyor. HTTPS&#39;li trafigin tarayicida cachelenmedigi konusunda ise haklisin. </div>

<div>Sifreleme islemi tarayici ile uzaktaki HTTP sunucusu (veya SSL destekli load balancer, v.b.) arasinda gerceklesiyor.</div>
<div>Burda aslinda problem HTTPS&#39;de degil de, sifrenin plaintext olarak uzaktaki makinada (veritabani, LDAP, v.b.) tutulma olasiligi ve belki de daha onemlisi sifrenin tarayiciya gonderilmesi. Profil guncelleme icin sifrenin gonderilmesine gerek yok.<br>
</div>
