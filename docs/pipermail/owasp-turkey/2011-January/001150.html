>> >> XSS'i sadece cookie çalmaya yönelik bir saldırı olarak görmemek lazım.
>> >> Cookie içindeki oturum bilgisini almak yapılabilecek saldırılardan
>> >> birisi. Örneğin facebook gibi popüler bir sistemde bulunabilecek bir
>> >> stored xss rahatlıkla <img src> tagleri ile DDOS yapmak için
>> >> kullanılabilir. Yine XSRF gibi yetkilendirilmiş bir kullanıcıya
>> >> yönelik salıdırılar XSS kullanılarak daha başarılı olacaktır. Bunların
>> >> hiç birinde de cookie ihtiyaç olmayacağı gibi hala önemli saldırılar
>> >> gerçekleştirilebilmektedir.
>> >>
>> >> aynı zamanda httponly korumasıda değişik yöntemler ile uygulamanın da
>> >> izin vermesi durumunda devre dışı bırakılabilir, bu sebeple onu nasıl
>> >> devre dışı bırakacağını bilen bir saldırganın oturum cookie'sini
>> >> çalmasını da engellemeyecektir. Bu nedenle yeterli düzeyde input
>> >> validation yapılmayan durumlarda XSS her zaman önemli bir risk
>> >> oluşturacaktır. dolayısı ile sadece çalınan cookie'nin önemi ile XSS
>> >> öneminin belirlenmesi yanlış olacaktır diye düşünüyorum.
>> >>
>> >> Web uygulama sorunlarını gidermek her halikarda uygulama geliştiricide
>> >> bitiyor, sorunları harici önlemler ile gidermeye çalışmak hiç bir
>> >> zaman yeterli olmayacaktır veya çok pahalıya mal olacaktır. XSS
>> >> problemi popüler olarak 2001 li yılların başında konuşulmaya başlandı
>> >> ve 2011 olmasına rağmen halen en sık karşılaşılan problem.
