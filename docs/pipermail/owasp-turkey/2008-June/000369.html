Herkese selam,

Hash(özet) fonksiyonlarının 3 temel özellikleri vardir:
1. Değişken veri boyutunu her zaman sabit bir boya yollarlar.
2. Tek yönlü fonksiyonlardır yani range kümesinden yola çıkarak
domain'in hangi elemanın inverse map'i oldugu bilinmemelidir.
3. Collision-free olmalılardır, yani 1-1 olmaya ne kadar yakın
olurlarsa o kadar iyidir. Range kümesine iyi fiffuse etmelidir.

Sizin istediginiz işlev için 2 ve 3 nolu maddelerin fail etme
olasiligi hayli fazla zira 1024 bit -> 32 bit map'de çok iyi bir
diffusion olacagini zannetmiyorum. Zaten 2^32+1 tane inoput sonra ki
bu 4.3 milyar civarında mutlaka çakışma olacaktır.

Bütün bunların yanında çözümün bir parçası olmak için "sihirli
sabitler" i içeren Robert Jenkins'in 32 bit hashini tavsiye
edebilirim.

uint32_t hash( uint32_t a)
{
   a = (a+0x7ed55d16) + (a<<12);
   a = (a^0xc761c23c) ^ (a>>19);
   a = (a+0x165667b1) + (a<<5);
   a = (a+0xd3a2646c) ^ (a<<9);
   a = (a+0xfd7046c5) + (a<<3);
   a = (a^0xb55a4f09) ^ (a>>16);
   return a;
}

Input'u modifiye ederek 32 bit çıktı alabilirsiniz. Umarım yardımcı
olmuşumdur ..
