pasif dinleyici durumu, arp poisining yada sniffing alanına girer. birde,
hangi os uzerinden de yapildigi onemli. Bu tur attact lar, yapabilmek icin,
pasif dinleyici diye tabir ettigin kisi yada sistemlerin sistemini dinlemesi
ve network agini zehirlemesi (fake mac adresleri ile sistemde gezinme yada
bagli bulundugu network ortaminda dns dagitan makineyi bulup onu sniff
etmesi)  gerekir.

Bana gore alternatiflerin su sekilde olmali
-- Windows ortaminda, mac adreslerinden yola çıkarak bulabilirsin (fake
adres varmi, sacma ama final kisminda isine yarayacak)
-- dns uzerinden giden paketleri kontrol etmen gerekebilir
-- dns uzerinde bir yavaslama, baglanti kaybi varmi, bunlari anlayabilmen,
-- yada, alakasiz gordugun portlari kapaman gerekebilir,
-- network adminisin, ve giden gelen paketlerin haddi hesabi yok, dersen de,
iste burada ilk bastaki secenek olan mac adres kontrolunu yapip, ilgili
pc'yi bulacak ve admin olarak o pc'yi kurcalaman gerekecek. Belki pc sahibi,
gizli biseyler yapiyor olabilir.
-- Linux uzerinde de ayni durum (fazla bilmiyorum tabii), yukaridakilere ek
olarak, bagli bulundugun sistemde script calistirman gerebilir, mesela bu
blogda var (
http://blog.stardothosting.com/2009/05/01/detect-arp-poisoning-on-lan/)
