Any file can be malicious data or no data :), in case of WMV files it can be
embedded with malicious content (shellcode) that gets executed when the
"loader" reads the file and vulnerability (overflow etc.. ) gets exploited.

Megha, just want to know what you want to do with the WMV files, if you want
to discover overflow vulnerbility then fuzzing and exception analysis is an
option, if you are doing an exploit analysis then you shd look for shellcode
scanners, that will try to mark out dangerous byte sequences.
