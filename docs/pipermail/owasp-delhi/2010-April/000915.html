It all depends on why you were considering the deployment earlier ? Also, I
am interested in knowing what do you need it for ?

SSL is a layer of security which lies between transport layer and
application layer. It is very useful for preventing man-in-the-middle
attacks in an organization i.e. the insider attacks. Deploying this on your
file server (sftp) would definitely slow down the file transfer process but
would ensure the secure transfer. SSL provides you resilience from attacks,
IDS provides you the property of detection & recovery. You would also look
at other security properties like Prevention and Deterrence.
