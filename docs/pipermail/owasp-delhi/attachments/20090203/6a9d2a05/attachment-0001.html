<br>
<br><font size=2 face="sans-serif">I can give you an example where both
CSRF and XSS vulnerabilities both are in action to amplify the severity
of attack.</font>
<br><font size=2 face="sans-serif">If one application is allowing to store
javascript from User Input and display the same somewhere in the site(e.g.
area 11) , then it is very easy to store </font>
<br><font size=2 face="sans-serif">&lt;img&gt; or &lt;iframe&gt; tag or
any other javascript to perform an unauthorized actions such as fund transfer
(actions which are normally performed with CSRF attack) ,stealing the authentication
cookie etc.</font>
<br><font size=2 face="sans-serif">Now if the application is vulnerable
to CSRF then those stored malicious javascript will be executed when the
victim visit the area (area 11) of the site after authentication which
is one of the prime requirement of CSRF attack.</font>
<br><font size=2 face="sans-serif">Often it is referred as Stored CSRF
attack but actually it is the CSRF in conjunction with XSS attack. </font>
