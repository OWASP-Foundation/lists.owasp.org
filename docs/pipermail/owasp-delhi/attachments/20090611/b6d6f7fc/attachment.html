<p class=MsoNormal><span style='font-size:11.0pt;font-family:"Calibri","sans-serif";
color:#1F497D'>    Every variable in the program has certain amount of memory space
allocated to it.  It is used to buffer data. When a very high volume of data is
supplied to this particular space and there’s no input validation in
application that checks for amount of data being inserted, it’s prone to buffer
overflow. It can lead to some verbose errors being displayed, memory corruption
or even application crashing.<o:p></o:p></span></p>

<p class=MsoNormal><span style='font-size:11.0pt;font-family:"Calibri","sans-serif";

<p class=MsoNormal><span style='font-size:11.0pt;font-family:"Calibri","sans-serif";

<p class=MsoNormal><span style='font-size:11.0pt;font-family:"Calibri","sans-serif";
color:#1F497D'>As far as DOS is concerned it’s simply Denial of Service of a
particular application, which becomes unaccessible to any legitimate user
creating business losses. Many reasons can cause DOS like:<o:p></o:p></span></p>

<p class=MsoNormal><span style='font-size:11.0pt;font-family:"Calibri","sans-serif";
color:#1F497D'>Buffer Overflow: Application Crashes<o:p></o:p></span></p>

<p class=MsoNormal><span style='font-size:11.0pt;font-family:"Calibri","sans-serif";
color:#1F497D'>Automated Submission: Using any automated program/tool which can
automatically submit any feedback/regisration form in the application several times
causing the line to be busy and thus not usable by any legitimate user. That’s
why CAPTCHAS are used.<o:p></o:p></span></p>

<p class=MsoNormal><span style='font-size:11.0pt;font-family:"Calibri","sans-serif";
color:#1F497D'>SQL Injection: They can also create DOS. Like if the query  ‘;
SHUTDOWN ‘ is executed by the application, the database will be shutdown
causing DOS.<o:p></o:p></span></p>
