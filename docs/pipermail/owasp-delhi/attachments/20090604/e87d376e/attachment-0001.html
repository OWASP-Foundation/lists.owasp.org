you &gt;&gt;have to try different things. You have to be more diverse.<o:p></o:p></span></i></p>

<p><span style='font-size:11.0pt;font-family:"Calibri","sans-serif";color:#660000'>It
means when you supply any payload inside browser, the browser first encodes it
into URLEncoded format and passes it to your application. Again your
application needs to decode it using decoding functions to convert it into original
form.<o:p></o:p></span></p>

<p><span style='font-size:11.0pt;font-family:"Calibri","sans-serif";color:#660000'>For
example: &lt;script&gt;alert(1);&lt;/script&gt; will be passed as %3cscript%3e….format
by browser to the application and application needs some URLDecode function to
convert it into original format to understand it:<o:p></o:p></span></p>

<p><span style='font-size:11.0pt;font-family:"Calibri","sans-serif";color:#660000'>&lt;script&gt;….&lt;/script&gt;<o:p></o:p></span></p>

<p><span style='font-size:11.0pt;font-family:"Calibri","sans-serif";color:#660000'>Again
it has to be escaped before displaying it as it is in executable format
&lt;script&gt;…&lt;/script&gt;.<o:p></o:p></span></p>

<p><span style='font-size:11.0pt;font-family:"Calibri","sans-serif";color:#660000'>The
sample code for URL decoding and output escaping process in PHP is:<o:p></o:p></span></p>

<p><code><span style='font-size:11.0pt;font-family:"Calibri","sans-serif";
color:#0000BB'>htmlspecialchars</span></code><code><span style='font-size:11.0pt;
font-family:"Calibri","sans-serif";color:#007700'>(</span></code><code><span
style='font-size:11.0pt;font-family:"Calibri","sans-serif";color:#0000BB'>urldecode</span></code><code><span
style='font-size:11.0pt;font-family:"Calibri","sans-serif";color:#007700'>(</span></code><code><span
style='font-size:11.0pt;font-family:"Calibri","sans-serif";color:#0000BB'>$URL
contents</span></code><code><span style='font-size:11.0pt;font-family:"Calibri","sans-serif";
color:#007700'>))<o:p></o:p></span></code></p>

<p><span style='font-size:11.0pt;font-family:"Calibri","sans-serif"'>Here the
URL contents are first decoded back by application and then output escaped by
htmlspecialchars() function so  that it can’t gets executed.<o:p></o:p></span></p>

<p><span style='font-size:11.0pt;font-family:"Calibri","sans-serif"'>As far as
