<p style="margin-bottom: 0in;">There are two possibility in case of
master server failed:</p>
<ol><li><p style="margin-bottom: 0in;">If  the session data is stored on
        the master server locally then the established connection with
        client will be disconnected becouse the redirected server has no
        session data.</p>
        </li><li><p style="margin-bottom: 0in;"> If the session data is stored on
        some centralized database and replicated using some technique then
        the established connection with client will not be disconnected due
        to the High Availability,  another server stands up as master server
        and shares session data from the centralized database. The Session
        Clustering module provides a comprehensive solution for
        synchronizing session data across a cluster. In this module the
        sessions that &quot;reside&quot; on the server where they were first
        created are subsequently, delivered to other servers in the cluster.
        This is done by having the alternate server, request the session
        data from the original server. Session Clustering HA (High
        Availability), is an additional safety layer for maintaining session
        information integrity in Web cluster environments. HA ensures that
        sessions will be serviced in case of a single failure. 
        </p>
</li></ol>
