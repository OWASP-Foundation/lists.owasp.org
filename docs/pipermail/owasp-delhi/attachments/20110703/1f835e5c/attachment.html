Hi All,<br><br>I&#39;ve released the first version of Jugaad library. 
&#39;libjugaad&#39; enables a binary to inject code (as a thread) within the 
context of another process on Linux. It exposes a simple API which is 
somewhat similar to CreateRemoteThread() on Windows and is very easy to 
use.<br><br><br><br>int create_remote_thread(pid_t pid,<br>                                     size_t stack_size,<br>                                     unsigned char * tpayload,<br>                                     size_t tpsize,<br>
                                     int thread_flags,<br>                                     int mmap_prot,<br>                                     int mmap_flags,<br>                                     void * bkpaddr);<br>
<br>For
 the knowledge hungry and for people who missed my talk at nullcon 
2011, the detail paper is uploaded on 
<a href="http://null.co.in/2011/07/03/project-jugaad/">http://null.co.in/2011/07/03/project-jugaad/</a>. If time permits I&#39;ll also 
post a video on the usage and internals of the library on SecurityTube.<br clear="all"><br>Why create the library? <br>Because there is no such (until now :p) functionality provided by the *nix platform for remote thread injection.<br>
<br>Why this email?<br>A request to test / use the API. Appreciate feedback from people on what distribution it fails or if there are any bugs - as
 I am yet to deliver it at defcon 2011. Your contribution (a valid bug report) would definately be mentioned as 
contributor in my presentation.<br><br>Source code?<br>Details: <a href="http://null.co.in/2011/07/03/project-jugaad-2/">http://null.co.in/2011/07/03/project-jugaad-2/</a><br>git repo: git://<a href="http://github.com/aseemjakhar/jugaad.git">github.com/aseemjakhar/jugaad.git</a><br clear="all">
