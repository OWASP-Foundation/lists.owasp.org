<div>After reading comments from different Experts i created a summary for this important attack:</div>
<div> </div>
<div><strong><u>Condition-1</u></strong> </div>
<div> </div>
<div><em><u>Both vulnerable website and e-mail account are opened within the mini tabs of the same parent browser:</u></em> </div>
<div> </div>
<div>In this case as a user clicks upon the crafted link in e-mail either by </div>
<div><br>&gt; right-clicking and opening it in in New Tab or in New Window<br></div>
<div>&gt; just click it simply</div>
<div> </div>
<div>In both the cases the XSS attack will occur. Again no matter which browser you are using.</div>
<div> </div>
<div><strong><u>Condition-2</u></strong> </div>
<div> </div>
<div>Both vulnerable website and e-mail account are opened within the Fresh Browsers Windows (and no new tab). </div>
<div>In this case as a user clicks upon the crafted link in e-mail either by </div>
<div><br>&gt; right-clicking and opening it in in New Tab or in New Window<br></div>
<div>&gt; just click it simply</div>
<div> </div>
<div><em>The attack will occur only if:</em></div>
<div><br>-&gt; A Browser is used which shares cookies for the same domain opened across different fresh browsers windows. ie: Chrome and Firefox 3.0.x handles cookies in same way even in tabs/Windows. </div>
<div> </div>
<div><strong>However many important websites especially financial do not allow users to open two instances of the same site for the same user which is an important security feature.</strong></div>
<div><strong></strong> </div>
