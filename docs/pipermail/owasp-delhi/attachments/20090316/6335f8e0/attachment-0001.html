<div> </div>
<div>As per Abrol Raj--</div>
<div>&gt; &quot;i am testing a website in which normal XSS script is not accepted by the application, it gives run time error. If i am encoded it into &gt;base64, it gets saved on server but not get executed.<br>&gt; what would be the possible reason that the script is not getting executed, even the script is saved by the application.&quot;</div>

<div> </div>
<div>Exactly....the application expects anything that can be again rendered back to the user after interpretation of the code. That is the case when this is exploited when there&#39;s no proper input validation in place and results in XSS execution.</div>

<div> </div>
<div>In case of supplying XSS vector in original form, the application knows how to render it back to the user-- If HTML encoding is there(output validation) it will just print the value as characters(i.e. no XSS); if HTML encodng is not there it will get executed(i.e. XSS). </div>

<div> </div>
<div>But in case of base64 encoding the application simply does not know &#39;what to do with the code&#39;. That payload is saved on the server instead of displaying it.</div>
<div> </div>
<div>However there are other ways to execute XSS with base64 encoding but they are rare. One exaple is with META tag, not with user input interaction:</div>
<div> </div>
<div>&lt;META HTTP-EQUIV=&quot;refresh&quot; CONTENT=&quot;0;url=data:text/html;base64,PHNjcmlwdD5hbGVydCgnWFNTJyk8L3NjcmlwdD4K&quot;&gt;</div>
