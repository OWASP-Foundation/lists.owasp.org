o prove the identity of someone accessing a service, such as a website. As the name suggests, KBA requires the knowledge of personal information of the individual to grant access to the protected material. There are two types of KBA: "static KBA", which is based on a pre-agreed set of "shared secrets"; and "dynamic KBA", which is based on questions generated from a wider base of personal information.</font></span></p><h2 style="font-weight:normal;margin:1em 0px 0.25em;overflow:hidden;padding:0px;border-bottom-width:1px;border-bottom-style:solid;border-bottom-color:rgb(170,170,170);background-image:none"><span style="background-color:rgba(255,255,255,0)"><font size="3" face="arial, helvetica, sans-serif">KBAPM Project Supports the NSTIC Guiding Principles</font></span></h2><p style="margin:0.5em 0px"><span style="background-color:rgba(255,255,255,0)"><font face="arial, helvetica, sans-serif">KBAPM Project is dedicated to developing KBA Standards in alignment the the NSTIC Guiding Principle. To that end KBAPM Project has answered the IDESG Standards Committee Solicitation to develop Knowledge Based Authentication Performance Metric Standards.</font></span></p><p style="margin:0.5em 0px"><span style="background-color:rgba(255,255,255,0)"><font face="arial, helvetica, sans-serif"><br></font></span></p><p style="margin:0.5em 0px"><font face="arial, helvetica, sans-serif">For more information contact the Project Leader, Ann Racuya-Robbins, at <a href="mailto:arr@worldknowldegebank.com" target="_blank">arr@worldknowldegebank.com</a>.</font></p><p style="margin:0.5em 0px"><font face="arial, helvetica, sans-serif"><br></font></p><p style="margin:0.5em 0px"><font face="arial, helvetica, sans-serif">Thanks</font></p><p style="margin:0.5em 0px"><font face="arial, helvetica, sans-serif">-----------------------------</font></p><p style="margin:0.5em 0px"><font face="arial, helvetica, sans-serif">Sandeep Singh</font></p><p style="margin:0.5em 0px"><font face="arial, helvetica, san
