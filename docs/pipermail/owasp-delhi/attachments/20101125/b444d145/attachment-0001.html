NoScript fails to detect the reflective XSS from trusted domains when an<br>
attack is conducted through SQLXSSI. The bypass in NoScript has been<br>
successfully conducted by using &quot;Reflective XSS&quot; through Union SQL<br>
poisoning attacks by exploiting the reverted errors in the browser. The<br>
attack string used to bypass is stated below<br>
<br>
<a href="http://www.example.com/news.php?news=12%27union%20select%201,2,3,4,5,6,7,0x3c7363726970743e616c657274282f73636861702f293c2f7363726970743e,9,10,11,12,version%28%29%20from%20tbl_news--+" target="_blank">http://www.example.com/news.php?news=12%27union%20select%201,2,3,4,5,6,7,0x3c7363726970743e616c657274282f73636861702f293c2f7363726970743e,9,10,11,12,version%28%29%20from%20tbl_news--</a><br>

<br>
The attacker can create a potential attack patterns using the above stated<br>
vector.<br>
<br>
The exploitation video has been released at SecNiche Security channel -<br>
<a href="http://www.youtube.com/watch?v=r-kgKNspqjQ" target="_blank">http://www.youtube.com/watch?v=r-kgKNspqjQ</a><br>
<br>
Disclosure: The bug was disclosed to the author on 24th November 2010. A<br>
new version of NoScript 2.0.6 is released today (25th November 2010).<br>
Further, NoScript 2.0.6 version fails to combat against this attack vector<br>
and can be bypassed with the same.<br><br><br>
