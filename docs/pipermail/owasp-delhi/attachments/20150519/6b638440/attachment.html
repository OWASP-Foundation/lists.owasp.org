<div dir="ltr">I got another interesting article on "<a href="https://wiki.mozilla.org/WebAppSec/Secure_Coding_Guidelines#Password_Storage">https://wiki.mozilla.org/WebAppSec/Secure_Coding_Guidelines#Password_Storage</a>".<div><br></div><div>The idea here is NOT to store "Simply Hashed Password" & "Salt".</div><div><br></div><div>As per the article:--</div><div><br></div><div><h3 style="color:black;margin:0.3em 0px 0px;overflow:hidden;padding-top:0.5em;padding-bottom:0px;border-bottom-style:none;font-size:20.1599998474121px;line-height:1.6;font-family:'Fira Sans','Trebuchet MS',sans-serif!important;background-image:none;background-repeat:initial"><span class="" id="Password_Storage">Password Storage</span></h3><p style="margin:0.5em 0px;line-height:23.0399990081787px;color:rgb(37,37,37);font-family:'Open Sans','Trebuchet MS',sans-serif;font-size:14.3999996185303px">Separate from the password policy, we should have the following standards when it comes to storing passwords:</p><ul style="line-height:1.5em;margin:0.3em 0px 1em 1.6em;padding:0px;color:rgb(37,37,37);font-family:'Open Sans','Trebuchet MS',sans-serif;font-size:14.3999996185303px"><li style="margin-bottom:0.1em">Passwords stored in a database should using the hmac+bcrypt function.</li></ul><p style="margin:0.5em 0px;line-height:23.0399990081787px;color:rgb(37,37,37);font-family:'Open Sans','Trebuchet MS',sans-serif;font-size:14.3999996185303px">The purpose of hmac and bcrypt storage is as follows:</p><ul style="line-height:1.5em;margin:0.3em 0px 1em 1.6em;padding:0px;color:rgb(37,37,37);font-family:'Open Sans','Trebuchet MS',sans-serif;font-size:14.3999996185303px"><li style="margin-bottom:0.1em">bcrypt provides a hashing mechanism which can be configured to consume sufficient time to prevent brute forcing of hash values even with many computers</li><li style="margin-bottom:0.1em">bcrypt can be easily adjusted at any time to increase the amount of work and thus provide protection against more powerful systems</li><li style="margin-bottom:0.1em">The nonce for the hmac value is designed to be stored on the file system and not in the databases storing the password hashes. In the event of a compromise of hash values due to SQL injection, the nonce will still be an unknown value since it would not be compromised from the file system. This significantly increases the complexity of brute forcing the compromised hashes considering both bcrypt and a large unknown nonce value</li><li style="margin-bottom:0.1em">The hmac operation is simply used as a secondary defense in the event there is a design weakness with bcrypt that could leak information about the password or aid an attacker</li></ul><p style="margin:0.5em 0px;line-height:23.0399990081787px;color:rgb(37,37,37);font-family:'Open Sans','Trebuchet MS',sans-serif;font-size:14.3999996185303px">A sample of this code is here: <a rel="nofollow" class="" href="https://github.com/fwenzel/django-sha2" style="color:rgb(102,51,102);text-decoration:none;padding-right:13px;background:linear-gradient(transparent,transparent) 100% 50% no-repeat,url(data:image/svg+xml)">https://github.com/fwenzel/django-sha2</a> </p><p style="margin:0.5em 0px;line-height:23.0399990081787px;color:rgb(37,37,37);font-family:'Open Sans','Trebuchet MS',sans-serif;font-size:14.3999996185303px">Keep in mind that while bcrypt is secure you should still enforce good passwords. As slow as an algorithm may be if a password is "123" it still would only take a short amount of time before somebody figures it out.</p></div></div><div class="gmail_extra"><br><div class="gmail_quote">On Tue, May 19, 2015 at 4:03 PM, Sushil Nehra <span dir="ltr"><<a href="mailto:sushil.nehra@gmail.com" target="_blank">sushil.nehra@gmail.com</a>></span> wrote:<br><blockquote class="gmail_quote" style="margin:0 0 0 .8ex;border-left:1px #ccc solid;padding-left:1ex"><div><div><div style="font-family:Calibri,sans-serif;font-size:11pt">Dear Sanjay<br><br>Salt is used in web app and also in Uni
