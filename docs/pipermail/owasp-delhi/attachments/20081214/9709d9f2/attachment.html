<br><font size=2 face="Arial">Pay specific effort to evaluate how key management
and encrypted data access are designed in the application architecture.</font>
<br>
<br><font size=2 face="Arial">Can an external attacker transparently decrypt
data without needing the key at all? Web applications might (highly possible)
have functionality that allows for transparent encryption and decryption
of data. Remember to evaluate how such transparent crypto functionality
might stand up to common web application attacks. Will a successful [SQL|HTML]
injection attack, return lots of unencrypted data because of crypto transparency?
This scenario will be a very common one. But we have to assess how resilient
the security architecture is, if there happens to be just one page with
SQL Injection vulnerability in the future.</font>
<br>
<br><font size=2 face="Arial">How resilient is the architecture towards
internal attackers? How good would the encryption and key management be
if we were to assume that a compromised laptop is plugged into the same
network?</font>
<br>
<br><font size=2 face="Arial">&gt; the consultants you hire have the right
skills for the job, else you might fail the certifying audit. </font>
<br><font size=2 face="Arial">&gt; If the certifying auditor himself turns
out to be a non-techie then God save our money ;)</font>
<br><font size=2 face="Arial">Thats a good point. In addition to tech skills,
I would always recommend consultants who would approach the application
as an adversary would. Dont hire a consultant who cant get his objective
right. The PCI consultant's goal should not be evaluating whether the PCI
Compliance certificate can hang in the auditee's reception. The goal should
be to assess if the application design and implementation have any vulnerabilities
that can lead to personally identifiable data.</font>
<br><font size=2 color=#000080 face="Arial">&nbsp;</font>
<br><font size=2 face="Arial">Check out the consultant's approach to PCI
audits. If it is mostly a checklist based one, with very little areas where
the attacker thinks like an adversary, the results might affect only the
auditee's reception :-)</font>
<br>
<br><font size=2 face="Arial">Just my 2 cents. YMMV.</font>
