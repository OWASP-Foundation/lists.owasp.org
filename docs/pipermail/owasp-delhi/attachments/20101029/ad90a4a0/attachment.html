<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>If I had to provide access to the enterprise portals to some
of the people in my partner organization (external users), I need to
authenticate them.<o:p></o:p></p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>If my internal users are using Active Directory for
authentication to the Enterprise Portals, will it be a good design to create
external users in the AD and allow authentication to the portal?<o:p></o:p></p>

<p class=MsoNormal>Or one should go with doing custom authentication of the
external users in Database and &#8220;not touch&#8221; Active Directory&#8221;?
<o:p></o:p></p>

<p class=MsoNormal>Or setup federated &#8220;trust&#8221; between organization
Active Directory (assuming both have same Active Directory technology)?<o:p></o:p></p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>My take is that Active Directory is the best product to
manage user authentication and should be used for creating external users too. <o:p></o:p></p>
