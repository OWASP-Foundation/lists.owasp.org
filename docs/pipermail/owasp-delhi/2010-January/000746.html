Precisely saying:

The Microsoft Internet Explorer vulnerability leveraged in this attack
allows for remote code execution, but does require user intervention (such
as following a hyperlink to a website, or opening an email attachment, etc).
Furthermore, the single exploit known to exist can be thwarted by Data
Execution Prevention (DEP), enabled by default in Internet Explorer 8 and
optionally in Internet Explorer 7.  Microsoft lists the following
combinations to be vulnerable: Internet Explorer 6 Service Pack 1 on
Microsoft Windows 2000 Service Pack 4, and Internet Explorer 6, Internet
Explorer 7 and Internet Explorer 8 on supported editions of Windows XP,
Windows Server 2003, Windows Vista, Windows Server 2008, Windows 7, and
Windows Server 2008 R2 are affected.
