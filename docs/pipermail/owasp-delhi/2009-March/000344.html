Hi Parmendra,

The first one is a CSRF for sure.
Unlike the common understanding that CSRF is used to make a change on the 
website, this one reads sensitive information.
Its a known gotcha with JSON.

The second one is not really a CSRF.
But what can be done is to guess passwords (and change them ) by 
overcoming the account lockout feature.
Its not a CSRF issue but a flaw with the change password feature, ideally 
there should be some account logoff feature or something after a series of 
failed 'change password' attempts.

Hope this helps.
