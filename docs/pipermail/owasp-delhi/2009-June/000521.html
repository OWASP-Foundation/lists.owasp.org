The split happens due carriage return i.e. CRLF Injection. HTTP Response
Splitting is one of the attacks originating from CRLF Injection. The other
one is Log Forging refer
http://www.acunetix.com/websitesecurity/crlf-injection.htm.

This attack is not as straight forward as XSS or SQL Injection to simulate.
Check out this resource for more information
http://www.owasp.org/index.php/HTTP_Response_Splitting

This article should help simulate a test
http://www.infosecwriters.com/text_resources/pdf/HTTP_Response.pdf
