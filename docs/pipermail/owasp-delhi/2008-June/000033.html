Hi All,

I was testing one of the application last night. The issue I am going to describe is nothing new to me but I was concerned about this for a long time and I did not get any chance to ask this before.

While testing the application against CSRF attack, I noticed that it is vulnerable to CSRF attack only if it is tested via. another tab of the same browser. However, if I test the application via.  different browser it does not work. 

This is how I tested:
I logged in to the application. I coded an html page with an image tag with source as the logout URL. (<img src="www.application.com/logout.php">). I sent this link to my email (just to make the scenario look real). I clicked on the link, it opened in the same browser in a different tab and I   got logged out of the application. However, if I use another browser window to open the URL, it did not work.

So, I wanted to know that how high is this risk and is there any way I can show that the application is also vulnerable by using different browser. Any suggestions/opinions will be highly appreciated.

Thank you.
Gunwant Singh.
