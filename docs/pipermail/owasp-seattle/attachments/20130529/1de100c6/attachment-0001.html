<br>
Title: Top Ten Web Defenses<br>
We cannot “firewall” or “patch” our way to secure websites. In the past,<br>
security professionals thought firewalls, Secure Sockets Layer (SSL),<br>
patching, and privacy policies were enough. Today, however, these<br>
methods are outdated and ineffective, as attacks on prominent,<br>
well-protected websites are occurring every day. Most every organization<br>
in the world have something in common – they have had websites<br>
compromised in some way. No company or industry is immune. Programmers<br>
need to learn to build websites differently. This talk will review the<br>
top coding techniques developers need to master in order to build a<br>
low-risk, high-security web application.<br>
<br>
<br>
BIO: Jim Manico is the VP of Security Architecture for WhiteHat<br>
Security, a web security firm. He authors and delivers developer<br>
security awareness training for WhiteHat Security and has a background<br>
building software as a developer and architect for over 20 years. Jim is<br>
also a global board member for the OWASP foundation where he helps drive<br>
the strategic vision for the organization. He manages and participates<br>
in several OWASP projects, including the OWASP cheat sheet series and<br>
several additional secure coding projects.<br>
<br>
<br>
<br>
After that, we have Brad Hill.<br>
<br>
Title: "HTML5 Security Realities"<br>
<br>
Abstract: It's almost conventional wisdom in the security biz that HTML5 is a nightmare of unnecessary complexity and attack surface, but should you believe the hype?  Brad challenges those notions in this talk with a report on the state of security in the modern browser and shows how to use emerging standards to solve some of the big problems of Web security.<br>

<br>
Bio: Brad Hill is an iSEC alum at PayPal, where he does security research and standards development, including serving as the co-chair of the Web Application Security Working Group at the W3C.<br>
