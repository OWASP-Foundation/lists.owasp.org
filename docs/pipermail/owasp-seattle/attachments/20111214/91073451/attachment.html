 I have been developing for Security Innovation with the help of O2

 me (as a developer) see security and its position on the 
development/management food-chain (btw on this topic, if you haven't you

<br></li>
<li>how O2 allows me to deal with real world problems such as: <br><br></li><ul><li>creating Unit Tests for jQuery/Ajax/WebServices based websites,</li><li>dealing
 with automation problems that ALL current browser automation engines 
have (WatiN, WatiR, Selenium, Cucumber, WebKit, QUnit, etc...) ,  and 
visualizing the data created using custom GUIs (note that O2 has native support for 
WatiN, NUnit and QUnit and has access/control to all of .NET's WinForms/WPF)</li><li>creating cached versions of the site (controlled by a built-inside-O2 web proxy), <br></li><li>direcly invoke/compile specific parts/components of the application (this is used to create targeted Unit Tests & fuzzing), </li>

<li>running consolidated (i.e. all available) NUnit tests using NUnit's GUI, command line and O2 scripts</li><li>dealing with complex webservices <br>
</li><li>view, analyse and test the server side RoleBase Authorization mappings (created using .NET Attributes) which affects the exposed WebServices <br><br></li></ul><li>how the APIs and tools created by O2 purely as 'developer aids' (i.e. not for security) are then massively important, useful and usable on the UnitTesting phase.</li>

</li><li><b>AppSensor  </b>- to allow TeamMentor to modify its behaviour depending on its current 'attack' level</li><li><b>OpenSAMM </b>- create a score card  </li><li><b>ZAP Proxy </b>- feed the existing O2 Browser automation scripts via ZAP's proxy and fire up its tests</li>
