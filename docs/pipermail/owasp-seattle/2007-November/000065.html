Someone mentioned at tonight's meeting that synergy does clipboard 
transferring similar to the application that Tom showed in his demo.

I found this on the synergy site 
(http://synergy2.sourceforge.net/security.html)

"Synergy does not do any authentication or encryption. Any computer can 
connect to the synergy server if it provides a screen name known to the 
server, and all data is transferred between the server and the clients 
unencrypted which means that anyone can, say, extract the key presses 
used to type a password. Therefore, synergy should not be used on 
untrusted networks."

It goes on to suggest port forwarding synergy traffic through ssh as a 
means to encrypt the traffic. Instructions are at the link above.
