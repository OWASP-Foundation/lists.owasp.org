The OWASP community at large has been very busy on a variety of open
source security related projects. Wanted to highlight a few of my
personal favorites:

 

1.       The OWASP Secure Coding Practices Quick Reference Guide: 
http://www.owasp.org/index.php/OWASP_Secure_Coding_Practices_-_Quick_Ref
erence_Guide

2.       The OWASP AppSensor Project: 
http://www.owasp.org/index.php/Category:OWASP_AppSensor_Project This
project is useful if you want to understand how to make your apps defend
against automated attacks.

3.       The OWASP Alchemist Project: 
http://www.owasp.org/index.php/OWASP_Alchemist_Project This project
enables a software development team in the realization of highly secure
and defensible application with built-in defense/controls against
security-related design, coding and implementation flaws.

4.       The OWASP Secure Web Application Framework Manifesto: 
http://www.owasp.org/index.php/OWASP_Secure_Web_Application_Framework_Ma
nifesto This project is a document detailing a specific set of security
requirements for developers of web application frameworks to adhere to
.Some of the things in it may feel familiar J

5.       The OWASP Application Security Skills Assessment: 
http://www.owasp.org/index.php/OWASP_Application_Security_Skills_Assessm
ent This is an online multiple-choice quiz built to help individuals
understand their strengths and weaknesses in specific application
security skills. My personal interest in this effort is to encourage
PCI/DSS auditors to find a simple way of telling when enterprises who
need to comply to PCI/DSS aren't fudging the truth and have people who
know what they are doing J

 

I encourage everyone to check out these projects and see if they add
value to your organization. If your organization finds value in OWASP,
maybe it will also consider becoming a member (
http://www.owasp.org/index.php/Membership). Please also consider
forwarding the above information to those who could benefit. 

 

I thank you in advance for doing your part to help make web application
security "visible"...
