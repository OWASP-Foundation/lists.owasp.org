<DIV><BR><SPAN style="BORDER-BOTTOM: #0066cc 1px dashed; BACKGROUND: none transparent scroll repeat 0% 0%; CURSOR: hand" id=lw_1252760468_11 class=yshortcuts>10:15 – 11:45</SPAN> “The Top 25 <SPAN id=lw_1252760468_12 class=yshortcuts>Software Security Vulnerabilities</SPAN>” by Chris Wysopal from Veracode. In this presentation, Chris takes the QA test perspective on finding the top 25 application <SPAN style="BORDER-BOTTOM: #0066cc 1px dashed; CURSOR: hand" id=lw_1252760468_13 class=yshortcuts>security vulnerabilities</SPAN> that are most likely to be exploited by attackers by combining static, dynamic and <SPAN id=lw_1252760468_14 class=yshortcuts>manual test techniques</SPAN>. By putting security vulnerabilities into context with other security issues, he will show what testing techniques are best suited for each category. The strengths and weaknesses of automated static and dynamic analysis as well as manual methods are examined. </DIV>

 security defects through dynamic and <SPAN style="BORDER-BOTTOM: #0066cc 1px dashed; CURSOR: hand" id=lw_1252760468_18 class=yshortcuts>static analysis</SPAN>. <BR></DIV>
