Hi folks,

I have been doing quite a bit of hacking on the WebGoat codebase, and I 
would appreciate testers taking it for a spin, and letting me know of 
any problems.

You can check what I have changed at 
http://dawes.za.net/gitweb.cgi?p=rogan/webgoat/webgoat.git;a=shortlog;h=master

You can get a copy of the repo at 
http://dawes.za.net/rogan/webgoat/webgoat-src-current.zip

Just create an eclipse workspace as described in " webgoat/main/HOW TO 
create the WebGoat workspace.txt".

At this stage, I would not advise using the ant build script to 
construct a .war file, as the build script has not been updated to cater 
for the per-user databases yet (see below).

Major changes from the previously published version:

* WebGoat now uses the HSQLDB database engine by default on all 
platforms. It constructs an in-memory database for each user, which is 
discarded when the server is restarted. This may cause problems for 
sites which have large classes all accessing a central webgoat server. 
Testing and feedback in this area is appreciated.

* Multi-stage lessons (primarily the labs) have been updated to allow 
direct access to each stage, even if the previous stages have not yet 
been completed.

* New Oracle-based lessons have been added, to demonstrate SQL Injection 
inside stored procedures, as well as data validation (Cross Site 
Scripting lesson). These lessons obviously depend on WebGoat being 
backed by an Oracle database. To experiment with this, edit web.xml to 
use the oracle database driver (oracle.jdbc.OracleDriver), update your 
connection string, and add DatabaseUser and DatabasePassword attributes. 
Also see WEB-INF/webgoat_oracle.sql. Note that the DatabaseUser 
parameter is actually a prefix. The user that webgoat will attempt to 
log in as will be ${DatabaseUser} + "_" + ${user}. e.g. webgoat_guest

Other changes were basically major clean ups and refactorings, which 
should not affect how WebGoat works. Any regressions will be eliminated 
on receipt of a bug report.

Happy testing!

Regards,

Rogan
