Trying to make WebGoat 5 HTTP splitting work gave me loads of trouble.
I got as far as using the hints, injecting the answer they flat out gave
me into the outgoing request, the HTTP header in the second request, and
the form itself, and it didn't help.  Maybe I'm doing it wrong, or maybe
it's busted.

WebGoat 5 also breaks horridly on Windows if you drop the WAR into
Tomcat 5.5 as anything but Webgoat (again, the HTTP Splitting lesson
doesn't work-- it flat out can't find the JSP file to submit to, because
it tries to go to WebGoat/ instead of WebGoat-5/).  Displaying
parameters and hints doesn't seem to work in that setting, so if you run
WebGoat 3-5 on one server for some reason no go.

WebGoat WAR on RedHat5/CentOS breaks for a different reason:  RedHat
messed up packaging Tomcat.  Point fingers at them.

Looking forward to WebGoat 6.  If you go through an RC cycle with the
Windows and Linux WAR files, I have environments to test both and it's
in my interest to point out bugs.  ;)
