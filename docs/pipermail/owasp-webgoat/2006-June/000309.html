An interesting "bug" is, when you do the following sequence of URL -
http://127.0.0.1/WebGoat (with any account) ... code 404,
http://127.0.0.1/WebGoat/main.jsp (with 500 - internal error) and
http://127.0.0.1/WebGoat/webgoat.jsp (you are in). (it's "discovered"
just by chance, because I firstly overlooked getting started.)  I think
that just http://127.0.0.1/WebGoat is more obvious and the
"getting_started" should be more visible. 


2) Documentation says "application running on default port 8080" - but
in fact it's just 80. I'm speaking about windows WebGoat 4.0 release run
and play release, not the UNIX one.

3) general tests - little troubles with "timing", but the java source
