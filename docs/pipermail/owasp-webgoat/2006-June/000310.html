Hi Kamal

Thank you for your feedback! You are correct, the main README file is 
missing from the Windows_WebGoat-4.0_*.zip distributions.  I will add the 
README to the distribution lists as soon as possible.  This is a HUGE 
oversight on my part.

There are some simple instructions for adding WebGoat user accounts to 
Tomcat  in the Standalone distribution.   The developer version has a "HOW 
TO create the WebGoat workspace.txt" which includes some basic instructions 
for setting up the IDE and running WebGoat but all the releases are missing 
the basic README file.

We are actively working on the WebGoat user guide 
(http://www.owasp.org/index.php/WebGoat_User_Guide_Table_of_Contents)  and 
hope to have that completed soon.  WebGoat is a community effort and the 
user guide work is often the hardest to complete.  We certainly would 
appreciate your help in updating this guide.


The lessons that specify "This lesson needs a creator" are lessons that I 
would like to get built but haven't had the time.  It would also be nice to 
have a forced browsing lesson :)

Regarding the basic authentication lesson... After you have entered your new 
credentials, the OLD JSESSIONID is sent with your NEW request which contains 
your NEW credentials.  This OLD JESSIONID is still a valid session id and 
therefore your new credentials are ignored.  You have to delete the OLD 
JSESSIONID when the request is sent with your new credentials...

I will work on improving the LAB instructions.


Bruce
