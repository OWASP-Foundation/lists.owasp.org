deepak louis wrote:
> Webgoat seems to be dependent on TomCat. Any pointers to making WebGoat 
> application server independent would be much appreciated. If anyone who 
> knows the code, please let us know how to change the code to make it 
> work on other application servers.
> Thanks
> Deepak Louis

There should be no inherent dependency on tomcat. Although it is 
"required" to build the war, this is only so we can have access to the 
servlet API jars.

Once the WAR is constructed, you *should* be able to deploy it to any 
application server. If this doesn't work, please let us know (with 
details), and we'll make a plan to fix it.
