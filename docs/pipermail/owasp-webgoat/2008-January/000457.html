I agree with Brian - I think the true solution is just better 
documentation. Explain what the prerequisites are.

Installing a JDK and setting JAVA_HOME has *nothing* to do with building 
applications. That is just general computer literacy, expected of even 
the most newbie developer.

*Contrast*:

The WebGoat developer edition requires a 1.5 or later JDK to be 
installed, and the JAVA_HOME environment variable set so that WebGoat 
can find where you have installed the JDK.

You can get the JDK for your platform from 
http://java.sun.com/<wherever>. Once it is installed, either:

(Windows) Go to "My Computer" -> Properties -> Advanced -> "Environment 
Variables". Look for a variable called JAVA_HOME. If it is set, check 
that it points to the directory that the JDK was installed to,

e.g. c:\program files\java\jdk1.5.0_06

Unix (sh, bash, ksh): JAVA_HOME=~/jdk1.5.0_06 export JAVA_HOME

or edit your login files to ensure that this variable is always set 
whenever you login

Users waiting for a 370MB download, installing/reinstalling that on 
students files, etc, as well as fighting with Google :-)
