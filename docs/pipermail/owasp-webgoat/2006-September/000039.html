Hi ML,

I've noticed a problem about WebGoat for Unix on the Sourceforge site.
During the starting phase I receive the follow error:

[root at wrynux home]# ./webgoat.sh start80
bash: ./webgoat.sh: /bin/sh^M: bad interpreter: No such file or directory

I've received this error because in the WebGoat files there is the ^M
character.

Patch:
You have to use dos2unix program to delete the ^M characters:

[root at wrynux home]# dos2unix webgoat.sh
dos2unix: converting file webgoat.sh to UNIX format ...
[root at wrynux home]# cd tomcat/bin/
[root at wrynux bin]# dos2unix *.sh
dos2unix: converting file catalina.sh to UNIX format ...
dos2unix: converting file digest.sh to UNIX format ...
dos2unix: converting file setclasspath.sh to UNIX format ...
dos2unix: converting file shutdown.sh to UNIX format ...
dos2unix: converting file startup.sh to UNIX format ...
dos2unix: converting file tool-wrapper.sh to UNIX format ...
dos2unix: converting file version.sh to UNIX format ...
