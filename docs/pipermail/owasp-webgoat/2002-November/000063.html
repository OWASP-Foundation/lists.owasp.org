I have a .net re-implementation of the 1.0 goat I call netgoat (see the
included image) that makes it much easier to deploy the goat on Windows
servers.

I've done this mainly to prove that although ASP.NET goes out of its way
to assist you doing things the right way by hiding session management,
supporting forms based auth directly, &c, you can still subvert security
by not paying attention.

The installation process for netgoat goes like this:

* Install the .Net framework (if you haven't already)
* Make a new virtual directory (like webgoat) in iisadmin
* Copy the netgoat content to the vdir location
* Establish the DSN and you're done

Once I start seeing webgoat 2.0 code, I'll bring up netgoat to the same
level of insecurity. :-)
