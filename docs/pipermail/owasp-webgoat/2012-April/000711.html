It appears that Tomcat is not able to listen on port 80, as shown in the line:

java.net.BindException: Permission denied:80

There are two things that could cause this:
1. Something else is already listening on port 80 (such as the Apache
web server)
2. Tomcat is starting as a non-root user.  You need root privileges to
listen on ports below 1024 on Linux.

I'm not sure what you are trying to do with WebGoat, but if you are
just looking to have it running (or even make changes to it), I
recommend you check out the OWASP Broken Web Applications VM
(www.owaspbwa.org).  It is an Ubuntu Linux based VM that includes
WebGoat and a bunch of other application already configured and
running as soon as you boot up the VM. (Full disclosure - I'm the
leader of that project)

Chuck
