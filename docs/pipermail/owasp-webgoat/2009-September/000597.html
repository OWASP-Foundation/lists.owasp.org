 I would need to go verify this in the code but I'm pretty sure in a shared
WebGoat environment user1 cannot see user2 messages. This was added so user1
could not XSS user2 in a classroom environment.  The logged in username is
stored with the database entry for the posted message and checked upon
message retrieval.

This being said, I'm positive there are other "features" in WebGoat that
would allow you to bypass this check but only a WebGoat source code change
would allow the functionality you've stated.
