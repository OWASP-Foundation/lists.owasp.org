If I recall, there is no code in WebGoat itself that does the login /
authentication process.  That is handled by the Application Server
(typically Tomcat for WebGoat).

If you are asking where you can set up additional users and assign
roles for WebGoat, then again is handled by the application server.
For Tomcat, that is the tomcat-users.xml file.
