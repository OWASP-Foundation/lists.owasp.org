Hello WebGoaters !

I am very pleased to announce that, after collaborative efforts from  community and amazing work from Nanne Baars, a "repository organization" improvement request that was proposed and approved on the last "Core Contributors" conference call has now been officially committed to Github's Unstable Repository / Master Branch [1] .

The changed implemented consisted into moving the "src" directory one-level down, transforming WebGoat into a multi-level maven project.

This was proposed and implemented in order to support the effort of decoupling the Core / Container / Classloader / Lessons modules of WebGoat, which  will allow easier development and improvement of the different modules of WebGoat independently. Additionally, this will enable us to start publishing WebGoat components to the Maven Central repository, once WebGoat 6.1 reaches a stable point.

The new directory structure is as follows:

WebGoat/
                - webgoat-container
                - webgoat-classloader
                - webgoat-release

For code contributors -> **** You must perform a git pull (or rebase) before you continue working on the source code ****

Running Webgoat:
mvn clean install
                mvn -pl webgoat-container tomcat7:run

Nanne was also kind enough to integrate with the Travis CI [2] service which is available on Github. This will ensure that after a check-in, a build is run, so before a pull request is merged, we could know if the build is stable.

Over the next few weeks, we will be improving the Build Process (and deployable artifact creation), the Wiki Documentation and the Jira Ticketing System ( for sprint planning and defect/enhancement requests tracking).

For those who would like to contribute (with coding, documentation or any other task), please feel free to contact us.
For all those who already contributed to the project, THANK YOU.  Major props to nbaars | misfir3 | mayhew64 | jwayman .
