Mitchell,

Here's the run down of what I have been doing (including list in case
others have input).  For background, I develop/test with
a stand-alone tomcat on Linux. Also, our build is a bit of a moving
target right now (which we are working to improve). I'm including the
list in case someone there wants to chime in or can benefit. Also cc'ing
Nanne as he can correct me if I mis-step below.

0) cd into the /webgoat-classloader and mvn clean install
1) mvn clean package << in webgoat 'container'* directory
2) mvn install << I think you can optionally just run 'mvn clean
install' and it should satisfy both steps here, but I often end up
running them discreetly
3) You should now be able to cd into the lessons project and
$ mvn clean package
4) You should be able to copy lesson jars into the webapp ...
.e.g.
$cp xpath-injection/target/xpath-injection-1.0.jar
$TOMCAT_HOME/webapps/$WEBGOAT_DIR/plugin_lessons/
5) I find that when running tomcat stand-alone as I do, I have to create
the $TOMCAT_HOME/webapps/$WEBGOAT_DIR/plugin_extracted directory
manually anytime I fully redeploy the 'container' application.

* The idea of 'container' and 'core' etc. is going to shuffle a little
(part of the effort to improve build process). There should be an email
coming to the list soon about that.

HTH,
Jason
