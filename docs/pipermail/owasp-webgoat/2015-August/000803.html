OK, the migration is done (see Doug's earlier email ... and Kudos again
to Nanne). I updated and just made one more pull/merge with the Cookies
Table/view re-enabled. Params, integrating this LessonInfo Service, then
some more menu work and some testing and clean-up.

As to issue management, we have been using JIRA (webgoat.atlassian.net,
will need to create you an account there). I have been very bad at it
myself as I've gotten used to just jumping in and handling most stuff
hands-on.  When I look at the decision between coding and working in
JIRA, I head for the code (for better or worse).  I'm asking the core
group what they think of moving issue management to github (and would be
curious of what new folks signing on think in that regard).

The one item that I can use help with currently that I have not put up
as a ticket yet is to:
Create a (we'll call it for now) LessonInfo Service to return basic info
about the lesson. Some would be redundant now, but could enable more
efficient (lazy) loading in the future. However, there is also a real
need for pulling the lesson title with the client-side routing via
Backbone now.

For starters (and this may be all we need) I'd like to create that
service which returns:

{
  lessonTitle:_TITLE_,
  numberHints:_#_,
  hasSource: T||F,
  hasSolution: T||F,
  hasPlan: T||F
}

Look at the HintService as a starting point

You'll want to fork this:
https://github.com/WebGoat/WebGoat
... and issue pull requests when you have a change. We can go into
minutia  off-list, but I'm posting this reply on-list as I think there's
some benefit to others.

Cheers,
Jason
