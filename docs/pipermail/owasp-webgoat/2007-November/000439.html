OWASP WebGoat maintainers,

There may be a fairly significant error in the WebGoat readme.txt.  Under INSTALLATION / Windows, the Note reads:

When intercepting request with IE7.  You must add a '.' to the end of localhost.  i.e. 
          http://localhost./WebGoat/attack           or
          http://localhost.8080/WebGoat/attack    if using a non standard port

I suggest that in all instances, that syntax should require a colon ':' rather than a period '.'.  Using a period fails to work in both IE6 and Firefox.  Users will be frustrated since WebGoat is unreachable using these instructions.  Please correct the instructions as indicated.
