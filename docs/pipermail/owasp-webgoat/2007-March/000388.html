The labs are intended to be difficult.  If you were examining a true application that had a broken access control mechanism you would need to consider some of the items below.  The hints in the lab are intended to guide you in a direction of thought.  It is difficult to draw the line between "solving" the lesson and getting the analyst to think about how the application should work and therefore how to exploit a potential flaw.

Could anybody pass this lesson, using these hints??

Many sites attempt to restrict access to resources by role.

Developers frequently make mistakes implementing this scheme.

Attempt combinations of users, roles, and resources.

How does the application know that the user selected the delete function?

How does the application know that the user selected any particular employee to view?

Note that the contents of the staff listing change depending on who is logged in.
