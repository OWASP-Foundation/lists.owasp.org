Hi there,

Thanks for pointing this out.

We are aware of the majority of the issues and are currently building
them into a new release which will be released by next week Monday.

Herewith are some comments on the points which have been raised.

> 1. Acquiring the Google API key: You will need a key from Google to
> have access to the Google API. You are limited 1000 requests a day.
> ...<SNIP>...

Google's API policy has changed in that new API keys have no longer been
issued as from December 5th, 2006.  Although we have not made a decision
as to the search API we will be using to replace the Google API, the new
version of Wikto will include a workaround / stop-gap method to provide
users without Google API keys the same functionality.  More details will
be made available as and when we release the new version.

> 2. HTTrack and HTTPrint  need to be downloaded from their respective
> locations and installed. You will need to configure Wikto to where it
> will use the executable.

Although HTTPrint and HTTrack are great products, the new version of
Wikto will no longer make use of these external programs (ie: it will no
longer be necessary to download and install them first).

Wikto will be fully functional after installation without the need for
downloading and installing any additional applications.

> 3. The entries in "Update sites" of Wikto section need to be changed
> since the  DB's and schema's don't exist in those locations anymore. 
> 
> a. With the latest version of Wikto (v1.63.2279.18538), the Nikto DB
> updated is pointing to
> http://www.cirt.net/nikto/UPDATES/1.34/scan_database.db this is
> INCORRECT and needs to me changed to
> http://www.cirt.net/nikto/UPDATES/1.36/scan_database.db . So do that
> if you want to get the latest Nikto Db 

The new version of Wikto uses a slightly modified Nikto db which reduces
the number of false-positives determined significantly whilst still
retaining compatibility with native Nikto databases.

Following the release of the new Wikto version, our modified Nikto
database updates will also be available from
www.sensepost.com/research/wikto/DB/nikto.db, and the application will
use this as it's default update path.

> b. The location of the GoogleHack DB is also incorrect. Actually, the
> schema.xml file does not exist anymore in XML format. It is kept
> ...<SNIP>...

Thanks for bringing this to our attention - we will ensure that this is
updated in the new version.

> c. The BackEnd DB updates are not available from SensePost anymore !
> Can anymore assist, SensePost do you still update teh BackEnd DB files
> and where can we obtain them from ? Or does anymore else have a recent
> copy of the updated DB ?

We still update the back-end databases and they are still available from
SensePost.  This is the first notice we have received that the updates
do not work.

I've tested the back-end updates on the current version of Wikto without
any problems.

Please note that in order to update the back-end from within Wikto,
please click the "Update From SensePost" button first, after which a
number of options will appear in the drop-down list.  Select the update
option you require (ie: Full / Quick etc) and click the "Update From
SensePost" button again.

Regards,

Ian
