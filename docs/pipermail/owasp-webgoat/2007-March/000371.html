> Hi,
>
> First off, in your example, you cannot be sure that 11617-XXX is the
> start of the other user's session ID. Note that your two possibilities
> are ..16 and ..19. The odds are good that you received one of ..17 or
> ..18 out of order, so just make sure that you are working on the right
> one.
>
> Next, there is a script in the ./scripts directory of a standard
> installation, that can be used to brute force the time component. It is
> called "webgoat-session-brute.bsh". If you are not using the -installer
> version of WebScarab, you can see the script online at:
>
> <
> http://dawes.za.net/gitweb.cgi?p=dawes.za.net/rogan/webscarab/webscarab.git;a=blob;f=scripts/webgoat-session-brute.bsh;h=8970bc5f5c162ac56ce71a793fa3845064646697;hb=HEAD
> >
>
> or (without the line numbers):
