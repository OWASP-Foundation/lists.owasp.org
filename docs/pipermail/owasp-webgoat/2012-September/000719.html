Did you install WebGoat yourself or use someone else's packaged setup? I
ask because I noticed that with a fresh SVN checkout, the exercise went
just fine, but a packaged version did not respond to the same malformed
input. If you're using Burp, try sending each of the POSTs you see to
Intruder and then using Sniper and the Character Block payload option to
throw LOOOOOONG (think multiples of 1024) sequences of characters at the
different form fields. One of the POSTs/fields makes the backend not happy
when you do that.

Regarding the Java question, the Webgoat code is designed to simulate a
vulnerable environment and I believe for this lesson you can pretend that
the back-end is something more insecure than it really is were it
"real-world".
