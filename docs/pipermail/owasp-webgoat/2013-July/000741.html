Hi Anirudh

What is the error you are getting from the eclipse import of the pom file?

If you have a .classpath, .project, or settings file/folder in your WebGoat
directory, you will need to delete those before the maven import.

Another thing to try to make sure the tree is synched correctly is to
issue: "mvn compile"  from a shell window in the webgoat-6.0 root.

Bruce
