There are multiple ways to build and run WebGoat.  I will cover some basics
here.  Enough to get people started.

1) Checkout the source code for google code:

svn checkout http://webgoat.googlecode.com/svn/branches/webgoat-6.0

Note: you will need https and a gmail account to check code in.  You will
also need to be a contributor to the WebGoat project.  Send me your gmail
address if you want to be a contributor.  See:
https://code.google.com/p/webgoat/source/checkout for checkin account info

2) Start eclipse (you will need to make sure m2e is installed.  m2e comes
with eclipse post 3.6 versions)

In Eclipse, from the File menu: File -> Import...

>From the import dialog's filter text box type in: maven
Choose: Existing Maven Projects
Browse to the webgoat-6.0 checkout folder
/pom.xml WebGoat:WebGoat:6.0:war   should be selected, select this pom file
if it is not
Click Finish

3) You can build from eclipse or from the command line.

>From the command line, cd to the webgoat-6.0 folder, there will be a
pom.xml file in this folder.
To Build: mvn compile

4) To run from the command line:
mvn tomcat:run-war

5) Browse to:  http://localhost:8080/WebGoat/attack


All of this can be done with eclipse as well, but I would need to add
pictures on how to set it up.  I will do that in the wiki in a few days.
 Or someone else can volunteer to create a wiki page on how to do that.

As always, let me know if there are questions.
