Hi Anirudh (and to all the other requests I've received)

WebGoat 6 is the newest branch and resides at:
https://code.google.com/p/webgoat/source/browse/#svn%2Fbranches%2Fwebgoat-6.0

This is the latest port of WebGoat to the SpringMVC framework.  The only
lesson that has been ported is HttpBasics, but the foundation for the port
has been laid.

There are several major things that are needed.

1) Redesign of the presentation layer.  Currently WebGoat uses a
combination of JSPs with apache ECS (Element Construction Set) for the
presentation layer.  The whole presentation layer needs to be moved to a
new technology that preferably is well known and has some demonstrable
weaknesses.

2) Port of all lessons to SpringMVC - this is a fairly trivial task.  If
the lesson seems outdated or no longer works we need to mark them for
deletion or rewrite.

3) Redesign of the session management and lesson architecture.  Not sure
how much work is needed here, when I went to port the SQL Injection
lessons, I realized some major refactoring was needed and I stopped.  We
can still port the lessons and do the refactoring later.

4) All the other items that are needed for a complete rewrite....


If you synch to the WebGoat 6.0 branch, you can import the project to
eclipse as a maven project and everything just works.  Let me know if you
need any help with this.

I really need someone to step up and start porting lessons.  Don't worry
about mistakes, we can get them fixed and learn as we go!

Bruce
