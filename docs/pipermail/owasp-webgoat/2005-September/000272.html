You can:

1) add a new user in <tomcat_root>/conf/tomcat-users.xml to specify a new user.  WebGoat will track all users individually.
2) delete the property files in <tomcat_root>/webapps/WebGoat/users/*.props

If you log into WebGoat as webgoat/webgoat you should see some multi-user admin support in the summary report card.  I believe I turned the multi-user feature on in the WebGoat 3.7 release
