After you force the browser to reauthenticate you, the page request is 
issued by the browser to the client.  Notice that the old session id is 
passed with the new request.  Why did this happen?  Think about the how and 
when cookies are sent to the server and the order of events that occur on 
the server side to ensure the client is authenticated.   Are you SURE you 
were authenticated as "basic"? Perhaps your authorization token was 
basic:basic but you are still authenticated as "guest" :)
