<DIV>&lt;disclaimer&gt;I wrote my first .NET application yesterday and liked it 
;-) &lt;/disclaimer&gt;</DIV>
<DIV>&nbsp;</DIV>
<DIV>I'm not sure I see the sense of having two versions.&nbsp; Why does it 
matter what language it is implemented in underneath?&nbsp; Do you think more 
people will use it if it's in .NET?&nbsp; I believe that ANY web security issue 
we want to teach people about can be easily implemented in the current 
framework.</DIV>
<DIV>&nbsp;</DIV>
<DIV>I also think that translating to .NET won't be trivial, since there are 
some complex features in WebGoat to enable plug-in lessons.&nbsp;Unless there's 
a technical reason to move to .NET, I suggest we make what we've got better 
instead of spending time switching horses midstream.</DIV>
<DIV>&nbsp;</DIV>
<DIV>Fundamentally, I really don't particularly care whether WebGoat is built on 
J2EE or .NET.&nbsp; Some things are going to have to be "simulated" either way 
(but the SQL injection *is* real).&nbsp; If there's a particular .NET flaw that 
you're interested in showing, could you describe it?&nbsp; My guess is that 
it'll be easy to implement in the current framework.</DIV>
