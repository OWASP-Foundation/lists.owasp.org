  <DIV><FONT size=2>Here's a very short cheat sheet.&nbsp; I'm not sure now 
  whether we should make this part of the distribution or not.&nbsp; 
  Thoughts?</FONT></DIV>
  <OL>
    <LI><FONT size=2>Http Basics -- use this one to demonstrate that HTTP is 
    stateless, request-reply protocol. Click show parameters to show what's 
    going with each request.</FONT> 
    <LI><FONT size=2>Fail Open Authentication -- try removing the password 
    parameter entirely using Achilles.&nbsp; Or you can do it on the URL with 
    (<A 
    href="http://localhost:8080/WebGoat/attack?Username=jeff">http://localhost:8080/WebGoat/attack?Username=jeff</A>) 
    -- when there's no password, an exception gets thrown and the authentication 
    step gets skipped.</FONT> 
    <LI><FONT size=2>HTML Clues -- view HTML and search for a comment with 
    admin:adminpw -- use this to authenticate</FONT> 
    <LI><FONT size=2>Parameter Injection --&nbsp;show how shell calls can be 
    used to run arbitrary code -- use the &amp; character to chain commands on 
    windows -- blah.txt &amp; netstat -a &amp; ipconfig</FONT> 
    <LI><FONT size=2>Unchecked Email -- enter a message that contains linked 
    offensive content -- Hi, check this out....&lt;A href="<A 
    href='http://www.google.com">Click'>http://www.google.com"&gt;Click</A> here 
    for Google"&lt;/A&gt; and it looks like it came from the website.</FONT> 
    <LI><FONT size=2>SQL Injection -- 101' or 1=1 to get the whole contents of 
    the db</FONT> 
    <LI><FONT size=2>Thread Safety -- this one's tricky -- use 'new window' to 
    get two browsers.&nbsp; Enter 'dave' in one window and 'jeff' in the 
    other.&nbsp; Then set it up so you click on the submit boxes as fast as 
    possible.&nbsp; If you do it fast enough, the application gets confused and 
    returns the second clicked account to both browsers.&nbsp; This is a 
    massively underreported problem in many many web apps.</FONT> 
    <LI><FONT size=2>Weak Authentication Cookie -- use achilles to add 
    'AuthCookie=ggfkggfk;' to the Cookie: header using Achilles. Note that the 
    cookie is username:password (jeff:jeff) backwords and one-letter up.</FONT> 
    <LI><FONT size=2>Database XSS -- enter &lt;script language="javascript" 
    type="text/javascript"&gt;alert("Ha Ha Ha");&lt;/script&gt; in the message 
    and then click on the name of the message below to show that anyone who 
    reads the message is attacked</FONT> 
    <LI><FONT size=2>Hidden Field Tampering -- there are a number of ways to 
    edit a hidden field.&nbsp; I like IEBooster, but you can use Achilles, or 
    even try to modify it in the URL (e.g. <A 
    href="http://localhost:8080/WebGoat/attack?Price=9.99">http://localhost:8080/WebGoat/attack?Price=9.99</A>).&nbsp; 
    Change the price of the TV from 4999.99 to 9.99.&nbsp; </FONT>
    <LI><FONT size=2>Weak Access Control -- this page is supposed to let you 
    access files in&nbsp;a certain directory.&nbsp; But if you use 
    ../topsecret.txt you can get other files.&nbsp; Would be nice to enhance 
    this with some url, hex, unicode encoding attacks.</FONT> 
    <LI><FONT size=2>Challenge 1 -- the challenge is tricky.&nbsp; To get past 
    the first page, you have to remember how to turn on showing of the java 
    code.&nbsp;Check the lesson plan link to find out that the lesson plan is 
    called ChallengeScreen.html.&nbsp; Then force browse to 
    /WebGoat/src/lessons/ChallengeScreen.java.&nbsp; Find the username and 
    password (youaretheweakestlink:goodbye).&nbsp; Note: there is another option 
    -- if you sniff the network, you'll see a UDP packet with the username and 
    password in the clear.&nbsp; Cool huh?</FONT> 
    <LI><FONT size=2>Challenge 2 -- you'll need to modify the 's' hidden field 
    to contain 101' or 1=1 (like the database lesson).&nbsp;Use Achilles or 
    IEBooster.&nbsp; This will give you all the passwords.</FONT> 
    <LI><FONT size=2>Challenge 3 -- you'll need to know a bit about Server Side 
    Includes here -- syntax is <FONT size=3>&lt;!--#exec tag1="dir" --&gt;&nbsp; 
    So you'll want to do something like &lt;!--#exec tag1="echo defaced &gt; 
    [PATH]\index.html" --&gt;.&nbsp; This will overwrite the index.html with 
    your content and win the challenge.</FONT></FONT></LI></OL>
