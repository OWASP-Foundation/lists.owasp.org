<DIV dir=ltr><FONT face=Arial size=2>Welcome aboard!&nbsp; The WebGoat Financials is slightly more complicated than the standard lesson architecture. I would recommend synching to the baseline at <A href="http://code.google.com/p/webgoat/">http://code.google.com/p/webgoat/</A>&nbsp;and following the instructions there&nbsp;for building the WebGoat environment.</FONT></DIV>
<DIV dir=ltr><FONT face=Arial size=2></FONT>&nbsp;</DIV>
<DIV dir=ltr><FONT face=Arial size=2>To build a simple lesson follow the instructions in the "New Lessons" category.&nbsp;&nbsp; There are no instructions on how to build a WebGoat Financials lesson. The basic control flow is:</FONT></DIV>
<DIV dir=ltr><FONT face=Arial size=2></FONT>&nbsp;</DIV>
<DIV dir=ltr><FONT face=Arial size=2>*** WebGoat Financials Control Flow ****</FONT></DIV>
<DIV dir=ltr><FONT face=Arial size=2></FONT>&nbsp;</DIV>
<DIV dir=ltr><FONT face=Arial size=2>Browser ----(HTTP Request)----&gt; Controller (HammerHead.java) ----(Delegates)----&gt;Action Handlers</FONT></DIV>
<DIV dir=ltr><FONT face=Arial size=2></FONT>&nbsp;</DIV>
<DIV dir=ltr><FONT face=Arial size=2>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Each lesson has a main delegate that acts like a Controller for that lesson's actions.&nbsp; </FONT></DIV>
<DIV dir=ltr><FONT face=Arial size=2>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; This controller will then delegate to all other actions in the current lesson.</FONT></DIV>
<DIV dir=ltr><FONT face=Arial size=2>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Action Handlers perform business logic and write data to WebSession</FONT></DIV>
<DIV dir=ltr><FONT face=Arial size=2>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Action Handlers interact with database (no official data layer)</FONT></DIV>
<DIV dir=ltr><FONT face=Arial size=2></FONT>&nbsp;</DIV>
<DIV dir=ltr><FONT face=Arial size=2>Controller(HammerHead.java) ----(forwards)----&gt; View(main.jsp) ----(HTTP Response)----&gt; Browser</FONT></DIV>
<DIV dir=ltr><FONT face=Arial size=2></FONT>&nbsp;</DIV>
<DIV dir=ltr><FONT face=Arial size=2>The main.jsp servlet is a nested JSP that has a lesson specific content area and will ask each lesson for lesson specific content via the AbstractLesson.java class.</FONT></DIV>
<DIV dir=ltr><FONT face=Arial size=2></FONT>&nbsp;</DIV>
<DIV dir=ltr><FONT face=Arial size=2>I will try to get some pictures in the WebGoat documentation for the 5.0 release.</FONT></DIV>
<DIV dir=ltr><FONT face=Arial size=2></FONT>&nbsp;</DIV>
<DIV dir=ltr><FONT face=Arial size=2></FONT>&nbsp;</DIV>
<DIV dir=ltr><FONT face=Arial size=2>*** WebGoat Database ***</FONT></DIV>
<DIV dir=ltr><FONT face=Arial size=2></FONT>&nbsp;</DIV>
<DIV dir=ltr><FONT face=Arial size=2>The data model structure can be found in org.owasp.webgoat.session.CreateDB.java which can be explored after running WebGoat.&nbsp; After running WebGoat look in &lt;deployed_root&gt;\webapps\WebGoat\database\webgoat.mdb.&nbsp; If you are running from your Eclipse workspace it will be located in: &lt;Eclipse_workspace_name&gt;\.metadata\.plugins\org.eclipse.wst.server.core\tmp0\webapps\WebGoat\database</FONT></DIV>
