<DIV dir=ltr><FONT face=Arial size=2>I've received a barrage of requests lately on&nbsp;how to configure WebGoat to be addressable by the host machine IP instead of localhost.&nbsp; This is a Tomcat configuration that is being overwritten every time WebGoat (i.e. Tomcat) is started.</FONT></DIV></DIV>
<DIV>
<P><FONT face=Arial size=2>In the webgoat.bat file, in the root directory, the following lines are executed:</FONT> </P>
<P><FONT face=Arial size=2>&nbsp;&nbsp;&nbsp; delete .\tomcat\conf\server.xml</FONT> <BR><FONT face=Arial size=2>&nbsp;&nbsp;&nbsp; copy .\tomcat\conf\server_80.xml .\tomcat\conf\server.xml</FONT> </P>
<P><FONT face=Arial size=2>This will overwrite any changes you may have made&nbsp;to server.xml file that addressed this issue....</FONT> </P>
<P><FONT face=Arial size=2>By changing the server_80.xml file (or by removing the above code from webgoat.bat, after making your changes)&nbsp; you can reflect your changes to&nbsp;the Tomcat configuration.&nbsp;&nbsp; You will need to change the IP address in the server_80.xml file to be the IP of the host machine.&nbsp; The following connectors should be modified</FONT></P>
<P><FONT face=Arial size=2>&nbsp;&nbsp;&nbsp; &lt;!-- Define a non-SSL HTTP/1.1 Connector on port 8080 --&gt;</FONT> <BR><FONT face=Arial size=2>&nbsp;&nbsp;&nbsp; &lt;Connector address="10.20.20.123" port="80"</FONT> <BR><FONT face=Arial size=2>...</FONT> </P><BR>
<P><FONT face=Arial size=2>&nbsp;&nbsp;&nbsp; &lt;!-- Define a SSL HTTP/1.1 Connector on port 8443 --&gt;</FONT> <BR><FONT face=Arial size=2>&nbsp;&nbsp;&nbsp; &lt;Connector address="10.20.20.123" port="443" maxHttpHeaderSize="8192"</FONT> <BR><FONT face=Arial size=2>.... </FONT></P>
<P><FONT face=Arial size=2>where the 127.0.0.1 will be replaced by your IP.&nbsp; In this case 10.20.20.123</FONT></P>
