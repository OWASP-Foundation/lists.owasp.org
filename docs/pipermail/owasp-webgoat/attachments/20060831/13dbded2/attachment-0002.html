                <p><b>OWASP Autumn Of Code 2006</b></p><p><b>London, United Kingdom, August 31, 2006</b></p><p>The Open Web Application Security Project (OWASP) is launching
today a new project aimed at financially sponsoring contributions to OWASP Projects.
</p><p>The new project, called "OWASP Autumn of Code 2006", aims to
sponsor 8 individuals (4 at $3,500 USD and 4 at $5,000 USD) to work on
existing OWASP Projects. The objective is to improve the selected
projects to a the level of completeness and professionalism required
for its wide use and deployment.
</p><p>There are no geographical, age or any other form of restrictions
of who can apply for an "OWASP Autumn of Code 2006" sponsorship. The
only requirement is that the candidate shows the potential to
accomplish the project's objectives and the commitment to dedicate the
time required to complete it in the allocated time frame (projects must
be completed by 31st December 2006).
</p><p>Prospective candidates should visit the page <a href="https://www.owasp.org/index.php/Owasp_Autumn_Of_Code_2006" class="external free" title="https://www.owasp.org/index.php/Owasp_Autumn_Of_Code_2006" rel="nofollow">https://www.owasp.org/index.php/Owasp_Autumn_Of_Code_2006</a> for project ideas, rules and information on how to apply.
</p><p><b>Project Schedule:</b></p><ol><li> 31th August - 'OWASP Autumn of Code' initiative is officially launched 
</li><li> 18th September - Deadline for project proposals
</li><li> 25th September - Publish of selected projects
</li><li> 1st October - Project starts 
</li><li> 15th October - Update of Project status on OWASP Conference in Seattle
</li><li> 15th November - Participants and to report on project status 
</li><li> 31st December - Project Completion 
</li></ol><p>The "OWASP Autumn of Code 2006" project leader is Dinis Cruz (based in London, UK) who can be contacted for further details.
</p><p><b>About OWASP</b></p><p>The Open Web Application Security Project (OWASP) is 501c3
not-for-profit foundation dedicated to enabling organizations to
develop, purchase, and maintain applications that can be trusted.
OWASP's open source projects and local chapters produce free, unbiased,
open-source documentation and tools. The OWASP community
also facilitates conferences, local chapters, papers, presentations,
and mailing lists. More information can be found at www.owasp.org.
