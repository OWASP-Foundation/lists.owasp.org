Hi Stephanie,

The installer is trying to download an obsolete version of Tomcat.  

I modified the install.props file that controls where the download 
occurs from.  I replaced  the "comp.0.fileset=" properties line with:
comp.0.fileset=http://ftp.epix.net/apache/jakarta/tomcat-4/v4.1.29/bin/jakarta-tomcat-4.1.29.exe

*NOTE* Webgoat has not been tested with this version of Tomcat.



1) Save the attached file into a directory called "installer".  
    The folder MUST be called "installer".  

2) Open the "install_WebGoat-2.0_windows.jar" using WinZip.  
    Drag the installer folder ( which contains install.props ) and 
    drop it on the WinZip window.  This will cause the install.props 
    file to be added into the "install_WebGoat-2.0_windows.jar" 
    file with a path of installer.  

3) Make sure you only have ONE install.props file in the .jar file. 
     If you have more than one install.props file, the file was not added properly.

4) Double click on the "install_WebGoat-2.0_windows.jar" file 
    and everything should work fine.

If doesn't work let me know.  I can build a new installer and send it to
you.  

Bruce.
