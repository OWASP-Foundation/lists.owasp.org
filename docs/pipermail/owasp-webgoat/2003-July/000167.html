  <disclaimer>I wrote my first .NET application yesterday and liked it ;-) </disclaimer>

  I'm not sure I see the sense of having two versions.  Why does it matter what language it is implemented in underneath?  Do you think more people will use it if it's in .NET?  I believe that ANY web security issue we want to teach people about can be easily implemented in the current framework.

  I also think that translating to .NET won't be trivial, since there are some complex features in WebGoat to enable plug-in lessons. Unless there's a technical reason to move to .NET, I suggest we make what we've got better instead of spending time switching horses midstream.

  Fundamentally, I really don't particularly care whether WebGoat is built on J2EE or .NET.  Some things are going to have to be "simulated" either way (but the SQL injection *is* real).  If there's a particular .NET flaw that you're interested in showing, could you describe it?  My guess is that it'll be easy to implement in the current framework.

  -- Jeff
