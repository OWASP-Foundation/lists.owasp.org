A lot of the structure depends where the guide/other docs are to live.
If it's in a readme type document included with the package, it makes
sense.  If it's to be the first html page presented to users when they
access WebGoat, it's probably less appropriate to have the install
information in the guide (hence a separate doc).  I'm always mindful of
the fact that nobody reads docs unless they have to - if the installer
is mostly self-explanatory, then they probably wont read the guide
before installing.
