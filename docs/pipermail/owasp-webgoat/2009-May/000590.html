Hello Group,
I'm a new user to webgoat and stuck at lesson 1, so it's not looking 
too promising. I understand proxies and web application attacks, but 
the problem is that neither web scarab or burp seem to intercept 
connections to the local host. If I make a connection to anything 
other than my macine (cnn, ebay, my routers web interface, etc) the 
proxy jumps in place and I can examine / modify headers and parameters 
but it just doesn't seem to catch things locally.
Again, in a nutshell, proxy intercepts when connecting to remote 
hosts, but doesn't see anything when I connect locally, I've used IP 
(127.0.0.1) and hostname (localhost). The only solution that comes to 
mind is setting up a VM, install webgoat on it, modify the config to 
allow remote connection. While that may work I'd prefer to solve the 
problem at hand.

What's the trick to intercepting the calls to webgoat with web scarab?

I'm using OS X 10.5.7 and the latest stable builds of WebScarab, burp 
and webgoat. I've tried with both Firefox and Safari and I get the 
same results.

TIA,
Ryan

Ryan

There used to be an IE (7?) issue where you had to add a "." to the url
after localhost in order to get the browser to use the proxy setting for
localhost.  However, I doubt you're using IE :) and I've not seen this
behavior with the other browsers.  Check to ensure the "no proxy for"
field is empty in FireFox connection settings and that any "bypass proxy
for localhost" settings are not configured for any browser.

Other than that, I don't have any other suggestions.  Sorry

Bruce
