> hi Tal,
>
> the most important message is: don't send sensitive data in the URL.
Adding to that, using GET method in this way violates the HTTP
specification requirement for safety and idempotency in the HTTP meaning
https://tools.ietf.org/html/rfc7231#section-4.2 GET should be not used
for any transactional state-changing operations for a number of reasons,
notably because it's automatically executed by browsers (result: CSRF)
and because GET requests may be transparently withheld or repeated by
caches, proxies, browser addons etc.
