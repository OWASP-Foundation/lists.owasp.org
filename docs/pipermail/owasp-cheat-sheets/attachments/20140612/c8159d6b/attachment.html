                <p class="MsoNormal"><br>
                   <b><br>
                    Only Support Strong Cryptographic Ciphers:</b><br>
                  …<br>
                  <br>
                  * use the very latest recommendations, they may be
                  volantile these days<br>
                  * Secure length for cryptographic keys and parameters
                  (like DH-parameter) >=2048 bits or equivalent
                  Elliptic Curves<br>
                  <br>
                  Example for a Policy to get a Whitelist for
                  recommenderd Ciphers:<br>
                  * Activate to set the Cipher Order by the Server <br>
                  * Highest Priority for Ciphers that support 'Forward
                  Secrecy'<br>
                  * Favor DHE over ECDHE, ECDHE lacks now of really
                  reliable Elliptic Curves, cf. <a
                    moz-do-not-send="true"
                    href="http://safecurves.cr.yp.to">
                    http://safecurves.cr.yp.to</a>; <br>
                  * Use RSA-Keys (no DSA/DSS, cf. <a
                    moz-do-not-send="true"
                    href="https://projectbullrun.org/dual-ec/tls.html">
                    https://projectbullrun.org/dual-ec/tls.html</a>)<br>
                  * Favor GCM over CBC regardless of the cipher size<br>
                  * Priorize the ciphers by the sizes of the Cipher and
                  the MAC<br>
                  * Disable weak ciphers without diabling latency
                  browsers and bots that have to be supported (find the
                  best compromise), actually the cipher
                  TLS_RSA_WITH_3DES_EDE_CBC_SHA (0xa) does this job.<br>
                  * Ciphers should be usable for DH-Pamameters >=
                  2048 bits, without blocking latency browsers (The
                  cipher ‘DHE-RSA-AES128-SHA’ is suppressed as some
                  browsers like to use it but are not capable to cope
                  with DH-Params > 1024 bits.)<o:p></o:p></p>
              </div>
              <div>
                <p class="MsoNormal">* Define a Cipher String that works
                  with different Versions of your encryption tool, like
                  openssl,
                  <br>
                  * Verify your cipher string <br>
                    ° with an autit-tool, like <a
                    moz-do-not-send="true"
                    href="https://www.owasp.org/index.php/O-Saft">OWASP
                    'O-Saft'</a><br>
                    ° listing it manually with your encryption software,
                  e.g. openssl ciphers -v <cipher-string> (the
                  result may differ by version), e.g.: openssl ciphers
                  -v
'EDH+aRSA+AESGCM:EDH+aRSA+AES:DHE-RSA-AES256-SHA:EECDH+aRSA+AESGCM:EECDH+aRSA+AES:ECDHE-RSA-AES256-SHA:ECDHE-RSA-AES128-SHA:RSA+AESGCM:RSA+AES+SHA:DES-CBC3-SHA:-DHE-RSA-AES128-SHA'<br>
                  #add optionally
                  ':!aNULL:!eNULL:!LOW:!MD5:!EXP:!PSK:!DSS:!RC4:!SEED:!ECDSA:!ADH:!IDEA'
                  to protect older Versions of OpenSSL<br>
                  <br>
                  * This results in this recommended Cpihers and their
                  Order:<o:p></o:p></p>
              </div>
              <div>
                <pre>0x00,0x9F - DHE-RSA-AES256-GCM-SHA384 TLSv1.2 Kx=DH       Au=RSA  Enc=AESGCM(256) Mac=AEAD<o:p></o:p></pre>
                <pre>0x00,0x9E - DHE-RSA-AES128-GCM-SHA256 TLSv1.2 Kx=DH       Au=RSA  Enc=AESGCM(128) Mac=AEAD<o:p></o:p></pre>
                <pre>0x00,0x6B - DHE-RSA-AES256-SHA256   TLSv1.2 Kx=DH       Au=RSA  Enc=AES(256)  Mac=SHA256<o:p></o:p></pre>
                <pre>0x00,0x39 - DHE-RSA-AES256-SHA      SSLv3 Kx=DH       Au=RSA  Enc=AES(256)  Mac=SHA1<o:p></o:p></pre>
                <pre>0x00,0x67 - DHE-RSA-AES128-SHA256   TLSv1.2 Kx=DH       Au=RSA  Enc=AES(128)  Mac=SHA256<o:p></o:p></pre>
                <pre>0xC0,0x30 - ECDHE-RSA-AES256-GCM-SHA384 TLSv1.2 Kx=ECDH     Au=RSA  Enc=AESGCM(256) Mac=AEAD<o:p></o:p></pre>
                <pre>0xC0,0x2F - ECDHE-RSA-AES128-GCM-SHA256 TLSv1.2 Kx=ECDH     Au=RSA  Enc=AESGCM(128) Mac=AEAD<o:p></o:p></pre>
                <pre>0xC0,0x28 - ECDHE-RSA-AES256-SHA384 TLSv1.2 Kx=ECDH     Au=RSA  Enc=AES(256)  Mac=SHA384<o:p></o:p></pre>
                <pre>0xC0,0x14 - ECDHE-RSA-AES256-SHA    SSLv3 Kx=ECDH     Au=RSA  Enc=AES(256)  Mac=SHA1<o:p></o:p></pre>
                <pre>0xC0,0x27 - ECDHE-RSA-AES128-SHA256 TLSv1.2 Kx=ECDH     Au=RSA  Enc=AES(128)  Mac=SHA256<o:p></o:p></pre>
                <pre>0xC0,0x13 - ECDHE-RSA-AES128-SHA    SSLv3 Kx=ECDH     Au=RSA  Enc=AES(128)  Mac=SHA1<o:p></o:p></pre>
                <pre>0x00,0x9D - AES256-GCM-SHA384       TLSv1.2 Kx=RSA      Au=RSA  Enc=AESGCM(256) Mac=AEAD<o:p></o:p></pre>
                <pre>0x00,0x9C - AES128-GCM-SHA256       TLSv1.2 Kx=RSA      Au=RSA  Enc=AESGCM(128) Mac=AEAD<o:p></o:p></pre>
                <pre>0x00,0x35 - AES256-SHA              SSLv3 Kx=RSA      Au=RSA  Enc=AES(256)  Mac=SHA1<o:p></o:p></pre>
                <pre>0x00,0x2F - AES128-SHA              SSLv3 Kx=RSA      Au=RSA  Enc=AES(128)  Mac=SHA1<o:p></o:p></pre>
                <pre>0x00,0x0A - DES-CBC3-SHA            SSLv3 Kx=RSA      Au=RSA  Enc=3DES(168) Mac=SHA1<o:p></o:p></pre>
                <pre>Remarks:
- According to my researches the most common browsers should be supported with this setting, too.
- Monitor the performance of your server, e.g. the TLS handshake with DHE hinders the CPU abt 2.4 times than ECDHE (cf. <a moz-do-not-send="true" href="http://vincent.bernat.im/en/blog/2011-ssl-perfect-forward-secrecy.html#some-benchmarks">[Vincent Bernat, 2011]</a>)<o:p></o:p></pre>
                <p class="MsoNormal"><br>
