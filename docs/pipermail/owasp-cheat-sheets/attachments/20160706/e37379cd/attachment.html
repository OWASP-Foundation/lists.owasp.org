        <div>With the standard browser function to allow the storing of
          passwords, it remembers it for anyone who uses the computer.
          There is not extra authentication required. With a password
          manager or vault, there is usually a mechanism to authenticate
          against.<br>
        </div>
        <div><br>
        </div>
        <div>Additionally, the browser password managers I have had
          experience with store passwords in plain text. Any time you
          store a password in plain text it is not a great idea. And if
          they don't store them in plain text, just remembering the
          password means I could use an interception proxy to steal your
          password, if I have control of your system.<br>
        </div>
        <div><br>
        </div>
        <div>I understand there is some difficulty in remembering
          passwords. And in some instances, if you understand the true
          risk, it is acceptable. However, as long as we have passwords,
          it is something we will have to deal with. I think the
          software engineers of the world have an obligation to help
          users out, I think designers owe it to their users to design
          better and easier user experiences, but I also think we need
          to train users to become more aware, which is what this is all
          about.<br>
        </div>

                                          <p><b>Description:</b>Passwords
                                            are the most common way to
                                            authenticate to systems,
                                            applications, services, etc.
                                            Authentication is how a
                                            person or system proves
                                            their identity. Three
                                            methods of authentications
                                            are: provide something you
                                            know, something you have, or
                                            something you are. Passwords
                                            fulfill the first condition,
                                            something you know. People
                                            and systems authenticate by
                                            providing something only
                                            they should know, therefore
                                            proving their identity. Weak
                                            password handling
                                            vulnerabilities are
                                            weaknesses in the handling,
                                            storage, and use of
                                            passwords. <br>
                                          </p>
                                          <p><b>Threats:</b>The exposure
                                            of passwords through
                                            mishandling or improper
                                            storage could allow
                                            discovery and use by a third
                                            party. <br>
                                          </p>
                                          <p><b>Impact:</b>Weak password
                                            handling can result in the
                                            unauthorized access and
                                            compromise of data or
                                            systems. <br>
                                          </p>
                                          <p><b>Recommendations:</b><br>
                                          </p>
                                          <ul>
                                            <li>Use Multi-factor
                                              Authentication, especially
                                              on important accounts <br>
                                            </li>
                                            <li>Use a Password Manager <br>
                                            </li>
                                            <li>Use Strong Passwords <br>
                                            </li>
                                            <li>Avoid using the same
                                              password across different
                                              accounts <br>
                                            </li>
                                            <li>Do not answer security
                                              questions with easily
                                              identifiable or enumerable
                                              answers <br>
                                            </li>
                                            <li>Do not allow browsers to
                                              store passwords <br>
                                            </li>
                                            <li>Do not share your
                                              passwords <br>
                                            </li>
                                            <li>Change default passwords
                                              <br>
                                            </li>
                                          </ul>
