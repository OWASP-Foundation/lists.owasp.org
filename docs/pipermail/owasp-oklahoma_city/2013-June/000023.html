Jason,

<2¢>

The idea is just plain awesome! There is a lot of hype around the 
RaspberryPI and it seems like everybody has one or can get one; so it 
seems like that would make it more accessible, more hackable and it is 
cheaper.

However, the BeagleBone is a better engineered product. BeagleBoard is 
much better suited for Real-Time linux and in the end this will be a 
giant advantage for any realtime analysis which will no doubt take 
place. I think thus, BeagleBone:

1) better engineered
2) likely to last longer 'in-field' due to #1
3) better suited for realtime linux
4) investment over time likely to be better due to #3

</2¢>

Note: I have been wrong before :-)
