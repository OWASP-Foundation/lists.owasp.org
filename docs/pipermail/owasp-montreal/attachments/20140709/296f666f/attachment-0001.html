                            <h3><font face="Helvetica, Arial, sans-serif" size="4" color="#D85D33">Featured OWASP Project</font></h3>
                            <p><font face="Helvetica, Arial, sans-serif" size="3"><a href="http://cts.vresp.com/c/?TheOWASPFoundation/7d3680ebcc/d1ed10e149/93bfc499c9" target="_blank">OWASP Java Encoder Project</a></font></p>


                            <p><font face="Helvetica, Arial, sans-serif">The OWASP Java Encoder is a Java 1.5+ simple-to-use drop-in high-performance encoder class with no dependencies and little baggage. This project will help Java web developers defend against Cross Site Scripting! The OWASP Java Encoder library is intended for quick contextual encoding with very little overhead, either in performance or usage. To get started, simply add the encoder-1.1.1.jar, import org.owasp.encoder.Encode and start encoding.</font></p>


                            <p><font face="Helvetica, Arial, sans-serif">For more information, please contact the Project Leaders, <a href="mailto:jeff.ichnowski@gmail.com" target="_blank">Jeff Ichnowski</a> and <a href="mailto:jim.manico@owasp.org" target="_blank"> Jim Manico</a></font></p>



                            <h3><font face="Helvetica, Arial, sans-serif" size="4" color="#D85D33">New OWASP Projects</font></h3>

                            <p><font face="Helvetica, Arial, sans-serif" size="3">OWASP Faux Bank</font></p>
                            <p><font face="Helvetica, Arial, sans-serif">Faux Bank has all 10 of the top vulnerabilities implemented, as well as fixes for these vulnerabilities. The idea is that developers can see a real-world system with vulnerabilities, so that they can see what to look for and how to write secure code. The OWASP Faux Bank wiki page <a href="http://cts.vresp.com/c/?TheOWASPFoundation/7d3680ebcc/d1ed10e149/ac92a8f519" target="_blank">can be found here.</a>   For more information, please contact the Project Leader, <a href="mailto:davie.elliott@owasp.org" target="_blank">Davie Elliott.</a></font></p>

                            <p><font face="Helvetica, Arial, sans-serif" size="3">OWASP Store Sheep Project</font></p>
                            <p><font face="Helvetica, Arial, sans-serif">OWASP Store Sheep is a work in progress application do demonstrate security concepts relating to Windows Store Apps. Store Sheep is a training app for Developers wishing to learn to securely code a Windows Store ('Metro Style') App, and Testers wanting to learn to test one. It contains a number of security vulnerabilities with explanations and fixes for them. The project page for the OWASP Store Sheep project <a href="http://cts.vresp.com/c/?TheOWASPFoundation/7d3680ebcc/d1ed10e149/509cfee45e" target="_blank">can be found here.<a> For more information, please contact the Project Leader, <a href="mailto:marion.mccune@owasp.org" target="_blank">Marion McCune.</a></a></a></font></p>

                            <p><font face="Helvetica, Arial, sans-serif" size="3">OWASP SonarQube Project</font></p>
                            <p><font face="Helvetica, Arial, sans-serif">OWASP Sonarqube Project consist to deliver a set of "standard" profile for security, like OWASP Top10 profile, ASVS profiles, PCI-DSS profile,ISO 27034ASC profile, ....who can be used by team with the support of OWASP Community. More than 20 programming languages are covered through plugins including Java, C#, C/C++, PL/SQL, Cobol, ABAP. The OWASP SonarQube Project is looking to expand the offered languages, and is looking for language experts in .NET, PHP and any other language. The project page for the OWASP SonarQube Project <a href="http://cts.vresp.com/c/?TheOWASPFoundation/7d3680ebcc/d1ed10e149/0064ee24f7" target="_blank">can be found here.<a> For more information, please contact the Project Leaders, <a href="mailto:sebastien.gioria" target="_blank">Sebastien Gioria.</a> and <a href="mailto:freddy.mallet@sonarsource.com" target="_blank">Freddy Mallet</a></a></a></font></p>

                            <p><font face="Helvetica, Arial, sans-serif" size="3">OWASP URL Checker</font></p>
                            <p><font face="Helvetica, Arial, sans-serif">OWASP URL Checker is an open source scrip-table tool to scan websites for URL's which may lead to information divulging, exploits and common attack patterns. This tool will check a user defined website for potentially exploitable/ vulnerable URL's by comparing them against the URL extensions in the database. The project page for the OWASP URL Checker <a href="http://cts.vresp.com/c/?TheOWASPFoundation/7d3680ebcc/d1ed10e149/20e785a0e2" target="_blank">can be found here.</a> For more information, please contact the Project Leader, <a href="mailto:craig.fox@owasp.org" target="_blank">Craig Fox.</a></font></p>


                            <h3><font face="Helvetica, Arial, sans-serif" size="4" color="#D85D33">Project Announcements</font></h3>


                            <p><font face="Helvetica, Arial, sans-serif" size="3">OWASP Security Shepherd New Version</font></p>
                            <p><font face="Helvetica, Arial, sans-serif">The new version of the OWASP Security Shepherd Project was released earlier this month. The project now has 50 lessons and challenges based on risks from both the Top Ten Mobile and Web App Security Risk lists. OWASP Security Shepherd is perfect for those who are looking to learn about appsec for the first time or are well seasoned in the arts of pen-testing and are looking for a challenge.</font></p>

                            <p><font face="Helvetica, Arial, sans-serif">More information can be found <a href="http://cts.vresp.com/c/?TheOWASPFoundation/7d3680ebcc/d1ed10e149/cee13b753d" target="_blank"> ON THE WIKI PAGE</a> or you can contact the project leader <a href="mailto:markdenihan@owasp.org" target="_blank">Mark Denihan</a></font></p>

                            <p><font face="Helvetica, Arial, sans-serif" size="3">Research Assistant Needed for the Developer guide</font></p>
                            <p><font face="Helvetica, Arial, sans-serif">The Developer Guide Project is looking for an honors student or masters student to replicate the 1979 paper by Morris and Thompson. It has been many years since we've had statistically sound research into the basic properties of the password. Morris and Thompson introduced countermeasures that we still use today (30 day password rotation, min six character passwords) that made sense for a PDP 11/870 back in 1979.
                                The project leaders would like a cryptographer research student or masters student to help look into session tokens, particularly RESTful API tokens. The basic topic would be a short paper on the necessary properties to protect against session prediction, session recovery, side channel attacks against sessions, and investigate a few sample session issuers, such as RESTful API in common use.
                                <p><font face="Helvetica, Arial, sans-serif">If you are interested in helping the Developer Guide, please contact <a href="mailto:vanderaj@owasp.org" target="_blank">Andrew van der Stock</a>.</font></p>

                                <p><font face="Helvetica, Arial, sans-serif" size="3">New Set of Architectural Security Principles</font></p>
                                <p><font face="Helvetica, Arial, sans-serif">The Reverse Engineering and Code Modification Prevention project has released a set of architectural security principles that enforce integrity preservation in mobile apps.  This is an updated list of principles / controls that security architects will find useful when enforcing code integrity within their mobile apps.</font></p>

                                <p><font face="Helvetica, Arial, sans-serif">For the complete list of the integrity controls and underlying security principles, <a href="http://cts.vresp.com/c/?TheOWASPFoundation/7d3680ebcc/d1ed10e149/56a44d0034" target="_blank">check out the Architectural Principles sub-project.</a></font></p>

                                <p><font face="Helvetica, Arial, sans-serif" size="3">New Dependency Check Version 1.2.3 Out Now</font></p>
                                <p><font face="Helvetica, Arial, sans-serif">On June 28th, the OWASP Dependency Check released version 1.2.3. Dependency Check can be used to analyze an applications dependent libraries (Java and .NET) to identify and report on any known, published vulnerabilities related to the libraries being used. The tool will be demoed during the Black Hat Arsenal in Las Vegas on Wednesday, August 6th.</font></p>

                                <p><font face="Helvetica, Arial, sans-serif">You can find the newest release of the OWASP Dependency Check <a href="http://cts.vresp.com/c/?TheOWASPFoundation/7d3680ebcc/d1ed10e149/9c56a58a78" target="_blank">on the project page.</a></font></p>

                                            <h3><font face="Helvetica, Arial, sans-serif" size="4" color="#00549E">Initiatives</font></h3>
                                    <p><font face="Helvetica, Arial, sans-serif" size="3">OWASP Winter Code Sprint</font></p>
                                    <p><font face="Helvetica, Arial, sans-serif">We are thrilled to announce the launch of OWASP Winter Code Sprint (OWCS) for this upcoming Autumn/Winter (Sept 14-March 15).</font></p>
                                    <p><font face="Helvetica, Arial, sans-serif"><strong>What is OWCS?</strong>
                                    <p><font face="Helvetica, Arial, sans-serif"> The OWCS is a program to involve students with Security projects. By participating in OCWS a student can get real life experience while contributing to an open source project and getting university credits.</font></p>

                                    <p><font face="Helvetica, Arial, sans-serif"><strong>How it works</strong></font></p>
                                    <p><font face="Helvetica, Arial, sans-serif">Any OWASP project that will give you university credits can participate in OCWS. Each project will be guided by an OWASP expert along with a professor. Students are graded by their University, based on success criteria identified at the beginning of the project.</font></p>

                                    <p><font face="Helvetica, Arial, sans-serif">Projects are focused on developing security tools. It is required that the code any student produces for those projects will be released as Open Source. Universities are free to specify their own requirements to projects, such as written reports. OWASP does not influence the way grades are allocated. The OWASP advisers will provide any information professors need in order to grade their students.</font></p>

                                        <p><font face="Helvetica, Arial, sans-serif"><strong>How to participate?</strong></font></p>
                                        <p><font face="Helvetica, Arial, sans-serif">As a Student:</font></p>
                                        <ol>
                                            <li>Review the list of OWASP Projects currently prticipating in OWCS</li>
                                            <li>Get in touch with the OWASP Project mentor of your choice</li>
                                            <li>Agree on deliverables with OWASP mentor and university professor</li>
                                            <li>Work away during Autumn/Winter 2014</li>
                                            <li>Rise to Open Source Development Glory!</li>
                                        </ol>
                                        <p><font face="Helvetica, Arial, sans-serif">As a Professor:</font></p>
                                        <ol>
                                            <li>Review the list of OWASP Projects currently prticipating in OWCS</li>
                                            <li>Get in touch with the OWASP Project mentor of your choice</li>
                                            <li>Promote the participating OWASP Projects among students</li>
                                            <li>Review student progress with help from OWASP mentors</li>
                                            <li>Grade student work according to university scoring system</li>
                                            <li>Provide student grade results to OWASP mentor/s</li>
                                        </ol>
