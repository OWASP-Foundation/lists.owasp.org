<br>
I am glad to inform that after one year of preparation, OWASP Python<br>
Security (Pysec) Project team has published the first release.<br>
<br>
OWASP Python Security (Pysec) is a project that aims at creating a<br>
hardened version of python that makes it easier for security<br>
professionals and developers to write applications more resilient to<br>
attacks and manipulations.<br>
<br>
We are publishing our code as PRE-ALPHA version, under heavy development<br>
and NOT to be used in any production environment. Code is being<br>
commented and we are drawing structural specifications, documentation<br>
will follow shortly after.<br>
<br>
Parallel to the code development we are working on a manual on secure<br>
coding in python that will be released as soon as we will have the first<br>
stable version, hopefully within six month.<br>
<br>
Still a long way from being ready for real use but we feel is a<br>
significant step forward in reaching our goals, and we hope some of you<br>
may get involved in this project as well.<br>

Some of OWASP Pysec features<br>
-----------------------------------------------<br>
+ Custom libraries in C and Python with functional controls<br>
+ Import functions with version control and hash check for integrity<br>
+ Logging supports flow controls, checkpoints and external handlers<br>
+ Error handling and detection using finite state automata<br>
+ Library for on-disk data structures and information storage<br>
+ File operation library with data verification and file system hooks<br>
+ Splitters for sequence-like objects to improve memory usage and speed<br>
<br>
Current OWASP Pysec "work in progress"<br>
-----------------------------------------------<br>
+ Multi-purpose Queue Library<br>
+ Logging thread-safe and multiprocess/multithread aware<br>
+ Hardened protocol libraries for POP3/SMTP/IMAP4/FTP<br>
+ Internal control library to enforce type and data verification<br>
+ Library to enable process and permission controls on OS objects<br>
+ Library to include OWASP ESAPI specifications and security controls<br>
<br>
We would love to hear your feedback on this project, bug reports,<br>
suggestions and contributions are all very welcome.<br>
