tion thinks are legitimate requests from the victim.</div><div><br></div><div>We will discuss what this issue is, a number of varieties of this issue along with methods for avoiding it in your application code and a demo of how you would examine a defence using burp.</div><div><br></div><div><a href="https://www.owasp.org/index.php/Top_10_2013-A8-Cross-Site_Request_Forgery_(CSRF)">https://www.owasp.org/index.php/Top_10_2013-A8-Cross-Site_Request_Forgery_(CSRF)</a></div><div><br></div><div>1 b). Top 10 2013-A9-Using Components with Known Vulnerabilities </div><div><br></div><div>Darren Fitzpartick</div><div><br></div><div>Components, such as libraries, frameworks, and other software modules, almost always run with full privileges. If a vulnerable component is exploited, such an attack can facilitate serious data loss or server takeover. Applications using components with known vulnerabilities may undermine application defenses and enable a range of possible attacks and impacts. </div><div><br></div><div>We will discuss how know vulnerabilities can be identified in a system and used to get access to other systems and data in your network. Mitigation techniques will also be discussed. </div><div><br></div><div><a href="https://www.owasp.org/index.php/Top_10_2013-A9-Using_Components_with_Known_Vulnerabilities">https://www.owasp.org/index.php/Top_10_2013-A9-Using_Components_with_Known_Vulnerabilities</a> </div><div><br></div><div>2. Practical Hands On Workshop</div><div><br></div><div>This section of the night will invoke our learning from the first phase and put it to practical use. We take our testing environment and use it to exploit some CSRF & component vulnerabilities on a safe, intentionally vulnerable website.</div><div><br></div><div>After giving some time for individually attempting to carry out the exploitation, a walk-through of the exploit technique will be given for each of the examples outlined. The OWASP team will be at hand to help with any issues that might arise through this phase.</div><div><br></div><div>Practical elements will cover the following two perspectives so that you leave with not only an understanding of the issues but also having had hands on practice in these areas: </div><div><br></div><div>1. Defensive - Seeing vulnerable code / configurations and investigating how the issues could be rectified.</div><div><br></div><div>2. Offensive - Attacking vulnerable sites from a malicious attacker or software tester's perspective.</div><div><br></div><div>Hope to see you there!</div><div><br></div><div>Darren & Fiona (OWASP Cork Team)</div></div><div><br></div><br clear="all"><div><div class="gmail_signature"><div dir="ltr"><div><br></div><div>Fiona Collins,</div><div>OWASP Cork Chapter Lead</div></div></div></div>
