            This was a conscientious decision, that I made, but I'm
            still on the fence about.  URL's do not always need the
            single-quote character encoded according to RFC3986--they're
            in the "reserved" set, so they may have special meaning
            depending on the part of the URL in which they appear.  So
            the thought was that if the URL is to be presented in an
            attribute context, it would require double encoding --
            forHtmlAttribute(forUri(data)), similarly if in a JavaScript
            context -- var url = "<%=
