<p class="MsoNormal"><span style="font-size:14.0pt">OWASP Java Encoder Project Team:<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:14.0pt">Hi – we are having trouble finding a standard library for JSON output encoding.  I was happy to see that OWASP had a program for output encoders to tackle common XSS exploits.  I was hoping there was one to
 handle our use case.<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:14.0pt"><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:14.0pt">OWASP XSS (Cross Site Scripting) Prevention Cheat Sheet confirms what our InfoSec team is telling us for output encoding for JSON payloads, namely:
</span><img width="840" height="47" id="Picture_x0020_1" src="cid:image001.png@01D3DDFE.26043D00"><span style="font-size:14.0pt"><o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:14.0pt">Though this talks about JavaScript specifically, I’ve seen where it applies to JSON.  IETF RFC 8259 section 7 talks about the correctness of supplying \uXXXX substitution if necessary.<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:14.0pt">I saw that the org.owasp.encoder.Encode.</span><b><span style="font-size:10.0pt;font-family:"Arial",sans-serif;color:#353833">
<span lang="EN">forJavaScript method does include JSON in its scope (from JavaDoc):<o:p></o:p></span></span></b></p>
<pre style="margin-left:0in;text-indent:-.25in;mso-list:l0 level3 lfo1;background:white"><![if !supportLists]><span lang="EN" style="font-size:10.0pt;font-family:Symbol;color:#353833"><span style="mso-list:Ignore">·<span style="font:7.0pt "Times New Roman"">         </span></span></span><![endif]><span style="font-size:14.0pt">“</span><span lang="EN" style="color:#353833">public static <a href="http://docs.oracle.com/javase/1.5.0/docs/api/java/lang/String.html?is-external=true" title="class or interface in java.lang">String</a> forJavaScript(<a href="http://docs.oracle.com/javase/1.5.0/docs/api/java/lang/String.html?is-external=true" title="class or interface in java.lang">String</a> input)<o:p></o:p></span></pre>
<p style="background:white"><span lang="EN" style="font-size:10.5pt;font-family:"Georgia",serif;color:#474747">Encodes for a JavaScript string. It is safe for use in HTML script attributes (such as
</span><code><span lang="EN" style="font-size:10.5pt;color:#474747">onclick</span></code><span lang="EN" style="font-size:10.5pt;font-family:"Georgia",serif;color:#474747">), script blocks, JSON files, and JavaScript source.<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:14.0pt">“<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:14.0pt">However the encoding provided does not change the encoding for non-alphanumeric characters to \uXXXX form but instead I see some characters (but not all I was hoping for – though the document confirms the
 smaller scope) converted to \xDD form.  For instance:<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:14.0pt">String jSON = forJavascript("{\"Name\" : \"!<[(!@Evil_Script#$)]>Вони наділені розумом і совістю</span><span style="font-size:14.0pt">嘴遧</span><span style="font-size:14.0pt">䔣</span><span style="font-size:14.0pt">\"}");<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:14.0pt">is converted to <o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:14.0pt">Encoded JSON = {\x22Name\x22 : \x22!<[(!@Evil_Script#$)]>Вони наділені розумом і совістю</span><span style="font-size:14.0pt">嘴遧</span><span style="font-size:14.0pt">䔣</span><span style="font-size:14.0pt">\x16\x22}<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:14.0pt"><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:14.0pt">If there are no current encoders for JSON as described above, are there any plans to support a JSON encoder that can implement the XSS Prevention talked about above?  Does alphanumeric mean [a-z,A-Z,0-9] only
 or is it alphanumeric in any language?  Our InfoSec team interprets the meaning of the guidelines as the smaller alpha scope.  Any advice your members can provide would be of great benefit to us.<o:p></o:p></span></p>
