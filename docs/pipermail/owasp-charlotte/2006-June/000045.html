I completely disagree.  It's a result of poor programming - it's just
like any other command injection - HTML Injection can be just as bad -
if I load a nassssty <applet> tag, or falsify information or
something.  Javascript isn't to blame, because bad guys will find new
ways to exploit the laziness of developers in Flash, HTML, Java, SQL,
command lines, XML, XSLT, LDAP, and a whole bunch of other TLA's.
Developers need to make sure what they send to the presentation layer
is appropriate for that type of presentation.

The beginning of it is technically correct - what we GENERALLY
classify as "Cross-site Scripting" is technically either HTML
Injection or Script Injection - cross-site scripting involves reading
script data from another site through frames or otherwise.  But
generally, that sort of cross-frame communication is not allowed by
the browsers anymore.
