Hi Everyone,

If we're talking about quick wins (< 4 hours), the HTML5 browser
compatibility tests, AppSensor documentation, and a Maven archetype
are great ideas. Creating a useful documentation stack may not be as
quick a win, but a big win just the same. We can chunk the tasks and
get started with the most popular technologies, frameworks, and
vulnerabilities. The documentation can provide code examples and some
suggestions (e.g. where to place AppSensor detection points). Maybe we
generate procedures that can also double as sources for technology
stories in an Agile methodology (e.g. every-sprint requirements in
SDL-Agile)? This could help us promote security in the development
life-cycle by suggesting what should be done, where, and when it
should be done.

Will, I think November would be good. I'd like to participate in a
couple of these working groups/meetups before the end of the year, if
possible.

Jay
