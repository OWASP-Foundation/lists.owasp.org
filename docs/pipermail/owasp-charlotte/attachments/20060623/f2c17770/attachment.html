application generates another application (think of a Java web app generating an 
HTML or JavaScript "application" to be evaluated by the user's browser) it is 
the sole responsibility of the first application to know what it is generating. 
Who would argue that SQL is to blame when an application uses String 
concatenation to create a SQL inject vulnerability when the designers could have 
otherwise used a prepared statement.</FONT></SPAN></DIV>
