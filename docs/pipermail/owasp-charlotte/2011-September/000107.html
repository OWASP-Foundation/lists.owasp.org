All,
Sorry for the delay in getting this typed up, but here are a few of my ideas
for how and where appsensor could be improved. I've tried to break them down
just a bit into categories in case folks have a preference. There are
definitely many, many more things that could be done, but hopefully these
will get the ball rolling. If you're interested in tackling any of these,
let me know and I'll get you setup.

Documentation
------------------------
- how to integrate appsensor into an application
- update wiki pages
- write getting started / how-to guides for users
- writeup faq for wiki based on mailing list discussions / bug list
- map detection points to modsecurity ruleset using those detection points
(cross-reference)
- think up new detection points and document them
- think up new response actions and document them

Coding
------------------------
- build appsensor in other languages (we discussed php)
- write more detection points into sample app / core code / new sample apps
- write more response actions into sample app / core code / new sample apps
- write more sample apps
- expand existing sample app (AppSensorDemo2)
- build out intrusion detector that communicates w/ an SIEM
- build trend analysis engine
- write different implementations of the existing interfaces (datastores for
different databases like oracle/mysql/etc., user implementation for
jsecurity/spring security/etc.)

Testing
------------------------
- write unit / integration tests
- build functional test cases for sample app(s)
- build new sample apps that use appsensor and find what breaks
- load test appsensor core code

Thanks,
John
