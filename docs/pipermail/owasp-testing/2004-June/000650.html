Here goes another suggestion

Do not mix "pure" web application issues with web application infrastructure issues, this is do not mix SQL injection which is a purerly programming (bad) practice with Web Server configuration issues like "HTTP Server banner". I've always fight for this disctintion to be made.

In my oppinion it is very important to separate OS, HTTP Server, application server (like a servlet engine) and application itself, they are different things and different ways to test them. For example, you hardly will perform a source review on a HTTP Server and only an automated blackbox testing for the "web software"* (very innacurate), but, blackbox tools are very good for http server issues (exploits) and source review for web application.

*Web Software = the web application itself, e.g. a portal made of some php pages (liked this term from Mark)

What do you think people?
