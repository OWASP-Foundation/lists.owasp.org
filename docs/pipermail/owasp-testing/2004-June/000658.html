I agree too.  A lot of the infrastructure related issues fall into the
'presentation tier' category, such as old/unnecessary files and server
components left in place, or platform vulnerabilities.  

Whilst these are equally important, the practicalities of testing and
division of responsibilities may mean that these are out of scope or control
of the app tester.

There are a number of crossover areas - e.g. the SQL injection example is an
application flaw, but if the user the application access the database with
is overly privileged, this is arguably a presentation (or data) tier issue.

Glyn.
