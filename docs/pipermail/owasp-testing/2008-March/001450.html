Yes, I think the AJAX section needs a lot of improvement, I'm starting 
to see demand on how to test rich client applications.

> 
> This is my idea for the new Testing Guide:
> http://www.owasp.org/index.php/OWASP_Testing_Guide_v3_Startup

But I don't think the chapter in the Testing Guide should be "AJAX 
testing" it should be "Rich client testing" and (try to) cover 
technologies such as Ajax, Java applets, Flex, Silverlight, Flash, and 
other tecnologies. Changing the name of the chapter would make it have a 
broader approach and it could start by talking about the generic issues 
in these type of applications and then cover samples with specific 
technologies.

As for your idea for the new Testing Guide I don't think you should 
strike out "Information gathering" and "Business logic testing". 
Information gathering is a quite important phase in testing and "info" 
vulnerabilities are vulnerabilities in themselves. Being able to 
fingerprint the web server, the application server, the database or any 
other technologies is important in making it possible to drive other 
attacks.

Just my 2c.

Javier
