Currently I'm performing a test on a web application.
I identified that a certain number (which consists out of 8 digits)
corresponds to the value of a cookie.
However, this number is encoded since the value of the cookie consists out
of 32 digits.

 f( [[:digit:]]{8} ) = [[:digit:]]{32}

It's not MD5. And I think that the server must be able to decode the value,
so it shouldn't be a one-way function.

Does anybody know of an algorithm or function which would generate this?
