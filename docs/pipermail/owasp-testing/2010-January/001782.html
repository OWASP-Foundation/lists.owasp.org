Hi,

Please see http://www.owasp.org/index.php/Common_OWASP_Numbering for what appears to be a generally agreed-upon new numbering scheme.

Please see the TOC of the following to "kick the tires", to expand/collapse the navigation tree to see how the numbering would work in practice: http://code.google.com/p/owasp-development-guide/wiki/Introduction

Next steps are basically to start using it on next versions of guides and whatnot. That's already done for the development guide, as evidenced by the above link. For the testing and other guides, the very next step is to start doing a mapping of the new numbers to legacy identifiers, put that in tables in document or wiki page somewhere, before doing any sort of updates or enhancements to baseline documents. Brad, I'll contact you individually about this, any other people interested in helping, please contact me directly, and we'll work that in a smaller group.
