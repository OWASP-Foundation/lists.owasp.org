Hi all,

It is necessary to bear in mind that the guide will be used by
pentesters that they will have to reflect in a report the results
obtained in every test.
In this sense, I believe that it would be interesting to include in
every test a paragraph that comments what results are expected and
probably even a template to present these results.
For example, in the SQL Injection test what do we expect to obtain? Only
data input to the application that are vulnerable? Or also it would be
necessary to know the type of possible injection (standard sql
injection, blind sql injection, union query sql injection, etc.),
affected functionalities of the application, etc.?
If the expected results are enumerated more homogeneous reports will be
created.

What does seem to you?

Regards,
