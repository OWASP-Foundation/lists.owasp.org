Hi,

There might be a recurring problem if using the term exploitation in the
definition of vulnerability.

Anyway, in the larger security context I always say that a vulnerability
is a feature that is available to a user, though it was not included in
the (security) design. IMO, it is important to convey the notion that
the designers intended this and that use, yet somewhere in the
development cycle new features where inadvertently added.

In our case, this could be followed by a more stringent definition, or a
set of "features" that attackers might gain.

I hope this helps.

Cheers,
Ariel
