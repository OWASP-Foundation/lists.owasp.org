On 14/11/06, Andrew van der Stock <vanderaj at owasp.org> wrote:
>
> For the major documentation projects (Guide, Top 10, Testing Guide),
> we need a PDF version along with a Wiki / Live CD version. The PDF
> versions have always been very popular.
>
> The OWASP Guide 3.0 has a publishing contract and when I get off my
> backside, it will be published. If you want, we can talk to our
> publisher about publishing the Testing Guide 2.0 in dead tree form.
> The profits from the sale of the book would allow OWASP to further
> OWASP's Goals such as run more AoC's in the future, and fun things
> like t-shirts for contributors, etc.
>
> If we want to do a PDF version, I have a Word template which I would
> like us to use (it's currently being used by the Top 10 2007). To do
> this, we have to use a tool to extract the text from the Wiki into
> Word format. It takes about 20 minutes a chapter to re-format basic
> Wiki extracts into the OWASP Word template. I have a full version of
> Acrobat, so I can create high quality, bookmarked ToC'd PDFs with a
> small size.
>
> Theoretically, it might be possible to use on of the few Wiki -> PDF
> generators out there to look like the Word version, but this would be
> an AWFUL lot of work to create the template to achieve an acceptable
> look and feel. The good news on this option is that once done, we
> could easily allow users to pump out up to the minute PDFs as long as
> we had adequate processing power. However, my past experience with
> this toolset is that it produces extremely poor quality PDFs, usually
> unreadable on screen, with no bookmarks or clickable TOCs. If that is
> the state of automated PDF creation, I'd prefer we didn't do it at
> all as it's like turning up to a job interview over-pierced, in torn
> jeans and a day old t-shirt covered in last night's fajitas.
>
> If we want to a book version, NSP have a very special Word and Open
> Office template (the ONLY formats they accept) which allows them to
> reflow the text into Framemaker. Again, it takes about 20 minutes per
> chapter to re-format the text. This template looks extremely
> ordinary, so we wouldn't use that for the PDF version of the document.
>
> thanks,
> Andrew
