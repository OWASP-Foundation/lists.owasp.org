I disagree slightly.

When describing an attack, which these are types of, I use attacker in the
3rd person.  For example, in the summary for XML Content Level testing, in
order to explain why we are testing, I state:

"An attacker can craft an XML document(SOAP message) that contains malicious
content in order to compromise the target system. Testing for proper content
validation should be included in the web application testing plan."

I think this has more impact (but of course, I'll bow to the wishes of the
group ;)
