Hi and thanks for your email,

In no way was I suggesting to describe what each request does; more to just
provide categories for the fuzz vectors. Something along the lines of:
"
1. Cross Site Scripting

2. SQL Injection (passive)

SELECT type of statements

3. SQL Injection (active)

INSERT type of statements
"
And perhaps include a few of the ones I encountered like:
"
4. Integer Overflows

-1
0
0x100
0x1000
0x3fffffff
0x7ffffffe
0x7fffffff
0x80000000
0xfffffffe
0xffffffff
0x10000
0x100000

5. Format String Errors

...

6. Buffer Overflows
"

It will still ruin complete copy-paste functionality for webscarab, but
ultimately give a user the ability to select the type of fuzzing they want to
perform, as well as provide a general categorisation demistifying this concept
of fuzzing a bit better for the wider public audience. 

Thanks,

Subere
