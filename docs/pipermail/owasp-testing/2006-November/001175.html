Granted in this case its acceptable to reference an attacker,  
especially if its in the context of stating how it may be exploited.
The point i was trying to make was that some of the sections have  
been written more like the hacking exposed books. E.g

To exploit this like an attacker, you need to encode the GET string  
using Unicode.

It would be better suited to write:

To test for this issue, the tester needs to encode the GET string  
using Unicode.

It may sound pedantic, but this guide was never meant to be a hacking  
web applications and using language which portrays it as that gives  
the wrong impression.
