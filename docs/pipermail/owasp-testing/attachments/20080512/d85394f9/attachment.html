<P><FONT SIZE=2>Hi all,<BR>
<BR>
actually, I have another proposal regarding Web Server / Application configuration.<BR>
<BR>
I'm internal auditor and normally we focus on different groups for responsibility on solving the issues we found.<BR>
<BR>
>From my point of view, OWASP testing is related with &quot;what any user can do&quot;, that is, interaction with the application. Sometimes the problem isn't the application itself, but the configuration of the servers (as an example: server headers or 404 Errors), so programmers can't solve the issues, but the system administrators can.<BR>
<BR>
If you want to review the configuration files, you have to ask for them, that is not &quot;what any user can do&quot;, you need access to the files, that is an infrastructure (web platform) issue.<BR>
<BR>
So, my proposal is to add (and maybe change some of the test) some tests focusing on configuration issues, not application issues, due to the person responsible for solve them are (and they should, from the point of view of segragation of duties) in different department.<BR>
<BR>
My two cents<BR>
<BR>
Regards,<BR>
