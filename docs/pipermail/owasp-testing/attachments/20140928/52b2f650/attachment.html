> version 4 improves on<br>
> version 3 in three ways:<br>
><br>
</span>> *1.* This version of the Testing Guide integrates with the two other<br>
<span class="">> flagship OWASP documentation products: the Developers Guide and the Code<br>
> Review Guide. To achieve this we aligned the testing categories and test<br>
> numbering with those in other OWASP products. The objective of the<br>
> Testing and Code Review Guides is to evaluate the security controls<br>
> described by the Developers Guide.<br>
><br>
</span>> *2.* All chapters have been improved and test cases expanded to 87 (64<br>
<span class="">> test cases in v3) including the introduction of four new chapters and<br>
> controls:<br>
> - Identity Management Testing<br>
> - Error Handling<br>
> - Cryptography<br>
> - Client Side Testing<br>
><br>
</span>> *3.* This version of the Testing Guide encourages the community not to<br>
<span class="">> simply accept the test cases outlined in this guide. We encourage<br>
> security testers to integrate with other software testers and devise<br>
> test cases specific to the target application. As we find test cases<br>
> that have wider applicability we encourage the security testing<br>
> community to share them and contribute them to the Testing Guide. This<br>
> will continue to build the application security body of knowledge and<br>
> allow the development of the Testing Guide to be an iterative rather<br>
> than monolithic process.<br>
