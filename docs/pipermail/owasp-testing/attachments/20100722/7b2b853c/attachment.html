<div class="im">&gt;  All session tokens in high value applications SHOULD be tied to a specific<br>
&gt; HTTP client instance (session identifier and IP address).<br>
<br>
</div>There are some clients that sit behind a multi-IP proxy where each request originates from a different IP address.  AOL used to employ such a system.  Those clients, if they&#39;re important, must have an alternate system available that doesn&#39;t rely on the IP address (or allows more than one).<br>

<div class="im"><br>
<br>
&gt; Each form or page nonce SHOULD be removed from the active list as soon as it<br>
&gt; is submitted.<br>
<br>
</div>If the nonce is rotated for each request, know that it will break the browser history (e.g. back button).  The impact to the user can be minimized by requiring they re-authenticate, then automatically perform the original request.<br>
