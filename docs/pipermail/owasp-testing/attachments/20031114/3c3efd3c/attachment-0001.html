  <h1>Sample and default files</h1>
  <h2>November 2003</h2>
  <h2>Dafydd Stuttard</h2>
  <h2><o:p>
  &nbsp;</o:p>
  </h2>
  <h1>Short Description of Issue</h1>
  <h1><span style="FONT-WEIGHT: normal; FONT-SIZE: 12pt; mso-bidi-font-weight: bold">Sample
  and default files are those which appear on a web server by default, as the
  result of common installation options, or as the result of installing
  additional software such as a web application development platform. They
  typically comprise vendor product documentation, small scripts to demonstrate
  particular functionality, and administrative interfaces.<o:p>
  </o:p>
  </span></h1>
  <h1><span style="FONT-WEIGHT: normal; FONT-SIZE: 12pt; mso-bidi-font-weight: bold">These
  type of files present various threats to the security of the web server
  itself, and in consequence to any web application running on it:<o:p>
  </o:p>
  </span></h1>
  <h1 style="MARGIN-LEFT: 35.45pt; TEXT-INDENT: -35.45pt; mso-list: l0 level1 lfo1"><span style="mso-list: Ignore; font-weight: normal; font-size: 8pt; font-family: Wingdings; mso-fareast-font-family: Wingdings; mso-bidi-font-weight: bold; mso-bidi-font-size: 12.0pt; mso-bidi-font-family: Wingdings">l<span style="FONT: 7pt 'Times New Roman'">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
  </span></span><span style="FONT-WEIGHT: normal; FONT-SIZE: 12pt; mso-bidi-font-weight: bold">Vendor
  documentation can provide detailed information about the precise version of
  the web server software, optional modules, and additional software
  installed. This allows an attacker to research vulnerabilities with this
  specific software, and increases the likelihood of a successful attack.<o:p></o:p>
  </span></h1>
  <h1 style="MARGIN-LEFT: 35.45pt; TEXT-INDENT: -35.45pt; mso-list: l0 level1 lfo1"><span style="mso-list: Ignore; font-weight: normal; font-size: 8pt; font-family: Wingdings; mso-fareast-font-family: Wingdings; mso-bidi-font-weight: bold; mso-bidi-font-size: 12.0pt; mso-bidi-font-family: Wingdings">l<span style="FONT: 7pt 'Times New Roman'">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
  </span></span><span style="FONT-WEIGHT: normal; FONT-SIZE: 12pt; mso-bidi-font-weight: bold">Sample
  scripts designed to demonstrate particular functionality frequently have
  serious vulnerabilities which may allow an attacker to compromise the entire
  server. These scripts are not intended to be run as “production” code, and
  are not subject to the same development and testing controls.<o:p>
  </o:p>
  </span></h1>
  <h1 style="MARGIN-LEFT: 35.45pt; TEXT-INDENT: -35.45pt; mso-list: l0 level1 lfo1"><span style="mso-list: Ignore; font-weight: normal; font-size: 8pt; font-family: Wingdings; mso-fareast-font-family: Wingdings; mso-bidi-font-weight: bold; mso-bidi-font-size: 12.0pt; mso-bidi-font-family: Wingdings">l<span style="FONT: 7pt 'Times New Roman'">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
  </span></span><span style="FONT-WEIGHT: normal; FONT-SIZE: 12pt; mso-bidi-font-weight: bold">Administrative
  interfaces may allow an attacker to gain remote administrative control of the
  server. For example, the common IIS program <i style="mso-bidi-font-style: normal">/iisadmpwd/aexp2.htr</i>
  may allow a password guessing attack against user accounts on the server.<o:p>
  </o:p>
  </span></h1>
  <h1>How to Test</h1>
  <h2>Black Box</h2>
  <h2><span style="FONT-WEIGHT: normal; FONT-SIZE: 12pt; mso-bidi-font-weight: bold">If
  it is already known which web server or other software is installed, then
  listings of sample and default files are likely to be publicly available. For
  example, the following are some common sample files on Microsoft IIS web
  server:<o:p>
  </o:p>
  </span></h2>
  <pre style="MARGIN-LEFT: 36pt">/<span class="SpellE">iissamples/issamples/fastq.idq
</span>/<span class="SpellE">iissamples/issamples/query.idq</span><span style="COLOR: black">
/<span class="SpellE">iissamples/sdk/asp/docs/CodeBrws.asp
</span></span>/scripts/samples/search/<span class="SpellE">qfullhit.htw
</span>/scripts/samples/search/<span class="SpellE">qsumrhit.htw
</span>/scripts/samples/search/<span class="SpellE">author.idq
</span>/scripts/samples/search/<span class="SpellE">filesize.idq
</span>/scripts/samples/search/<span class="SpellE">filetime.idq
</span>/scripts/samples/search/<span class="SpellE">queryhit.idq
</span>/scripts/samples/search/<span class="SpellE">simple.idq</span></pre>
  <h2><span style="FONT-WEIGHT: normal; FONT-SIZE: 12pt; mso-bidi-font-weight: bold">If
  it is feasible, perform a clean lab installation of the software being tested, and
  examine this to identify all sample and default files. Check whether each file
  is present on the web server being tested. If so, research any <span class="GramE">vulnerabilities</span>
  that may exist in the files found.<o:p>
  </o:p>
  </span></h2>
  <h2><span style="FONT-WEIGHT: normal; FONT-SIZE: 12pt; mso-bidi-font-weight: bold">If
  it is not already known which software is installed on the server, it is
  necessary to use a list of common files for a variety of servers and software
  products. Security scanners like <span class="SpellE">Nessus</span> perform
  checks for common files from several products.<o:p>
  </o:p>
  </span></h2>
  <h2><span style="FONT-WEIGHT: normal; FONT-SIZE: 12pt; mso-bidi-font-weight: bold"><o:p>
  &nbsp;</o:p>
  </span></h2>
  <h2>White Box</h2>
  <h2><span style="FONT-WEIGHT: normal; FONT-SIZE: 12pt; mso-bidi-font-weight: bold">White
  box testing for sample and default files is typically straightforward. Browse the
  file system of the web server and identify all resources within the <span class="SpellE">webroot</span>.
  Check each resource to confirm whether it is actually required for the
  application, or may be safely removed.<o:p>
  </o:p>
  </span></h2>
