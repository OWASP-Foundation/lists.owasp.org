<p class="MsoNormal"><font><span style="line-height:normal;background-color:rgba(255,255,255,0)">Hi folks,<u></u><u></u></span></font></p><p class="MsoNormal"><font><span style="line-height:normal;background-color:rgba(255,255,255,0)">  As I've recently been appointed as Matteo's Test Guide wingman I thought I would introduce some things I've been working on in relation to the Test Guide.<u></u><u></u></span></font></p>
<p class="MsoNormal"><font><span style="line-height:normal;background-color:rgba(255,255,255,0)"><u></u> <u></u></span></font></p><p class="MsoNormal"><font><span style="line-height:normal;background-color:rgba(255,255,255,0)">The brainstorming is largely complete and we've started reviewing the efficacy of test cases. You'll see some suggested ones have disappeared or changed categories. There were some very cool suggestions that I'd recommend folks consider during testing (e.g. prod data in pre-prod) but reflect procedural vulnerabilities rather than software vulnerabilities. Where possible I've retained the existing test case wording to reduce the stress of transitioning from v3 to v4, but there are some significant changes. The major changes include:<u></u><u></u></span></font></p>
<p class="MsoNormal"><font><span style="line-height:normal;background-color:rgba(255,255,255,0)"><u></u> <u></u></span></font></p><p class="MsoNormal"><font><span style="line-height:normal;background-color:rgba(255,255,255,0)">1) Common Numbering - I've leveraged work done by Dave Wichers and Andrew vd Stock in relation to this and started migrating OWASP test cases into a new numbering scheme that aligns with their earlier work. The new scheme is based on defining security control categories and then defining test cases that test for vulnerabilities related to each control category. However, you'll find that some of these categories relate to technology rather than a security control. The inconsistent application of this principle was done in the interests of usability and familiarity.<u></u><u></u></span></font></p>
<p class="MsoNormal"><font><span style="line-height:normal;background-color:rgba(255,255,255,0)"><u></u> <u></u></span></font></p><p class="MsoNormal"><font><span style="line-height:normal;background-color:rgba(255,255,255,0)">2) Test Case template - I've been working on creating a test case template that makes understanding, executing, responding to and reporting tests easier and more consistent.  <u></u><u></u></span></font></p>
<p class="MsoNormal"><font><span style="line-height:normal;background-color:rgba(255,255,255,0)">For example, it would be helpful if test cases were titled according to:<u></u><u></u></span></font></p><p class="MsoNormal">
<font><span style="line-height:normal;background-color:rgba(255,255,255,0)">[Verb] [Object] for [Vulnerability] (e.g. "Test Inputs for SQL Injection")<u></u><u></u></span></font></p><p class="MsoNormal"><font><span style="line-height:normal;background-color:rgba(255,255,255,0)">It would also be helpful to describe a test case by addressing:<u></u><u></u></span></font></p>
<p class="MsoNormal" style="text-indent:0px"><font><span style="line-height:normal;background-color:rgba(255,255,255,0)">a) summary of vulnerability,<u></u><u></u></span></font></p><p class="MsoNormal" style="margin-left:36pt">
<font><span style="line-height:normal;background-color:rgba(255,255,255,0)">b) summary of test,<u></u><u></u></span></font></p><p class="MsoNormal" style="margin-left:36pt"><font><span style="line-height:normal;background-color:rgba(255,255,255,0)">c) objective of test,<u></u><u></u></span></font></p>
<p class="MsoNormal" style="margin-left:36pt"><font><span style="line-height:normal;background-color:rgba(255,255,255,0)">d) how to test,<u></u><u></u></span></font></p><p class="MsoNormal" style="margin-left:36pt"><font><span style="line-height:normal;background-color:rgba(255,255,255,0)">e) example of test,<u></u><u></u></span></font></p>
<p class="MsoNormal" style="margin-left:36pt"><font><span style="line-height:normal;background-color:rgba(255,255,255,0)">f) tools,<u></u><u></u></span></font></p><p class="MsoNormal" style="margin-left:36pt"><font><span style="line-height:normal;background-color:rgba(255,255,255,0)">g) references to the Dev Guide for fixing the issue, and<u></u><u></u></span></font></p>
<p class="MsoNormal" style="margin-left:36pt"><font><span style="line-height:normal;background-color:rgba(255,255,255,0)">h) vulnerability references.<u></u><u></u></span></font></p><p class="MsoNormal"><font><span style="line-height:normal;background-color:rgba(255,255,255,0)">I'm not inclined to change the legacy test case titles just yet, but I ask for contributors to consider formatting the content test cases in this way.<u></u><u></u></span></font></p>
<p class="MsoNormal"><font><span style="line-height:normal;background-color:rgba(255,255,255,0)"><u></u> <u></u></span></font></p><p class="MsoNormal"><font><span style="line-height:normal;background-color:rgba(255,255,255,0)">3) New control categories - Identity Management, Error Handling, Cryptography, Logging and Client-side security have emerged that either better categorise existing and new test cases. As with the existing test cases, not all will apply to every engagement, but we should aim to be as comprehensive as possible.<u></u><u></u></span></font></p>
<p class="MsoNormal"><font><span style="line-height:normal;background-color:rgba(255,255,255,0)"><u></u> <u></u></span></font></p><p class="MsoNormal"><font><span style="line-height:normal;background-color:rgba(255,255,255,0)">4) Above all, we're not suggesting you stick to the OWASP list of test cases. Generic test cases will only get us so far, and it is imperative that security testers develop their own test cases for functionality that is either highly specific or absent from the OWASP test case list. The Test Guide is the starting point, not the finishing point. But we hope that you'll be able to apply the methodology and templates to these as yet unknown test cases. If you develop a test case you think is particularly relevant then please consider submitting it to the Test Guide.<u></u><u></u></span></font></p>
