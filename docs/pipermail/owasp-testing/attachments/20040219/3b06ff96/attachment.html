<p style='margin-left:.5in'><font size=2 color=black face="Times New Roman"><span

think it has never been more important to get the OWASP Testing Guide 1.0 out
into the public. At this point you can only imagine that any guides they

know from bitter experience (and testing using WebGoat) that those tools

web security issues. </span></font><font color=black><span style='color:black;

<p style='margin-left:.5in'><font size=2 color=black face="Times New Roman"><span
style='font-size:10.0pt;color:black'>Testing security of applications must be
about testing the development process, and implementation and not just pen
testing which is after the fact, &quot;hit and miss&quot; and costly. It has to
focus on the big picture. I know that the majority of organizations in the US I
speak to are trying to move away from the pen test mentality having seen the
futility and I hope the OWASP Testing Guide can help layout the foundation for
an enterprise testing framework for everyone.&nbsp; </span></font><font
color=black><span style='color:black;mso-color-alt:windowtext'><o:p></o:p></span></font></p>
