</span></font></span></span></font></i></b><![endif]><span dir=LTR>Causes<o:p></o:p></span></p>

<p class=MsoNormal dir=LTR style='text-align:left;direction:ltr;unicode-bidi:
embed'><font size=3 face="Times New Roman"><span style='font-size:12.0pt'>The
source for this problem is often inexperienced IT personnel, unaware of the
importance of changing default passwords on installed infrastructure components,
programmers, leaving backdoors so they can easily access and test the
application, later forgetting to remove them, application administrators and
users that chose an easy username and password for themselves, and application
with built in, non-removable default accounts with a preset username and
password. Another problem is blank passwords, which are simply a result of
security unawareness and willingness to simplify things.<o:p></o:p></span></font></p>

<p class=StyleHeading2 dir=LTR style='mso-margin-top-alt:12.0pt;margin-right:
0in;margin-bottom:3.0pt;margin-left:.4in;text-align:left;direction:ltr;
unicode-bidi:embed'><![if !supportLists]><b><i><font size=4 face=Arial><span

</span></font></span></span></font></i></b><![endif]><span dir=LTR>Blackbox
Testing<o:p></o:p></span></p>

<h3 dir=LTR style='mso-margin-top-alt:12.0pt;margin-right:0in;margin-bottom:
3.0pt;margin-left:.5in;text-align:left;direction:ltr;unicode-bidi:embed'><![if !supportLists]><b><font
size=4 face=Arial><span style='font-size:13.0pt;mso-fareast-font-family:Arial'><span
style='mso-list:Ignore'>1.2.1<font size=1 face="Times New Roman"><span
style='font:7.0pt "Times New Roman"'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </span></font></span></span></font></b><![endif]><span
dir=LTR>Manual<o:p></o:p></span></h3>

<p class=MsoNormal dir=LTR style='text-align:left;direction:ltr;unicode-bidi:
embed'><i><font size=3 face="Times New Roman"><span style='font-size:12.0pt;
font-style:italic'>Note - before approaching the test, it is important to
understand whether the environment you are testing is a production environment.
Modifying passwords or locking out users on a production environment can lead
to undesired consequences. <o:p></o:p></span></font></i></p>

<p class=MsoNormal dir=LTR style='text-align:left;direction:ltr;unicode-bidi:
embed'><i><font size=3 face="Times New Roman"><span style='font-size:12.0pt;
font-style:italic'><o:p>&nbsp;</o:p></span></font></i></p>

<p class=MsoNormal dir=LTR style='text-align:left;direction:ltr;unicode-bidi:
embed'><font size=3 face="Times New Roman"><span style='font-size:12.0pt'>In
blackbox testing we know nothing about the application, its underlying
infrastructure, and any username and/or password policies. Often this is not
the case and some information about the application is provided – simply skip
the steps that refer to obtaining information you already have.<o:p></o:p></span></font></p>

<p class=MsoNormal dir=LTR style='text-align:left;direction:ltr;unicode-bidi:
embed'><font size=3 face="Times New Roman"><span style='font-size:12.0pt'><o:p>&nbsp;</o:p></span></font></p>

<p class=MsoNormal dir=LTR style='text-align:left;direction:ltr;unicode-bidi:
embed'><font size=3 face="Times New Roman"><span style='font-size:12.0pt'>When
testing a known application interface, such as a Cisco router web interface, or
Weblogic admin access, check the known usernames and passwords for these
devices. This can be done either by Google, or using one of the references in
the Further Reading section.<o:p></o:p></span></font></p>

<p class=MsoNormal dir=LTR style='text-align:left;direction:ltr;unicode-bidi:
embed'><font size=3 face="Times New Roman"><span style='font-size:12.0pt'><o:p>&nbsp;</o:p></span></font></p>

<p class=MsoNormal dir=LTR style='text-align:left;direction:ltr;unicode-bidi:
embed'><font size=3 face="Times New Roman"><span style='font-size:12.0pt'>When
facing a homegrown application, to which we do not have a list of default and
common user accounts, we need to test it manually, following these guidelines:<o:p></o:p></span></font></p>

<p class=MsoListBullet dir=LTR style='mso-margin-top-alt:0in;margin-right:0in;
margin-bottom:0in;margin-left:.25in;margin-bottom:.0001pt;text-align:left;
direction:ltr;unicode-bidi:embed'><![if !supportLists]><font size=3
face=Symbol><span style='font-size:12.0pt;font-family:Symbol;mso-fareast-font-family:
Symbol;mso-bidi-font-family:Symbol'><span style='mso-list:Ignore'>·<font
size=1 face="Times New Roman"><span style='font:7.0pt "Times New Roman"'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
</span></font></span></span></font><![endif]><span dir=LTR>Try the following
usernames - &quot;admin&quot;, &quot;administrator&quot;, &quot;root&quot;,
&quot;system&quot;, or &quot;super&quot;. These are popular among system
administrators and are often used. Additionally you could try &quot;qa&quot;,
&quot;test&quot;, &quot;test1&quot;, &quot;testing&quot;, and similar names.
Attempt any combination of the above in both the username and the password
fields. If the application is vulnerable to username enumeration, and you
successfully managed to identify any of the above usernames, attempt passwords
in a similar manner.<o:p></o:p></span></p>

<p class=MsoListBullet dir=LTR style='mso-margin-top-alt:0in;margin-right:0in;
margin-bottom:0in;margin-left:.25in;margin-bottom:.0001pt;text-align:left;
direction:ltr;unicode-bidi:embed'><![if !supportLists]><font size=3
face=Symbol><span style='font-size:12.0pt;font-family:Symbol;mso-fareast-font-family:
Symbol;mso-bidi-font-family:Symbol'><span style='mso-list:Ignore'>·<font
size=1 face="Times New Roman"><span style='font:7.0pt "Times New Roman"'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
</span></font></span></span></font><![endif]><span dir=LTR>Application
administrative users are often named after the application. This means if you
are testing an application named &quot;Obscurity&quot;, try using <i><span
style='font-style:italic'>obscurity/obscurity </span></i>as the username and
password.<o:p></o:p></span></p>

<p class=MsoListBullet dir=LTR style='mso-margin-top-alt:0in;margin-right:0in;
margin-bottom:0in;margin-left:.25in;margin-bottom:.0001pt;text-align:left;
direction:ltr;unicode-bidi:embed'><![if !supportLists]><font size=3

</span></font></span></span></font><![endif]><span dir=LTR>When performing a
test for a customer, attempt using names of contacts you have received as
usernames.<o:p></o:p></span></p>

<p class=MsoListBullet dir=LTR style='mso-margin-top-alt:0in;margin-right:0in;
margin-bottom:0in;margin-left:.25in;margin-bottom:.0001pt;text-align:left;

</span></font></span></span></font><![endif]><span dir=LTR>Attempt using all
the above usernames with blank passwords.<o:p></o:p></span></p>

<p class=MsoNormal dir=LTR style='text-align:left;direction:ltr;unicode-bidi:
embed'><font size=3 face="Times New Roman"><span style='font-size:12.0pt'><o:p>&nbsp;</o:p></span></font></p>
