Hi folks,<br><br>As those of you also on the Leaders list will have seen I've started a <a href="https://www.owasp.org/index.php/Web_Application_Security_Testing_Cheat_Sheet" target="_blank">Web Application Security Testing Cheat Sheet</a>.<br>
<br>To quote from that page:<br><h1 style="margin-left:40px">
<font size="4"><span>Introduction </span></font></h1>
<p style="margin-left:40px">This cheat sheet provides a checklist of tasks to be performed when performing a blackbox security test of a web application.
</p>
<h1 style="margin-left:40px"> <span> <font size="4">Purpose </font></span></h1>
<p style="margin-left:40px">This checklist is intended to be used as an aide memoire for experienced pentesters and should be used in conjunction with the <a href="https://www.owasp.org/index.php/Category:OWASP_Testing_Project" title="Category:OWASP Testing Project" target="_blank">OWASP Testing Guide</a>. It will be updated as the <a href="https://www.owasp.org/index.php/OWASP_Application_Testing_guide_v4" title="OWASP Application Testing guide v4" target="_blank">Testing Guide v4</a> is progressed.
</p><p style="margin-left:40px">The intention is that this guide will be available as an XML 
document, with scripts that convert it into formats such as pdf, Media 
Wiki markup, HTML etc. 
</p><p style="margin-left:40px">This will allow it to be consumed within security tools as well as being available in a format suitable for printing.
</p><p style="margin-left:40px">It is currently at a very early stage, <span style>but any feedback or offers of help will be appreciated.</span></p><p><br></p><p>As a lot of it comes from the Testing Guide v3, all of the guide authors are credited (as a group rather than individually).<br>
</p><p></p><p>My initial draft has already been improved by various people (thanks!), but I'm sure theres plenty more improvements possible.<br></p><p>So please have a look at it, post any feedback to this thread, and feel free to add more content to the wiki!<br>
