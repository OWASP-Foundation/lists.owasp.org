  <h1>Old, Backup and Unreferenced Files</h1>
  <h2>November 2003</h2>
  <h2>Dafydd Stuttard</h2>
  <h2><o:p>
  &nbsp;&nbsp;
  </h2>
  <h1>Short Description of Issue</h1>
  <h1><span style="FONT-WEIGHT: normal; FONT-SIZE: 12pt; mso-bidi-font-weight: bold">Old,
  backup and unreferenced files present various threats to the security of a web
  application:<o:p>
  </o:p>
  </span></h1>
  <h1 style="MARGIN-LEFT: 35.45pt; TEXT-INDENT: -35.45pt; mso-list: l3 level1 lfo2"><span style="mso-list: Ignore; font-weight: normal; font-size: 8pt; font-family: Wingdings; mso-fareast-font-family: Wingdings; mso-bidi-font-weight: bold; mso-bidi-font-size: 12.0pt; mso-bidi-font-family: Wingdings">l<span style="FONT: 7pt 'Times New Roman'">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
  </span></span><span style="FONT-WEIGHT: normal; FONT-SIZE: 12pt; mso-bidi-font-weight: bold">Unreferenced
  files may disclose sensitive information that can facilitate a focused attack
  against the application; for example include files containing database
  credentials,<a name="wsftp"> configuration files containing</a></span><span style="font-size: 10pt; color: black; font-family: Arial; mso-bookmark: wsftp">
  </span><span style="FONT-WEIGHT: normal; FONT-SIZE: 12pt; mso-bidi-font-weight: bold">references
  to other hidden content, absolute file paths, etc.<o:p>
  </o:p>
  </span></h1>
  <h1 style="MARGIN-LEFT: 35.45pt; TEXT-INDENT: -35.45pt; mso-list: l3 level1 lfo2"><span style="mso-list: Ignore; font-weight: normal; font-size: 8pt; font-family: Wingdings; mso-fareast-font-family: Wingdings; mso-bidi-font-weight: bold; mso-bidi-font-size: 12.0pt; mso-bidi-font-family: Wingdings">l<span style="FONT: 7pt 'Times New Roman'">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
  </span></span><span style="FONT-WEIGHT: normal; FONT-SIZE: 12pt; mso-bidi-font-weight: bold">Unreferenced
  pages may contain powerful functionality that can be used to attack the
  application; for example an administration page that is not linked from
  published content but can be accessed by any user who knows where to find it.<o:p>
  </o:p>
  </span></h1>
  <h1 style="MARGIN-LEFT: 35.45pt; TEXT-INDENT: -35.45pt; mso-list: l3 level1 lfo2"><span style="mso-list: Ignore; font-weight: normal; font-size: 8pt; font-family: Wingdings; mso-fareast-font-family: Wingdings; mso-bidi-font-weight: bold; mso-bidi-font-size: 12.0pt; mso-bidi-font-family: Wingdings">l<span style="FONT: 7pt 'Times New Roman'">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
  </span></span><span style="FONT-WEIGHT: normal; FONT-SIZE: 12pt; mso-bidi-font-weight: bold">Old
  and backup files may contain vulnerabilities that have been fixed in more
  recent versions; for example <span class="SpellE"><i style="mso-bidi-font-style: normal">viewdoc.old.jsp</i></span>
  may contain a directory traversal vulnerability that has been fixed in <span class="SpellE"><i style="mso-bidi-font-style: normal">viewdoc.jsp</i></span>
  but can still be exploited by anyone who finds the old version.<o:p>
  </o:p>
  </span></h1>
  <h1 style="MARGIN-LEFT: 35.45pt; TEXT-INDENT: -35.45pt; mso-list: l3 level1 lfo2"><span style="mso-list: Ignore; font-weight: normal; font-size: 8pt; font-family: Wingdings; mso-fareast-font-family: Wingdings; mso-bidi-font-weight: bold; mso-bidi-font-size: 12.0pt; mso-bidi-font-family: Wingdings">l<span style="FONT: 7pt 'Times New Roman'">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
  </span></span><span style="FONT-WEIGHT: normal; FONT-SIZE: 12pt; mso-bidi-font-weight: bold">Backup
  files may disclose the source code for pages designed to execute on the
  server; for example requesting <span class="SpellE"><i style="mso-bidi-font-style: normal">viewdoc.bak</i></span>
  may return the source code for <span class="SpellE"><i style="mso-bidi-font-style: normal">viewdoc.jsp</i></span>,
  which can be reviewed for vulnerabilities that may be difficult to find by
  making blind requests to the executable page.<o:p>
  </o:p>
  </span></h1>
  <h1 style="MARGIN-LEFT: 35.45pt; TEXT-INDENT: -35.45pt; mso-list: l3 level1 lfo2"><span style="mso-list: Ignore; font-weight: normal; font-size: 8pt; font-family: Wingdings; mso-fareast-font-family: Wingdings; mso-bidi-font-weight: bold; mso-bidi-font-size: 12.0pt; mso-bidi-font-family: Wingdings">l<span style="FONT: 7pt 'Times New Roman'">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
  </span></span><span style="FONT-WEIGHT: normal; FONT-SIZE: 12pt; mso-bidi-font-weight: bold">Backup
  archives may contain copies of all files within (or even outside) the <span class="SpellE">webroot</span>.
  This allows an attacker to quickly enumerate the entire application, including
  unreferenced pages, source code, include files, etc.<o:p>
  </o:p>
  </span></h1>
  <h1 style="MARGIN-LEFT: 35.45pt; TEXT-INDENT: -35.45pt; mso-list: l3 level1 lfo2"><span style="mso-list: Ignore; font-weight: normal; font-size: 8pt; font-family: Wingdings; mso-fareast-font-family: Wingdings; mso-bidi-font-weight: bold; mso-bidi-font-size: 12.0pt; mso-bidi-font-family: Wingdings">l<span style="FONT: 7pt 'Times New Roman'">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
  </span></span><span style="FONT-WEIGHT: normal; FONT-SIZE: 12pt; mso-bidi-font-weight: bold">Log
  files may contain sensitive information about the activities of application
  users, for example sensitive data passed in URL parameters, session IDs, URLs
  visited (which may disclose additional unreferenced content), etc. Other log
  files (e.g. ftp logs) may contain sensitive information about the maintenance
  of the application by system administrators.<o:p>
  </o:p>
  </span></h1>
  <h1>How to Test</h1>
  <h2>Black Box</h2>
  <h2><span style="FONT-WEIGHT: normal; FONT-SIZE: 12pt; mso-bidi-font-weight: bold">Testing
  for unreferenced files uses both automated and manual techniques, and
  typically involves a combination of the following:<o:p>
  </o:p>
  </span></h2>
  <h2><i style="mso-bidi-font-style: normal"><span style="FONT-WEIGHT: normal; FONT-SIZE: 12pt; mso-bidi-font-weight: bold">(<span class="SpellE">i</span>)<span style="mso-tab-count: 1">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
  </span>Inference from the naming scheme used for published content<o:p>
  </o:p>
  </span></i></h2>
  <h2 style="MARGIN-LEFT: 36pt"><span style="FONT-WEIGHT: normal; FONT-SIZE: 12pt; mso-bidi-font-weight: bold">If
  not already done, enumerate all of the application’s pages and
  functionality. This can be done manually using a browser, or using an
  application <span class="SpellE">spidering</span> tool. Most applications use
  a recognisable naming scheme, and organise resources into pages and
  directories using words that describe their function. From the naming scheme
  used for published content, it is often possible to infer the name and
  location of unreferenced pages. For example, if a page <span class="SpellE"><i style="mso-bidi-font-style: normal">viewuser.asp</i></span>
  is found, then look also for <span class="SpellE"><i style="mso-bidi-font-style: normal">edituser.asp</i></span>,
  <span class="SpellE"><i style="mso-bidi-font-style: normal">adduser.asp</i></span>
  and <span class="SpellE"><i style="mso-bidi-font-style: normal">deleteuser.asp</i></span>.
  If a directory <i style="mso-bidi-font-style: normal">/app/user</i> is found,
  then look also for <i style="mso-bidi-font-style: normal">/app/admin</i> and <i style="mso-bidi-font-style: normal">/app/manager</i>.<o:p>
  </o:p>
  </span></h2>
  <h2><i style="mso-bidi-font-style: normal"><span style="FONT-WEIGHT: normal; FONT-SIZE: 12pt; mso-bidi-font-weight: bold">(ii)<span style="mso-tab-count: 1">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
  </span>Other clues in published content<o:p>
  </o:p>
  </span></i></h2>
  <h2 style="MARGIN-LEFT: 36pt"><span style="FONT-WEIGHT: normal; FONT-SIZE: 12pt; mso-bidi-font-weight: bold">Many
  web applications leave clues in published content that can lead to the
  discovery of hidden pages and functionality. These clues often appear in the
  source code of HTML and JavaScript files. The source code for all published
  content should be manually reviewed to identify clues about other pages and
  functionality. For example:<o:p>
  </o:p>
  </span></h2>
  <h2 style="MARGIN-LEFT: 36pt"><span style="FONT-WEIGHT: normal; FONT-SIZE: 12pt; mso-bidi-font-weight: bold">Programmers’
  comments and commented-out sections of source code may refer to hidden
  content:<o:p>
  </o:p>
  </span></h2>
  <h2 style="MARGIN-LEFT: 72pt"><span style="FONT-WEIGHT: normal; FONT-SIZE: 10pt; FONT-FAMILY: 'Courier New'; mso-bidi-font-weight: bold">&lt;!--
  &lt;A HREF=&quot;<span class="SpellE">uploadfile.jsp</span>&quot;&gt;Upload a
  document to the server&lt;/A&gt; --&gt;<br>
  &lt;!-- Link removed while bugs in <span class="SpellE">uploadfile.jsp</span>
  are fixed<span style="mso-spacerun: yes">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
  </span>--&gt;<o:p>
  </o:p>
  </span></h2>
  <h2 style="MARGIN-LEFT: 36pt"><span style="FONT-WEIGHT: normal; FONT-SIZE: 12pt; mso-bidi-font-weight: bold">JavaScript
  may contain page links that are only rendered within the user’s GUI under
  certain circumstances:<o:p>
  </o:p>
  </span></h2>
  <h2 style="MARGIN-LEFT: 72pt"><span style="FONT-WEIGHT: normal; FONT-SIZE: 10pt; FONT-FAMILY: 'Courier New'; mso-bidi-font-weight: bold" class="SpellE">var</span><span style="FONT-WEIGHT: normal; FONT-SIZE: 10pt; FONT-FAMILY: 'Courier New'; mso-bidi-font-weight: bold">
  <span class="SpellE">adminUser</span>=false;<br>
  :<br>
  if (<span class="SpellE">adminUser</span>) <span class="SpellE">menu.add</span>
  (new <span class="SpellE">menuItem</span> (&quot;Maintain users&quot;,
  &quot;/admin/<span class="SpellE">useradmin.jsp</span>&quot;));<o:p>
  </o:p>
  </span></h2>
  <h2 style="MARGIN-LEFT: 36pt"><span style="FONT-WEIGHT: normal; FONT-SIZE: 12pt; mso-bidi-font-weight: bold">HTML
  pages may contain <span class="SpellE">FORMs</span> that have been hidden by
  disabling the SUBMIT element:<o:p>
  </o:p>
  </span></h2>
  <h2 style="MARGIN-LEFT: 72pt"><span style="FONT-WEIGHT: normal; FONT-SIZE: 10pt; FONT-FAMILY: 'Courier New'; mso-bidi-font-weight: bold">&lt;FORM
  action=&quot;<span class="SpellE">forgotPassword.jsp</span>&quot;
  method=&quot;post&quot;&gt;<br>
  <span style="mso-tab-count: 1">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </span>&lt;INPUT
  type=&quot;hidden&quot; name=&quot;<span class="SpellE">userID</span>&quot;
  value=&quot;123&quot;&gt;<br>
  <span style="mso-tab-count: 1">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </span>&lt;!--
  &lt;INPUT type=&quot;submit&quot; value=&quot;Forgot Password&quot;&gt; --&gt;<br>
  &lt;/FORM&gt;<o:p>
  </o:p>
  </span></h2>
  <h2 style="MARGIN-LEFT: 36pt"><span style="FONT-WEIGHT: normal; FONT-SIZE: 12pt; mso-bidi-font-weight: bold">Another
  source of clues about unreferenced directories is the /<i style="mso-bidi-font-style: normal">robots.txt</i>
  file used to provide instructions to web robots:<o:p>
  </o:p>
  </span></h2>
  <h2 style="MARGIN-LEFT: 72pt"><span style="FONT-WEIGHT: normal; FONT-SIZE: 10pt; FONT-FAMILY: 'Courier New'; mso-bidi-font-weight: bold">User-agent:
  *<br>
  Disallow: /Admin<br>
  Disallow: /uploads<br>
  Disallow: /backup<br>
  Disallow: /~<span class="SpellE">jbloggs</span><br>
  Disallow: /include<o:p>
  </o:p>
  </span></h2>
  <h2><i style="mso-bidi-font-style: normal"><span style="FONT-WEIGHT: normal; FONT-SIZE: 12pt; mso-bidi-font-weight: bold">(iii)<span style="mso-tab-count: 1">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
  </span>Blind guessing</span></i><span style="FONT-WEIGHT: normal; FONT-SIZE: 12pt; mso-bidi-font-weight: bold"><o:p>
  </o:p>
  </span></h2>
  <h2 style="MARGIN-LEFT: 36pt"><span style="FONT-WEIGHT: normal; FONT-SIZE: 12pt; mso-bidi-font-weight: bold">In
  its simplest form, this involves running a list of common filenames through a
  request engine in an attempt to guess files and directories that exist on the
  server. The following <span class="SpellE">netcat</span> wrapper script will
  read a wordlist from <span class="SpellE">stdin</span> and perform a basic
  guessing attack:<o:p>
  </o:p>
  </span></h2>
  <h2 style="MARGIN-LEFT: 72pt"><span style="FONT-WEIGHT: normal; FONT-SIZE: 10pt; FONT-FAMILY: 'Courier New'; mso-bidi-font-weight: bold">#!/bin/bash<br>
  <br>
  server=www.targetapp.com<br>
  port=80<br>
  <br>
  while read <span class="SpellE">url</span><br>
  do<br>
  echo -<span class="SpellE">ne</span> &quot;$<span class="SpellE">url</span>\t&quot;<br>
  echo -e &quot;GET /$<span class="SpellE">url</span> HTTP/1.0\nHost:
  $server\n&quot; | <span class="SpellE">netcat</span> $server $port | head -1<br>
  done | tee <span class="SpellE">outputfile</span><o:p>
  </o:p>
  </span></h2>
  <h2 style="MARGIN-LEFT: 36pt"><span style="FONT-WEIGHT: normal; FONT-SIZE: 12pt; mso-bidi-font-weight: bold">Depending
  upon the server, GET may be replaced with HEAD for faster results. The <span class="SpellE">outputfile</span>
  specified can be <span class="SpellE">grepped</span> for “interesting”
  response codes. The response code 200 (OK) usually indicates that a valid
  resource has been found (provided the server does not deliver a custom “not
  found” page using the 200 code). But also look out for 301 (Moved), 302
  (Found), 401 (Unauthorized), 403 (Forbidden) and 500 (Internal error), which
  may also indicate resources or directories that are worthy of further
  investigation.<o:p>
  </o:p>
  </span></h2>
  <h2 style="MARGIN-LEFT: 36pt"><span style="FONT-WEIGHT: normal; FONT-SIZE: 12pt; mso-bidi-font-weight: bold">The
  basic guessing attack should be run against the <span class="SpellE">webroot</span>,
  and also against all directories that have been identified through other
  enumeration techniques. More advanced/effective guessing attacks can be
  performed as follows:<o:p>
  </o:p>
  </span></h2>
  <h2 style="MARGIN-LEFT: 71.45pt; TEXT-INDENT: -35.45pt; mso-list: l1 level1 lfo4"><span style="mso-list: Ignore; font-weight: normal; font-size: 8pt; font-family: Wingdings; mso-fareast-font-family: Wingdings; mso-bidi-font-weight: bold; mso-bidi-font-size: 12.0pt; mso-bidi-font-family: Wingdings">l<span style="FONT: 7pt 'Times New Roman'">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
  </span></span><span style="FONT-WEIGHT: normal; FONT-SIZE: 12pt; mso-bidi-font-weight: bold">Identify
  the file extensions in use within known areas of the application (e.g. <span class="SpellE">jsp</span>,
  <span class="SpellE">aspx</span>, html), and use a basic wordlist appended
  with each of these extensions (or use a longer list of common extensions if
  resources permit).<o:p>
  </o:p>
  </span></h2>
  <h2 style="MARGIN-LEFT: 71.45pt; TEXT-INDENT: -35.45pt; mso-list: l1 level1 lfo4"><span style="mso-list: Ignore; font-weight: normal; font-size: 8pt; font-family: Wingdings; mso-fareast-font-family: Wingdings; mso-bidi-font-weight: bold; mso-bidi-font-size: 12.0pt; mso-bidi-font-family: Wingdings">l<span style="FONT: 7pt 'Times New Roman'">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
  </span></span><span style="FONT-WEIGHT: normal; FONT-SIZE: 12pt; mso-bidi-font-weight: bold">For
  each file identified through other enumeration techniques, create a custom
  wordlist derived from that filename. Get a list of common file extensions
  (including <span class="SpellE">bak</span>, txt, <span class="SpellE">src</span>,
  dev, old, inc, etc.) and use each extension before, after, and instead of, the
  extension of the actual filename.<o:p>
  </o:p>
  </span></h2>
  <h2><i style="mso-bidi-font-style: normal"><span style="FONT-WEIGHT: normal; FONT-SIZE: 12pt; mso-bidi-font-weight: bold">(iv)<span style="mso-tab-count: 1">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
  </span>Information obtained through server vulnerabilities and <span class="SpellE">misconfiguration</span><o:p>
  </o:p>
  </span></i></h2>
  <h2 style="MARGIN-LEFT: 36pt"><span style="FONT-WEIGHT: normal; FONT-SIZE: 12pt; mso-bidi-font-weight: bold">The
  most obvious way in which a <span class="SpellE">misconfigured</span> server
  may disclose unreferenced pages is through directory listing. Request all
  enumerated directories to identify any which provide a directory listing.<o:p>
  </o:p>
  </span></h2>
  <h2 style="MARGIN-LEFT: 36pt"><span style="FONT-WEIGHT: normal; FONT-SIZE: 12pt; mso-bidi-font-weight: bold">Numerous
  vulnerabilities have been found in individual web servers which allow an
  attacker to enumerate unreferenced content, for example:<o:p>
  </o:p>
  </span></h2>

  </span></span><span style="FONT-WEIGHT: normal; FONT-SIZE: 12pt; mso-bidi-font-weight: bold">Apache

  </span></span><span style="FONT-WEIGHT: normal; FONT-SIZE: 12pt; mso-bidi-font-weight: bold">Various

  <h2><i style="mso-bidi-font-style: normal"><span style="FONT-WEIGHT: normal; FONT-SIZE: 12pt; mso-bidi-font-weight: bold">(v)<span style="mso-tab-count: 1">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
  </span>Use of publicly available information<o:p>
  </o:p>
  </span></i></h2>
  <h2 style="MARGIN-LEFT: 36pt"><span style="FONT-WEIGHT: normal; FONT-SIZE: 12pt; mso-bidi-font-weight: bold">Pages
  and functionality in Internet-facing web applications that are not referenced
  from within the application itself may be referenced from other public domain
  sources. There are various sources of these references:<o:p>
  </o:p>
  </span></h2>
  <h2 style="MARGIN-LEFT: 71.45pt; TEXT-INDENT: -35.45pt; mso-list: l4 level1 lfo8"><span style="mso-list: Ignore; font-weight: normal; font-size: 8pt; font-family: Wingdings; mso-fareast-font-family: Wingdings; mso-bidi-font-weight: bold; mso-bidi-font-size: 12.0pt; mso-bidi-font-family: Wingdings">l<span style="FONT: 7pt 'Times New Roman'">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
  </span></span><span style="FONT-WEIGHT: normal; FONT-SIZE: 12pt; mso-bidi-font-weight: bold">Pages
  that used to be referenced may still appear in the archives of Internet search
  engines. For example, <i style="mso-bidi-font-style: normal">1998results.asp</i>
  may no longer be linked from a company’s website, but may remain on the
  server and in search engine databases. This old script may contain
  vulnerabilities that could be used to compromise the entire site. The
  following <span class="SpellE">Google</span> search query will return all
  pages for the target website: site:www.targetapp.com -<span class="SpellE">zzz</span><o:p>
  </o:p>
  </span></h2>
  <h2 style="MARGIN-LEFT: 71.45pt; TEXT-INDENT: -35.45pt; mso-list: l4 level1 lfo8"><span style="mso-list: Ignore; font-weight: normal; font-size: 8pt; font-family: Wingdings; mso-fareast-font-family: Wingdings; mso-bidi-font-weight: bold; mso-bidi-font-size: 12.0pt; mso-bidi-font-family: Wingdings">l<span style="FONT: 7pt 'Times New Roman'">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
  </span></span><span style="FONT-WEIGHT: normal; FONT-SIZE: 12pt; mso-bidi-font-weight: bold">In
  addition, <span class="SpellE">Google</span> and Yahoo keep cached versions of
  pages found by their robots. Even if <i style="mso-bidi-font-style: normal">1998results.asp</i>
  has been removed from the target server, a version of its output may still be
  stored by these search engines. The cached version may contain references to,
  or clues about, additional hidden content that still remains on the server.<o:p>
  </o:p>
  </span></h2>
  <h2 style="MARGIN-LEFT: 71.45pt; TEXT-INDENT: -35.45pt; mso-list: l4 level1 lfo8"><span style="mso-list: Ignore; font-weight: normal; font-size: 8pt; font-family: Wingdings; mso-fareast-font-family: Wingdings; mso-bidi-font-weight: bold; mso-bidi-font-size: 12.0pt; mso-bidi-font-family: Wingdings">l<span style="FONT: 7pt 'Times New Roman'">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
  </span></span><span style="FONT-WEIGHT: normal; FONT-SIZE: 12pt; mso-bidi-font-weight: bold">Content
  that is not referenced from within a target application may be linked to by
  third-party websites. For example, an application which processes online
  payments on behalf of third-party traders may contain a variety of bespoke
  functionality which can (normally) only be found by following links within the
  web sites of its customers.<o:p>
  </o:p>
  </span></h2>
  <h2><span style="FONT-WEIGHT: normal; FONT-SIZE: 12pt; mso-bidi-font-weight: bold"><o:p>
