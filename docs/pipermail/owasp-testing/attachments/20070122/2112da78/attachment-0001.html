<p class=MsoNormal><font size=3 face="Times New Roman"><span style='font-size:
12.0pt'>&gt; of this one) but most issues should be detectable. <br>
<br>
<o:p></o:p></span></font></p>

<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;
font-family:Arial'>I agree we should have a better framework for analyzing code
for simple issues. &nbsp;LAPSE is interesting, but is really a one-trick pony.&nbsp; LAPSE
does source-to-sink dataflow analysis, so it&#8217;s pretty good for analyzing
things like SQL injection and XSS. &nbsp;But it has no ability to analyze
encryption, logging, access control, authentication, error handling,
concurrency, etc&#8230; And it only works on Java.<o:p></o:p></span></font></p>

<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;
font-family:Arial'><o:p>&nbsp;</o:p></span></font></p>

<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;
font-family:Arial'>I think &#8220;most issues should be detectable&#8221; is
too aggressive (I&#8217;ve done quite a lot of work in this space). &nbsp;That&#8217;s
what the commercial static analysis tool vendors are trying to do. &nbsp;I suggest
we focus on tools that assist the manual code reviewer, and DO NOT try to find
problems automatically.<o:p></o:p></span></font></p>

<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;
font-family:Arial'><o:p>&nbsp;</o:p></span></font></p>

<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;
font-family:Arial'>For example, a tool that finds and flags all the encryption
code is easy and valuable. &nbsp;Maybe it helps me navigate the code with &#8220;security
goggles&#8221; on. &nbsp;A tool that attempts to analyze the encryption code and
determine if it is sound is ridiculously hard and will have lots of false
alarms.<o:p></o:p></span></font></p>

<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;
font-family:Arial'><o:p>&nbsp;</o:p></span></font></p>

<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;
font-family:Arial'>The tool must have some compiler-like features &#8211; at least
symbol resolution, because grep is too inaccurate.<o:p></o:p></span></font></p>
