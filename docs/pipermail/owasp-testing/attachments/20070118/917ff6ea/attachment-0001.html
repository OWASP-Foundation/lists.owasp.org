<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;
font-family:Arial'>I&#8217;d strongly recommend using bytecode for Java and
.NET. &nbsp;The compiler handles a <st1:place w:st="on">LOT</st1:place> of sticky
issues for you &#8211; like resolving symbols, so you can make much more
sophisticated rules.&nbsp; For example if you see a call to runtime.exec() in a Java
program, what type is the &#8220;runtime&#8221; variable &#8211; probably Runtime,
but it could be something else. &nbsp;All variables are resolved in the bytecode.<o:p></o:p></span></font></p>

Gosling&#8217;s Jackpot engine is really really cool. &nbsp;It&#8217;s basically an
API for source code. See <a href="http://jackpot.netbeans.org/">http://jackpot.netbeans.org/</a>.
&nbsp;You can use it to find problems, and ALSO to actually transform the code.&nbsp; It&#8217;s
integrated into netbeans, but I spent a few hours to extract it and make a
command line version. &nbsp;Never did anything with it though.&nbsp; If anyone wants to
