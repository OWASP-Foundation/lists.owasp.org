A good question, and it might be useful to add these somewhere (the
wiki?) as a list of topics to be considered, so we don't lose the
information?

I'm working with Vinay and Shankar as on:

    ENISA Common Assurance Maturity Model (name changed from wiki page URL)
    http://www.owasp.org/index.php/Industry:ENISA_Cloud_Computing_Common_Assurance_Metrics

ENISA/CAM have asked the OWASP contributors to work on the "software
development" sub-domain to identify security controls.  Specifically
they are interested what's in CSA docs, ENISA IAF, ISO 27001, ITIL,
COBIT and PCIDSS, but we're tryinmg to use SAMM with SafeCODE and
BSIMM to get greater coverage.  These other "standards" do address
some operational issues more widely.  But I wonder if anyone has done
any work identifying the overlap between SAMM and these, especially
BSIMM(2)?
