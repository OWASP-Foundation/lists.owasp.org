Kevin,
Do you have an example you can share?
Right now in the Roadmap tab, the four phases are not defaulting to the
interview value, so if the Interview answer is higher than the current
answers in the future phases it can show a decline in scoring.

The intention with the Roadmap tab is to project the answer for security
practices in future phases.
This is the first iteration and it is largely a manual process at the
moment.
Answers for each of the phases will need to be set based on projects and
initiatives underway, sometimes the answers will stay the same and some
times they should change (for better or worse).

If you look at the phases for the security practices in question, do the
answers in future phases show in red?
If so they just need to be adjusted to align with the answered question if
they aren't truly degrading.

I considered reseting all the phase answers to match the interview answer
any time it changes, but I didn't want to mess up projections that people
didn't intend to change.  I didn't want to get into complicated logic in
Excel so I didn't implement changing defaults, that's something that will
likely be done in a webapp in the future.

Please let me know if that answers your question or not.

Thanks,
Brian
