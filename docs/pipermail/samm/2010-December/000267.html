I was thinking about making a publicy available version of the SAMM 
1.0 document that would be readable on ebook readers like Kindle. 
This requires an OPF file with metadata etc which is trivial and 
properly formatted content - from what I've seen plain HTML works 
best.

I started (just to see how it would work) by converting PDF to TXT 
and then reformatting the content in HTML with diagrams as PNG 
images. My questions to the group would be:

1) maybe anyone did that previously? (I found a paperback version 
of SAMM on Amazon, but no ebook)
2) if not, what are your thoughts about myself doing that - and 
maybe there's someone willing to help?
3) can you think of any better source data to start with? (most 
conversions from ODF or PDF I've tried produces content that looks 
ugly on ebook readers though)

The OPF source would be eventually publicly available, so that 
everyone can convert it to whatever format they use (EPUB, MOBI).
