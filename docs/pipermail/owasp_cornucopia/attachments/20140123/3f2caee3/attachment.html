<div dir="ltr">Thanks for all the work on this project.  In the past I've printed the EOP game on tarot sized cards on <a href="http://printerstudio.com">printerstudio.com</a> and played it a few times.  I LOVE the idea of an enterprise webpage edition, so thanks for the effort to pull this off.<div>
<br></div><div>I targeted the tarot sized card because the bigger size would help accommodate the amount of text.  Their sizing guide was very helpful in figuring out the exact size: <a href="http://www.printerstudio.com/pops/pc-temp-guide.html">http://www.printerstudio.com/pops/pc-temp-guide.html</a></div>
<div><br></div><div>I tried a hundred permutations of this, so here are instructions on how I got it to work. The challenge is the web-page auto-scales to the size of their card, but this puts the number and right side of the text in the "cut-zone" meaning it might get cut off.  So these are the steps I took to create the images and resize them:</div>
<div><br></div><div>1. Start with the print ready pdf zip file </div><div>-  (I've no MS office software so this zip file was very helpful)</div><div><br></div><div>2. Create individual images from the pdf using gimp (it doesn't support pdf import)</div>
<div> a. open the pdf files with gimp - a pdf import wizard will pop up</div><div> b. select all images</div><div> c. select open pages as "images" instead of "layers"</div><div> d. set the height to 2994</div>
<div>   - the width will snap to 1794 to preserve the ratio</div><div>   - (2994 is double 1497, the height of a tarot card from the printing bleed-area, not the cut edge.  It's double because the page complains of low resolution if I give the exact dimensions)</div>
<div> e. click import, (see attached picture)</div><div> e. export each image.  This is a bit tedious.  <br></div><div><br></div><div>3. Scale each image to tarot size</div><div>  a. Install gimp-plugin-registry to get "David's Batch Processor" </div>
<div>    - on ubuntu: sudo apt-get install gimp-plugin-registry</div><div>  b. open "David's Batch Processor"</div><div>    - open gimp</div><div>    - click "Filter -> Batch -> Batch Process</div>
<div>  c. add all file on the "input" tab</div><div>  d. add a prefix or postfix on the "rename" tab, something like "scaled"</div><div>  e. On the Resize tab do this:</div><div>   - click enable</div>
<div>   - click absolute</div><div>   - set width = 1794 and height = 2994</div><div>   - set Fit = "Exact"</div><div>   (changes ratio to match tarot)</div><div>  f. click "Start"</div><div><br></div>
<div>4. Resize again with padding (I tried lots of ways to remove this step but I didn't like the results)</div><div>  a. clear input files, </div><div>  b. add "-scaled" images to input</div><div>  c. change prefix of postfix on the "rename" tab, something like "padded"</div>
<div>  d. On the Resize tab do this:</div><div>    - change width to 1894 x 3094</div><div>    - change Fit = "Padded"</div><div>  f. click "Start"</div><div> </div><div>(Step four shows my inability to wield the power of gimp - I feel like a toddler with a chain saw.  I'm not sure why it works. It still isn't quite right, but its close enough)</div>
<div><br></div><div>5. import the padded-scaled images into printerstudio on a tarot size card project with custom front and back images.</div><div><br></div><div>I hope this helps someone.  Thanks again.</div><div><br></div>
