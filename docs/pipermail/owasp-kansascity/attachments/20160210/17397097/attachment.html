<p><b>OWASP ZSC</b></p>

<p>We are preparing to start developing a powerful obfuscation tool OWASP ZSC and looking for some volunteers to contribute the tool project.</p>

<p>OWASP ZSC Project<br>
OWASP ZSC is an open source software in python language which lets you generate customized shellcodes and convert scripts to an obfuscated script. This software can be run on Windows/Linux/OSX under python.</p>

<p>Usage of shellcodes<br>
Shellcodes are small codes in assembly which could be use as the payload in software exploiting. Other usages are in malwares, bypassing anti viruses, obfuscated codes and etc.</p>

<p>Usage of Obfuscate Codes<br>
Can be use for bypassing antiviruses, code protections, same stuff etc…</p>

<p>Why use OWASP ZSC ?<br>
According to other shellcode generators such as metasploit tools and etc, OWASP ZSC using new encodes and methods which antiviruses won't detect. OWASP ZSC encoders are able to generate shellcodes with random encodes that lets you to get thousands of new dynamic shellcodes with the same job in just a second, it means you will not get a same code if you use random encodes with same commands, and that makes OWASP ZSC one of the bests! otherwise it's going to generate shellcodes for other operation systems in the next versions. It’s the same story for the code obfuscation.</p>

<p>There are more details about <a href="http://cts.vresp.com/c/?TheOWASPFoundation/be9a26312a/91418889ff/17d4d5d4ac" target="_blank">how it works and user guides</a> and also <a href="http://cts.vresp.com/c/?TheOWASPFoundation/be9a26312a/91418889ff/1ffdcbd2f7" target="_blank">how to develop</a>. </p>

<p>And whole developer and users guide documents are available for download in <a href="http://cts.vresp.com/c/?TheOWASPFoundation/be9a26312a/91418889ff/c70442c62a" target="_blank">gitbooks</a>.</p>

<p>Developers can add new features and if you don’t have idea but like to develop, you can find the issue which software needed to be fix/add/done <a href="http://cts.vresp.com/c/?TheOWASPFoundation/be9a26312a/91418889ff/1bd7bf2a82" target="_blank">HERE</a>.</p>

<p>After fix/add or develop something, please send your pull request and remember that your code must be compatible with python2 and python3.</p>

<p><b>New ESAPI Release</b></p>

<p>ESAPI project co-leader, Kevin Wall announced his team has just tagged (and signed) a new ESAPI release. The tag name is esapi-2.1.0.1. There are 36 GitHub issues that were closed. You can find full details at: <a>https://github.com/ESAPI/esapi-java-legacy/blob/master/documentation/esapi4java-core-2.1.0.1-release-notes.txt</a>.</p>

<p>Note that there are also some important changes made to the GitHub repo itself. Specifically, we have chosen to adopt a git workflow based on this blog: <a href="http://cts.vresp.com/c/?TheOWASPFoundation/be9a26312a/91418889ff/dd7765f039" target="_blank">http://nvie.com/posts/a-successful-git-branching-model/</a>, where all the new development work will be done on the 'develop' branch and the 'master' branch will henceforth reflect the latest official ESAPI release.</p>

<p>To accommodate this,

</p><ul><li>The 'develop' branch has now been made the DEFAULT branch.</li>
<li>The 'master' branch has now been made a PROTECTED branch.</li></ul>

<p>Chris Schmidt will be uploading this to Maven sometime later this day, probably once he's through with his day job.  Lastly, a special shout-out to Matt Seil and Jeremiah Stacey for their help with Git and some nasty JUnit concurrency issues.<br> </p>

<p style="margin-top:2em"><strong>EVENTS: Upcoming AppSec Events</strong></p>

<p style="text-align:center"><a href="http://cts.vresp.com/c/?TheOWASPFoundation/be9a26312a/91418889ff/51bee3e7f2" target="_blank"><img src="http://pr.ak.vresp.com/a8b0a2106/www.itway.com/generale/newsletter/2015/Business-e/appseceu_en.jpg"></a></p>

<p style="color:#cc9900;text-align:center">The European OWASP Conference is going to be one of the best ever.<br>
Come to hear and share ideas with the experts!<br>
27 June - 1 July 2016</p>

<p>Read the latest news on the next <b>OWASP AppSecEU</b> on the conference site: <a href="http://cts.vresp.com/c/?TheOWASPFoundation/be9a26312a/91418889ff/097d656140" target="_blank">http://2016.appsec.eu/</a></p>

<p>Important keynote speakers will be present at the Marriott Park Hotel in Rome, Italy.
Our special guest will be Charlie Miller, who will present the keynote talk "Bugs ruin everything". In his speech, Miller will discuss some popular methods for finding vulnerabilities and why it is so difficult to spot them.</p>

<p><b>Charlie Miller is a senior security engineer at Uber ATC, a hacker, and a gentleman.</b> Back when he still had time to research, he was the first with a public remote exploit for both the iPhone and the G1 Android phone. He is a four-time winner of the CanSecWest Pwn2Own competition. He has authored three information security books and holds a PhD from the University of Notre Dame. He has hacked browsers, phones, cars, and batteries.</p>

<p>The <b>Open Web Application Security Project</b> is an open-source project for application security. OWASP provides advice on the creation of secure Internet applications and testing guides.</p>

<p>It boasts a strong global community with more than 45,000 participants, more than 55 corporate members and 20 academic supporters through 249 active local chapters in 6 continents and 97 countries.</p>

<p>More than 800 people are expected at the event, with 3 days of training followed by the 2-day conference that includes:</p>

<ul>
<li>Five parallel talks with focus on the OWASP core mission (Dev, Ops, Hack, CISO and Research);</li>
<li>Keynotes from industry leaders;</li>
<li>Exhibition spaces that offer innovative solutions for the needs of companies.</li>
</ul>

<p>Do not miss the opportunity to participate  this important conference,  mentioned in Tripwire as a <a href="http://cts.vresp.com/c/?TheOWASPFoundation/be9a26312a/91418889ff/d971013ace" target="_blank">TOP 11 SECURITY CONFERENCE IN 2016</a>.</p>
