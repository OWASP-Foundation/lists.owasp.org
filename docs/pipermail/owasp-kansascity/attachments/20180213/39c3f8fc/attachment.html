<p class="MsoNormal" style="margin:0in 0in 0.0001pt;font-size:12pt;font-family:"Times New Roman",serif"><b>Latest example of why
you should stop using browser extensions<span></span></b></p>

<p class="MsoNormal" style="margin:0in 0in 0.0001pt;font-size:12pt;font-family:"Times New Roman",serif"><b><span> </span></b></p>

<p class="MsoNormal" style="margin:0in 0in 0.0001pt;font-size:12pt;font-family:"Times New Roman",serif"><a href="https://thehackernews.com/2018/02/grammar-checking-software.html" target="_blank" style="color:blue;text-decoration:underline"><b><span style="font-size:10.5pt;font-family:Arial,sans-serif;color:rgb(56,135,190);background:white">Critical Flaw in Grammarly Spell Checker Could
Let Attackers Steal Your Data </span></b></a><span style="font-size:10.5pt;font-family:Arial,sans-serif;color:rgb(51,51,51);background:white">(<i>February 5,
2018</i>)</span><span style="font-size:10.5pt;font-family:Arial,sans-serif;color:rgb(51,51,51)"><br>
<span style="background:white">Google Project Zero researcher Tavis Ormandy
discovered a vulnerability in the “Grammarly” spell-checking extension for
Chrome and Firefox. The vulnerability affects approximately 22 million
Grammarly users by exposing authentication tokens to all websites that a user
visits. A threat actor could exploit this vulnerability could steal said
authentication tokens with a measly four lines of code, according to
researchers.</span><br>
<b><span style="background:white">Recommendation:</span></b><span style="background:white"> Grammarly has since issued a patch for this
vulnerability, and updates are available for Chrome and Firefox browsers. At
the time of this writing, Grammarly states that does not appear that anyone has
been affected by this vulnerability. However, with Ormandy’s proof-of-concept
code for exploitation of this vulnerability publicly available it is likely
threat actors will attempt to use said code for malicious purposes. Web
browsers and extensions should always be kept up-to-date with the latest
security patches because as this story portrays, exploitation code is often
made available on open sources.</span><br>
<b><span style="background:white">Tags:</span></b><span style="background:white"> Vulnerability,
Grammarly, Data theft</span></span><span></span></p>

DNS servers!!!<span></span></b></p>

Credit Card Data via DNS </span></b></a><span style="font-size:10.5pt;font-family:Arial,sans-serif;color:rgb(51,51,51);background:white"><span style="font-variant-ligatures:normal;font-variant-caps:normal;text-align:start;text-decoration-style:initial;text-decoration-color:initial;float:none;word-spacing:0px">(</span><i style="box-sizing:border-box;line-height:inherit;font-variant-ligatures:normal;font-variant-caps:normal;text-align:start;text-decoration-style:initial;text-decoration-color:initial;word-spacing:0px">February 8, 2018</i><span style="font-variant-ligatures:normal;font-variant-caps:normal;text-align:start;text-decoration-style:initial;text-decoration-color:initial;float:none;word-spacing:0px">)</span></span><span style="font-size:10.5pt;font-family:Arial,sans-serif;color:rgb(51,51,51)"><br style="box-sizing:border-box;font-variant-ligatures:normal;font-variant-caps:normal;text-align:start;text-decoration-style:initial;text-decoration-color:initial;word-spacing:0px">
<span style="background:white"><span style="font-variant-ligatures:normal;font-variant-caps:normal;text-align:start;text-decoration-style:initial;text-decoration-color:initial;float:none;word-spacing:0px">A Point-of-Sale (POS) malware, dubbed “UDPOS,” is being
distributed by threat actors by impersonating a service pack provided by
“LogMeln,”. According to Forcepoint researchers one sample, of UDPOS, has been
observed in the wild, and there is evidence to suggesting it is being used.
Evidence consists of LogMeln file names and Command and Control (C2) URLs. If
the fake package is downloaded and the file named “update.exe” is executed, the
content of the file is automatically launched with 7-Zip’s “RunProgram”
feature. The malware then creates a new service to maintain persistence on an
affected system. UDPOS is designed to steal magnetic stripe data (Track 1 and
Track 2) from debit and credit cards by scraping the memory of running
processes.</span></span><br style="box-sizing:border-box;font-variant-ligatures:normal;font-variant-caps:normal;text-align:start;text-decoration-style:initial;text-decoration-color:initial;word-spacing:0px">
<b style="box-sizing:border-box;line-height:inherit;font-variant-ligatures:normal;font-variant-caps:normal;text-align:start;text-decoration-style:initial;text-decoration-color:initial;word-spacing:0px"><span style="background:white">Recommendation:</span></b><span style="background:white"><span style="font-variant-ligatures:normal;font-variant-caps:normal;text-align:start;text-decoration-style:initial;text-decoration-color:initial;float:none;word-spacing:0px"> POS networks should carefully be monitored for unusual
activity, thus keeping logs of what typical network activity looks like is very
important. In the case of strange activity, taking POS systems offline and
repopulating them is a safe mediation step in order to avoid possible loss of
reputation, or lawsuit by individuals who had their credit card information
stolen, and possibly used by threat actors.</span></span><br style="box-sizing:border-box;font-variant-ligatures:normal;font-variant-caps:normal;text-align:start;text-decoration-style:initial;text-decoration-color:initial;word-spacing:0px">

service uses a JavaScript that “adds speech, reading, and translation to websites.”

script to the Browsealoud one that would limit the processing power of the

approximately 4,200 websites were infected as a result of this incident.</span></span><br style="box-sizing:border-box;font-variant-ligatures:normal;font-variant-caps:normal;text-align:start;text-decoration-style:initial;text-decoration-color:initial;word-spacing:0px">
