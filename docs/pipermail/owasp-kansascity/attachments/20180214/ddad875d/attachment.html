*Critical Flaw in Grammarly Spell Checker Could Let Attackers Steal Your<br>
Data * <<a href="https://thehackernews.com/2018/02/grammar-checking-software.html" rel="noreferrer" target="_blank">https://thehackernews.com/2018/02/grammar-checking-software.html</a>>(*February<br>
5, 2018*)<br>
Google Project Zero researcher Tavis Ormandy discovered a vulnerability in<br>
the ?Grammarly? spell-checking extension for Chrome and Firefox. The<br>
vulnerability affects approximately 22 million Grammarly users by exposing<br>
authentication tokens to all websites that a user visits. A threat actor<br>
could exploit this vulnerability could steal said authentication tokens<br>
with a measly four lines of code, according to researchers.<br>
*Recommendation:* Grammarly has since issued a patch for this<br>
vulnerability, and updates are available for Chrome and Firefox browsers.<br>
At the time of this writing, Grammarly states that does not appear that<br>
anyone has been affected by this vulnerability. However, with Ormandy?s<br>
proof-of-concept code for exploitation of this vulnerability publicly<br>
available it is likely threat actors will attempt to use said code for<br>
malicious purposes. Web browsers and extensions should always be kept<br>
up-to-date with the latest security patches because as this story portrays,<br>
exploitation code is often made available on open sources.<br>
*Tags:* Vulnerability, Grammarly, Data theft<br>

*UDPOS ? Exfiltrating Credit Card Data via DNS *<br>
<<a href="https://blogs.forcepoint.com/security-labs/udpos-exfiltrating-credit-card-data-dns" rel="noreferrer" target="_blank">https://blogs.forcepoint.com/security-labs/udpos-exfiltrating-credit-card-data-dns</a>><br>
(*February 8, 2018*)<br>
A Point-of-Sale (POS) malware, dubbed ?UDPOS,? is being distributed by<br>
threat actors by impersonating a service pack provided by ?LogMeln,?.<br>
According to Forcepoint researchers one sample, of UDPOS, has been observed<br>
in the wild, and there is evidence to suggesting it is being used. Evidence<br>
consists of LogMeln file names and Command and Control (C2) URLs. If the<br>
fake package is downloaded and the file named ?update.exe? is executed, the<br>
content of the file is automatically launched with 7-Zip?s ?RunProgram?<br>
feature. The malware then creates a new service to maintain persistence on<br>
an affected system. UDPOS is designed to steal magnetic stripe data (Track<br>
1 and Track 2) from debit and credit cards by scraping the memory of<br>
running processes.<br>
*Recommendation:* POS networks should carefully be monitored for unusual<br>
activity, thus keeping logs of what typical network activity looks like is<br>
very important. In the case of strange activity, taking POS systems offline<br>
and repopulating them is a safe mediation step in order to avoid possible<br>
loss of reputation, or lawsuit by individuals who had their credit card<br>
information stolen, and possibly used by threat actors.<br>
*Tags:* POS malware, UDPOS, Credit card data theft<br>

*Thousands of Government, Orgs? Websites Found Serving Crypto Mining<br>
Script *<br>
<<a href="https://www.helpnetsecurity.com/2018/02/12/websites-found-serving-crypto-mining-script/" rel="noreferrer" target="_blank">https://www.helpnetsecurity.com/2018/02/12/websites-found-serving-crypto-mining-script/</a>><br>
(*February 12, 2018*)<br>
Security researcher Scott Helme discovered that the U.K.?s Information<br>
Commissioner?s Office (ICO) had a crypto miner installed on it. Further<br>
investigation led Helme to discover that the cause of this incident was the<br>
compromise of the ?Browsealoud? service run by U.K. firm Texthelp; the<br>
service uses a JavaScript that ?adds speech, reading, and translation to<br>
websites.? Texthelp?s script server was compromised and threat actors added<br>
an obfuscated script to the Browsealoud one that would limit the processing<br>
power of the crypto miner to assist in staying hidden. Researchers believe<br>
that approximately 4,200 websites were infected as a result of this<br>
incident.<br>
*Recommendation:* Sometimes webmasters discover that one of their sites has<br>
been compromised months after the initial infection. Websites, much like<br>
personal workstations, require constant maintenance and upkeep in order to<br>
adapt to the latest threats. In addition to keeping server software up to<br>
date, it is critical that all external facing assets are monitored and<br>
scanned for vulnerabilities. The ability to easily restore from backup,<br>
incident response planning, and customer communication channels should all<br>
be established before a breach occurs.<br>
*Tags:* Compromised server, Texthelp, Infected websites, Cryptocurrency<br>
miner<br>
