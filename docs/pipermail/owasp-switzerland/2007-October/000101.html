Hi Adrian,
unfortunately I don't had the possibility to test the newest version of
any of the big commercial WebAppSec scanners out there yet so I can't
give my own feedback on them here but perhaps one of the following links
could help you:

Jordan Wiens did a quiet good job on reviewing the most popular
WebAppSec scanners out there. Perhaps you'll find something interesting
there:
http://www.networkcomputing.com/rollingreviews/Web-Applications-Scanners/

The "Web Application Security Consortium" (WASC) has recently started a
project called "Web Application Security Scanner Evaluation Criteria"
but nothing is released yet. I think this might become a very good
resource for information like you're looking for in the near future:
http://webappsec.org/projects/wassec/

Just as an additional hint, Acunetix has released a free XSS scanner
which you can use but AFAIK it has no JavaScript or as you asked for,
"AJAX-Support":
http://www.acunetix.com/cross-site-scripting/scanner.htm

- From my point of view it really depends on your needs, which scanner is
the one you should use for your work but I also fully agree with Jan ("I
prefer a mixed approach: Manual hacking, source code and configuration
inspection, scanning and interview.").

Btw. perhaps you should ask such a question also to the OWASP Testing ML
(https://lists.owasp.org/mailman/listinfo/owasp-testing) because there
are much more WebAppSec pentesters and also some scanner developers :)

Regards,
Sven
