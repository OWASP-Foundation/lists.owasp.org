*IMPORTANT* : If you'd like to attend the meeting, please send an email
to Thomas Risch (Thomas_Risch at swissre.com) with your full name and company.

Date and time:
The meeting take place at the 8. September 2008 and will be from 17:00 -
ca. 19:00


Where:
Swiss Re
Mythenquai 50 / Zurich

http://maps.google.com/maps?f=q&hl=en&geocode=&q=mythenquai+50+z%C3%BCrich

Thanks to Thomas Risch and the Swiss Re for sponsoring the meeting room.


Who:
As at all of our meetings, everyone is welcome, so if you know someone
who could also be interested in participate, ask him/her to come too :)


What attendees should bring:
  - Laptop
  - Patch cable or WLAN card
  - Firefox


What:
This time we have a hands-on meeting so *bring your notebook with you*

- - Quality of services for web applications (Hands-On Workshop)
  by Pacal Buchbinder - United Security Providers

  - Presentation of mod_qos/Apache (http://mod-qos.sourceforge.net/)
  - Playing around with mod_qos on a Live-CD


- - XML Security (Hands-On Workshop)
  by Cyrill Brunschwiler - Compass Security

  Theory...
  - Introduction In the use of web service In B2B environments and
  backend integration
  - Introduction into XML, DTD and XML Schema
  - Shows attacks against XML generator and XML parser.
  - Presents how the Xerces parsers can be hardened to prevent these attacks

  Hands-on labs include excercises for...
  - XML Injection
  - XML URL enumeration
  - XML directore traversal
  - XML port scanning

  What the talk is not about...
  - XML security standards and protocols (e.g. SAML)


And there will be a information slot about:
- - ISC2/Application security
  by Sandro Moretti - UBS Investment Bank


Dinner:
There will be a sponsored dinner again. This time Compass Security
(http://www.csnc.ch/), a company based in Rapperswil (Saint Gall)
Switzerland specialising in security assessments and forensic
investigations. Bring appetite with you.

- -------------------------------------------------------------------------

As usual, if there are any further questions, don't hesitate to ask.
