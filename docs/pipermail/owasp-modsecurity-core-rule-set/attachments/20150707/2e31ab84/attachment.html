<div style="font-family: Calibri, sans-serif;">So there are two questions here and I’ll try and address them independently (unfortunately they are slightly intertwined). </div>
<div style="font-family: Calibri, sans-serif;">There is a common mistake people make when dealing with removing/modify rules  so I’ll get that out of the way first. There are two ways to do this. One can either do it with a ctl: action (<a href="https://github.com/SpiderLabs/ModSecurity/wiki/Reference-Manual#ctl">https://github.com/SpiderLabs/ModSecurity/wiki/Reference-Manual#ctl</a>)
 such as </div>
<div style="font-family: Calibri, sans-serif;">
<pre style="box-sizing: border-box; overflow: auto; font-family: Consolas, 'Liberation Mono', Menlo, Courier, monospace; font-size: 13.6000003814697px; margin-top: 0px; margin-bottom: 16px; line-height: 1.45; padding: 16px; border-top-left-radius: 3px; border-top-right-radius: 3px; border-bottom-right-radius: 3px; border-bottom-left-radius: 3px; word-wrap: normal; color: rgb(51, 51, 51); widows: 1; background-color: rgb(247, 247, 247);">SecRule REQUEST_URI "@beginsWith /index.php" "phase:1,t:none,pass, \
  nolog,ctl:ruleRemoveTargetById=981260;ARGS:user</pre>
</div>
<div style="font-family: Calibri, sans-serif;"> Or via the way I showed you previously </div>
<div>
<pre style="font-size: 13.6000003814697px; box-sizing: border-box; overflow: auto; margin-top: 0px; margin-bottom: 16px; line-height: 1.45; padding: 16px; border-top-left-radius: 3px; border-top-right-radius: 3px; border-bottom-right-radius: 3px; border-bottom-left-radius: 3px; word-wrap: normal; color: rgb(51, 51, 51); widows: 1; background-color: rgb(247, 247, 247);"><span style="background-color: white; font-size: 10.5pt; line-height: 1.45;"><font face="Consolas">SecRuleUpdateTargetById 950907 !ARGS:test</font></span></pre>
</div>
<div style="font-family: Calibri, sans-serif;">The placement of these depends on which one you use. The rule version, featuring the ctl action must come BEFORE the rule in question. Anywhere really will work. In the CRS 3.0 world there is a file called REQUEST-00-LOCAL-WHITELIST.conf.example.
 Renaming this file to REQUEST-00-LOCAL-WHITELIST.conf and adding it there is what it was designed for. There is no reason that you can’t do this with an earlier version of OWASP too. The key being that it just needs to be loaded before the rest of the rules.</div>
<div style="font-family: Calibri, sans-serif;">A similar file exists for SecRuleUpdateTargetById type directives. These must be places AFTER all the rules (it has to do with when these are run … at startup versus at time of rule check). CRS 3.0 also has a file
 just for this named RESPONSE-99-EXCEPTIONS.conf.example. This can also be renamed RESPONSE-99-EXCEPTIONS.conf and the SecRuleUpdateTargetById directive could be included there.</div>
<div style="font-family: Calibri, sans-serif;"><br>
</div>
<div style="font-family: Calibri, sans-serif;">Now onto your second question how to upgrade to CRS 3.0. Fortunately this is the easier part. CRS is just a number of configuration files. So you can find those typically in your apache config area… depending on
 the OS this might be /etc/httpd/. Grep/find will be your friend in this instance. Once you’ve found all the files it is simply a matter of deleting (or moving) them. You can now download CRS 3.0 from the following URL: <a href="https://github.com/SpiderLabs/owasp-modsecurity-crs/tree/v3.0.0-dev">https://github.com/SpiderLabs/owasp-modsecurity-crs/tree/v3.0.0-dev</a>.
 Extract these and move these rules to the same area.</div>
<div style="font-family: Calibri, sans-serif;">You will then have to find the file from where CRS was included (either httpd.conf, modsecurity.conf, sometimes if installed via yum this can be named 10-modsecurity.conf (or something like that).  In the end you
 are looking for something like this (which you can grep). Note, these might have a folder path in front of them:</div>
<div style="font-family: Calibri, sans-serif;"><span style="background-color: rgb(243, 243, 243);">Include modsecurity_crs_10_setup.conf</span></div>
<div style="font-family: Calibri, sans-serif;"><span style="background-color: rgb(243, 243, 243);">Include activated_rules/*.conf</span></div>
<div style="font-family: Calibri, sans-serif;"><span style="background-color: rgb(246, 246, 246);"><br>
</span></div>
<div style="font-family: Calibri, sans-serif;"><span style="background-color: rgb(246, 246, 246);"><br>
</span></div>
<div style="font-family: Calibri, sans-serif;">Simply change these to reflect the new folder layout of 3.0. I.E</div>
<div style="font-family: Calibri, sans-serif;">
<div><span style="background-color: rgb(243, 243, 243);"><br>
</span></div>
<div><span style="background-color: rgb(243, 243, 243);">Include modsecurity_crs_10_setup.conf</span></div>
<div><span style="background-color: rgb(243, 243, 243);">Include rules/*.conf</span></div>
<div><span style="background-color: rgb(243, 243, 243);"><br>
</span></div>
<div> That should do it, make sure the rules load correctly and you are using 3.0. Hopefully this helps!</div>
