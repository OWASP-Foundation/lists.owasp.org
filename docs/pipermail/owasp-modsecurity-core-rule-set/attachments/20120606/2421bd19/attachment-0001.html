<p class="MsoNormal"><font size="1" color="black" face="Calibri"><span style="font-size:7.0pt;font-family:Calibri;color:black">One of the items we are
considering with the CRS v3.0 update is to start from scratch with all of the
rule IDs.  We have a defined range reserved for the OWASP CRS - </span></font></p>

</div>

<div>

<p class="MsoNormal"><font size="1" color="black" face="Calibri"><span style="font-size:7.0pt;font-family:Calibri;color:black">The rationale for this
change is to make it easier to users to do exceptions and disable groups of
rules by using SecRuleRemoveById, etc That directive has the capability to
remove <b><span style="font-weight:bold">ranges</span></b> of rule IDs,
however over the years the rule IDs were added ad-hoc and they are not properly
grouped in this manner.  For instance, groupings could be something
similar to this -</span></font></p>

</div>

<div>

<p class="MsoNormal"><font size="1" color="black" face="Calibri"><span style="font-size:7.0pt;font-family:Calibri;color:black"> </span></font></p>

</div>

<ul type="disc">
 <li class="MsoNormal" style="color:black"><font size="1" color="black" face="Calibri"><span style="font-size:7.0pt;font-family:Calibri">Protocol Violation 
     900000-900099</span></font></li>
 <li class="MsoNormal" style="color:black"><font size="1" color="black" face="Calibri"><span style="font-size:7.0pt;font-family:Calibri">Protocol Policies -
     900100900199</span></font></li>
 <li class="MsoNormal" style="color:black"><font size="1" color="black" face="Calibri"><span style="font-size:7.0pt;font-family:Calibri">Cross-site Scripting -
     900200900299</span></font></li>
 <li class="MsoNormal" style="color:black"><font size="1" color="black" face="Calibri"><span style="font-size:7.0pt;font-family:Calibri"></span></font></li>
</ul>

<div>

<p class="MsoNormal"><font size="1" color="black" face="Calibri"><span style="font-size:7.0pt;font-family:Calibri;color:black">We would give each rule
group approx 100 rule IDs.  The initial number would be less than that
which would allow some growth.  After some analysis of the current rules,
we really shouldn't ever exceed those allotments as there ends up being rule
logic duplication and you just end up with bloated rule amounts.</span></font></p>

</div>

<div>

<p class="MsoNormal"><font size="1" color="black" face="Calibri"><span style="font-size:7.0pt;font-family:Calibri;color:black"> </span></font></p>

</div>

<div>

<p class="MsoNormal"><font size="1" color="black" face="Calibri"><span style="font-size:7.0pt;font-family:Calibri;color:black">Keep in mind that the
active rule IDs were going to change a bit anyways with this update as we will
be consolidating many rules into highly optimized regexes, etc..  </span></font></p>

</div>

<div>

<p class="MsoNormal"><font size="1" color="black" face="Calibri"><span style="font-size:7.0pt;font-family:Calibri;color:black"> </span></font></p>

</div>

<div>

<p class="MsoNormal"><font size="1" color="black" face="Calibri"><span style="font-size:7.0pt;font-family:Calibri;color:black">The only concern that I
have with making this move would be if users have extensively implemented
exceptions based on the current rule IDs.  They would need to be aware of
this when they migrate to v3.0.</span></font></p>

</div>
