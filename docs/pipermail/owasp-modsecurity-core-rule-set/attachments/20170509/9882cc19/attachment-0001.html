<div id="divtagdefaultwrapper" style="font-size:12pt;color:#000000;font-family:Calibri,Arial,Helvetica,sans-serif;" dir="ltr">
<p>Rule 920440 looks to be in two parts but basically is this:</p>
<p><br>
</p>
<p></p>
<div>
<div>    SecRule REQUEST_BASENAME "\.(.*)$" "id:920440,chain,block...etc."</div>
<div><span class="Apple-tab-span" style="white-space:pre"></span>SecRule TX:EXTENSION "@within %{tx.restricted_extensions}" "..."</div>
<div><br>
</div>
</div>
<div>This basically says is there a dot in the request basename?</div>
<div>And, if so, is the Extension of that request a restricted one?</div>
<div><br>
</div>
<div></div>
<div><span style="font-size: 12pt;">When you are running this:</span><br>
</div>
<div><span style="font-size: 12pt;"><br>
</span></div>
<div><span style="font-size: 12pt;"><span style="color: rgb(31, 73, 125); font-family: Calibri, sans-serif, serif, EmojiFont; font-size: 14.6667px;">SecRuleUpdateActionById 920440 "deny,ctl:ruleEngine=On"</span><br>
</span></div>
<p></p>
<p><br>
</p>
<p>You are overwriting the whole "action" part of first rule in this chain - and removing the "chain" part of this, so breaking the chain and splitting this into two rules:</p>
<p></p>
<div style="font-family: Calibri, Arial, Helvetica, sans-serif, EmojiFont, "Apple Color Emoji", "Segoe UI Emoji", NotoColorEmoji, "Segoe UI Symbol", "Android Emoji", EmojiSymbols; font-size: 16px;">
<br>
</div>
<div style="font-family: Calibri, Arial, Helvetica, sans-serif, EmojiFont, "Apple Color Emoji", "Segoe UI Emoji", NotoColorEmoji, "Segoe UI Symbol", "Android Emoji", EmojiSymbols; font-size: 16px;">
    SecRule REQUEST_BASENAME "\.(.*)$" "id:920440,deny,ctl:ruleEngine=On"</div>
<div style="font-family: Calibri, Arial, Helvetica, sans-serif, EmojiFont, "Apple Color Emoji", "Segoe UI Emoji", NotoColorEmoji, "Segoe UI Symbol", "Android Emoji", EmojiSymbols; font-size: 16px;">
    SecRule TX:EXTENSION "@within %{tx.restricted_extensions}" "..."</div>
<br>
<p></p>
<p>Effectively you're ignoring the second rule in the chain completely, s<span style="font-size: 12pt;">o EVERY request with a dot in the basename is suddenly matching.</span></p>
<p><br>
</p>
<p>I'd imagine if you did something like this, to keep this as a chained rule, then it should work:</p>
<p><br>
</p>
<p></p>
<p style="font-family: Calibri, Arial, Helvetica, sans-serif, EmojiFont, "Apple Color Emoji", "Segoe UI Emoji", NotoColorEmoji, "Segoe UI Symbol", "Android Emoji", EmojiSymbols; font-size: 16px;">
</p>
<div><span style="font-size: 12pt;"><span style="color: rgb(31, 73, 125); font-family: Calibri, sans-serif, serif, EmojiFont; font-size: 14.6667px;">SecRuleUpdateActionById 920440 "deny,chain,ctl:ruleEngine=On"</span><br>
</span></div>
<div><span style="font-size: 12pt;"><span style="color: rgb(31, 73, 125); font-family: Calibri, sans-serif, serif, EmojiFont; font-size: 14.6667px;"><br>
</span></span></div>
<p></p>
Though not tried this myself admittedly.
<p></p>
<p><br>
</p>
<p>In fact the ModSecurity Reference Manual (<a href="https://github.com/SpiderLabs/ModSecurity/wiki/Reference-Manual#SecRuleUpdateActionById" class="OWAAutoLink" id="LPlnk643687" previewremoved="true" style="font-size: 12pt;">https://github.com/SpiderLabs/ModSecurity/wiki/Reference-Manual#SecRuleUpdateActionById</a>)
 does state:</p>
<p><br>
</p>
<p><span style="color: rgb(36, 41, 46); font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Helvetica, Arial, sans-serif, "Apple Color Emoji", "Segoe UI Emoji", "Segoe UI Symbol"; font-size: 16px; font-style: italic; font-weight: 600;">Note : If the
 target rule is a chained rule, you must currently specify chain in the SecRuleUpdateActionById action list as well. This will be fixed in a future version.</span><br>
</p>
<p></p>
<p><br>
</p>
