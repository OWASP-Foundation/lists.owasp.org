You are running in blocking mode with anomaly scoring off.<br>
<br>
This is the hardest mode to tune and it will block immediately<br>
if something is amiss.<br>
<br>
I suggest you run in blocking mode with anomaly scoring on and<br>
a high anomaly limit (-> 1K or more).<br>
<br>
The rule which blocked your request is 958291. It is known for<br>
a lot of false positives and it is not one of my favorite rules.<br>
<br>
You can switch it off completely with<br>
SecRuleRemoveByID 958291<br>
<br>
But be assured given your config, the next rule will bite immediately.<br>
<br>
