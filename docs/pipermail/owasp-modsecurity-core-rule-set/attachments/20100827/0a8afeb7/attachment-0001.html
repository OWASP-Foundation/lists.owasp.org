<FONT FACE="Calibri, Verdana, Helvetica, Arial"><SPAN STYLE='font-size:11pt'>Greetings everyone,<BR>
I wanted to announce the availability of the OWASP ModSecurity CRS v2.0.8. <BR>
<BR>
DOWNLOADING -<BR>
Download page - <a href="http://www.owasp.org/index.php/Category:OWASP_ModSecurity_Core_Rule_Set_Project#tab=Download">http://www.owasp.org/index.php/Category:OWASP_ModSecurity_Core_Rule_Set_Project#tab=Download</a><BR>
You can also use the util/rules-updater.pl script to auto-download the latest ZIP archive (see the rules-updater-example.conf file for Repo data).<BR>
<BR>
TESTING -<BR>
We have integrated the new CRS into the Demo page to help facilitate community testing -<BR>
<a href="http://www.modsecurity.org/demo/">http://www.modsecurity.org/demo/</a><BR>
<BR>
CHANGES -<BR>
--------------------------<BR>
Version 2.0.8 - 08/27/2010<BR>
--------------------------<BR>
<BR>
Improvements:<BR>
- Updated the PHPIDS filters<BR>
- Updated the SQL Injection filters to detect boolean attacks (1&lt;2, foo == bar, etc..)<BR>
- Updated the SQL Injection filters to account for different quotes<BR>
- Added UTF-8 encoding validation support to the modsecurity_crs_10_config.conf file<BR>
- Added Rule ID 950109 to detect multiple URL encodings<BR>
- Added two experimental rules to detect anomalous use of special characters<BR>
<BR>
Bug Fixes:<BR>
- Fixed Encoding Detection RegEx (950107 and 950108)<BR>
- Fixed rules-updater.pl script to better handle whitespace<BR>
&nbsp;&nbsp;<a href="https://www.modsecurity.org/tracker/browse/MODSEC-167">https://www.modsecurity.org/tracker/browse/MODSEC-167</a><BR>
- Fixed missing pass action bug in modsecurity_crs_21_protocol_anomalies.conf<BR>
&nbsp;&nbsp;<a href="https://www.modsecurity.org/tracker/browse/CORERULES-55">https://www.modsecurity.org/tracker/browse/CORERULES-55</a><BR>
- Fixed the anomaly scoring in the modsecurity_crs_41_phpids_filters.conf file<BR>
&nbsp;&nbsp;<a href="https://www.modsecurity.org/tracker/browse/CORERULES-54">https://www.modsecurity.org/tracker/browse/CORERULES-54</a><BR>
- Updated XSS rule id 958001 to improve the .cookie regex to reduce false postives<BR>
&nbsp;&nbsp;<a href="https://www.modsecurity.org/tracker/browse/CORERULES-29">https://www.modsecurity.org/tracker/browse/CORERULES-29</a> &nbsp;<BR>
