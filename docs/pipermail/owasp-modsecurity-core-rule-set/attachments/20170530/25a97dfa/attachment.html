<p class="MsoNormal">I’m deploying modsecurity 2.9.1 with the OWASP CRS 3.0.2 rules.<o:p></o:p></p>
<p class="MsoNormal"><o:p> </o:p></p>
<p class="MsoNormal">I’d like to update a few rules to have different anomaly scores. This is quite easy for a normal rule: eg.<o:p></o:p></p>
<p class="MsoNormal">SecRuleUpdateActionById 920350 "setvar:tx.anomaly_score=+100"<o:p></o:p></p>
<p class="MsoNormal"><o:p> </o:p></p>
<p class="MsoNormal">However, for chained rules this type of update will only update the first rule in the chain. Eg.
<o:p></o:p></p>
<p class="MsoNormal">SecRuleUpdateActionById 920440 "setvar:tx.anomaly_score=+100,chain"<o:p></o:p></p>
<p class="MsoNormal"><o:p> </o:p></p>
<p class="MsoNormal">Since the action is non-disruptive the score addition takes place whenever the first rule in the chain matches (as referenced
<a href="http://blog.modsecurity.org/2008/07/modsecurity-tri.html">http://blog.modsecurity.org/2008/07/modsecurity-tri.html</a>).<o:p></o:p></p>
<p class="MsoNormal"><o:p> </o:p></p>
<p class="MsoNormal">Has anything been done to allow SecRuleUpdateActionById to specify offsets (eg. as suggested at
<a href="https://github.com/SpiderLabs/ModSecurity/issues/190">https://github.com/SpiderLabs/ModSecurity/issues/190</a>)?<o:p></o:p></p>
<p class="MsoNormal"><o:p> </o:p></p>
<p class="MsoNormal">I tried this:<o:p></o:p></p>
<p class="MsoNormal">SecRuleUpdateActionById 920440:1 "setvar:tx.anomaly_score=+100,chain"<o:p></o:p></p>
<p class="MsoNormal">and there is no error about the Rule Id not existing, but it doesn’t seem like the update has taken place. I’m assuming this feature suggestion hasn’t been implemented yet and the update action is ignored because there is no rule id “920440:1”?<o:p></o:p></p>
<p class="MsoNormal"><o:p> </o:p></p>
<p class="MsoNormal">So my question is: Is it possible to change the anomaly score for a specified chained rule in the CRS without editing the CRS files directly?<o:p></o:p></p>
