<br>
Really only use Apache error files rather than ModSecurity audit logs since they are structured to be parsed easier, so sometimes then have to look at Audit log for details but sufficient to raise the alert in first place.<br>
<br>
This has the nice side effective of not slowing down the webserver or ModSecurity processing as run as a separate instance on a separate server and only the Splunk forwarded runs on the webserver.<br>
