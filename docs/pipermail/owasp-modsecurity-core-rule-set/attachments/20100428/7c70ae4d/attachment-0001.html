<P><FONT SIZE=2>Yes, that is correct: 10 is loaded, then 15 (custom) is loaded before all of the succeeding rules (20+).<BR>
<BR>
I've learned from the mailing lists that customization in modsecurity_crs_15_customrules.conf allows you to import all of the original files (10, 20+) without having to modify them. This is useful for maintenance updates!<BR>
<BR>
So long as your Apache configuration for modsecurity sources the rule files (10, 15, 20+) -- they can be located anywhere.<BR>
<BR>
I've also learned from the mailing list that placing the base and optional rules in the same directory is fine, so I've simplified my configuration to do that now, here's a partial example:<BR>
<BR>
#etc. (set up omitted)<BR>
&lt;IfModule mod_security2.c&gt;<BR>
&nbsp;# Basic Configuration Defaults and Rule Customizations<BR>
&nbsp;Include conf/modsecurity-core-rules/modsecurity_crs_10_config.conf<BR>
&nbsp;Include conf/modsecurity-core-rules/modsecurity_crs_15_customrules.conf<BR>
&nbsp;#etc. (omitted)<BR>
&nbsp;# Rule Sets: Basics (including CRS optional rules)<BR>
&nbsp;Include conf/modsecurity-core-rules/modsecurity_crs_2*.conf<BR>
&nbsp;Include conf/modsecurity-core-rules/modsecurity_crs_3*.conf<BR>
&nbsp;#etc. (omitted)<BR>
&nbsp;&lt;IfModule mod_headers.c&gt;<BR>
&nbsp; Include conf/modsecurity-core-rules/modsecurity_crs_49_header_tagging.conf<BR>
&nbsp;&lt;/IfModule&gt;<BR>
&nbsp;#etc. (omitted)<BR>
&lt;/IfModule&gt;<BR>
#etc. (other set up omitted)<BR>

Yes, you want to learn and set the variables ending in _anomaly_score,<BR>
which are defined in modsecurity_crs_10_config.conf. You could adjust<BR>
critical_anomaly, since it defines the score which blocks a request.<BR>
<BR>
This is touched on in the documentation, see section &quot;Rules - Detection and<BR>
Management&quot; under &quot;CRS 2.0 - Collaborative Detection&quot; on<BR>
<A HREF="http://www.owasp.org/index.php/Category:OWASP_ModSecurity_Core_Rule_Set_Project#tab=Documentation">http://www.owasp.org/index.php/Category:OWASP_ModSecurity_Core_Rule_Set_Project#tab=Documentation</A><BR>
<BR>
<BR>
Rather than modify that file, I have adjusted them by overriding their<BR>
values (I redefine them) in my own modsecurity_crs_15_customrules.conf<BR>
<BR>
E.g.:<BR>
<BR>
# Adjustment (2.0.6) modsecurity_crs_10_config.conf: double notice from 5<BR>
to 10<BR>
SecAction &quot;phase:1,t:none,nolog,pass, \<BR>
setvar:tx.critical_anomaly_score=20, \<BR>
setvar:tx.error_anomaly_score=15, \<BR>
setvar:tx.warning_anomaly_score=10, \<BR>
setvar:tx.notice_anomaly_score=10&quot;<BR>
<BR>
Hope this helps, I'm still learning the details of the CRS.<BR>
