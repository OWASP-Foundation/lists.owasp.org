On Wed, Mar 21, 2018 at 02:25:32PM +0000, Hiranmayi Palanki wrote:<br>
> What is the recommended Paranoia Level for an enterprise Internet facing<br>
> application, that does not break the application functionality?<br>
<br>
I'd say you should put it at least on level 2. This will bring some false<br>
positives and you deal with those as described in the documentation.<br>
<br>
> Let’s say if I have the default configuration enabled, and the encoded XSS<br>
> attacks are getting through, what is the best way to block the encoded XSS<br>
> attacks without enabling the higher Paranoia Levels?<br>
<br>
If you do not want to use CRS rules, you need to write your own XSS rules<br>
that do a better job than CRS. There are very good XSS documents around,<br>
many of them several pages long with lots of example payloads. You need to<br>
cover all these without breaking your application.<br>
<br>
Most people find it easier to use CRS and deal with the false positives.<br>
