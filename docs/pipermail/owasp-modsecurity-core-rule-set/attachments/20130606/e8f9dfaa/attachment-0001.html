<div>The reason this does not work is because rule ID 960010 runs in phase:1 and by placing the SecRuleRemoveById directive within an Apache scope location, it won't execute until the end of phase:2.</div>
<div><br>
</div>
<div>You might want to do this instead in a custom rules file that runs before the other rules -</div>
<div><br>
</div>
<div>SecRule REQUEST_FILENAME "@beginsWith /iredadmin/login/" "phase:1,id:123,t:none,nolog,pass,ctl:ruleRemoveById=960010"</div>
