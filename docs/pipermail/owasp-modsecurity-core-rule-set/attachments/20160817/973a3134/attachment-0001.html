<p class="MsoPlainText">The OWASP Core Rule Set team is proud to announce the first of two planned release candidates for the upcoming OWASP ModSecurity Core Rule Set v3.0.0.<o:p></o:p></p>
<p class="MsoPlainText"><o:p> </o:p></p>
<p class="MsoPlainText">This new release represents a huge step forward in terms of both capabilities and protections including:<o:p></o:p></p>
<p class="MsoPlainText"><o:p> </o:p></p>
<p class="MsoPlainText">- A 95% reduction in false positives for a typical CRS deployment using the default configuration.<o:p></o:p></p>
<p class="MsoPlainText">- Extended effectiveness and detection capabilities in numerous areas; namely Remote Command Execution and PHP injections (Walter Hop).<o:p></o:p></p>
<p class="MsoPlainText">- A simple to use, adjustable paranoia level that allows users to tailor their ruleset experience.<o:p></o:p></p>
<p class="MsoPlainText">- The capability to allow existing sites to try out the Core Rules by enabling the rules for only limited percentage of requests (Christian Folini).<o:p></o:p></p>
<p class="MsoPlainText"><o:p></o:p></p>
<p class="MsoPlainText">Please see the CHANGES document for a detailed list of new features and improvements. (<a href="https://github.com/SpiderLabs/owasp-modsecurity-crs/blob/v3.0.0-rc1/CHANGES">https://github.com/SpiderLabs/owasp-modsecurity-crs/blob/v3.0.0-rc1/CHANGES</a>)
<o:p></o:p></p>
<p class="MsoPlainText"> <o:p></o:p></p>
<p class="MsoPlainText">Our desire is to see the Core Rules project as a baseline security feature, effectively fighting OWASP TOP 10 weaknesses with few side effects. As such we attempt to cut down on false positives as much as possible in the default install.
 This RC1 therefore provides an opportunity for individuals to provide feedback and to report any other issues they may face. This is no longer aimed at ModSecurity experts. This is the Core Rules for the rest of us.<o:p></o:p></p>
<p class="MsoPlainText"><o:p></o:p></p>
<p class="MsoPlainText">Please use the CRS GitHub (<a href="https://github.com/SpiderLabs/owasp-modsecurity-crs/releases/tag/v3.0.0-rc1">https://github.com/SpiderLabs/owasp-modsecurity-crs/releases/tag/v3.0.0-rc1</a>) or the Core Rules mailing list to tell
 us about your experiences, including false positives or other issues with this release candidate. Our current timeline is to seek public feedback on RC1 for the next month, followed by an RC2 and subsequently a release.<o:p></o:p></p>
<p class="MsoPlainText"><o:p></o:p></p>
<p class="MsoPlainText"> For more information, please see the following blog post accompanying this release:<o:p></o:p></p>
<p class="MsoPlainText"><a href="https://www.trustwave.com/Resources/SpiderLabs-Blog/OWASP-ModSecurity-CRS-Version-3-0-RC1-Released/">https://www.trustwave.com/Resources/SpiderLabs-Blog/OWASP-ModSecurity-CRS-Version-3-0-RC1-Released/</a><o:p></o:p></p>
