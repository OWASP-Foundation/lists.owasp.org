<div>    Seems no one mentioned that the rule should be "placed" after the rule your are going to disable. Therefore, according to the CRS repository, we can find the rule with the id "960009" is located in "modsecurity_crs_21_protocol_anomalies.conf". You have to put your whitelist rule after it, thus, you may create a file named "mod security_crs_60_disable_rid.conf" and put your whitelist rule into it. While the rule file is ready, you can restart your web server and everything should works well. <br></div>
