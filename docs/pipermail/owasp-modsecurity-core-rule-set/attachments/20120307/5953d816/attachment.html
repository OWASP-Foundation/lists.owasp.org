<div style="font-family: Calibri, sans-serif; font-size: 14px; ">There are three ModSecurity related items at AppSecDC -</div>
<ol>
<li><b>Virtual Patching Workshop</b> – this is a 2 day training session that I will be running where we will be covering various virtual patching topics and also work extensively in hands-on labs using ModSecurity to help mitigate vulnerabilities found in the
 OWASP Broken Web Application Project.  Training details here: <a href="https://www.owasp.org/index.php/OWASP_AppSec_DC_2012/Training/Virtual_Patching_Workshop">https://www.owasp.org/index.php/OWASP_AppSec_DC_2012/Training/Virtual_Patching_Workshop</a>.  Registration
 is here: <a href="http://www.regonline.com/builder/site/Default.aspx?EventID=1021433">http://www.regonline.com/builder/site/Default.aspx?EventID=1021433</a></li><li><b>Web Application Defense with Bayesian Attack Analysis</b> – this is a talk I am giving that demonstrates how to use the OSBF-Lua Bayesian Classifier module to add in bayesian analysis to ModSecurity to help detect web attacks.  <a href="https://www.owasp.org/index.php/OWASP_AppSec_DC_2012/Web_Application_Defense_with_Bayesian_Attack_Analysis">https://www.owasp.org/index.php/OWASP_AppSec_DC_2012/Web_Application_Defense_with_Bayesian_Attack_Analysis</a></li><li><b>Dynamic DAST/WAF Integration</b> – this is another talk I am giving that will demonstrate a working PoC integration between ModSecurity and the Arachni Scanner RPC service (www.arachni-scanner.com).  In the integration, ModSecurity can pass URL+Param+Cookie
 data to Arachni in real-time using the Lua API for targeted, on-demand assessments of resources.  ModSecurity will then check back in with Arachni to pull scan reports and save any vulnerability data to the RESOURCE collection for each URL.  With this vulnerability
 intelligence, we can then become more aggressive in blocking attacks targeting known vulnerable vectors.</li></ol>
