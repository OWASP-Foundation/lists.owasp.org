<div style="color: rgb(0, 0, 0);">Hey Richard, first I’ll echo what Christian said, but also what you have described is just a LIMITED case of SQL injection. SQL Injection can really deal with any area in which SQL may be introduced by an attacker and it will
 then be executed later. There is no need to limit this to a where clause. For instance it could be in a like clause, it could be the clause itself, it could be the statement (select for instance) etc. The underlying idea is that anywhere a user can control
 a SQL statement would be bad. In fairness to libinjection it isn’t factoring in all these situations it is designed with parameters in mind and SQL injections more a kin to the type you are describing. This is why we leave in complex regex patterns as well
 that can detect SQL patterns like the one you have described.</div>
<div style="color: rgb(0, 0, 0);">Another situation where it would be valuable to detect such statements would be in the case of stacked sql queries. Although not all engines support it, in some you may so things like:</div>
<div style="color: rgb(0, 0, 0);"><br>
</div>
<div><font color="#2672ec" style="color: rgb(0, 0, 0);"><span style="font-size: 12pt;">select * from User where id = </span> [ input]</font> to <font><span style="font-size: 12pt;">select * from User where id =<font color="#7f0000"> 1; </font></span></font><b><font color="#7f0000"><span style="font-size: 12pt;">select </span><span style="font-size: 12pt;"><font size="4">a
 from uid2</font></span></font></b></div>

<div><br>
</div>
<div>My understanding for SQL injection case is  that careless web developers take the web request parameters and add them to their "where clause".</div>
<div><br>
</div>
<div>So if the input is  '1 union select a from uid2" that would cause successful injection because the <span style="font-size: 12pt;">careless</span><span style="font-size: 12pt;"> </span><span style="font-size: 12pt;">developer's SQL was modified from </span></div>
<div><br>
</div>
<div><font color="#2672ec"><span style="font-size: 12pt;">select * from User where id = </span> [ input]</font> to <font color="#d24726"><span style="font-size: 12pt;">select * from User where id = 1</span><span style="font-size: 12pt;"> union select a from
 uid2</span></font></div>
<div><font color="#d24726"><span style="font-size: 12pt;"><br>
</span></font></div>
<div><span style="font-size: 12pt;">In above case, the SQL statement was </span>successfully modified with correct syntax. Thus I can accept this is a successful injection.</div>
<div><font color="#d24726"><span style="font-size: 12pt;"><br>
</span></font></div>
<div><span style="font-size: 12pt;">But if the input is "select </span><span style="font-size: 12pt;"><font size="4">a from uid2", then </font></span><span style="font-size: 12pt;">careless</span><span style="font-size: 12pt;"> </span><span style="font-size: 12pt;"><font size="4">developers
 SQL is as:</font></span></div>
<div><b style="font-size: 12pt;"><font size="4"><br>
</font></b></div>
<blockquote style="margin: 0 0 0 40px; border: none; padding: 0px;">
<div><font color="#ac193d"><b>select * from User where id = <span style="font-size: 12pt;">select </span><span style="font-size: 12pt;"><font size="4">a from uid2</font></span></b></font></div>
</blockquote>
<div><br>
</div>
<div>The above SQL statement is not correct in syntax. So this injection will not be successful. Maybe due to this consideration, the libinjection does not consider input of "<span style="font-size: 12pt;">select </span><span style="font-size: 12pt;"><font size="4">a
 from uid2" is a SQL injection although its SQL pattern is so obvious.</font></span></div>
