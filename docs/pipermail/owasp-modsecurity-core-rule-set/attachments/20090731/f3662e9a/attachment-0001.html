follow the information here to make sure that you understand the changes <br>

<p style="-qt-paragraph-type:empty; margin-top:0px; margin-bottom:0px; margin-left:0px; margin-right:0px; -qt-block-indent:0; text-indent:0px; -qt-user-state:0;"><br></p>1) New Home for CRS<br>
The Core Rule Set is now an official OWASP Project!  Here is the new project <br>
site - <br>
http://www.owasp.org/index.php/Category:OWASP_ModSecurity_Core_Rule_Set_Project.  <br>
<p style="-qt-paragraph-type:empty; margin-top:0px; margin-bottom:0px; margin-left:0px; margin-right:0px; -qt-block-indent:0; text-indent:0px; -qt-user-state:0;"><br></p>This is the new home of the CRS.  The main goal of moving the CRS to OWASP is <br>
to better facilitate documentation and development of the rules.  As you know, <br>
the OWASP pages are wiki-based so you all can go in there and help to document <br>
them :)  I will add some example template pages soon to help get the ball <br>
rolling however my thinking is that we should emulate what Snort Sigs DB used <br>
to do and document the goal of each group of rules, what are they looking for, <br>
how are they looking for it and any false positive/exception fixes, etc...<br>
<p style="-qt-paragraph-type:empty; margin-top:0px; margin-bottom:0px; margin-left:0px; margin-right:0px; -qt-block-indent:0; text-indent:0px; -qt-user-state:0;"><br></p>Here is the new Download link page -<br>
http://www.owasp.org/index.php/Category:OWASP_ModSecurity_Core_Rule_Set_Project#tab=Download<br>
<p style="-qt-paragraph-type:empty; margin-top:0px; margin-bottom:0px; margin-left:0px; margin-right:0px; -qt-block-indent:0; text-indent:0px; -qt-user-state:0;"><br></p>2) Changes to the CRS<br>
The latest version of the CRS is v2.0.0 and there are significant changes.  The <br>
most important ones are related to running in an anomaly scoring mode which <br>
allows the rules to collaborate to an overall anomaly score.  This will allow <br>
users to set appropriate thresholds for their sites for logging/blocking.  <br>
There are too many other changes to mention directly here so please review the <br>
CHANGELOG file - http://voxel.dl.sourceforge.net/project/mod-<br>
security/modsecurity-crs/0-CURRENT/CHANGELOG<br>
<p style="-qt-paragraph-type:empty; margin-top:0px; margin-bottom:0px; margin-left:0px; margin-right:0px; -qt-block-indent:0; text-indent:0px; -qt-user-state:0;"><br></p>3) Rule Update Tracking for the CRS<br>
While the new OWASP project site will mainly be used for documentation <br>
purposes, all CRS rule issues will be tracked by using our Jira app - <br>
https://www.modsecurity.org/tracker/.  We want to track all bugs, false <br>
positives and false negatives (if there are any bypass evasion issues that you <br>
find), etc...<br>
