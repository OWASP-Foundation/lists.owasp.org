<p class="MsoNormal"><span style="color:#1F497D">Many agree that deploying CSP for Web apps is a hassle, one of which requires externalizing all inline JS.</span></p>
<p class="MsoNormal"><span style="color:#1F497D"> </span></p>
<p class="MsoNormal"><span style="color:#1F497D">I would like to discuss about the idea of making this process automated by ModSecurity Core rule set with assistant of Lua script, can anyone help me validate this idea? Here is workflow:</span></p>
<p class="MsoNormal"><span style="color:#1F497D"> </span></p>
<p class="MsoNormal"><span style="color:#1F497D">In “learning mode”, creating Lua script to:</span></p>
<p class="MsoListParagraph" style="text-indent:-18.0pt"><span style="color:#1F497D"><span style="">1-<span style="font-style:normal; font-variant:normal; font-weight:normal; font-size:7pt; line-height:normal; font-family:'Times New Roman'">    
</span></span></span><span style="color:#1F497D">Intercept and scan response data, parses HTML and identify inline JS and JS embedded in HTML events.
</span></p>
<p class="MsoListParagraph" style="text-indent:-18.0pt"><span style="color:#1F497D"><span style="">2-<span style="font-style:normal; font-variant:normal; font-weight:normal; font-size:7pt; line-height:normal; font-family:'Times New Roman'">    
</span></span></span><span style="color:#1F497D">Then, creating a new .js file to include identified JS, store(e.g. FTP) it in one of web location can be accessed by public domain.</span></p>
<p class="MsoListParagraph" style="text-indent:-18.0pt"><span style="color:#1F497D"><span style="">3-<span style="font-style:normal; font-variant:normal; font-weight:normal; font-size:7pt; line-height:normal; font-family:'Times New Roman'">    
</span></span></span><span style="color:#1F497D">Remove inline JS blocks from response.
</span></p>
<p class="MsoListParagraph" style="text-indent:-18.0pt"><span style="color:#1F497D"><span style="">4-<span style="font-style:normal; font-variant:normal; font-weight:normal; font-size:7pt; line-height:normal; font-family:'Times New Roman'">    
</span></span></span><span style="color:#1F497D">Replace HTML event with new JS function for event call from response.</span></p>
<p class="MsoListParagraph" style="text-indent:-18.0pt"><span style="color:#1F497D"><span style="">5-<span style="font-style:normal; font-variant:normal; font-weight:normal; font-size:7pt; line-height:normal; font-family:'Times New Roman'">    
</span></span></span><span style="color:#1F497D">Calculate CSP content based on original response as well as adding location for newly generated .js file.</span></p>
<p class="MsoListParagraph" style="text-indent:-18.0pt"><span style="color:#1F497D"><span style="">6-<span style="font-style:normal; font-variant:normal; font-weight:normal; font-size:7pt; line-height:normal; font-family:'Times New Roman'">    
</span></span></span><span style="color:#1F497D">Append generated CSP headers to response.</span></p>
<p class="MsoListParagraph" style="text-indent:-18.0pt"><span style="color:#1F497D"><span style="">7-<span style="font-style:normal; font-variant:normal; font-weight:normal; font-size:7pt; line-height:normal; font-family:'Times New Roman'">    
</span></span></span><span style="color:#1F497D">Save metadata info (i.e. new .js file name, hash of original response, request URL to local data store for caching (performance consideration)</span></p>
<p class="MsoNormal"><span style="color:#1F497D">“Learning mode” is referring to Core rule set performing above actions ONLY when it sees request from trusted source such as given IP address.</span></p>
<p class="MsoNormal"><span style="color:#1F497D">Automated requested can be sent by open sources web crawlers available out there, or web browsers manually.</span></p>
<p class="MsoNormal"><span style="color:#1F497D"> </span></p>
<p class="MsoNormal"><span style="color:#1F497D">Thanks,</span></p>
<p class="MsoNormal"><span style="color:#1F497D">R.S.</span></p>

<div>First of all – I agree with your problem statement.  While CSP has value, orgs have a difficult time retro-fitting their apps to support it.  If you have not see it, please see my recently blog post on how to implement CSP as a part of virtual patching
 remediation for XSS vulns -</div>
<div><a href="http://blog.spiderlabs.com/2013/07/modsecurity-advanced-topic-of-the-week-mitigating-xss-vulnerabilities-using-targeted-csp-enforcement.html">http://blog.spiderlabs.com/2013/07/modsecurity-advanced-topic-of-the-week-mitigating-xss-vulnerabilities-using-targeted-csp-enforcement.html</a></div>
<div><br>
</div>
<div>I like your idea however it would not be easy…  As Achim mentioned, doing proper parsing/extraction of JS data, etc… would be challenging.</div>
<div><br>
</div>
<div>Due to the extensive respnose data changes needed here, this would probably best suited for some new additional engine vs. Lua API.</div>
<div><br>
</div>
<div>I do think this idea is work exploring however.</div>
