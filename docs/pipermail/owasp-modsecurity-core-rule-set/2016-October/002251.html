> I use Splunk (https://www.splunk.com) to collect log files. I can then
> set up alerts based on those. Don't alert on everything (agree with Cosimo
> on that!) but have a dashboard of web server healthiness (of which
> ModSecurity alerts is one measure) and individual alerts for some
> rules/URLs that are sensitive and I want to know about alerts for. Splunk
> is quite nice to drill down into things (it's basically a massive Grep tool
> in a web front end with dashboarding and alerting).
>
> Really only use Apache error files rather than ModSecurity audit logs
> since they are structured to be parsed easier, so sometimes then have to
> look at Audit log for details but sufficient to raise the alert in first
> place.
>
> This has the nice side effective of not slowing down the webserver or
> ModSecurity processing as run as a separate instance on a separate server
> and only the Splunk forwarded runs on the webserver.
