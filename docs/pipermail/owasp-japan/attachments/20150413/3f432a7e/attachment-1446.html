<div dir="ltr">All,<br><br>The OWASP dependency-check team is pleased to announce the release of version 1.2.10! <span style="font-size:13.3333330154419px">Please visit the</span><a href="http://jeremylong.github.io/DependencyCheck/" style="font-size:13.3333330154419px" rel="nofollow"> documentation site</a><span style="font-size:13.3333330154419px"> for information on obtaining the new version (</span><a href="http://jeremylong.github.io/DependencyCheck/dependency-check-cli/installation.html" style="font-size:13.3333330154419px" rel="nofollow">CLI</a><span style="font-size:13.3333330154419px">, </span><a href="http://jeremylong.github.io/DependencyCheck/dependency-check-maven/usage.html" style="font-size:13.3333330154419px" rel="nofollow">Maven Plugin</a><span style="font-size:13.3333330154419px">, </span><a href="http://jeremylong.github.io/DependencyCheck/dependency-check-ant/installation.html" style="font-size:13.3333330154419px" rel="nofollow">Ant Task</a><span style="font-size:13.3333330154419px">, </span><a href="https://wiki.jenkins-ci.org/display/JENKINS/OWASP+Dependency-Check+Plugin" style="font-size:13.3333330154419px" rel="nofollow">Jenkins Plugin</a><span style="font-size:13.3333330154419px">).<br><br>Summary of changes:<br></span><ul>
<li><span style="font-size:13.3333330154419px">New logo thanks to Hugo Costa!!!</span></li>
<li><span style="font-size:13.3333330154419px">Fixed issue 210 that caused a different number of findings to be identified under Java 7 vs. Java 8. The issue was JAXB parsing of the POM works better under Java 8 - to avoid this and other JAXB issues the POM.xml parser was written using a SAX handler to only extract the needed elements.</span></li>
<li><span style="font-size:13.3333330154419px">Resolved issue #206 - the Evidence comparison had some incorrect logic that caused the hint analyzer to think some JAR files were related to the Spring Framework.</span></li>
<li><span style="font-size:13.3333330154419px">Resolved issues with Nexus APIs including ensuring that the SHA1 hashes were lower case (issue 202) and the redirects from a local Nexus will be correctly followed so that the POM.xml file can be correctly downloaded.</span></li>
<li><span style="font-size:13.3333330154419px">Added an update only option to the CLI and Ant Task and added an update-only goal to the Maven plugin.</span></li>
</ul>
<span style="font-size:13.3333330154419px">Again, thanks for the PRs and please open a github issue if you find any false positives or false negatives.<br><br>Best Regards,<br><br>The OWASP dependency-check team<span><font color="#888888"><br></font></span></span>
