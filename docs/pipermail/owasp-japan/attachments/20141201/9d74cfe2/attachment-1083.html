<br>
　先日の OWASP アジアツアー 開催時に、twitter の中で<br>
<br>
「一部の Web サイトで burpSuite や ZAP で https が<br>
　うまくつながらない事がある」<br>
<br>
という話題が出ていました。<br>
<br>
　この問題は、おそらく最近の JRE のバージョンアップの際に<br>
 SNI Extension の実装がされたことが原因と思われます。<br>
<br>
　具体的には、Webアプリケーション側で https の SNI 実装<br>
が甘いサイトへ接続しようとすると、JRE として https の<br>
セッションを開始しないわけです。<br>
<br>
　本来は、SNI Extension として正しい挙動をしているわけです<br>
が、このままでは local proxy が使えないことになってしまう<br>
ので、以下のサンプルのように CLI でオプション指定して、<br>
SNI Extension を抑止した状態で local proxy を起動することで、<br>
この問題を回避することができます。<br>
<br>
"\Program Files (x86)\Java\jre7\bin\javaw.exe" -Djsse.enableSNIExtension=false -Xmx1024m -jar \Users\whereYouLive\Documents\burpsuite\burpsuite_pro_v1.6.09.jar<br>
<br>
　上記「-Djsse.enableSNIExtension=false」が SNI Extension<br>
を抑止するオプションパラメータとなります。このサンプルは burpSuite<br>
での例となっていますが、ZAP の場合でも同様の設定ができると思います。<br>
