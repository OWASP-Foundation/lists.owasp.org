Sorry - I missed this one.

No, there is no current general purpose method for detecting cookie
modification. The AttackDetectorUtils class is where the current
detection points are generally covered. If you'd like to build
something general purpose and contribute it, that'd be great. For lots
of detection points, we've found they are going to be application
specific, so we don't add example code for them. I'd also recommend
Kevin's HMAC approach as it's going to be reliable and fairly simple
and clean. Other cookie modification detection approaches add
complexity by trying to be clever.

Thanks,
John
