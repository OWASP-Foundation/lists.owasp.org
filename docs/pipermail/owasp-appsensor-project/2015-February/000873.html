> I just read your intro for developers. I find it very accurate. I like the
> AOP suggestion. Even though I understand this is not a “selling brochure” I
> would like to suggest to add a few “buzz” words in it. For instance, one big
> advantage I see with AppSensor is that I could have one AppSensor instance
> that tracks users across many cloud applications or many web front ends for
> one application, that “cloud compatibility” should be noted somewhere ;

> developers understand they do not need one instance per front end for

> few words. Another buzz word could be “honeypot” ; AppSensor is a very good
> tool to manage honeypot pages across applications. This was one of the
