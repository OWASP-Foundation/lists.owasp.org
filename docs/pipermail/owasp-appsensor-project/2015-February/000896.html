Ok, great. I think there's a straightforward notification mechanism in JMX we could use that shouldn't be too much trouble. 

As for your larger question, I want to support what people need. If that's JMX, that's fine. My guess is that the largest need throughout the industry would be to have the data in some structured format that's parseable into their tool of choice. As Louis mentioned, some people will want to use the UI we build, but many enterprises are going to use whatever tool they have in house that already manages this data. If we can send data out in CEF format, for instance, and your tool consumes CEF, then we're good. What I don't know right now is what tools most people use and what their preferred formats are. I imagine over the long run we want to build a number of these output integrations, but I want to prioritize them as needed.
