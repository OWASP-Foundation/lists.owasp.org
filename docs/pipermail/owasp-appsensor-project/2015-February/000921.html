John / All

I did begin a little on this yesterday, so it is timely to get some more
encouragement (pressure!) to get on with it.

When John created the text for the developer flyer, and we got feedback on
it, that made it really easy for me to do the subsequent design part. At
the moment I feel stuck on some fundamental things that everyone on this
list could help decide:

   1. The developer flyer is a two-sided A4 or US letter leaflet, which is
   great for handing out in large quantities
   2. I wondered if the CISO document should be something a bit
   thicker/plusher since it is a different audience
      - What size? (see Lulu options...
      http://connect.lulu.com/t5/Product-Pricing-Information/Binding-options-and-their-page-counts/ta-p/33673
we
      used Crown Quarto for the Guide, but that isn't an option for
<32 pages) -
      US Letter or Square?
      - How many pages - just two or a little booklet say 10 12 or 16 sides
      - How bound (coil, saddle stitch or hardcover) see
      http://www.lulu.com/create/books/ for dependencies on sizes, binding,
      colours and number pages
      - Maybe colour inside?

Once we have a feel for the size of the thing, it will be easier to
determine what text can go in it.

What is the key messaging we want to push? Some buzzy phrase ideas:

   - Real time
   - Telemetry and instrumentation
   - Intelligence driven security
   - Instrument applications
   - Active defense
   - Integrates with existing monitoring systems and network defences
   - ???
   - ???
   - Already used at scale

We could then put some draft text up in a Google doc for editing?

Colin
