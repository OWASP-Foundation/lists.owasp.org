All,

Wanted to get the community opinion on something I've been thinking about
recently. What does everyone think about having a separate website for
appsensor? I know we have the standard page (
https://www.owasp.org/index.php/AppSensor), which has improved greatly in
both form and function, but wondering if others felt a standalone website
would be useful.

I ran across the OpenSAMM website (http://www.opensamm.org/) tonight and
compared it to the OWASP OpenSAMM project page (
https://www.owasp.org/index.php/Category:Software_Assurance_Maturity_Model).
I thought the standalone website had some useful differences.

There seems to be advantages to having both. In particular, I started
thinking about this when thinking about building out user documentation for
the v2 code project, which I think could benefit from a standalone site
simply for visualization and flow compared to the wiki.

Not sold on this idea, just looking for general feedback.

Thanks,
