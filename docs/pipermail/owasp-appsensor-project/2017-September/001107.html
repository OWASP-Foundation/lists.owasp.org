2017-09-06 5:17 GMT+02:00 John Melton <jtmelton at gmail.com>:

> Tomás,
>
> Thanks for reaching out. I'm going to try and give a few thoughts here:
>
> - We try to help support any related projects we can, and have worked with
> students on a number of occasions to extend the project successfully.
>
> - One of the extensions done by a former student was to add a rules-based
> analysis engine. This engine allows more complex rules to be formed, and
> provides a more precise and flexible model for detecting specific attacks.
>
> - Generally, the goal for appsensor is to collect relatively few events.
> The idea is that "suspicious" or "malicious" events should be relatively
> small relative to normal usage if you have appropriate security controls in
> place (an explicit assumption of the appsensor model). Having said that,
> folks have used appsensor in a number of different ways, and at times for
> larger data sets. I'm not aware of anyone using it for data sets large
> enough to require hadoop, or more generally, the "big data" ecosystem.
>
> - This summer, we're actually working with a student who is building both
> simple statistical and machine learning based approaches for detecting
> suspicious behavior in web server logs. Certainly web server logs can be
> large depending upon the environment. Though these are not the standard
> appsensor data input format, it's still an interesting problem, and serves
> to help the same users of our project, primarily developers and security
> professionals.
>
> - Lastly, if you let me know what kind of documentation you would like, I
> can try and help. Generally we either point people to the user guide, or
> the website: appsensor.org
>
> Hope this helps, and please reach out with any questions or further
> discussion.
>
> Thanks,
> John
