<p dir="auto">AppSensor Team,</p>

<p dir="auto">I'd like to create a child project within AppSensor called "Automated<br>
Attack Prevention". This project will be a study of current techniques to<br>
thwart automated attacks against application. The deliverable will be well<br>
documented knowledge and best practices (just like appsensor). The<br>
deliverable could be both stand alone and also as a chapter within the<br>
overall AppSensor book.</p>

<p dir="auto">This idea is very similar to the defensive nature of AppSensor; however,<br>
the differences of the material and approach warrants it to be slightly<br>
separated as a child project.</p>

<p dir="auto">Here are a few examples of material we would cover.<br>
- Brute Force Login Protection :<br>
-- Defensive options: Threshold based captcha, IP blocking, Response time<br>
delay<br>
-- Triggers: Can be tied to attacker (IP) or victim (targeted account).<br>
Example: Multiple failed logins from a single IP against any number of<br>
target accounts OR multiple failed logins from multiple IPs against the<br>
same username</p>

<ul>
<li><p dir="auto">Hashcash concepts and how they might apply in web apps (<br>
<a href="http://en.wikipedia.org/wiki/Hashcash">http://en.wikipedia.org/wiki/Hashcash</a>)</p></li>
<li><p dir="auto">Detection of common web scanner requests and optional response tactics<br>
(drop, ignore, block ip, feed fake response data)</p></li>
<li><p dir="auto">and more</p></li>
</ul>

<p dir="auto">I think the material is related enough to fly under the AppSensor umbrella,<br>
but different enough to become a child project. Thoughts?</p>
