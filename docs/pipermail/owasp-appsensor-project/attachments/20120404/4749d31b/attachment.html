Dinis, <br>No, there is no documentation at all for doing this specifically for spring apps. Just as a heads up, the project is currently going through a re-architecturing (not a word). The current version is constrained by lots of factors, but it's fairly easy to add appsensor to any java app, not just a spring app. The basic idea for configuring the current version is: <br>


1. Perform steps to add ESAPI (we have a pretty tight dependency on ESAPI in the current architecture)<br>2. Add the appsensor jar file to WEB-INF/lib (latest is always in maven central)<br>3. Add an appsensor.properties file to the .esapi directory that contains your appsensor configuration - a sample file can be retrieved from the test code source or demo app source<br>


4. Start adding detection points, which involves adding the java code in the appropriate place, and configuration matching that detection point code: a couple simple examples are found here: <br>    <a href="https://www.owasp.org/index.php/AppSensor_Developer_Guide#Using_AppSensor_in_your_project" target="_blank">https://www.owasp.org/index.php/AppSensor_Developer_Guide#Using_AppSensor_in_your_project</a><br>
