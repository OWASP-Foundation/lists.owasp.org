> be considered a true "AppSensor" imlementation.  The key piece here is<br>
> that most WAFs work as "attack detection systems" for a very narrow<br>
> category of attacks - mainly Injection flaws.  In order to have good<br>
> AppSensor coverage, users need to customize the detection points for their<br>
> application.  It is this key piece that has caused me to re-think strategy<br>
> here for ModSecurity.  What I am considering is to still have basic<br>
> mappings in the OWASP CRS to AppSensor detection points, but to possibly<br>
> develop a completely separate "AppSensor" ruleset for ModSecurity.  These<br>
> rules would utilize the Lua API to implement the detection points and it<br>
