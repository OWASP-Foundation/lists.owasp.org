<br>A coworker of mine tested an application once that locked out his

vulnerabilities (e.g., logic flaws, not using SSL, HTTPOnly not set,
etc.).<br>
<br>One more detection point you could add is a list of common
usernames that do not exist in the application.  For example, the
AppSensor would detect an attack when I tried to log in as
&quot;administrator&quot; and &quot;admin&quot;, because those are not real usernames for
this application.<br>
<br>Another detection idea is to insert fake parameters (e.g. uID=0),
to see if they are ever modified.  This is more of a honey pot
approach, but I guess you don&#39;t need to detect every attack, just one
(good is the enemy of great).<br>
<br>Have you considered integrating this with ESAPI?  It seems like you
