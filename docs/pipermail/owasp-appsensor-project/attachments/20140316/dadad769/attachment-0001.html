              Note: the reason I asked about the separate website in the
              first place was for the documentation of the new version
              v2 of the code. However, want to make sure I'm doing

            </div>
            The code for the project is being completely redone for v2
            so that it can work not only in "local" mode, but also in
            client-server style mode (many client applications
            instrumented with detection points sending events to a
            central server that analyses them and informs clients of
            appropriate responses to take). This enables us to support
            much more than just Java. Also, the dependency on ESAPI is
            completely removed. I have some code to push probably
            tonight that will put the new code in a place where the
            initial base features are all there and we'll have a couple
            trivial sample reporting apps. From here on out, it'll be
            adding new components. <br>
            <br>
            I've had several conversations about the next version of the
            code on the mailing lists (spanning a couple years b/c it
            took a while for me to get going :>). <br>
            <br>
          </div>
          <div>If anybody has thoughts or ideas, send them my way, and
            I'll try to get them added, or feel free to add issues to
            github. I'll also try to close out that final issue in
            google code so it points to v2 of the code. v1 was honestly
            mostly a POC or alpha project at best (the code, not the
            book).<br>
