      Jim, <br>
      From my perspective, it should be pretty easy.&nbsp; If you had a
      webapp w/ esapi and wanted to add appsensor, here are the steps as
      of the current release.<br>
      1. get appsensor jar and add to project<br>
      2. modify esapi.properties file to point to appsensor intrusion
      detector instead of default<br>
      3. add appsensor.properties file<br>
      ... and ...<br>
      4. modify your code and/or the properties files to add new
      detection points and their associated threshold configurations
      (this step is optional, technically)<br>
      <br>
      If you want to add appsensor to a project that's not using esapi,
      you'd have a couple extra steps, but it's still possible.<br>

                  Jim, <br>
                  Not sure I agree with #1 - not sure why we need to
                  stop using ESAPI.&nbsp; I'm not sure how maven deals w/
                  circular dependencies, but ESAPI's interaction w/
                  appsensor for the most part would be through the
                  intrusion detector mechanism ...
                  ESAPI.intrusionDetector().addException(this), which is
                  already implemented in the EnterpriseSecurityException
                  constructor and maybe 1 or 2 other exceptions.&nbsp; I
                  admit, though, I'm not sure how maven deals w/ the
                  dependencies so that could certainly be an issue - any
                  maven experts can you chime in?<br>
                  <br>
                  We do use a few of the esapi classes.&nbsp; <br>
                  - IntrusionDetector interface<br>
                  - SecurityConfiguration class <br>
                  - I think we might use the StringUtilities in a place
                  or 2, but we could remove these if need be<br>
                  - use the logger through an adapter<br>
                  - use the authenticator through an adapter.<br>
                  <br>
                  We've planned ahead for integration w/ ESAPI or w/
                  other frameworks.&nbsp; In the current incarnation of
                  AppSensor, we do depend on ESAPI, but you don't have
                  to <b>use</b> ESAPI.&nbsp; You can use a different
                  mechanism for all the controls that ESAPI would
                  normally cover, but if you use AppSensor, we're still
                  gonna use some ESAPI code behind the scenes.&nbsp; Having
                  said that, our only implementations for certain pieces
                  are written for ESAPI integration
                  (logger/authenticator), but an app could easily
                  implement their own.<br>
