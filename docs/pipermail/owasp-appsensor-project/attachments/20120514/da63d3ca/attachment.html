>> AppSensor Experts & Enthusiasts,<br>
>><br>
>> Last fall at the AppSensor summit we hashed out the framework for the next version of the AppSensor book.  Based on several years of feedback and the collective expertise of core contributors, and those with fresh new perspectives, we arrived at a good skeleton. But, alas, we don't have a single page written so far!<br>



>><br>
>> I'd like to discuss how we can collectively go about creating the next great app sensor book. The traditional approach is for a few of us to sit down and crank out the whole book over 12-14 months and have some pier review - but I just don't think that truly leverages our open source and collective skills effectively.<br>



>><br>
>> I'm going to propose an alternative approach - community driven book writing.<br>
>><br>
>> Approach<br>
>> 1. Collectively agree on the skeleton / table of contents (pretty close here)<br>
>> 2. Determine writing style guides / tone / language (region for spelling purposes)<br>
>> 3. Determine a small number of core authors<br>
>> 4. Determine a larger number of contributing writers<br>
>> 5. Divide responsibilities (e.g. book sections/ chapters) to core authors<br>
>> 6. Core authors map out overall flow of their sections<br>
>> 7. Core authors and contributing writers create their sections<br>
>> 8. Continual review from core authors across their sections<br>
>> 9. Review, adjust, repeat<br>
>><br>
>><br>
>> Core Author<br>
>> - Subject matter expert on their section<br>
>> - Responsible for entire chapter or section<br>
>> - Drives the main ideas covered in the sections<br>
>> - Responsible for pier review of all content in respective section, organization of ideas<br>
>> - Works with contributing writers to arrange material, flow<br>
>> - Nomination and acceptance process for all Core Authors<br>
>><br>
>> Contributing Writers<br>
>> - Contributes a small (or larger) amount of material within any section they feel comfortable<br>
>> - Expert within the areas they are contributing text<br>
>> - Stays within the defined style guidelines<br>
>><br>
>> Attribution:<br>
>> Everyone that contributes would be recognized. We would emphasize individuals that contribute substantial amounts of work.<br>
>><br>
>><br>
>> Think of a core author as an architect and a contributing writer as a developer (except we have an architect per major book section & there are pier review between the architect and all developers in the respective section)<br>
