John has a good point that we aren't ready to implement this into the
code, but we can still list it as a detection point in the wiki.&nbsp; Our
AppSensor information project provides someone with all the knowledge
to roll their own AppSensor. The AppSensor code that we are developing
is one implementation in Java with ESAPI integration. So although we
may not support this detection point in our code, its still a good idea
for someone else to consider.<br>

Source<br>
-----------------------------------<br>
Just another idea, but based on discussions about creating an Internet<br>
Threat Exposure (ITE) index by the UK Cybersecurity KTN in 2008.<br>
    <br>
Description<br>
-----------------------------------<br>
The general threat level (e.g. general risk of attack from the<br>
Internet, or specific targetted attacks against an organisation) is<br>
elevated. &nbsp;This could also be used to change response sensitivity due<br>
to short-term effects such as application upgrades/patching. &nbsp;This<br>
input could be used to alter thresholds for AppSensor responses.<br>
    <br>
Suggested categorization<br>
-----------------------------------<br>
In the suggested new category "Reputation" (see RP1 Suspicious User IP
Address)<br>
RP5 Change to Environment Threat Level<br>
