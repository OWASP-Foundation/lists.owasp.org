 detection RegExes -<o:p></o:p></span></p>
</div>
<div>
<p class="MsoNormal"><span style="font-size:10.5pt;font-family:"Calibri","sans-serif";color:black"><a href="http://lists.owasp.org/pipermail/owasp-appsensor-project/2012-February/000342.html">http://lists.owasp.org/pipermail/owasp-appsensor-project/2012-February/000342.html</a><o:p></o:p></span></p>

<div>
<p class="MsoNormal"><span style="font-size:10.5pt;font-family:"Calibri","sans-serif";color:black">I think that we (OWASP) need to develop a Blacklist Regex Repository for detecting common attack payloads (SQL injection, XSS, RFI, etc…).  Something similar
 to this old Validation RegEx Repo but for attacks - <o:p></o:p></span></p>
</div>
<div>
<p class="MsoNormal"><span style="font-size:10.5pt;font-family:"Calibri","sans-serif";color:black"><a href="https://www.owasp.org/index.php/OWASP_Validation_Regex_Repository">https://www.owasp.org/index.php/OWASP_Validation_Regex_Repository</a><o:p></o:p></span></p>
</div>
<div>
<p class="MsoNormal"><span style="font-size:10.5pt;font-family:"Calibri","sans-serif";color:black"><o:p> </o:p></span></p>
</div>
<div>
<p class="MsoNormal"><span style="font-size:10.5pt;font-family:"Calibri","sans-serif";color:black">My thinking is that we should focus on the RegEx Repo and then various other projects can import/use them (AppSensor, ModSecurity CRS, etc..).  I would like to
 get good participation from the Breaker community to help vet the RegExs.  I know they will never be 100% foolproof but looking at some of the "example" blacklist RegExs floating around in various project code makes me cringe…  We can do better.<o:p></o:p></span></p>
</div>

<p class="MsoNormal"><span style="font-size:10.5pt;font-family:"Calibri","sans-serif";color:black">Not sure if this should be a stand-alone project or not (probably) but I would like your feedback.<o:p></o:p></span></p>
