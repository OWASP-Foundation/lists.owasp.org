<div><br></div><div>The concept is that normal/real users have an amount of delay between requesting dynamic resources as they need time to read the pages and fill in FORM elements, etc… So this detection point would be measuring the time interval between the application sending back a web page with form elements and when the application then receives a follow-up request with parameters.</div>

<div><br></div><div>I have conducted some minimal testing with this concept and ModSecurity and have found that it also catches many CSRF attacks as it forces the user&#39;s browser to send an <span style="font-weight: bold;">immediate</span> request back to the server once the html page loads and the user&#39;s browser is tricked into executing the CSRF code.  See the last section here - <a href="http://blog.spiderlabs.com/2011/01/detecting-malice-with-modsecurity-csrf-attacks.html" target="_blank">http://blog.spiderlabs.com/2011/01/detecting-malice-with-modsecurity-csrf-attacks.html</a></div>
