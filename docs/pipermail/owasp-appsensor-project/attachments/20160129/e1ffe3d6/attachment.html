<div dir="ltr"><div><div>All, <br><br></div>Just a periodic update on what's been happening with appsensor development. <br><br></div><div>As I mentioned in the last update, we have a number of folks who are contributing great work as of late. Here are a few of the things being done: <br><br>- docker-ization of our apps and product<br>- adding support for per-client-application detection point overrides (allow different client applications on the same appsensor server to have different configurations for the same detection point)<br></div><div>- new analysis engines being built (2 folks working on this)<br><br></div><div>While other folks are leading the charge on the efforts above (yay!), I've been working on another common feature request: detection points. One of the biggest things to consider when integrating appsensor is the level of effort to build detection points into your application. While many will be custom, there are some that are generic enough to be built once and re-used. <br><br>To that end, I've created a related sub-project on github called appsensor-reverse-proxy (<a href="https://github.com/jtmelton/appsensor-reverse-proxy/">https://github.com/jtmelton/appsensor-reverse-proxy/</a>). The tool provides a few primary features: <br><br></div><div>- blocks requests based on a blacklist (this requires some straightforward integration with the appsensor server and a "block store")<br></div><div>- implements several canned detection points (the readme specifies which ones), and there are more to come<br></div><div>- provides simple integration points to do more detection points, ie. it's very simple to contribute code to this one<br></div><div>- collects basic summarized statistical information that can be used for more intelligent data analysis<br></div><div>- everything except the reverse proxy feature (blocking, detection points, trending info) is disabled by default and must be explicitly enabled. <br><br></div><div>I'm pretty happy with how it's been running so far. It's still new, but in the testing I've done, it's been pretty solid. Feel free to view the readme, try it out and give us feedback. Excited to see how people use it. If you have any ideas about how to improve it, or a new feature request, feel free to email or add a github issue. <br><br></div><div>As always, if you have any questions, let me know.<br></div><div><br></div><div>Thanks,<br></div><div>John<br></div></div>
