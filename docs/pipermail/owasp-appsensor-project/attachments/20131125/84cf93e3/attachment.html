<div dir="ltr"><div><div>AppSensor Team,<br><br></div>I'd like to create a child project within AppSensor called "Automated Attack Prevention". This project will be a study of current techniques to thwart automated attacks against application. The deliverable will be well documented knowledge and best practices (just like appsensor). The deliverable could be both stand alone and also as a chapter within the overall AppSensor book.<br>
<br></div><div>This idea is very similar to the defensive nature of AppSensor; however, the differences of the material and approach warrants it to be slightly separated as a child project. <br></div><div><div><div><br></div>
<div>Here are a few examples of material we would cover.<br></div><div>- Brute Force Login Protection : <br></div><div>-- Defensive options: Threshold based captcha, IP blocking, Response time delay<br></div><div>-- Triggers: Can be tied to attacker (IP) or victim (targeted account). Example: Multiple failed logins from a single IP against any number of target accounts OR multiple failed logins from multiple IPs against the same username<br>
<br></div><div>- Hashcash concepts and how they might apply in web apps (<a href="http://en.wikipedia.org/wiki/Hashcash">http://en.wikipedia.org/wiki/Hashcash</a>)<br><br></div><div>- Detection of common web scanner requests and optional response tactics (drop, ignore, block ip, feed fake response data)<br>
<br></div><div>- and more<br></div><div><br></div><div><br></div><div>I think the material is related enough to fly under the AppSensor umbrella, but different enough to become a child project. Thoughts? </div><div><br></div>
