Dear Team OWASP,


I am very excited to contribute in open source community through the
platform of GSoC 2012. Last year, I contributed "Debugger Visualizers" to
MonoDevelop which had successfully become a part of Mono Develop.



I have seen project ideas for GSoC 2012 at
https://www.owasp.org/index.php/GSoC2012_Ideas#AppSensor. I am interested
in "*SOAP web service server implementation*" for AppSensor. The reason for
my interest is that I have good working experience of web services
(SOAP/REST/Axis2), Java, Servlets and related technologies.



My understanding of the requirements is as follows:



1. Coordinate with the REST team to develop a common standard that will be
used in the SOAP/REST Web service development.

1. Implement a Soap based Web service using Java.

2. The Web service will expose AppSensor engine's functionality via WSDL.

3. The Web service shall talk to AppSensor core engine and will be consumed
by AppSensor clients to utilise the functionality of AppSensor

4. Liaise with other GSoC project (Project 003) to expose the web service
to clients.



Are you going to work on Axis2 to implement this web service?


Your kind feedback would help me in better recognising the core project
requirements and develop a good proposal.


Regards,



Rauf
