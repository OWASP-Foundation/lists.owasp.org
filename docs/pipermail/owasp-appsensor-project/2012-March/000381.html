Very good. Glad to meet you (virtually) and glad to hear you're
excited about the project.
>
> At first I've subscribed at your project's mailing list and then I joined
> your irc room with the nickname "vervas". I've also made a checkout from the
> subversion repository but I'm more familiar with GIT, so I wonder if it is
> better to have a look at subversion or set up a GIT repository.
The mailing list is used. To my knowledge, IRC is not - at least I
never use it. We have the current version of the code in subversion.
I'm not sure where the new version of the code will be (might be git),
but for the time being, we're using svn.
>
> At this point I check the tutorials and other material related to the
> project trying to get a taste, as I wasn't involved in the past. Any
> suggestions about where to focus my attention on are welcome.
If you're wanting to learn about the project in general, look at the
book (https://www.owasp.org/images/2/2f/OWASP_AppSensor_Beta_1.1.pdf).
If you want to learn about the code, pull the code down and have a
look. You can also check out the dev guide
(https://www.owasp.org/index.php/AppSensor_Developer_Guide)
>
> If there is something that I have to know about the project, or other stuff
> that I have to take care of before writing and submitting my application
> please let me know.
In my opinion, you should know Java. You should understand the basics
of security. You should at least briefly read the basics of intrusion
detection (just to get an idea about it). Depending on what you plan
to try and work on, you might want to read about web services (using
SOAP and/or REST). I wouldn't necessarily spend a lot of time on these
(except Java), but it will help to at least introduce yourself to the
concepts.
