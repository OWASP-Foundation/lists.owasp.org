Cosmin,

Thanks very much for your interest in the project! I'm the lead
developer on appsensor. Looks like you have some good experience that
you could contribute to the project. As for getting connected to the
project, I'd suggest you take a look at the book (linked off this
page: https://www.owasp.org/index.php/OWASP_AppSensor_Project). It
will describe a lot of the principles in detail if you're interested.
Also, feel free to pull down the source from google code (svn). As for
what is most important to us in the update, I'd say the soap/rest
service/client implementations are definitely the largest and most
important tasks. One thing I did want to point out, though, is that
both the "client" and "server" pieces will be application servers.
There is no user interface development to what we're building. This is
all back-end communication and building services to expose data (the
reporting services).

If you're interested in submitting for the project, use the google
link to submit your proposal. OWASP provided a sample proposal you can
use as a basis.

Looking forward to working with you if everything works out! If you
have any further questions, feel free to respond to the list or me
directly.

Thanks,
John
