BTW I think Joseph Feiman's original publication of this name was:

   Apr 2012 Runtime Application Self-Protection: A Must-Have, Emerging
Security Technology

https://www.gartner.com/doc/1994816/runtime-application-selfprotection-musthave-emerging

But I found a more recent paper, which has some interesting points:

   Sep 2014 Stop Protecting Your Apps: Its Time for Apps to Protect
Themselves

https://www.gartner.com/doc/2856020/maverick-research-stop-protecting-apps

which is also "free" from the vendor referred to in the SC magazine article:

    http://info.waratek.com/gartner-maverick-research

Someone else's opinion about RASP here:

   AppSec: RASP is Dead in the Water
   http://richard-quinn.com/2014/08/22/appsec-rasp-is-dead-in-the-water/

I put a little suggested text about RASP in the draft AppSensor CISO
