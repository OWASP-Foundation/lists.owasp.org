I was wondering if we can somehow sum up the difference. This came to mind:

DRAFT: "The RASP product category currently focuses on vendor-offerings,
and as such excludes the greater opportunities available of deeper
integration within applications and enterprise-wide security intelligence
sharing, described and implemented by AppSensor. Organisations can
instrument their own applications and analysis engine, or use existing
event monitoring systems and network security devices, or build upon the
AppSensor reference implementation."

Is that true? Is it fair? Can it be improved?

> Yeah, good find. RASP is probably the best fit for appsensor in gartner
> speak. The one major difference is the "runtime" implication. I think the
> term is getting overloaded, but at the end of the day, the "runtime" part
> is leading people to think the .NET or JVM runtimes (to the exclusion of
> languages without a runtime, ie. native code or dynamic languages). There's
> nothing in appsensor to prevent any language / runtime from benefitting
> from the concepts, so that's where there is a standout difference.
