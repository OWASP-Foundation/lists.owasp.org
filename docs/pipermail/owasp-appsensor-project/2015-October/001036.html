All, 

A topic arose today around a "blacklist" or set of signatures for XSS and SQLi that could be used in appsensor. It would be difficult and a waste of resources to define these on a per-project basis. It would be nice to have a list of these to pull from for detection purposes. 

 

Ryan Barnett pointed to this project as an existing project specifically intended to fill this requirement: https://www.owasp.org/index.php/Category:OWASP_Blacklist_Regex_Repository

An example of the type of thing we're looking at building is here: https://github.com/SpiderLabs/owasp-modsecurity-crs/blob/v3.0.0-dev/util/regression-tests/tests.xml

(this could also serve as inspiration: https://github.com/SpiderLabs/owasp-modsecurity-crs/tree/master/base_rules)

2 questions: 

- do folks have other ideas? does something like this already exist?

- who wants to contribute here? This project would benefit us as well as other projects

 

Thanks,

John
