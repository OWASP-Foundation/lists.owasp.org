> -        Are there any long term plans to AppSensor development?
>
The short answer is yes.

The focus areas are analysis engines and visualization.

For analysis engines, we just added the rules engine (boolean logic for
configured rules), and the next steps will be expanding the machine
learning work from GSoC last year and building a trend analysis engine.

For visualization, there are a host of things to handle. In the past we've
generally had users that pulled the data out of appsensor into some other
system in house for visualization (SIEM, or data warehouse). We've gotten
more requests recently though to provide a real UI, something for analysts
to use full-time. Also, the more complex rules engine just added could use
a nice UI for building those rules.

In addition to the 2 main focus areas, there's always work in making the
project more user-friendly, and projects like the "server project
generator" would fall into that bucket. Additionally, integration work
comes up frequently. As an example, there's some work to integrate with
repsheet (a great open source tool with a similar idea, but different
deployment model and tech stack)

> -        Any other ideas what could be used to convince my colleagues to
> invest their time in AppSensor integration?
>
Almost universally, every user of our system has had good luck starting
with a very small integration / pilot project. They've targeted a specific
case to prove value (e.g. XSS detection, account takeover, fraud detection,
etc.) and shown that it is possible, simple and actually works. Also note
that it's not always about the response. Several of our users started just
with detection in order to show value.

I generally recommend people start with business logic exceptions, whether
they are security related or not. When you hit an exceptional condition,
fire an event. Others recommend starting with exceptions in security
filters (global checks for sqli, xss, certain headers being set, etc.).
Certainly with new tech AND the new concept of appsensor, small provable
value-add has been most effective from what I've seen.
