Just scribbling some more draft text for the v2 book. Apart from the
new web services version, the book is probably going to reference
other "example implementations", and thus the existing /old Java
version. People might want to use different
languages/methods/architectures, so I think we ought to keep them all
available. The links that seem to be Java version are currently:

 •	Code repository
	http://code.google.com/p/appsensor/
•	Getting Started Guide
	https://www.owasp.org/index.php/AppSensor_GettingStarted
•	Developer Guide
	https://www.owasp.org/index.php/AppSensor_Developer_Guide

Wiki pages can be created freely, but I was wondering what the plans
are for source code?

- Will the URL code.google.com/p/appsensor/ change for the 2010 Java version?
- Where will the new web services' code be?
- Where might we put other future code examples?
- How will they be differentiated?
- Do we have preferred names for the previous and new versions (e.g.
Integrated Java implementation, and SOAP Web Services)?

It doesn't have to decided now, but I am hoping we don't lose
older/other versions. I'm just flagging the issue up.
