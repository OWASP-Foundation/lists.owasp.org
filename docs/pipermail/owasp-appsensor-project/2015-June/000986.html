For the "view from afar dashboard", I think some BIG NUMBERS styled
using CSS-alone would be sufficient. I have uploaded an idea:

   https://drive.google.com/file/d/0ByNJ8mfWALwjTXBodHBZRlpzSE0/view?usp=sharing

This displays a trend over 5 time periods:

- last hour
- last shift (8 hours)
- last day
- last week
- last month

The numbers are the average detection events (above line), and
responses (below line), per hour in the period. Note that all periods,
include all data from the shorter periods, so that the every period is
a rolling minute-by-minute period that all include the last minute. It
needs to auto-update (every minute?). The user could select to
application or application group, or maybe it should just aggregate
everything they are assigned to. All text apart from the numbers ought
to be very small, and will not be legible from a distance. There would
not be any drill down options from this screen. Maybe these things
should be config options:

- colour RGB values of black, red, green
- percentage increase/decrease that changes the colour from black
(mock up suggests 20% more, 20% less)
- the averaging period for the numbers ("per hour" in the mock up)

Would have to cater for "display overflow", maybe by determining how
many digits there are in the rounded outputs, and set the CSS font
size to suit.

This summary info/ layout won't be suitable for many situations, but
it is a demo and people can modify it.  I could do some layout CSS in
due course.

Colin
