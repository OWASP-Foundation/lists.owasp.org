        <p class="MsoNormal">Hi Folks,<o:p></o:p></p>
        <p class="MsoNormal">I did some searching on all the archives
          for struts before posting, I will search for “forward” next<o:p></o:p></p>
        <p class="MsoNormal">We are trying to add csrfguard 3.1 to a
          very large, legacy (spaghetti) web app, (actually many, but
          focusing on 1 for now)<o:p></o:p></p>
        <p class="MsoNormal">We are using struts 1.1 , weblogic 12.1.3
          etc<o:p></o:p></p>
        <p class="MsoNormal">I’m able to get happy path for a fairly
          complex CRUD flow with a combo of unprotected urls, as well as
          turning off token per page and token rotate properties<o:p></o:p></p>
        <p class="MsoNormal">, which is in essence, just another session
          token , OWASP_CSRFTOKEN, which is similar to JSESSIONID and
          struts token and can be forged is similar matter. We need a
          per page token or rotating token to<o:p></o:p></p>
        <p class="MsoNormal">After some careful analysis on (what looks
          like) the latest and greatest source
          <a moz-do-not-send="true"
            href="https://github.com/aramrami/OWASP-CSRFGuard">https://github.com/aramrami/OWASP-CSRFGuard</a><o:p></o:p></p>
        <p class="MsoNormal">The problem is with the struts forward and
          flow is as follows:<o:p></o:p></p>
        <p class="MsoListParagraph"
          style="text-indent:-.25in;mso-list:l1 level1 lfo1"><!--[if !supportLists]--><span
            style="mso-list:Ignore">1.<span style="font:7.0pt
              "Times New Roman"">     
            </span></span><!--[endif]-->Initial request using (wonky
          javascript) and csrfguard taglib<br>
          openIwin("createSomeEntityForward.do?<csrf:token
          uri="createSomeEntityForward.do"/>", "Create
          Organization");<o:p></o:p></p>
        <p class="MsoListParagraph"
          style="text-indent:-.25in;mso-list:l1 level1 lfo1"><!--[if !supportLists]--><span
            style="mso-list:Ignore">2.<span style="font:7.0pt
              "Times New Roman"">     
            </span></span><!--[endif]-->CSRFGuard Filter (using token
          per page ) validates/allows the the 1<sup>st</sup> request
           uri
          <i><span
              style="font-size:10.0pt;font-family:Consolas;color:#2A00FF">createSomeEntityForward</span></i><o:p></o:p></p>
        <p class="MsoListParagraph"
          style="text-indent:-.25in;mso-list:l1 level1 lfo1"><!--[if !supportLists]--><span
            style="mso-list:Ignore">3.<span style="font:7.0pt
              "Times New Roman"">     
            </span></span><!--[endif]-->But when Filter intercepts 2nd
          forwarded path “<i><span
              style="font-size:10.0pt;font-family:Consolas;color:#2A00FF">/WEB-INF/jsp/…/createEntity.jsp"</span></i><span
            style="font-size:10.0pt;font-family:Consolas;color:teal">/>
          </span><span style="font-size:10.0pt;font-family:Consolas">The
            token in the request is not found in the csrfguard uri/token
            map so it checks against the session token which fails</span><o:p></o:p></p>
        <p class="MsoListParagraph"
          style="text-indent:-.25in;mso-list:l1 level1 lfo1"><!--[if !supportLists]--><span
            style="mso-list:Ignore">4.<span style="font:7.0pt
              "Times New Roman"">     
            </span></span><!--[endif]--><span
            style="font-size:10.0pt;font-family:Consolas">I also tried
            turning off token per page and turned on token rotate, the
            session token got updated, but the orig request has the now,
            stale token</span><o:p></o:p></p>
        <p class="MsoListParagraph"
          style="text-indent:-.25in;mso-list:l1 level1 lfo1"><!--[if !supportLists]--><span
            style="mso-list:Ignore">5.<span style="font:7.0pt
              "Times New Roman"">     
            </span></span><!--[endif]--><span
            style="font-size:10.0pt;font-family:Consolas">If there was
            some other normal (non csrf) validation, like a missing
            field, it would be forwarded to yet another page which would
            also break</span><o:p></o:p></p>
        <p class="MsoNormal"><o:p> </o:p></p>
        <p class="MsoNormal">Re-writing the webapp is not an option at
          this time, nor is upgrading to more modern version of struts<o:p></o:p></p>
        <p class="MsoNormal"><o:p> </o:p></p>
        <p class="MsoNormal">Proposed work-around:<o:p></o:p></p>
        <p class="MsoListParagraph"
          style="text-indent:-.25in;mso-list:l0 level1 lfo2"><!--[if !supportLists]--><span
            style="mso-list:Ignore">1.<span style="font:7.0pt
              "Times New Roman"">     
            </span></span><!--[endif]-->We were thinking of writing our
          own filter to attached token to each outbound request (and
          using the csrfguard api), to inject (add NV pair to URL if GET
          or body if POST) and forgoing javascript injection or csrf
          taglib, since we have hundreds of complex pages). <o:p></o:p></p>
        <p class="MsoListParagraph"
          style="text-indent:-.25in;mso-list:l0 level1 lfo2"><!--[if !supportLists]--><span
            style="mso-list:Ignore">2.<span style="font:7.0pt
              "Times New Roman"">     
            </span></span><!--[endif]-->When the csrfguard filter now
          picks up the inbound request, it validates each request,
          regardless if forward<o:p></o:p></p>
        <p class="MsoNormal"><o:p> </o:p></p>
        <p class="MsoNormal">Another workaround:<o:p></o:p></p>
        <p class="MsoListParagraph"
          style="text-indent:-.25in;mso-list:l2 level1 lfo3"><!--[if !supportLists]--><span
            style="mso-list:Ignore">1.<span style="font:7.0pt
              "Times New Roman"">     
            </span></span><!--[endif]-->Somehow get a handle to the orig
          request, the .do request, then change the
          CSRFGuard.verifyPageToken() method to check the URI/Token map
          for both the forwarded URI (.jsp) OR the orig URI (.do) for a
          match<o:p></o:p></p>
        <p class="MsoNormal"><o:p> </o:p></p>
        <p class="MsoNormal">We are open to other ideas as well?<o:p></o:p></p>
        <p class="MsoNormal"><o:p> </o:p></p>
        <p class="MsoNormal">Here’s a snippet of struts-config.xml
          (names fudged a bit)<o:p></o:p></p>
