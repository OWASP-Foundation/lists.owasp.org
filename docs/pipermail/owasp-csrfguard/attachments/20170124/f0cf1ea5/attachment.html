<p class="MsoNormal"><span style="font-size:10.0pt">Dear OWASP CSRFGuard Mailing List:<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:10.0pt"><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:10.0pt">Be advised, after obtaining the latest version I could locate (3.1.0) of CSRFGuard from github, I discovered that the correction identified in
<a href="https://github.com/aramrami/OWASP-CSRFGuard/issues/20">https://github.com/aramrami/OWASP-CSRFGuard/issues/20</a> to resolve a ClassNotFoundException at startup was NOT part of the zip file, so I modified the following line in the
<i>org.owasp.csrfguard.CsrfGuard.retrieveNewConfig()</i> method (and repackaged the executable JAR):<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:10.0pt">From<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:10.0pt">                              String configurationProviderFactoryClassName = this.properties.getProperty(<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:10.0pt">                                                            "org.owasp.csrfguard.configuration.provider.factory", PropertiesConfigurationProvider.class.getName());<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:10.0pt">To <o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:10.0pt">                              String configurationProviderFactoryClassName = this.properties.getProperty(<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:10.0pt">                                                            "org.owasp.csrfguard.configuration.provider.factory", PropertiesConfigurationProvider<b>Factory</b>.class.getName());<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:10.0pt"><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:10.0pt">However, I still ran into ClassLoader difficulties here in the next two lines in the same class/method:<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:10.0pt">                              Class<ConfigurationProviderFactory> configurationProviderFactoryClass = CsrfGuardUtils.forName(configurationProviderFactoryClassName);<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:10.0pt">                             
<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:10.0pt">                              ConfigurationProviderFactory configurationProviderFactory = CsrfGuardUtils.newInstance(configurationProviderFactoryClass);<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:10.0pt"><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:10.0pt">To “quickly” get around this, I then modified these, removing the reflective instantiation:<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:10.0pt">                              //Class<ConfigurationProviderFactory> configurationProviderFactoryClass = CsrfGuardUtils.forName(configurationProviderFactoryClassName);<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:10.0pt">                             
<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:10.0pt">                              ConfigurationProviderFactory configurationProviderFactory = new PropertiesConfigurationProviderFactory(); //CsrfGuardUtils.newInstance(configurationProviderFactoryClass);<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:10.0pt"><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:10.0pt">This in turn brought me to an issue in the
<i>org.owasp.csrfguard.config.PropertiesConfigurationProvider.PropertiesConfigurationProvider(Properties)</i> method, whereby I received the “action class  has not yet been specified” exception. After some time spent in debugging this issue, I finally determined
 that the properties example provided here: <a href="https://www.owasp.org/index.php/CSRFGuard_3_Configuration">
https://www.owasp.org/index.php/CSRFGuard_3_Configuration</a> and in the github zip file contains an error, specifically: org.owasp.csrfguard.action.<b>class</b>.Redirect = org.owasp.csrfguard.action.Redirect. After analyzing the code, the ONLY way I could
 get past this exception was by altering that property entry to: org.owasp.csrfguard.action.Redirect = org.owasp.csrfguard.action.Redirect (removing the .<b>class</b> node).<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:10.0pt"><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:10.0pt">I am now past the point of initialization of the framework.<o:p></o:p></span></p>
<p class="MsoNormal"><span style="font-size:10.0pt"><o:p> </o:p></span></p>
<p class="MsoNormal"><span style="font-size:10.0pt">Is there a repository with later versions than 3.1.0, which per the documentation, has been reworked and strengthened from 3.0?<o:p></o:p></span></p>
