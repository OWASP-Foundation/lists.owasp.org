Hey Chris,
the problem is that clients are downloading "full featured" code to their 
mobiles, as opposed to a form (in HTML), sort of like a thick client, but 
maybe not so thick.
The server side is not performing proper security but a better 
architecture is to make the client thinner and perform all DV on the 
server.
Simply put, thick client written in Java can be decompiled, altered or 
studied. 

This altered code can be uploaded to other clients who think they are 
using the real thing, like a phishing attack.??

The decompiled code can be reviewed to find weakness. Lets say there is 
code on the server doing security, but its simply a clone of the client 
side code, that would be no good is a weakness was found in the client.

The server may not be doing any App Sec, hoping client will take care of 
that (a big no no). (What you are saying?)

and loads more stuff, people more clever than myself can come up with.
