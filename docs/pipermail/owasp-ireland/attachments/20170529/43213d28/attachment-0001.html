                                                <p><a href="http://cts.vresp.com/c/?TheOWASPFoundation/a6cf2b52a7/0a86f0b8bf/cc5ea4dbee"><img height="100" src="https://c.na48.content.force.com/servlet/servlet.ImageServer?id=0150B000005aZIo&oid=00DU0000000IvqV&lastMod=1495673752000" style="width: 187px; height: 100px;" /></a></p><p>The <a href="http://cts.vresp.com/c/?TheOWASPFoundation/a6cf2b52a7/0a86f0b8bf/45fb9b3a47" style="color:#00549d">OWASP Code Sprint 2017</a> is a program that aims to provide incentives for students to contribute to OWASP projects. By participating in the OWASP Code Sprint 2017 a student can get real-life experience contributing to an open source project. A student who successfully completes the program will receive in total $1500. You can read more on the <a href="http://cts.vresp.com/c/?TheOWASPFoundation/a6cf2b52a7/0a86f0b8bf/f7c9efb704" style="color:#00549d">Code Sprint 2017 wiki page</a>.</p><b>How it works</b><p> </p><p>Any code/tool project can participate in the OWASP Code Sprint. Each project will be guided by an OWASP mentor. Students are evaluated in the middle and at the end of the coding period, based on success criteria identified at the beginning of the project. Successful students will receive $750 after each evaluation, a total of $1500 per student.</p><p><a href="http://cts.vresp.com/c/?TheOWASPFoundation/a6cf2b52a7/0a86f0b8bf/f64d886229" style="color:#00549d">ALL STUDENTS PLEASE APPLY HERE</a></p><hr /></td>

<p><img src="https://c.na48.content.force.com/servlet/servlet.ImageServer?id=0150B000005ab8M&oid=00DU0000000IvqV&lastMod=1495765331000" style="width: 500px; height: 113px;" width="500" /></p><p>OWASP had another successful project summit at AppSec Europe Belfast. The Project Summit is a combination project working session and program improvement session for OWASP Projects. In addition to <a href="http://cts.vresp.com/c/?TheOWASPFoundation/a6cf2b52a7/0a86f0b8bf/6b257c645d/usp=sharing" style="color:#00549d">presenting their projects</a> to the summit, the leaders discussed ways for their projects to collaborate and there was a proposal from one leader to combine two existing projects with some additional work into a new third project. Project leaders were very excited about up coming improvements to OWASP and Projects, including Discourse, Project Handbook changes and gamification options that will be soon available.</p>

                                                <p>In addition to focusing on programmatic changes and their projects, leaders and other community members came together to review seven applications for projects wishing to graduate to the next level of project maturity.</p>

                                                <p>If you would like to attend the next <a href="http://cts.vresp.com/c/?TheOWASPFoundation/a6cf2b52a7/0a86f0b8bf/e1ac0ba651" style="color:#00549d">AppSec Project Summit</a>, see the below for details.</p>

                                                <h2 style="font-size: 14pt; color: #767a4c;letter-spacing:0.3px;padding:0;margin:6px 0px 0px 0px;">Chapter Leader Workshop</h2>

                                                <p>Before every Global AppSec Event we host a Leaders’ Workshop. These workshops serve to allow the foundation staff (and board) to hear from Chapter and Project Leaders and to catch them up on big occurrences in the OWASP Foundation. This year there was an unusually large portion of the meeting dedicated to sharing information from the Foundation.</p>

                                                <p>The OWASP Staff have been analyzing the Foundation’s technical debt and engaged in deep listening activities to understand how to prioritize our actions. This activity has led to the staff prioritizing 4 large changes to our structure: The Website Reboot, the AMS update, a new system for mailing lists, and a volunteer management program. These new systems will bring more insight and control to Project and Chapter Leaders.</p>

                                                <p>The Leaders’ Workshop is recapped in three blog posts: <a href="http://cts.vresp.com/c/?TheOWASPFoundation/a6cf2b52a7/0a86f0b8bf/58b2c8c94e" style="color:#00549d">OWASP Leaders’ Workshop Pt 1: 4 Major Changes and Leader Insight and Control</a>, about the 4 major updates; <a href="http://cts.vresp.com/c/?TheOWASPFoundation/a6cf2b52a7/0a86f0b8bf/ff829869a1" style="color:#00549d">OWASP Code Sprint 2017</a>, answering the question about the code sprint; and Chapter Questions from the LW, discussing the other two questions leaders brought up: (1)Discussion on move from 2 to 4 meetings per year, (2)What does the foundation look at when judging if an event can be charged for or not? The final blog post is not yet up.</p>
