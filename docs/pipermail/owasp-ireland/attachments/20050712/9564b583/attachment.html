<br><font size=3 face="Arial">A tool for hacking web services:</font>
<br>
<br><font size=3 face="Arial">WSDigger and is written to run on C# for
.NET 1.1 (Win32).</font>
<br><font size=2 face="sans-serif"><br>
</font><a href=http://www.foundstone.com/resources/s3i_tools.htm target=_blank><font size=3 color=blue face="Arial"><u>http://www.foundstone.com/resources/s3i_tools.htm</u></font></a><font size=3 face="Arial"><br>
<br>
The user specifies a UDDI and a search criteria such as &quot;weather&quot;,
the tool<br>
determines and displays the possible available services. The user then<br>
selects a service to connect to and the tool gets the WSDL and displays
the<br>
methods such as getHumidty(); or getTemp(); The user can then apply a<br>
payload such as SQL Injection or XPATH injection and determine if the web<br>
service has common vulnerabilities. The tool is written to accept plugins<br>
(we ship with 3 sample plugins for XSS, SQL Injection and XPATH injection).<br>
There will be a Sourceforge CVS tree to submit plugins for the framework.
It<br>
should be very easy to write any number of fuzzing type plugins. The code
is<br>
in the download and will in the CVS at</font><font size=3 color=blue face="Arial"><u><br>

this week.<br>
