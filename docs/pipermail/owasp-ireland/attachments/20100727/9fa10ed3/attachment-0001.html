<br><font size=3 color=#000080><tt><b>Security Implications for Web Applications
based on SOA</b></tt></font><font size=3><tt><br>
<br>
The main point of SOA (in this context) is combining systems and applications
to make new applications, or a big 'overall' application.<br>
<br>
This higher inter-operability does (by default) lower security. For a start,
a request originating from a web user might end up at several back end
systems, which do not know who or what the request came from.<br>
<br>
Each back end system might have no access to the customer data, have a
different security models, and serve serveral front end. Each of the above
systems could be under different ownership, thus the owners have different
concerns and priorities. Also, the basic solution at a technical level
include single sign on, or security as a service. This can be costly, give
limited coverage and have a performance hit. But is pretty much the only
way to do it. The other thing to do (probably in tandem) is strict management,
and delegation of authority.</tt></font>
<br>
<br><font size=3><tt>John has about 13 years in IT. Most of this in distributed
systems and 'Middleware' integration software. Including BEA (now owned
by Oracle). Mainly working on Enterprise Java and more recently on Microsoft
BizTalk. Various industries, incuding financials, public services, and
a fish farm.<br>
</tt></font>

About Security</b></tt></font>
<br>
<br><font size=3><tt>The web2py creators clearly kept security in mind
from the design phase, and it shows in the end result. web2py is immune

<br><font size=3 color=#000080><tt><b>5. Timing Attacks Explained</b></tt></font>
<br>
<br><font size=3><tt>Probably the most interesting bit of security research
that happened this week was the dropping of this little bombshell by Taylor
Nelson of Root Labs:</tt></font>
<br>
<br><font size=3><tt>Every OpenID implementation I have checked this far
has contained timing dependent compares in the HMAC verification, allowing
a remote attacker to forge valid tokens.</tt></font>

<br><font size=3 color=#000080><tt><b>6. Taint Mode for Python</b></tt></font>
<br>
<br><font size=3><tt>Juanjo Conti has done some fantastic work for OWASP
implementing a taint mode in Python through a library. Taint mode is a

user input through the code by tracking the “taintedness” of variables.</tt></font>
