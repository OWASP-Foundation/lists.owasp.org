Hi Shivi,<div><br></div><div>Detective measures can not (normally) be examined using pen testing such as :</div><div>(Some of these would not be performed during runtime testing as the tests may be too invasive on production systems.)</div>
<div><br></div><div>CRLF/Audit attacking: Damaging log integrity or logging untrusted input to logs directly without any input validation, sanitisation, length checking. Logger DoS: Log overflow which may cause availability issues on some systems due to poor error handling.</div>
<div><br></div><div>Auditing/Logging: at the correct place, information leakage, privacy, logging the right things.</div><div><br></div><div>Denial of service: Generally you wont test for this during a penetration test but you can detect this very easily with code review: broken logic, infinite loops, buffer overflows etc</div>
