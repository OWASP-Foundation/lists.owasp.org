Moth is a VMware image with a set of vulnerable Web Applications and<br>
scripts, that you may use for:<br>
    - Testing Web Application Security Scanners<br>
    - Testing Static Code Analysis tools (SCA)<br>
    - Giving an introductory course to Web Application Security<br>
<br>
The motivation for creating this tool came after reading<br>
&quot;anantasec-report.pdf&quot; which is included in the release file which you<br>
are free to download. The main objective of this tool is to give the<br>
community a ready to use testbed for web application security tools.<br>
For almost every web application vulnerability in existance, there is<br>
a test script available in moth.<br>
<br>
Other tools like this are available but they lack one very important<br>
feature: a list of vulnerabilities included in the Web Applications!<br>
In our case, we used the results gathered in the anantasec report to<br>
solve this issue without any extra work.<br>
<br>
There are three different ways to access the web applications and<br>
vulnerable scripts:<br>
    - Directly<br>
    - Through mod_security<br>
    - Through PHP-IDS (only if the web application is written in PHP)<br>
<br>
Both mod_security and PHP-IDS have their default configurations and<br>
they show a log of the offending request when one is found. This is<br>
very useful for testing web application scanners, and teaching<br>
students how web application firewalls work. The beauty is that a user<br>
may access the same vulnerable script using the three methods; which<br>
helps a lot in the learning process.<br>
