<p class=MsoNormal><span style='font-size:10.0pt;font-family:"Arial","sans-serif"'>Just
wanted to pick up on the topic of arrays and pointers ala &#8220;char
*buf[MAXLEN];&#8221; as this is one area that is commonly misunderstood.<o:p></o:p></span></p>

<p class=MsoNormal><span style='font-size:10.0pt;font-family:"Arial","sans-serif"'><o:p>&nbsp;</o:p></span></p>

<p class=MsoNormal><span style='font-size:10.0pt;font-family:"Arial","sans-serif"'>Assuming
this is C code&#8230;it is worth pointing out that due to C&#8217;s operator
precedence &#8220;char *buf[MAXLEN];&#8221; defines an array of pointers to
type char &#8211; not a pointer to an array of chars as one may expect.<o:p></o:p></span></p>

<p class=MsoNormal><span style='font-size:10.0pt;font-family:"Arial","sans-serif"'>&#8220;char
buf[MAXLEN];&#8221; defines an array of chars as expected. <o:p></o:p></span></p>

<p class=MsoNormal><span style='font-size:10.0pt;font-family:"Arial","sans-serif"'><o:p>&nbsp;</o:p></span></p>

<p class=MsoNormal><span style='font-size:10.0pt;font-family:"Arial","sans-serif"'><o:p>&nbsp;</o:p></span></p>

<p class=MsoNormal><span style='font-size:10.0pt;font-family:"Arial","sans-serif"'>The
obvious difference is that sizeof(buf) will be 4096 i.e. 4 times MAXLEN, not
1024 as expected (assuming pointers are 4 bytes as determined by build
environment)<o:p></o:p></span></p>

<p class=MsoNormal><span style='font-size:10.0pt;font-family:"Arial","sans-serif"'>There
are other subtle differences that can catch the unwary related to this point e.g.
how buf is interpreted if passed to a C function. <o:p></o:p></span></p>

<p class=MsoNormal><span style='font-size:10.0pt;font-family:"Arial","sans-serif"'>This
subtlety can and does lead to nasty bugs.<o:p></o:p></span></p>

<p class=MsoNormal><span style='font-size:10.0pt;font-family:"Arial","sans-serif"'><o:p>&nbsp;</o:p></span></p>

<p class=MsoNormal><span style='font-size:10.0pt;font-family:"Arial","sans-serif"'>This
topic is covered extensively in Peter Van Der Linden&#8217;s excellent book &#8220;Expert
C Programming - Deep C Secrets&#8220; &nbsp;The chapter is entitled &#8220;The <em><span
style='font-family:"Arial","sans-serif"'>Shocking Truth</span></em>: C Arrays
and Pointers Are NOT the Same!&#8221;<o:p></o:p></span></p>
