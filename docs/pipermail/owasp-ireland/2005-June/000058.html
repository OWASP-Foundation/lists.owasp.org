1.
An interesting document from Watchfire (from AppScan fame) which explains 
a type of attach called HTTP Smuggling.
HTTP smuggling is in effect placing more than one HTTP request in a single 
HTTP request??!!*#
The different parsing methods of various devices on a network (Firewall 
HTTP server, Appserver) cause discrepancies which can be taken advantage 
of.

http://www.watchfire.com/resources/HTTP-Request-Smuggling.pdf
