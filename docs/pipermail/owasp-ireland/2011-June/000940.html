Hi Brian,

Burp is a commercial tool that costs like $150 and would be a really good
tool to start with if you wanted to make the comparison between free and
commercial tools..There has just been a new version released which has
incorporated allot of the functionality that has been missing for a long
time. Cenzic is also a more robust automated solution. I have a contact
there who could possibly give you a few weeks trial if you were interested.
He might see the benefit to the company of your research.

Owen

> Hi,
>
> I'm doing a practicum for my course in DCU on the area of penetration
> testing for web applications. I thought this would be a good place to tell
> about it since it concerns web application security.
>
> I plan to investigate and examine the current tools/frameworks available to
> pen-testers out now, and their effectiveness in sample tests against some
> web services. Right now, I have the sample Hacme services available from
> Foundstone ( http://www.mcafee.com/us/downloads/free-tools/index.aspx ).
> Since these are intentionally filled with potential exploits, they are the
> best web services to test these tools on.
>
> What I plan to do is the following:
>
> 1. Develop my own process for penetration testing. From my research I do
> not
> see a standard approach to this. I am currently going through and plan to
> use some aspects of the OWASP Application Security Verification Standard in
> this, as well as the OWASP Top 10 in my attempt to develop an approach to
> testing.
>
> 2. Test these sample web services with two tools, w3af and Metasploit Pro.
> W3af ( http://w3af.sourceforge.net/ ) is open source tool, while
> Metasploit
> Pro ( http://www.rapid7.com/products/metasploit-pro.jsp ) is a commercial
> tool.
>
> 3. Test the web services using the approach I developed.
>
> 4. Compare the results of all 3 methods used.
>
> During the process of this I will be researching and studying everything
> security related for web applications so I should come out the other end
> with some good knowledge of web app exploits, how to run a pentest
>
> Is there anything obvious that I am missing that should be in a study of
> penetration testing of web applications and its available tools?
>
> Would anyone be interested in the results from this investigation?
>
> Thanks,
>
> Brian

> Hi Brian,
> ASVS is a great start as the OWASP Top 10 was never and is not a standard
> but a survey with lots of great data and remediation hanging off of it.
> The OWASP Testing guide may help with you developing an approach. Many
> approaches are not shared as they are considered internal information.
> Last time I looked metasploit is not a pure web app sec tool but rather a
> exploit tool with some nice auxillary scanners etc.
>
> Burp Proxy and Zap are very common web proxies used in industry today
> coupled with commercial tools such as IBM Appscan, HP Webinspect or Cenzic
> hailstorm not to forget Netsparker but to name a few.
>
> The more mature approach is secure coding, development process and code
> review coupled with runtime testing.
> Industry is realising that a pen test alone is not enough as it does not
> cover all aspects.
>
> Also the idea of scanning for known vulnerabilities vs fault injection
> these
> are different concepts.
> Many free/Open source tools do not do "known vulnerabilities" as well if at
> all as this requires constant rule base updating as new issues are
> discovered and the developers focus on improving accuracy of the result
> parsing and injection rather than making new rules to reproduce an exploit.
>
> -ek
