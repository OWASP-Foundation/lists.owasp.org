On 01/02/07, Stephen O'Sullivan <sosullivan at gcxcorp.com> wrote:
>
>  Good morning All
>
> Anyone on this mailing list know of SFTP options for Windows 2003?
>

Free: Cygwin + openssh should work fine. If you want a package for this, you
could update the sshwindows project (http://sshwindows.sf.net), which is a
bit outdated; currently at openssh 3.8.1p1, needs repackaging, not a complex
task.

Free: alternative to sftp might be a host-to-host IPSec, with a policy
restricting access to FTP flows. Advantage of this is that no extra software
is required, but management complexity might be a factor if you're planning
on growing this to a large number of sites. Perhaps no more complex that
managing ssh client and key deployments. Of course, you're still using ftp
and your requirements may be different from the security characteristics
afforded by a host-to-host IPSec solution.

OpenSource or Commercial  - I don't mind..... once its tried, tested,
> reliable, and secure.....
>

> - Tumbleweed SecureTransport Server
> - Globalscape Secure FTP Server
>

Not familiar with any of these as cygwin+openssh has proved reliable.
However, you could also try ssh.com as another commercial source.
