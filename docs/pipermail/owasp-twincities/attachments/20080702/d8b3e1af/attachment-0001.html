<a name="11ae4ba96fad2d12_Topic_Overview"></a><h4> <span> Topic Overview </span></h4>
<p>SOA and Web services promise wonderful interoperability, but
distributed systems create lots of room for fantastic failures. This
session will explore the gory details of unique vulnerabilities at each
layer of the SOA stack - from the WSDL interfaces to XML processing
(XSD, XPath and XQuery), to the implementation languages liike Java and
C#, to new security standards like WS-Security and SAML.
</p><p>Gunnar gave this talk with Brian Chess at the <a href="https://cm.rsaconference.com/US08/catalog/controller/catalog" title="https://cm.rsaconference.com/US08/catalog/controller/catalog" rel="nofollow" target="_blank">2008 RSA Conference</a>.</p>
<br>Gunnar was also interviewed by Gary McGraw:<br><br>&quot;On the 27th episode of The Silver Bullet Security Podcast, Gary McGraw interviews software security expert <span class="nfakPe">Gunnar</span> Peterson, a Managing Principal at Arctec Group. Gary and <span class="nfakPe">Gunnar</span>
begin with the age-old question, "What is security?" They go on to
discuss how Web 2.0 and SOA security is progressing, the big idea
behind "federated identity," whether all market verticals can follow
the software security lead of the financial services industry, and the
inherent badness of the color purple.&quot;<br>
<br>
<a href="http://www.cigital.com/silverbullet/show-027/" target="_blank">http://www.cigital.com/silverbullet/show-027/</a><br><br><p><a name="11ae4ba96fad2d12_Speaker_Bio"></a></p><h4> <span> Speaker Bio </span></h4>
<p>Gunnar Peterson is a Managing Principal at Arctec Group focused on
architecture consulting and training. Peterson is an Associate Editor
for IEEE Security &amp; Privacy Journal; leads the OWASP XML Security
Gateway Evaluation Criteria project; and contributor to the SEI and DHS
Build Security In portal on software security. He maintains a blog at <a href="http://1raindrop.typepad.com/" title="http://1raindrop.typepad.com" rel="nofollow" target="_blank">http://1raindrop.typepad.com</a>.
