<h1><b><font size="6" face="Verdana"><span style="font-size: 24pt; font-family: Verdana;">April 27th OWASP Meeting –Gunnar

<p><b><font size="2" face="Verdana"><span style="font-weight: bold; font-size: 10pt; font-family: Verdana;">Topic 
Overview:</span></font></b><font face="Verdana"><span style="font-family: Verdana;"></span></font></p>
<p class="MsoPlainText"><b><font size="5" face="Verdana"><span style="font-weight: bold; font-size: 16pt; font-family: Verdana;">OWASP Top Ten 
Web Services</span></font></b></p>
<p class="MsoPlainText"><font size="2" face="Verdana"><span style="font-size: 10pt; font-family: Verdana;">What do Web apps, Web 2.0, Cloud 
Computing, SOA, and Rest all have in common? They all use Web services for 
functionality, data access and integration. Unfortunately, by default Web 
services also lack a security model. The OWASP Top Ten Web Services goes into 
the technical details of the vulnerabilities, remediations, and examples of 
common  </span></font></p>
<p class="MsoPlainText"><font size="2" face="Verdana"><span style="font-size: 10pt; font-family: Verdana;">Web services security issues like 
authentication and authorization flaws, how sensitive data is disclosed, and why 
security standards like WS-Security and SAML can be your best friend or your 
worst nightmare.</span></font></p>
