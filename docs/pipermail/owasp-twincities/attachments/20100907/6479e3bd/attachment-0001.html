<p><strong>The Talk: </strong></p>
<p><strong></strong></p><strong>
<h3>How to Do Application Logging Right</h3></strong>
<p></p>
<div>
<div>
<p><a href="http://chuvakin.blogspot.com/" target="_blank"><span style="TEXT-DECORATION: underline"><font color="#0066cc">Anton Chuvakin</font></span></a> and Gunnar have a paper in the current IEEE Security &amp; Privacy Journal - &quot;<a href="http://arctecgroup.net/pdf/howtoapplogging.pdf" target="_blank"><span style="TEXT-DECORATION: underline"><font color="#0066cc">How to Do Application Logging Right</font></span></a>.&quot; The paper explores app logging from a developer&#39;s perspective. There are various standards that mandate logging, most famously PCI DSS. PCI mandates a regime around log storage and security and provides an event model and log format for certain data (like financial data). For developers, though, it does not provide guidance on audit logger placement nor what&#39;s useful to include and not include in event payloads. Gunnar will be providing a talk modeled around this work.</p>
