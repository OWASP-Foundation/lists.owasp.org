>From my days of converting files into fax documents, I remember that
each file usually has an identifier within the first 20 bytes of the
file itself. I did a rough check of some samples and find that --<br>
&nbsp; -- readily identify GIF and PNG by %GIF and %PNG in the first 4
characters.<br>
&nbsp; -- readily identify JPG by looking for the signature of JFIF. <br>
<br>
Obviously, just like locks on doors, you will only catch the "honest"
person. But there is one other way in which we detected files for
conversion (and we, also, did not depend on the external file name
extension). We actually ran the file through a "helper" application
that if valid, would produce a rendition into the file type we want.
For instance, a program that only reads GIF and JPG would render
correctly (return code 0) and then we would export to the
"intermediate" format necessary for conversion into a fax later in the
process.<br>
