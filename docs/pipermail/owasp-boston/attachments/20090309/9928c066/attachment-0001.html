<P><FONT face=Arial><FONT size=2><B>Main Presentation</B> </FONT></FONT></P>
<P><FONT face=Arial><FONT size=2><B>Framework-Level Threat Analysis - Adding 
science to the art of source code review</B> </FONT></FONT></P>
<P><FONT face=Arial size=2>A traditional Threat Model is an effective tool for 
determining the threats that pose a risk to the architectural components of an 
application. But what if we wish to enumerate the threats that face the 
developmental components? Framework-Level Threat Analysis is a systemic approach 
to code review that speaks to the development staff by examining the underlying 
object model of an application. </FONT></P>
<P><FONT face=Arial size=2>In Framework-Level Threat Analysis, you are reviewing 
the application&#8217;s source code and breaking it down to it&#8217;s various components 
(servlet container, servlets, controller, delegate, command/business object, 
DAO, etc&#8230;), very similar to how you break down an app into its various 
architectural components in a standard TM. You then analyze a good number of use 
cases and model the way that data moves between the various components. The next 
step is to analyze each of the components in the path (can be as detailed as 
reviewing class by class) and document which security controls are happening 
where in the call flow. Once you&#8217;ve done that, it becomes visually apparent 
which security controls are missing in each use case. </FONT></P>
