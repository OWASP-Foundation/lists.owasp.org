So a lot of your typical weaknesses are going to apply - make sure not to
trust unsantized input.

SWFIntruder was released this year - you'll definitely want to check it out:
http://www.owasp.org/index.php/Category:SWFIntruder

Also, if you're trying to reverse engineer SWF's, the following are
helpful:

Flare - actionscript decompiler - dumps all the script
Flasm - actionscript dissambler (and assembler) - dumps bytecode

Also make sure to check the crossdomain.xml -
http://kb2.adobe.com/cps/142/tn_14213.html
