Well a starting point should be that whoever owns the system needs to
have control of all passwords associated with it. If you have a system
manager who is the only person who knows the password to the system,
you have a serious problem if they either fall under a bus or decide
to attempt extortion. Both happen.

Access rights for scripts have always been something of a problem.
They don't really fit in the standard paradigm for access control. My
preference would be to create a separate account for each script and
to give the account system privileges to allow access to every
resource it requires.

That approach does not really work well across machine boundaries
though. For that I would probably want to use kerberos combined with a
primary authentication mechanism that allows whatever credentials are
in use to be tied to a particular machine in such a way that they can
never be exported.
