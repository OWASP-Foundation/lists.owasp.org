<p><span style='font-size:10.0pt;font-family:"Verdana","sans-serif"'>Dear
Members,<o:p></o:p></span></p>

<p><span style='font-size:10.0pt;font-family:"Verdana","sans-serif"'>Aiming to
better identify bugs that could lead to security issues, Microsoft announced on
Wednesday that it planned to release a tool to help developers classify and
assess program crashes.<o:p></o:p></span></p>

<p><span style='font-size:10.0pt;font-family:"Verdana","sans-serif"'>The tool,
known as !exploitable and pronounced &quot;bang exploitable,&quot; is a plugin
for the Windows debugger that categorizes crash information using two hashes,
members of Microsoft's Security Science group told <cite><span
style='font-family:"Verdana","sans-serif"'>SecurityFocus</span></cite> in a
briefing. Using the hashes, which Microsoft called major and minor hashes, the
tool can group crashes that are caused by the same bug, even if the program
executed in different ways to produce the crashes. The tool also estimates the
exploitability of the vulnerability, rating them as Exploitable, Probably
Exploitable, Possibly Exploitable, and Unknown.<o:p></o:p></span></p>
