<p class=MsoNormal>The Software Assurance Maturity Model (SAMM) is an open
framework to help organizations formulate and implement a strategy for software
security that is tailored to the specific risks facing the organization. The
resources provided by SAMM will aid in:<o:p></o:p></p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>◊ Evaluating an organization’s existing<o:p></o:p></p>

<p class=MsoNormal>software security practices<o:p></o:p></p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>◊ Building a balanced software security program<o:p></o:p></p>

<p class=MsoNormal>in well-defined iterations<o:p></o:p></p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>◊ Demonstrating concrete improvements<o:p></o:p></p>

<p class=MsoNormal>to a security assurance program<o:p></o:p></p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>◊ Defining and measuring security-related activities<o:p></o:p></p>

<p class=MsoNormal>within an organization<o:p></o:p></p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>SAMM was defined with flexibility in mind such that it can
be utilized by small, medium, and large organizations using any style of
development. Additionally, this model can be applied organization-wide, for a
single line-of-business, or even for an individual project.<o:p></o:p></p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>As an open project, SAMM content shall always remain
vendor-neutral and freely available for all to use.<o:p></o:p></p>
