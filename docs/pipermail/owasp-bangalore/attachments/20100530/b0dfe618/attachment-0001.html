<P><FONT SIZE=2>Hi,<BR>
<BR>
This is regarding multiple XSS vulnerabilities in core components of the administrative section of Joomla!.<BR>
<BR>
* Project: Joomla!<BR>
* SubProject: All<BR>
* Severity: High<BR>
* Versions: 1.5.17 and all previous 1.5 releases<BR>
* Exploit type: XSS Injection<BR>
* Reported Date: 2010-May-13<BR>
* Fixed Date: 2010-May-28<BR>
* Fixed Version: Joomla! 1.5.18<BR>
* Update Download Link: <A HREF="http://www.joomla.org/download.html">http://www.joomla.org/download.html</A><BR>
* Info URL: <A HREF="http://developer.joomla.org/security/news/314-20100501-core-xss-vulnerabilities-in-back-end.html">http://developer.joomla.org/security/news/314-20100501-core-xss-vulnerabilities-in-back-end.html</A><BR>
<BR>
Vulnerability Details:<BR>
<BR>
User can execute arbitrary JavaScript code within the vulnerable application.<BR>
<BR>
<BR>
The vulnerability arises due to the administrator core components failing to properly sanitize user-supplied input in the &quot;search&quot; variable. Successful exploitation of this vulnerability could result in, but not limited to, compromise of the application, theft of cookie-based authentication credentials, arbitrary url redirection, disclosure or modification of sensitive data and phishing attacks.<BR>
<BR>
<BR>
An attacker can send a link with the exploit to an administrator whose access could compromise the application.<BR>
<BR>
All admins who use Joomla! version 1.5.17, please update to the latest version.<BR>
