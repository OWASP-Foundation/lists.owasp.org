&gt;&gt; From what I know an App cannot be certified to ISO 27001 because there is<br>
&gt;&gt; no provision in the standard to certify an application for security.<br>
&gt;&gt;<br>
&gt;&gt;<br>
&gt;&gt;<br>
&gt;&gt; The standard sets up the ISMS in the organization and will ask that<br>
&gt;&gt; applications be secured and tested for security vulnerabilities. However it<br>
&gt;&gt; does not go into the nitty gritty of application testing and this is left<br>
&gt;&gt; for the organization to decide upon. If you see this is the case with Risk<br>
&gt;&gt; Management, Asset Management etc in ISO 27k1 too.<br>
&gt;&gt;<br>
&gt;&gt;<br>
&gt;&gt;<br>
&gt;&gt; So for your client to be asking why OWASP when ISO is there the answer is<br>
&gt;&gt; that ISO is a standard for ISMS but OWASP is the framework to build / test<br>
&gt;&gt; security in the application.<br>
&gt;&gt;<br>
