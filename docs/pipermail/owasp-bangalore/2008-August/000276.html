	An interesting read from the Mumbai list, wanted to share with
all here.
	 
	A new threat has been discovered against user supplied content.
Basically you can concatenate an image and a jar together and then
upload it to a server that allows for image upload.  This type of file
is called a gifar (gif + jar), but can be a jpg or other file types and
a jar.

	The server can perform fairly comprehensive data validation on
the gifar beyond simply checking the file name extension, including
actually looking at the image header, etc. and conclude that it is an
image, completely missing the jar tacked onto the end.  You can unzip
the file and see the jar, unzip will ignore the image part of the file.

	A browser (IE and Firefox both) will render the image portion of
the file via an image tag, and will also execute the jar portion of the
file via the applet tag.  Note that the applet tag can be on any page,
the important thing is to be able to upload the applet (embedded in the
image) and be able to reference its location (must be predictable).

	Once the applet is running, it can contact the server, so the
applet can send any arbitrary and arbitrarily complex HTTP and HTTPS
requests back to the server the applet was retrieved from.  And, the
kicker is that the applet has access to the cookies related to the
server it was retrieved from, so it can masquerade as the user.  Evil.
