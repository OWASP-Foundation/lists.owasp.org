> On Wednesday, 5 February 2014 22:14:39 UTC+5:30, karniv0re wrote:
>
> > *Please note that all Bangalore null meets are free for anyone to attend.
> > There are absolutely no fees. Just come with an open mind and willingness
> > to share and learn.*
> >
> > The schedule for this month's meet is as outlined below:
> >
> > 09:30 - 10:10:  Web Application Security for Beginners: CSRF - Jayesh
> Singh
> > This is a multipart series on Web Application Security. This session will
> > cover Cross Site Request Forgery (CSRF) attacks, the identification and
> > concepts behind it. The speaker will also cover the creation of a PoC and
> > the mitigation for this vulnerability.
> >
> > 10:10 - 10:20:  A Introductions
> >
> > 10:20 - 10:50: Getting started with iptables - Sriram Narayanan
> > The Linux iptables firewall is usually considered either a scary or a
> > complicated topic. Ram will cover some basics, introduce patterns of
> > thinking and approaching iptables, and show some examples of how this is
> > used. This should get you started with exploring more about this tool.
> >
> > 10:50 - 11:20:  Struts Validation Framework: Part 3 - Satish
> > This session is the third part of the talk on Validation frameworks.
> These
> > frameworks are used to secure information from entering business model in
> > an MVC architecture. ?Struts Validation framework? is a set of predefined
> > plugin codes which have proven best practices in Data validation. We will
> > take a look at the working of the framework and understand how malicious
> > data is treated.
> >
> > 11:20 - 11:40:  Networking session followed by a break
> >
> > 11:40 - 12:20:  The fall of a domain - Riyaz Walikar
> > This is a demo of a simulated environment that was akin to a domain
> > controller compromise from the network during a pentest that Riyaz
> > performed. He will show the tools and the tricks employed to go from a
> > local user to a domain administrator in this fast paced demo.
> >
> > 12:20 - 12:40: XMLChor - XPath Injection Exploitation Tool (Demo) -
> > Harshal Jamdade
> > XMLChor is a web based tool which will allows attackers to find and
> > exploit XPATH injection vulnerabilities in vulnerable web applications.
> > This tool automates XPATH injection.
> >
> > 12:40 - 13:10: Web Application Security: The pitfalls and the brickwalls,
> > a developer perspective - Insecure Direct Object Reference - Vamsi
> Krishna
> > This is a multi-part series on common developer mistakes that result in
> > major security vulnerabilities. This month we will see how authentication
> > and authorization are implemented in current systems where a developer
> does
> > not anticipate a permission model for objects resulting in unauthorized
> > access to data resulting in an Insecure Direct Object Reference.
> >
> > 13:10 - 13:30:  Feedback and Topic discussion for next month meet
> >
> >
