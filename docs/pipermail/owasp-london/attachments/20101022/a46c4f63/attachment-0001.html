    Hi dear fellow OWASP London members, <br>
    <br>
    many of the security problems we face derive from weaknesses and
    inconsistencies in the operation of the browsers and the HTTP
    protocol. And as OWASP, we have been pretty good at mitigating these
    problems and working around these deficiencies. But many times I
    felt like we actually should try to change the underlying root of
    the problem rather then having to live and work within it. <br>
    <br>
    The IETF (Internet Engineering Task Force) set up a new working
    group WEBSEC (<a
      href="http://datatracker.ietf.org/wg/websec/charter/">http://datatracker.ietf.org/wg/websec/charter/</a>)
    to identify and tackle these underlying issues and develop standards
    to solve them wherever possible. It will also work closely with the
    W3C Web Application Security Working Group to get the new changes on
    the road and implemented. <br>
    <br>
    These two working groups will provide the unique opportunity to
    actually change some of the underlying mechanisms and try to solve
    some of our biggest security challenges at the root. But for that to
    accomplish, it is important to understand what the problem and what
    should be done and what better place then OWASP, with the best Web
    Application Security minds, to ask what is good and what should be
    changed and corrected and in which way.<br>
    <br>
    So I would like to invite you to provide input for this and join the
    WEBSEC working group (note: the IETF is an open organization like
    OWASP and all WGs are open for everybody to join freely). <br>
    <b>You can join the group and its mailing-list here: <a
        href="https://www.ietf.org/mailman/listinfo/websec">https://www.ietf.org/mailman/listinfo/websec

      </a></b><br>
    If you are new to the IETF, there is a small info page Tao of the
    IETF (<a href="http://www.ietf.org/tao.html">http://www.ietf.org/tao.html</a>)<br>
    <br>
    <b>So what are your ideas to help mitigate many of the OWASP top
      ten?</b><br>
    What are the biggest problems? <br>
    What should be done? <br>
    What HTTP headers, protocol and policies would you suggest we change
    or correct? <br>
    Do we need to standardize new headers and certain browser behavior
    across the vendors? <br>
    How should browsers behave to be safe or how should their behavior
    change for use to be able to build more secure web applications? <br>
    <br>
    The time frame for this opportunity of change is within the next few
    months, so if you have ideas and suggestions please come forward and
    we need to work on them now!<br>
    <br>
    Just as an example of the first items that we discuss are Media Type
    Sniffing, Web Origin Concept, integrity of the browser and server,
    usage of DNSSEC, Strict Transport Security, X-FRAME-OPTIONS, but
    there's a lot more to do and we need your input as OWASP experts!<br>
    <br>
    So if you have ideas, please join the WG and/or get in touch with
    me. <br>
    And please forward this to interested chapters and people!<br>
    <br>
