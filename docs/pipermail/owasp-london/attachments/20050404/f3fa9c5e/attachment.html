scope is to wide since they don't really define what&nbsp; an authorized

<i><b><b>" ... <br>

And if you then look at the CMA definition of 'secures access to any

many of you have this level of EXPLICIT consent when you are accessing
your systems.<br>

When was the last time you saw a document that EXPLICITLY defined what

program or data</i>"<br>
<br>
Maybe an interesting exercise for us (as a community) to do, would be
do create a series of 'Use Case' scenarios and map them to the CMA. For
example:<br>
<br>
&nbsp;&nbsp;&nbsp; - "Using a computer at work"<br>
&nbsp;&nbsp;&nbsp; - "Hot-desking (i.e. using different computers to work)"&nbsp; <br>
&nbsp;&nbsp;&nbsp; - "Helping a colleague (on his computer) to perform an
action(correct something, configure something, change something)"<br>
&nbsp;&nbsp;&nbsp; - "Accessing a server (managed by a colleague) to install or
configure a software"<br>
&nbsp;&nbsp;&nbsp; - "Using an Public Internet access point (at a Cyder cafe, Leisure
Club, etc.."<br>
&nbsp;&nbsp;&nbsp; - "Browsing the web, accessing an FTP site"<br>
&nbsp;&nbsp;&nbsp; - "Using a Server which is used to host different websites"<br>
<br>
The big question is: "On each of these scenarios where is the line
drawn between and authorized access and an unauthorized one?" <br>
