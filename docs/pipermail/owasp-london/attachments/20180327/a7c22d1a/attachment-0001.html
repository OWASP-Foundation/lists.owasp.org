    All,<br>
    <br>
    We encourage you to present talks at our upcoming London Chapter
    events. <br>
    <p>
      OWASP mainly focuses on these 3 tracks: <br>
      <br>
       
      * Builders <br>
       
      * Breakers <br>
        * Defenders <br>
      <br>
      As a guideline we invite talks on the following topics (not an
      exhaustive list): <br>
      <br>
        * Novel web/mobile vulnerabilities and countermeasures (note: no
      zero-days, any vulnerabilities must be responsibly disclosed prior
      to presenting at OWASP events) <br>
        * New technologies, paradigms, frameworks, tools <br>
        * OWASP tools or projects - new and existing, best practices and
      experience of implementing them in your company <br>
        * Agile Security and Continuous Integration <br>
        * Secure development: frameworks, best practices, secure coding,
      methods, processes, standards, SDLC, code review <br>
        * AppSec Pipeline - Automation and Orchestration <br>
        * Threat modelling <br>
        * AI and Machine Learning in Application Security <br>
        * Browser security <br>
        * IoT security and AppSec <br>
        * JavaScript/Node.JS security <br>
        * Mobile security and security for the mobile web <br>
        * Threat Intelligence & AppSec <br>
        * Cloud security, Container security <br>
        * REST/SOAP and API security <br>
        * Security of frameworks <br>
        * Payment Security, PCI impact on AppSec, tools, frameworks,
      best practices <br>
        * Blockchain & AppSecurity <br>
        * Cryptography and its use (and abuse) in Mobile and Web Apps <br>
        * Data Privacy, Governance
      , GDPR & AppSec, GDPR & Developers<br>
        * Management topics: Managing Secure Software Development Life
      Cycle (SDLC), Managing Application Security initiatives, Developer
      Training <br>
      <br>
      Recommended talk duration is 45 minutes with additional 10-15
      minutes for any live demos and Q&A. <br>
      <br>
      We also encourage short “lightning” talks: 5-10 minutes in
      duration. <br>
    </p>
    <p>
      If you would like to present a talk at future OWASP London Chapter
      meetings please review and agree with the OWASP Speaker Agreement
      here: <br>
      <br>
      <a class="moz-txt-link-freetext"
        href="https://www.owasp.org/index.php/Speaker_Agreement">https://www.owasp.org/index.php/Speaker_Agreement</a>
      <br>
      <br>
      and contact the OWASP London Chapter Leaders via e-mail: <a
        class="moz-txt-link-abbreviated"
        href="mailto:owasplondon@owasp.org">owasplondon@owasp.org</a>
      with the following details: <br>
      <br>
      1. The title of your proposed talk (up to 12 words)<br>
      2. The abstract summarising your proposed talk (70-150 words) <br>
      3. The speaker bio (70-150 words)<br>
      <br>
      Please note that you can also pair up with a colleague and present
      a joint talk. </p>
    <p>Please ensure that your talk is objective, stresses open source
      approaches, and avoids references to any commercial offerings of
      your company. </p>
    <p>We are looking forward to your submissions.<br>
