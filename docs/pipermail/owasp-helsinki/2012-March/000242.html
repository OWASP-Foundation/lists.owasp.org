Lopputalven tervehdys sinulle OWASP Helsingin listan jäsen!
 
OWASP Helsinki -jaos kokoontuu vuoden ensimmäiseen tapaamiseensa keskiviikkona 21.3.2012 web-sovelluksen pääsynhallinnan merkeissä.
 
Paikkana on Marttakeskus Kampin metroaseman tuntumassa. Tarkka osoite on Malminrinne 1 B, 7. krs, Helsinki (ovisummeri "Kokoustilat). Kyllä, pääsemme tutustumaan Marttaliiton yhteydessä toimiviin tiloihin (http://www.ravintolamartta.fi/kokoustilat). 
 
Kokoontumisen aiheena ovat kokemukset web-sovelluksen pääsynhallinnan suunnittelusta.
 
Illan esiintyjällä Jim Manicolla on pitkä kokemus sovelluskehityksestä ja hän on toiminut vuoden 2008 alusta OWASP:n Connections Committee Chair -roolissa. Hänen laajasta tehtäväkentästään voi lukea OWASP:n sivuilta osoitteesta https://www.owasp.org/index.php/User:Jmanico . Jimin blogiin voit tutustua  osoitteessa manicode.blogspot.com . Jim on luvannut kertoa erityisesti auditoinneissa esiin tulleista pääsynhallinnan puutteista ja siitä millä menetelmillä niitä voidaan kokonaisvaltaisesti korjata.

17:50 Web Application Access Control Design Excellence / Jim Manico, OWASP
 
Abstract:
Access Control is a necessary security control at almost every layer within a web application. This talk will discuss several of the key access control anti-patterns commonly found during website security audits. These access control anti-patterns include hard-coded security policies, lack of horizontal access control, and "fail open" access control mechanisms. In reviewing these and other access control problems, we will discuss and design a positive access control mechanism that is data contextual, activity based, configurable, flexible, and deny-by-default - among other positive design attributes that make up a robust web-based access-control mechanism.
