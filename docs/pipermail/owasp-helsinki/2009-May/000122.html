Kommentti CSRF:n ongelmalähtöiseen kuvaukseen:
Puutteellinen tietokantakyselyn rakenteen säilyvyys on enemmän suomea,
SQL voi hyvin tulla ilmi otsikon yhteydessä olevasta SQL Injectionista

Kannatan kuitenkin seuraavia muotoja:

XSS:
Ongelma - Verkkosivun rakenne ei säily
Hyökkäys - Verkkosivun rakenteen muuttaminen välityshyökkäyksellä

CSRF:
Ongelma - Puutteellinen pyynnön alkuperän tarkistus
Hyökkäys - Pyyntöhuijaus toiseen sivustoon

Injektio:
Ongelma - Taustajärjestelmäkyselyn rakenne ei säily
Hyökkäys - Tietokantakyselyn rakenteen muuttaminen


XSS, CSRF ja SQL Injection kannattaa tosiaan pitää otsikoissa esim.  
suluissa mukana jo helpomman hahmottamisenkin kannalta.
