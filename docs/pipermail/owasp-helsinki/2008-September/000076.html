Tämä menee liian yksityiskohtaiselle tasolle tarkoituksiani varten.

Kyse on siis siitä, että sovelluskehittäjien on usein vaikea ymmärtää 
mitä tietoturvariskillä tarkoitetaan. Tietoturvavaatimustenkin 
"kääntäminen" riskimuotoon tuntuu olevan haasteellista. Siksipä 
tietoturvariskinalyysin tueksi tarvitaan kuvaus "yleisimmistä" 
tietoturvariskeistä - ihan vain ihmisten saamiseksi oikeaan moodiin.

> Top Ten suomennos on vielä alkutekijöissä ainakin listan osalta. Tälle
> syynä on tietysti minun saamattomuuteni ja kun ei ole asetettu
> aikarajaa käännökselle niin ei vielä edistytty. Tässä olisikin sopiva
> tilaisuus kysyä vapaa-ehtoisia käänöstöihin. Lähettäkää minulle postia
> niin edistetään sovellustietoturvatietoisuutta.

Mahdollisesti teen toptenistä tiivistetyt käännökset. Jos saan ne 
aikaiseksi, niin lähetän sinulle tarpeen tullen käytettäväksi.
