Tein SecurityFocuksen vulnerability-listaan analyysin, minkä tyylisiä
tietoturva-aukkoja löytyy erilaisilla ohjelmistokielillä tehdyissä
sovelluksissa. Lähinnä kiinnosti, miten J2EE:n ja php:n tietoturva-aukot
eroavat toisistaan. SF:n vulnerability tietokannan data ei ole kovin kattava,
materiaalini valinta oli kaikkea muuta kuin tilastollisesti pätevä,
luokittelu myös enemmänkin "mutu" kuin tieteellisesti pätevä.

JSP/Java sovellukset sisältävät myös sovelluspalvelimet kuten Tomcatin.
Periaate oli, että nappasin kaikki sellaiset sovellukset, jotka vastasivat
http-pyyntöihin jollain tavalla.

PHP:ssa SQL injection on pääosassa, koska kehittäjät käyttävät useimmiten
suoraan distron php-asennusta, jossa magic_quotes_gpc on päällä. Markkinoille
yrittävän konsultin kannattaa ensin ottaa satunnainen avoimen lähdekoodin
php-softa, kääntää magic_quotes_gpc pois päältä, iskee SQL:ää ja läpi
on kävelty. Remot file inclusion tapahtuu samaan tyyliin register_globals
-optiolla.

Tästä listasta puuttuu kokonaan CSRF, joka on myös melko tyypillinen oikeassa
elämässä. Sitä ei olla kuitenkaan taidettu nähdä kunnon
tietoturvaongelmana kuin vasta viime aikoina.
