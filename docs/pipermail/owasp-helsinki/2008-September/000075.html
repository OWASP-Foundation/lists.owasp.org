Moi,

yksi mahdollinen lähde voisi olla OWASPIN CLASP projekti jossa on
määriteltynä 104 tyypillistä virhettä lähdekooditasolla. Eivät ehkä
ole aivan yksiselitteisiä, mutta ainakin jotain sieltä voisi löytyä.
http://www.owasp.org/index.php/Category:OWASP_CLASP_Project ja
tarkemmin Vulnerability lexicon. Lisäksi löytyy Excel lista
edellämainituista
http://www.owasp.org/images/9/9a/VulnChecklist.v1.zip

Top Ten suomennos on vielä alkutekijöissä ainakin listan osalta. Tälle
syynä on tietysti minun saamattomuuteni ja kun ei ole asetettu
aikarajaa käännökselle niin ei vielä edistytty. Tässä olisikin sopiva
tilaisuus kysyä vapaa-ehtoisia käänöstöihin. Lähettäkää minulle postia
niin edistetään sovellustietoturvatietoisuutta.
