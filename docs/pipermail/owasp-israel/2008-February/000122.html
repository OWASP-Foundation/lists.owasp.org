Guy, 

To start with, the password space is a bit smaller - passwords are not case sensitive and special chars are not allowed.

IMHO you don't adress the real problem.

MY BANK HAS A ROTTEN PASSWORD POLICY!!! 

When you issue a corporate password policy, your user will tell you that you don't make sense: even the bank does not insist on more than 6 characters! And why use special characters? Why use mixed case?  If a simple password is good for the bank, surely it is good enough for me, no?

It makes you wonder what else they may have missed in their security procedures.

  The question is if limiting the number of possible passwords to 6 characters instead of the option to use any number of characters (6 or more in this case) is a security problem.

  IMHO, if you implement a good mechanism to lock the user after 3-5 bad login attempts and set policy to choose a new password every time a user locks down you'll see that brute force is not a real security issue in this case.

  After all - 6 characters give ~60^6 password options (why 60? All the lower and upper case, add 1-0 and you'll get 62 and I didn't mentioned the special characters)  and with a 5 bad login attempts lock - It is impossible to brute force.
