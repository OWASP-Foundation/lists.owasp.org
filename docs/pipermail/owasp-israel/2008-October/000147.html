שלום לכולם,

בתאריך 11.11.08 (שלישי בערב) אעביר מצגת בקהילת אבטחת אבטחת מידע למפתחים של מיקרוסופט בנושא מאוד מעניין, בעקבות מחקר שביצעתי – כיצד ניתן לשנות את שפת הפיתוח (לדוגמא C#) ולגרום לקוד אשר רץ תחת ה Framework לבצע דברים אחרים ממה שכתוב בקוד! נראה כיצד ניתן לשנות ע"י טכניקות של reverse engineering את המימוש הפנימי של ה Framework, וכן כיצד תוקף יכול לנצל עובדה זו על מנת לשתול קוד זדוני, כגון Rootkit או Backdoor, עמוק בתוך תשתית ה .NET כך שהיא תישאר בלתי ניתנת לזיהוי. 

במהלך המצגת נבחן כיצד ניתן לעקוף את מנגנוני הגנה קיימים של ה Framework, וכן נצפה במגוון דוגמאות כגון ביצוע Reverse shell, שליחת מידע רגיש לתוקף, ניטרול CAS, קיבוע מפתחות הצפנה וכדומה.

בנוסף, יחשף כלי בשם .NET-Sploit אשר פותח במיוחד לצורך ביצוע השינויים הנ"ל, אשר מסוגל לשתול Rootkits בתוך DLL של ה .NET Framework

הנך מוזמן למפגש קהילת אבטחת מידע - .Net security user group

שיתקיים בתאריך ה 11-11-08

התכנסות ב-17:30 בבית מיקרוסופט, רח' הפנינה 2 רעננה.

שם המפגש: .NET Root kits - Backdoors Inside Your Framework  

 

 

בהמשך להרצאה המצוינת שהעביר ארז מטולה (מומחה אבטחת אפליקציות, 2BSecure ) בכנס OWASP 2008 Conference, אני שמח שוב לארח בקבוצה את ארז על מנת שיעביר את המצגת במתכונתה המלאה. 

למי שלא היה בכנס או שכח (אני עדין לא שכחתי...) , המפגש יעסוק במחקר מאוד מעניין שביצע ארז ובסופו הוכיח כי ניתן לבצע פעולות זדוניות למול ה .Net Framework שכולנו כל כך אוהבים לבצע בו שימוש וחשבנו אותו לבטוח. 

הבעיות שארז גילה במחקר שביצע, חושפות לפנינו איומים חדשים שלא חשבנו שנאלץ להתמודד מולם עד היום, וכל הנראה נצטרך לדעת להתמודד מולם גם בעתיד וזאת וטרם התקבל מענה מחייב כל שהוא מחברת Microsoft לביצוע תיקונים במנגנוני האבטחה של ה Framework למול האיומים שנמצאו במחקר. 

 

להלן תיאור המפגש באנגלית: 

Following his interesting discovery on the Israeli OWASP 2008 Conference, I'm happy to have Erez Metula as a speaker for the next UG meeting. On the next meeting, Erez will talk about a very interesting method of modifying the .NET language, and changing its internal implementation! 

 

Description:

It is considered relatively easy to reverse engineer compiled .NET EXE's and DLL's. This is a well known fact. 

But what happens if we will use this fact to reverse engineer and modify .NET Framework DLL's and recompile the code..? We can change the .NET language! 

Erez Metula will expose the methods required to modify the Framework, and how to bypass its own protection mechanisms.

We will talk about the various ways to develop rootkits for the .NET framework, so that every EXE/DLL that runs on a modified Framework will behave differently than what it's supposed to do. Code reviews will not detect backdoors installed inside the framework since the payload is not in the code itself, but rather it is inside the framework implementation. 

We will see how to write rootkits for the framework, that will enable the attacker to install a reverse shell inside the framework, to steal valuable information, to fixate encryption keys, disable security checks and more. 

In addition, a new tool for building MSIL rootkits will be released that will enable the user to inject preloaded/custom payload to the Framework core DLL.
