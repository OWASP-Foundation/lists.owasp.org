Following his interesting discovery on the Israeli OWASP 2008 Conference, I'm happy to have Erez Metula as a speaker for the next UG meeting. On the next meeting, Erez will talk about a very interesting method of modifying the .NET language, and changing its internal implementation! 

 

Description:

It is considered relatively easy to reverse engineer compiled .NET EXE's and DLL's. This is a well known fact. 

But what happens if we will use this fact to reverse engineer and modify .NET Framework DLL's and recompile the code..? We can change the .NET language! 

Erez Metula will expose the methods required to modify the Framework, and how to bypass its own protection mechanisms.

We will talk about the various ways to develop rootkits for the .NET framework, so that every EXE/DLL that runs on a modified Framework will behave differently than what it's supposed to do. Code reviews will not detect backdoors installed inside the framework since the payload is not in the code itself, but rather it is inside the framework implementation. 

We will see how to write rootkits for the framework, that will enable the attacker to install a reverse shell inside the framework, to steal valuable information, to fixate encryption keys, disable security checks and more. 

In addition, a new tool for building MSIL rootkits will be released that will enable the user to inject preloaded/custom payload to the Framework core DLL.
