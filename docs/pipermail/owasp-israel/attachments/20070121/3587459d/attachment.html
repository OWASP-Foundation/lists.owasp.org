<p dir=LTR><font size=3 face="Times New Roman"><span style='font-size:12.0pt'><br>
<b><span style='font-weight:bold'>17:15 - Gathering, Socializing and Pizzas</span></b>
<o:p></o:p></span></font></p>

<p dir=LTR><font size=3 face="Times New Roman"><span style='font-size:12.0pt'><br>
</span></font><b><font size=4><span style='font-size:13.5pt;font-weight:bold'>17:30
- Source Code Analysis and Application Security - Cheating the maze</span></font></b>
<o:p></o:p></p>

<p dir=LTR><b><font size=3 face="Times New Roman"><span style='font-size:12.0pt;
font-weight:bold'>Maty Siman, Founder &amp; CTO, <a
href="http://www.checkmarx.com/" title="http://www.checkmarx.com/">Checkmarx</a></span></font></b>
<o:p></o:p></p>

<p dir=LTR><font size=3 face="Times New Roman"><span style='font-size:12.0pt'>During
the last few years automatically analyzing source code in order to find
security vulnerabilities became a popular method in the field of Application
Security. The presentation will discuss the theory and research of static code
analysis, the application of static code analysis for security, comparing this
method to other application security defense technologies and will demonstrate
the use of static code analysis for application security. <o:p></o:p></span></font></p>

<p dir=LTR><font size=3 face="Times New Roman"><span style='font-size:12.0pt'><br>
</span></font><b><font size=4><span style='font-size:13.5pt;font-weight:bold'>18:30
- Security Implications of .Net 3.0 and the Windows Communication Foundation
(WCF)</span></font></b> <o:p></o:p></p>

<p dir=LTR><b><font size=3 face="Times New Roman"><span style='font-size:12.0pt;
font-weight:bold'>Emmanuel Cohen-Yashar (Manu), Senior .NET technology
consultant, <a href="http://www.sela.co.il" title="http://www.sela.co.il">Sela
Group</a></span></font></b> <o:p></o:p></p>

<p dir=LTR><font size=3 face="Times New Roman"><span style='font-size:12.0pt'>Windows
Communication Foundation (WCF) is the new Microsoft communication framework
bundled as part of of .NET Framework 3.0, the new .NET Windows API succeeding
Win32 with the release of Windows Vista. WCF programming model unifies Web
Services, .NET Remoting, Distributed Transactions, and Message Queues into a
single Service-oriented programming model for distributed computing. The
presentation will describe the tenets of SOA – Service Oriented Architecture,
introduce WCF and discuss the security implications of this broad new
communication paradigm. <o:p></o:p></span></font></p>

<p dir=LTR><font size=3 face="Times New Roman"><span style='font-size:12.0pt'><br>
</span></font><b><font size=4><span style='font-size:13.5pt;font-weight:bold'>19:30
- Analysis of the Universal XSS PDF vulnerability - Cause, Solutions and Fun stuff</span></font></b>
<o:p></o:p></p>

<p dir=LTR><st1:PersonName w:st="on"><b><font size=3 face="Times New Roman"><span
 style='font-size:12.0pt;font-weight:bold'>Ofer Shezaf</span></font></b></st1:PersonName><b><span
style='font-weight:bold'>, CTO, <a href="http://www.breach.com"
title="http://www.breach.com">Breach Security</a>, Leader or <a
href="http://www.modsecurity.org/projects/rules/index.html"
title="http://www.modsecurity.org/projects/rules/index.html">ModSecurity Core
Rule Set</a> open source project </span></b><o:p></o:p></p>

<p dir=LTR><font size=3 face="Times New Roman"><span style='font-size:12.0pt'>Recently
a new vulnerability was discovered in commonly used versions of Adobe Acrobat
software. Unlike common XSS attacks that require a specific vulnerability in
the attacked web site, in this case the vulnerability in Acrobat is sufficient
and no fault is required in the attacked web site, and any site that serves PDF
files is vulnerable. Therefore it is called &quot;universal XSS&quot; vulnerability.
<o:p></o:p></span></font></p>

<p dir=LTR><font size=3 face="Times New Roman"><span style='font-size:12.0pt'>The
presentation will describe the vulnerability, the theoretical and practical
solutions for the vulnerability as well as some very funny stories about the
dynamics of such a high profile vulnerability, or in other words, what happens
when you try to get a car mechanic to fix an application security
vulnerability. <o:p></o:p></span></font></p>
