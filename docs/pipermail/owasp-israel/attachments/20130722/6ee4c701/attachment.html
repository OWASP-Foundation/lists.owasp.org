<div dir="ltr"><div><div><div>Hi all,<br><br>I'm pleased to announce the 0.6.0 release of Pythonect: <a href="http://www.pythonect.org" target="_blank">http://www.pythonect.org</a><br><br>Pythonect
 is a new, experimental, general-purpose dataflow programming language 
based on Python. It provides both a visual programming language and a 
text-based scripting language. The text-based scripting language aims to
 combine the quick and intuitive feel of shell scripting, with the power
 of Python. The visual programming language is based on the idea of a 
diagram with “boxes and arrows”. <br><br>Pythonect interpreter (and 
reference implementation) is a free and open source software written 
completely in Python. It is available under the the BSD 3-Clause 
License.<br><br></div>Earlier this year I gave a talk at a few conferences called 'Hacking Like 2013' (slides: <a href="http://www.ikotler.org/HackLikeIts2013_RVASec.pdf">http://www.ikotler.org/HackLikeIts2013_RVASec.pdf</a> and <a href="https://conference.hitb.org/hitbsecconf2013ams/materials/D1LAB%20-%20Itzik%20Kotler%20-%20Hack%20Like%20It%27s%202013.pdf">https://conference.hitb.org/hitbsecconf2013ams/materials/D1LAB%20-%20Itzik%20Kotler%20-%20Hack%20Like%20It%27s%202013.pdf</a>) in there I explain and show my vision for Pythonect as a tool for IT Security Professionals. <br>

<br></div><div>Among the examples you can find in the slides are simple web pentest scripts, fuzzers/fuzzing scripts, and some basic malware analysis automation.<br><br></div>Pythonect is still a work in progress and I'd be happy to hear your thoughts and feedback on it. Also, I am looking for interesting case studies and uses for Pythonect, and I am willing to do some development from my side to make it work.<br>
