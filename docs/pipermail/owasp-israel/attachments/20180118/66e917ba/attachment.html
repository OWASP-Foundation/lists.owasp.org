<p class="MsoNormal" style="margin:0in 0in 0.0001pt;font-size:12pt;font-family:Calibri,sans-serif"><span style="font-size:11pt"><u>Title</u>: </span><span style="font-size:11pt">Jumping into Heaven’s Gate -
Yarden Shafir</span></p>

<p class="MsoNormal" style="margin:0in 0in 0.0001pt;font-size:12pt;font-family:Calibri,sans-serif"><span style="font-size:11pt"><span> </span></span></p>

<p class="MsoNormal" style="margin:0in 0in 0.0001pt;font-size:12pt;font-family:Calibri,sans-serif"><span style="font-size:11pt"><u style="">Abstract:</u></span></p>

<p class="MsoNormal" style="margin:0in 0in 0.0001pt;font-size:12pt;font-family:Calibri,sans-serif"><span style="font-size:11pt">The old days of 32bit
applications are long bygone, nowadays most Operating Systems are running in a
64bit environment, requiring 64bit applications.<span></span></span></p>

<p class="MsoNormal" style="margin:0in 0in 0.0001pt;font-size:12pt;font-family:Calibri,sans-serif"><span style="font-size:11pt">So how can a 64bit Operating
System run a 32bit legacy Application?<span></span></span></p>

<p class="MsoNormal" style="margin:0in 0in 0.0001pt;font-size:12pt;font-family:Calibri,sans-serif"><span style="font-size:11pt">The native 64bit environment
cannot directly support the execution of a 32bit Application.<span></span></span></p>

<p class="MsoNormal" style="margin:0in 0in 0.0001pt;font-size:12pt;font-family:Calibri,sans-serif"><span style="font-size:11pt">32bit Applications expect
several surrounding pillars which help it perform necessary actions,<span></span></span></p>

<p class="MsoNormal" style="margin:0in 0in 0.0001pt;font-size:12pt;font-family:Calibri,sans-serif"><span style="font-size:11pt">and those no longer exist in
a 64bit environment.<span></span></span></p>

<p class="MsoNormal" style="margin:0in 0in 0.0001pt;font-size:12pt;font-family:Calibri,sans-serif"><span style="font-size:11pt">However, in practice Windows
contains many secrets, and one of those secrets is the WoW64<span></span></span></p>

<p class="MsoNormal" style="margin:0in 0in 0.0001pt;font-size:12pt;font-family:Calibri,sans-serif"><span style="font-size:11pt">subsystem.<span></span></span></p>

<p class="MsoNormal" style="margin:0in 0in 0.0001pt;font-size:12pt;font-family:Calibri,sans-serif"><span style="font-size:11pt">The Wow64 Subsystem supplies
a natural environment for the legacy 32bit Application and enables anyone to
run them on newer 64bit Operating Systems without any trouble.<span></span></span></p>

<p class="MsoNormal" style="margin:0in 0in 0.0001pt;font-size:12pt;font-family:Calibri,sans-serif"><span style="font-size:11pt">How the subsystem actually
does this remains a question to many.<span></span></span></p>

<p class="MsoNormal" style="margin:0in 0in 0.0001pt;font-size:12pt;font-family:Calibri,sans-serif"><span style="font-size:11pt">Any Application, whatever its
type, begins its execution in 64bit mode.<span></span></span></p>

<p class="MsoNormal" style="margin:0in 0in 0.0001pt;font-size:12pt;font-family:Calibri,sans-serif"><span style="font-size:11pt">The Operating System then
relentlessly moves forward to the 32bit world by loading the WoW64 Subsystem,
in order to let the 32bit Application execute freely.<span></span></span></p>

<p class="MsoNormal" style="margin:0in 0in 0.0001pt;font-size:12pt;font-family:Calibri,sans-serif"><span style="font-size:11pt">In this talk we will dive
into the WoW64 Subsystem and explain how a 32bit Application performs 64bit
(native) system calls.<span></span></span></p>

<p class="MsoNormal" style="margin:0in 0in 0.0001pt;font-size:12pt;font-family:Calibri,sans-serif"><span style="font-size:11pt">We will also see how it is
possible to exploit this mechanism in order to create smarter malware that
evade Next-Generation and Previous-Generation AV products and integrates with
the Cuckoo Sandbox to “detonate” the user-supplied specimen in a controlled
environment to observe active mutex objects and their effects on the malicious
program.<span></span></span></p>

<p class="MsoNormal" style="margin:0in 0in 0.0001pt;font-size:12pt;font-family:Calibri,sans-serif"><span style="font-size:11pt"><span> </span></span></p>

<p class="MsoNormal" style="margin:0in 0in 0.0001pt;font-size:12pt;font-family:Calibri,sans-serif"><span style="font-size:11pt"> <span></span></span></p>

<p class="MsoNormal" style="margin:0in 0in 0.0001pt;font-size:12pt;font-family:Calibri,sans-serif"><span style="font-size:11pt"><u>Title:</u> </span><span style="font-size:11pt">Breaking obfuscations - Tomer
Zait</span></p>

<p class="MsoNormal" style="margin:0in 0in 0.0001pt;font-size:12pt;font-family:Calibri,sans-serif"><span style="font-size:11pt"><span> </span></span></p>

<p class="MsoNormal" style="margin:0in 0in 0.0001pt;font-size:12pt;font-family:Calibri,sans-serif"><span style="font-size:11pt"><u>Abstract:<span></span></u></span></p>

<p class="MsoNormal" style="margin:0in 0in 0.0001pt;font-size:12pt;font-family:Calibri,sans-serif"><span style="font-size:11pt">During my journey in
de-obfuscating malicious scripts, such as JavaScript and PowerShell, I have
realized that there is a lack of good one-stop-shop solution. Researchers still
perform this tedious task manually while encountering exploit kits, web injects,
PowerShell and python post exploitation agents as well as different legitimate
JavaScript products.<span></span></span></p>

<span style="font-size:11pt;font-family:Calibri,sans-serif">During this Session I will demonstrate working with de-obfuscation tools I
created, of-the-shelf tools and how to create similar tools on your own . In
addition, I will touch Android de-obfuscation in practice and the obfuscation
attack surface each language provides.</span>    <span style="font-size:11pt;font-family:Calibri,sans-serif"><br></span></div>
