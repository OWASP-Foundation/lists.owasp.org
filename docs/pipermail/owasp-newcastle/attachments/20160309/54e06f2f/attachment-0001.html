<a href="https://wiki.jenkins-ci.org/display/JENKINS/OWASP+Dependency-Check+Plugin" rel="nofollow" style="font-size:13.3333px" target="_blank">Jenkins Plugin</a><span style="font-size:13.3333px">).</span><br></div><div><span style="font-size:13.3333px"><br></span></div><div><span style="font-size:13.3333px"><u>Release Notes</u></span></div><div><br></div><div>In addition to the general minor bug fixes and false positive reductions here are the release notes for 1.3.5 (and 1.3.4 - I forgot to post these previously):</div><div><br></div><div>Version 1.3.5</div><div><ul><li>False negative reduction - ore entries are being parsed from the manifest in JAR files. In some cases, this may appear to cause false positives. However, these extra entries generally represent code that was copied from another project (possibly via shading) and as such, the vulnerability may also exist in the combined JAR. <br></li><li>The aggregate goal for the Maven plugin is <a href="http://jeremylong.github.io/DependencyCheck/dependency-check-maven/configuration.html" target="_blank">still problematic</a>; changes have been made to try and resolve some of the issues. However, if the aggregate goal is used in site reporting a blank report will be produced for anything beyond site:site (i.e. site:stage and site:deploy will likely have blank reports for the aggregate goal).<br></li><li>If using a <a href="http://jeremylong.github.io/DependencyCheck/data/database.html" target="_blank">centralized database</a>, forward compatibility has been implemented. Previously, if you have many installations using the same database when you upgraded to a new version that made an update to the database schema all installations would need to be upgraded as well. With version 1.3.5 when a new version of dependency-check comes out that makes a change to the database schema 1.3.5 will still be able to use the same centralized database.<br></li><li>The Maven plugin now supports the <a href="https://maven.apache.org/settings.html#Proxies" target="_blank">non-proxy hosts</a>, as configured in the settings.xml.<br></li><li>Resolved an issue that occurred in some CI environments that forked builds, which caused multiple threads to hit a non-threadsafe method. The method has now been marked synchronized.<br></li><li>Support for additional databases has been added when using a centralized database. Specifically, MySQL, Postgres, and Oracle are now possible.<br></li></ul></div><div><br></div><div>Version 1.3.4</div><div><ul><li>The Maven plugin now supports <a href="http://maven.apache.org/guides/mini/guide-encryption.html" target="_blank">encrypted passwords</a>; useful when using a centralized database.<br></li><li>In the Maven plugin the autoupdate configuration was changed to autoUpdate. If you have set this to false and upgrade the autoupdate will likely default to true unless you fix the casing in the configuration.<br></li></ul></div><div><br></div><div><div>Best Regards,<br></div><div><br></div><div>The OWASP dependency-check team</div></div></div></div></div>
