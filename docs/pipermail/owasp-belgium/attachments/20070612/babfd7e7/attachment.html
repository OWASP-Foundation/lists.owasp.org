    <p><font size=3 color=black face="Times New Roman"><span lang=EN-GB
    style='font-size:12.0pt;color:black'>The threat landscape has seen some
    drastic changes over the last few years.<br>
    The fading perimeter, format rendering vulnerabilities and targeted attacks<br>
    have all changed the way we deal with technical controls and how we choose <br>
    to mitigate risk. Organizations are having a hard time dealing with these<br>
    new threats, and are increasingly looking to formal security intelligence<br>
    processes for assistance. Maarten will introduce the intelligence
    principles <br>
    that underlie the way we deal with security information, apply them to a<br>
    number of recent security incidents, and show how they can be and are being<br>
    used by organizations and security vendors to more effectively deal with <br>
    threats to information security.</span></font><o:p></o:p></p>
    </div>
    <p><b><u><font size=4 color=black face=Arial><span lang=EN-GB
    style='font-size:13.5pt;font-family:Arial;color:black;font-weight:bold'>Mr
    Didier Stevens, Security Consultant at Contraste Europe</span></font></u></b><font
    size=2 color=black face=Arial><span lang=EN-GB style='font-size:10.0pt;
    font-family:Arial;color:black'> </span></font><o:p></o:p></p>
    <p><font size=3 color=black face="Times New Roman"><span lang=EN-GB
    style='font-size:12.0pt;color:black'>Didier Stevens (CISSP, MCSD .NET,
    MCSE/Security) is an IT Security Consultant currently working at a large
    Belgian financial corporation. He is employed by Contraste Europe NV, an IT
    Consulting Services company. (note from Tomas : &quot;His blog and his IT
    Security Research and Open Source tools are renown around the world, but he
    is Mr Modesty in person!&quot;) </span></font><o:p></o:p></p>
    <p><b><u><font size=3 color=black face="Times New Roman"><span lang=EN-GB
    style='font-size:12.0pt;color:black;font-weight:bold'>abstract : </span></font></u></b><o:p></o:p></p>
    <p><font size=2 color=black face=Arial><span lang=EN-GB style='font-size:
    10.0pt;font-family:Arial;color:black'>Windows XP (and later Windows
    versions) keeps a list of all programs executed by a user. This list is
    kept in the registry (UserAssist keys). For each executed program, a
    running counter and a timestamp is maintained. <br>
    The data in the UserAssist registry keys is used by Windows to display the
    Most Frequently Used Programs list on the start menu, but it can also be
    used to analyse which programs a user has executed.<br>
    Exploiting these keys for forensic analysis is a little known technique,
    because there is no publicly available official documentation from
    Microsoft about these registry keys. This presentation will detail Didier's
    findings and the use of the Open Source tool he wrote specifically&nbsp;to
    reveal&nbsp;this info in a userfriendly way. </span></font><o:p></o:p></p>
