<div><br></div><div><br></div><div>   Jim Manico</div><div><br></div><div>   Bio:</div><div><br></div><div>   Jim Manico is the VP of Security Architecture for WhiteHat Security. Jim is</div><div>   part of the WhiteHat Static Analysis Software Testing (SAST) team, leading</div>

<div>   Security and others.</div><div><br></div><div><br></div><div>   Abstract:</div><div><br></div><div>   Access Control is a necessary security control at almost every layer within</div><div>   a web application. This talk will discuss several of the key access control</div>
<div>   anti-patterns commonly found during website security audits. These access</div><div>   control anti-patterns include hard-coded security policies, lack of</div><div>   horizontal access control, and "fail open" access control mechanisms. In</div>
<div>   reviewing these and other access control problems, we will discuss and</div><div>   design a positive access control mechanism that is data contextual, activity</div><div>   based, configurable, flexible, and deny-by-default - among other positive</div>
<div>   design attributes that make up a robust web-based access-control mechanism.</div><div><br></div><div><br></div><div>   Programme:</div><div><br></div><div>   18:00 - 18:45  Registration & Snack</div><div><br></div>
