<h1 align="center">Preliminary Call For Papers&nbsp;&nbsp;</h1>
<h1 align="center">Refereed Papers Track at OWASP AppSec Europe 2007 Conference&nbsp;</h1>
<p align="center"><b>Date</b>: April or May 2007<br>
<b>Location</b>: Italy</p>
<p align="center">(Precise date and location to be confirmed, please consult <a href="http://www.owasp.org/index.php/6th_OWASP_AppSec_Conference_-_Italy_2007">http://www.owasp.org/index.php/6th_OWASP_AppSec_Conference_-_Italy_2007</a>
for details as they become available)</p>
<p align="left"><br>
<br>
The Open Web Application Security Project (OWASP, <a href="http://www.owasp.org">http://www.owasp.org</a>) is dedicated to finding and
fighting the causes of insecure software. OWASP has dozens of projects and over 50 chapters worldwide
focused on application security. Our high quality tools and documentation are used everywhere,
including the freely available book-length "Guide to Secure Web Applications and Services", the leading
web application penetration testing tool called &quot;WebScarab&quot;, and an advanced web application security
training application called &quot;WebGoat&quot;.&nbsp;</p>
<p align="left">The OWASP Foundation, a not-for-profit charitable organization, ensures the ongoing availability and
support for this work.&nbsp;<br>
<br>
The OWASP AppSec conferences (<a href="http://www.owasp.org/index.php/Category:OWASP_AppSec_Conference">http://www.owasp.org/index.php/Category:OWASP_AppSec_Conference</a>)
bring together application security experts, researchers and practitioners from all over the world.
Industry and academia can meet to discuss open problems and new solutions in application security.
The conferences offer tutorials, keynotes, and invited presentations.&nbsp;<br>
As in 2006, the OWASP AppSec Europe 2007 conference will feature a refereed papers track. The goal of
the refereed papers track is twofold:&nbsp;</p>
<ol>
  <li>
    <p align="left">to give academic researchers in web application security the opportunity
    to share their research results with practitioners, and&nbsp;</li>
  <li>
    <p align="left">to give industry people the possibility to share experiences with the OWASP community.</li>
</ol>
<p align="left">Hence both research papers as well as experience papers pertaining to all aspects of web application
security are solicited. Papers should describe new ideas, new implementations, or experiences related to web
application security.&nbsp;<br>
<br>
Topics of interest include, but are not limited to:&nbsp;</p>
<ul>
  <li>
    <p align="left">Web application security&nbsp;</li>
  <li>
    <p align="left">Threat modeling of web applications&nbsp;</li>
  <li>
    <p align="left">Vulnerability analysis of web applications (code review,
    pentest, static analysis, scanning)&nbsp;</li>
  <li>
    <p align="left">Countermeasures for web application vulnerabilities&nbsp;</li>
  <li>
    <p align="left">Secure coding techniques&nbsp;</li>
  <li>
    <p align="left">Static and dynamic analysis of web application technologies&nbsp;</li>
  <li>
    <p align="left">Platform or language (e.g. Java, .NET) security features   that help secure web applications&nbsp;</li>
  <li>
    <p align="left">Open source framework features that help secure web applications</li>
  <li>
    <p align="left">How to use databases securely in web applications&nbsp;</li>
  <li>
    <p align="left">Experiences or new ideas on Secure Development Lifecycles (SDLC)&nbsp;</li>
  <li>
    <p align="left">Experiences using web application security scanning or code   analysis tools&nbsp;</li>
  <li>
    <p align="left">Access control in web applications&nbsp;</li>
  <li>
    <p align="left">Trusted computing solutions for web applications</li>
  <li>
    <p align="left">Non-repudiation in web applications</li>
  <li>
    <p align="left">Web services security&nbsp;</li>
  <li>
    <p align="left">AJAX security</li>
  <li>
    <p align="left">Security of Service Oriented Architectures</li>
</ul>
<p align="left">It is explicitly allowed to submit papers that have already been published, but in a publication channel with a different
audience. In particular, papers that have already been presented at academic conferences are welcomed, and will be refereed on
how interesting and valuable they are to an OWASP audience. Authors are
encouraged to motivate in the paper why they consider the paper relevant for the
OWASP audience.<br>
<br>
For accepted papers, and where allowed by possibly existing copyrights on the paper, the papers will be published in a
proceedings distributed as a technical report from the Katholieke Universiteit Leuven, Belgium.<br>
<br>

is decided):&nbsp;<br>
