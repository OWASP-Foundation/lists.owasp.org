<h1 style="color:rgb(37,37,37);margin:0px 0px 10px;padding:0px;border:0px;font:inherit;vertical-align:baseline;line-height:1.2;font-family:Georgia,serif"><b>Java zero-day leads to Internet Explorer zero-day</b></h1><p class="summary" style="color:rgb(37,37,37);margin:0px 0px 10px;padding:0px;border:0px;font:inherit;vertical-align:baseline;font-style:italic;line-height:21px;font-family:Georgia,serif">
<span style="margin:0px 3px 0px 0px;padding:0px;border:0px;font-size:14px;font:inherit;vertical-align:baseline;color:rgb(150,21,21);font-weight:bold">Summary:</span> Following the trail of attackers exploiting vulnerabilities in Java led one security researcher to discover a new zero-day vulnerability in Internet Explorer.</p>
<div class="byline" style="color:rgb(37,37,37);margin:0px;padding:10px 0px;border-width:1px 0px;border-top-style:dotted;border-bottom-style:dotted;border-top-color:rgb(198,198,198);border-bottom-color:rgb(198,198,198);font:inherit;vertical-align:baseline;zoom:1">
<div style="margin:0px;padding:0px;border:0px;font-size:12px;font:inherit;vertical-align:baseline"><span style="color:rgb(108,115,118)">By</span><span style="color:rgb(108,115,118)"> </span><a href="http://www.zdnet.com/meet-the-team/au/michael.lee/" rel="author" style="margin:0px;padding:0px;border:0px;font:inherit;vertical-align:baseline;color:rgb(0,83,153);text-decoration:none">Michael Lee</a><span style="color:rgb(108,115,118)"> </span><span style="color:rgb(108,115,118)">|  </span>September 17, 2012 -- 01:29 GMT (18:29 PDT)</div>
</div><div class="storyBody" style="margin:0px 0px 10px;padding:0px;border:0px;font:inherit;vertical-align:baseline;zoom:1;color:rgb(37,37,37);font-family:Verdana,Arial,sans-serif;line-height:18px;background-color:rgb(255,255,255)">
<p style="margin:0px 0px 15px;padding:0px;border:0px;font-size:12px;font:inherit;vertical-align:baseline">While looking around a compromised server that was being used to <a href="http://www.zdnet.com/java-zero-day-skyrockets-blackhole-exploit-success-rates-7000003467/" style="margin:0px;padding:0px;border:0px;font-size:12px;font:inherit;vertical-align:baseline;color:rgb(0,83,153);text-decoration:none">exploit Java vulnerabilities</a>, a security researcher stumbled upon another exploit that he claims affects fully patched versions of Microsoft Internet Explorer 7, 8 and 9.</p>
<p style="margin:0px 0px 15px;padding:0px;border:0px;font-size:12px;font:inherit;vertical-align:baseline">Eric Romang <a href="http://eromang.zataz.com/2012/09/16/zero-day-season-is-really-not-over-yet/" style="margin:0px;padding:0px;border:0px;font-size:12px;font:inherit;vertical-align:baseline;color:rgb(0,83,153);text-decoration:none">found four files on the server</a>: an executable, a Flash Player movie and two HTML files called exploit.html and protect.html</p>
<p style="margin:0px 0px 15px;padding:0px;border:0px;font-size:12px;font:inherit;vertical-align:baseline">When users visit the exploit.html page, it loads the Flash movie, which in turn loads the other HTML page, protect.html. Together, they help drop the executable on to the victim's computer. At this point, attackers have everything they need to drop whatever applications they like on the victim's machine, whether it is to join a botnet or conduct attacks. In this case, the dropper executable installs another program when the victim next logs in.</p>
<p style="margin:0px 0px 15px;padding:0px;border:0px;font-size:12px;font:inherit;vertical-align:baseline">Romang discussed the zero-day with other security researchers, who also came to the same conclusion that this was a vulnerability in Internet Explorer.</p>
<p style="margin:0px 0px 15px;padding:0px;border:0px;font-size:12px;font:inherit;vertical-align:baseline">However, Romang's presence has not gone unnoticed by those behind the exploit. Shortly after Romang discovered the zero-day, the exploit authors removed the files from the server, replacing them with a text file containing Romang's Twitter handle, "eromang". They also removed the previous Java exploit that was on the server.</p>
<p style="margin:0px 0px 15px;padding:0px;border:0px;font-size:12px;font:inherit;vertical-align:baseline">The vulnerability has also been picked up by developers working on the Metasploit exploit framework, and an early version of a module exploiting the zero-day has already been created.</p>
