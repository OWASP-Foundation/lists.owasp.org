<div dir="ltr"><div class="gmail_quote"><div dir="ltr"><div>Hello.</div><div><br></div><div>Many websites have revoked or will revoke their SSL certificates due to the Heartbeat bug. Browsers will protect you using OCSP, Online Certificate Status Protocol, but only if you tell them to check. Below is how to set up your browsers to check if a certificate has been revoked. </div>

<div><br></div><div>Chrome: </div><div>Click on menu <i>Settings</i>. </div><div>On the bottom of the page click on <i>Show advanced settings</i>. </div><div>Make sure this is checked: <i>Check for server certificate revocation</i>.</div>

<div><br></div><div>Firefox:</div><div>Click on menu <i>Tools</i> then <i>Options</i>. </div><div>On the pop-up, click on <i>Certificate</i> tab, then click on <i>Validate</i> button. </div><div>On the pop-up, make sure this is checked: <i>When an OCSP server connection fails, treat the certificate as invalid</i>.</div>

<div><br></div><div>Internet Explorer: </div><div>Click on menu <i>Internet Options</i>. </div><div>On the pop-up, click on <i>Advanced</i> tab. </div><div>Scroll down to <i>Security</i> and make sure this are checked: <i>Check for publisher's certificate revocation</i> and <i>Check for server certificate revocation</i>. </div>

<div><br></div><div>Safari: </div><div>Open <i>Applications -> Utilities -> Keychain Access. </i>Under Keychain Access menu, </div><div>Select Preferences...</div><div>Select the Certificates tab</div><div>Set "Online Certificate Status Protocol (OCSP)" to "Best Attempt"</div>

<div>Set "Certificate Revocation List (CRL)" to "Best Attempt"</div><div>Set "Priority" to "OCSP"</div><div><br></div><div>How do you know if you have done it right? Just click on this link, <a href="https://www.cloudflarechallenge.com/" target="_blank">https://www.cloudflarechallenge.com/</a>. Your browser is protected if you see "Cannot connect to the real <a href="http://www.cloudflarechallenge.com" target="_blank">www.cloudflarechallenge.com</a>".</div>
</div></div><div><br></div><div>Happy Easter!</div><div><br></div><div><br></div>-- <br><div dir="ltr">Helen Gao, CISSP</div>
