Hello.

Many websites have revoked or will revoke their SSL certificates due to the
Heartbeat bug. Browsers will protect you using OCSP, Online Certificate
Status Protocol, but only if you tell them to check. Below is how to set up
your browsers to check if a certificate has been revoked.

Chrome:
Click on menu *Settings*.
On the bottom of the page click on *Show advanced settings*.
Make sure this is checked: *Check for server certificate revocation*.

Firefox:
Click on menu *Tools* then *Options*.
On the pop-up, click on *Certificate* tab, then click on *Validate* button.
On the pop-up, make sure this is checked: *When an OCSP server connection
fails, treat the certificate as invalid*.

Internet Explorer:
Click on menu *Internet Options*.
On the pop-up, click on *Advanced* tab.
Scroll down to *Security* and make sure this are checked: *Check for
publisher's certificate revocation* and *Check for
server certificate revocation*.

Safari:
Open *Applications -> Utilities -> Keychain Access. *Under Keychain Access
menu,
Select Preferences...
Select the Certificates tab
Set "Online Certificate Status Protocol (OCSP)" to "Best Attempt"
Set "Certificate Revocation List (CRL)" to "Best Attempt"
Set "Priority" to "OCSP"

How do you know if you have done it right? Just click on this link,
https://www.cloudflarechallenge.com/. Your browser is protected if you see
"Cannot connect to the real www.cloudflarechallenge.com".
