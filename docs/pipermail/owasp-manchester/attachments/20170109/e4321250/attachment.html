As a guideline we invite talks on the following topics (not exhaustive list):<br>
<br>
  * Novel web/mobile vulnerabilities and countermeasures (note: no 
zero-days, any vulnerabilities must be responsibly disclosed prior to 
presenting at OWASP events)<br>
  * New technologies, paradigms, frameworks, tools<br>
  * OWASP tools or projects - new and existing, best practices and experience of implementing them in your company<br>
  * Agile Security and Continuous Integration<br>
  * Secure development: frameworks, best practices, secure coding, methods, processes, standards, SDLC, code review<br>
  * AppSec Pipeline - Automation and Orchestration<br>
  * Threat modelling<br>
  * AI and Machine Learning in Application Security<br>
  * Browser security<br>
  * IoT security and AppSec<br>
  * JavaScript/Node.JS security<br>
  * Mobile security and security for the mobile web<br>
  * Threat Intelligence & AppSec<br>
  * Cloud security, Container security<br>
  * REST/SOAP and API security<br>
  * Security of frameworks<br>
  * Payment Security, PCI impact on AppSec, tools, frameworks, best practices<br>
  * Blockchain & AppSecurity<br>
  * Cryptography and its use (and abuse) in Mobile and Web Apps<br>
  * Data Privacy, Governance<br>
  * Management topics: Managing Secure Software Development 
LifeCycle (SDLC), Managing Application Security initiatives, Developer 
Training<br>
<br>
Recommended talk duration is 30 to 45 minutes with additional 15 minutes for any live demos and Q&A.<br>
