> 1)      How to protect against SQL injection attacks.
"SQL Injection Attacks" book by Justin Clarke.

"Seven Deadliest Web Attacks" book by Mike Shema (First chapter on XSS)

We call this "HTTP Header Injection" now. The tool, Burp Suite
Professional is very good at finding these types of vulnerabilities
and only costs 165 British Pounds (GBP)

> 4)      Best web security practices using Javascript.
"Web 2.0 Security: Defending Ajax, RIA, and SOA" book by Shreeraj Shah

> 5)      How to secure Flash objects and Java applets.
"Web Application Hacker's Handbook" by Dafydd Stuttard; Marcus Pinto
-- this book also covers the HTTP Header Injection issues above

> 8)      The inside scoop on what an auditor or hacker does and how they attempt to break the applications.

You'll have to ask me. "Auditor" is an incorrect word (and so is
"hacker"). Most people refer to these best as "application security
consultants" or "appsec consultants".

I prefer to use the terminology "application assessment" or "app
assessment" instead of "source code audit" or "penetration-test" or
"ethical hacking".

Good appsec consultants will perform threat-modeling (see the
Microsoft book with the same name) in order to provide adversarial
security testing. This involves using a tool such as Burp Suite
Professional (mentioned above) to walk the web app. After a walk,
"insertion points" are identified, which usually consist of URLs,
parameters, forms, queries, cookies, and HTTP headers. Some "insertion
points" need to be identified separately by using a web browser (note:
Burp Pro requires a web browser, so you're already using one) and
downloading Javascript or SWF files.

After using the web browser and Burp Suite Professional to identify
insertion points, a good appsec consultant will map the URLs to source
code through a process known as "code traversal". This requires deep
knowledge of the target application, its source code architecture,
internal code flow, and the ability to perform both "data flow
analysis" and "code flow analysis". I use two tools to do this: 1)
Source Insight, which is a code editor/viewer, and 2) OWASP
CodeCrawler, another code viewer that requires Visual Studio
Professional 2008. Source Insight is just over 200 US dollars
(although a 30-day trial is downloadable for free) and Visual Studio
Professional 2008 can be had for as little as 70 US dollars
(especially now that Visual Studio Professional 2010 is available).

It is required to have lots of books available in order to understand
all of the code intricacies -- I suggest subscriptions to
SafariBooksOnline, Books24x7, Kindle for PC, and Google Books (Google
Editions Book Store will be out soon, as well).
