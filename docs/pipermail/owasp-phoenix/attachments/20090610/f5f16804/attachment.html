<br>
We are formally introducing the OWASP PCI Project to the Web<br>
Application Security community! The industry needs a workspace for PCI<br>
QSAs* and Application Security experts to work constructively together<br>
- the OWASP PCI Project will serve as the platform in building<br>
community consensus.<br>
<br>
The PCI Project drives focused discussion and awareness, promoting a<br>
thorough understanding of how to ensure safety in online payments.<br>
Our mission is to:<br>
+Make payment application security requirements achievable,<br>
+QSA perspective and audit points accessible,<br>
+A unified and mutually agreed upon approach to secure payment<br>
applications, and sustainable compliance<br>
<br>
The scope of this group will ultimately extend beyond PCI, becoming a<br>
scalable software risk management framework for other regulations.  By<br>
focusing on managing risk – we are ensuring web sites, applications,<br>
and web enabled software of any type are secured the right way (and<br>
making compliance a natural and sustainable byproduct).<br>
<br>
Now is the time to get involved!  Visit the project site and sign up!<br>
 We are starting to build the project roadmap, we need YOUR help in<br>
framing this project!<br>
&lt;<a href="https://lists.owasp.org/mailman/listinfo/owasp-pci-project" target="_blank">https://lists.owasp.org/mailman/listinfo/owasp-pci-project</a>&gt;<br>
<br>
Proposed projects include:<br>
+PCI Application Security Scoping Guidance,<br>
+Application Security Development Guidance,<br>
+PCI Application Security Auditor’s Playbooks,<br>
+More to come!<br>
