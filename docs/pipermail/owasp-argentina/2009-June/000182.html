Para no liarme con versiones de archivos puse todo en un svn:
http://svn.juanjoconti.com.ar/dyntaint/

La revisión 0 coincide con los archivos que envié a la lista.
La revisión 1 cambia para soportar multi-vulnerabilidad.

Ahora el ej2.py falla por que se marca al sumidero como sensible a
todas las vulnerabilidades y la variable que llega solo fue limpiada
contra SQLI. El ej3.py ejecuta OK por que se especifica que el
sumidero es solo sensible a SQLI:

@ssinc(SQLI)
def guardarDB(valor):
    print "Guardando en la BD:", valor

Problemas a intentar resolver:

- operaciones sobre strings, mencionado por Ezequiel. Creo que puedo
resolverlo, mantenganse sintonizados a la revisión 2 :)
- usabilidad, mencionado por Andrés. Para esto la solución parece ser
tener fuentes y sumideros identificados y decorarlos al cargar el
módulo dyntaint.

PS: para los que les gustó el tema de decoradores en Python, no dejen
de mirar las modificaciones a dyntaint.py. Funciones decoradores con
parámetros no es fácil de agarrar a la primera.
