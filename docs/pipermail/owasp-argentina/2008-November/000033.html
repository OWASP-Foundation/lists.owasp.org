El trabajo que hicimos usaba un approach de taint-mode analisis pero
haciendo analisis de flujo.Tambien tuvimos un prototipo super inicial de una
idea que un investigador del LIFIA estaba desarrollando de analisis basado
en gramaticas.

Nos encontramos con varios obstaculos en el camino:

 - Java es enorme como para hacer un analisis exhaustivo del flujo de un
programa, asi que tuvimos que "recortar" el lenguaje en lo que al analisis
respecta.
- El taint analisis combinado con el analisis del flujo (usando por ejemplo
el AST de un programa) te hace tener que evaluar entre profundizar algun
path de ejecucion o podar con alguna heurisitica, si no el problema puede
volverse intratable.
