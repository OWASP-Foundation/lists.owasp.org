Te tiro una idea bien loca: Que te parece un plugin para un IDE
popular (sea el que sea) que te detecte vulnerabilidades en tiempo
real? En tiempo real, me refiero a mientras el desarrollador está
escribiendo el código. Primero lo explico mejor con un ejemplo, y
despues te digo como se implementaría en la realidad.

Ejemplo:
Paso 1- El desarrollador esta escribiendo un programa en PHP, y viene
tipeando lo siguiente:

Line 1: <?
Line 2: $user_input = $_GET['nombre'];

Paso 2- El desarrollador sigue tipeando, y escribe lo siguiente:

Line 3: echo $user_input;

Paso 3- Cuando el desarrollador tipea el ";" de fín de línea del paso
2, se detecta que el código es vulnerable a Cross Site Scripting; y de
alguna manera (ventana pop-up, ventana de log en la parte de abajo del
texto, etc.) se le dice: "Hey pascual! Fijate que estas escribiendo
código vulnerable en la línea número 3!"

Espero que se haya entendido la idea! Bueno... ahora con la
implementación. Básicamente esto es un analizador de código del tipo
"tainted mode". Lo que haría el programa es analizar el código que se
va escribiendo, y por ejemplo al terminar de escribirse la línea 2, el
estado del analizador debería ser:
    - Variables manchadas: ['user_input']
    - Variables no manchadas: []

Luego, cuando llegue a la línea 3, y la misma se finalice (con el
enter despues del ";") el programa va a aplicar sus reglas de lógica y
va a matchear la que dice: "si uso una variable manchada, como
parametro de un echo entonces hay un XSS".

Por las dudas, te hago el ejemplo en el cual el programa no detectaría
una vulnerabilidad:

Paso 1- El desarrollador esta escribiendo un programa en PHP, y viene
tipeando lo siguiente:

Line 1: <?
Line 2: $user_input = $_GET['nombre'];

    - Variables manchadas: ['user_input']
    - Variables no manchadas: []

Paso 2- El desarrollador sigue tipeando, y escribe lo siguiente:

Line 3: $user_input = htmlentities($user_input);

    - Variables manchadas: []
    - Variables no manchadas: ['user_input']

Paso 3- El desarrollador tipea:

Line 4: echo $user_input;

Nada sucede, no hay vulnerabilidad.

La verdad que no se si esto es investigación, o desarrollo de una
herramienta, pero te puedo decir que no es nada facil hacer que esto
funcione, algunas de las razones:
    - Idealmente, deberían hacerlo para un par de lenguajes: PHP,
Python, Ruby, ASP.NET
    - La lógica es muy compleja, y hay unas cuantas vulnerabilidades a
tener en cuenta
    - Hay que tener en cuenta cuando se pasa una variable de manchada
a no manchada, y viceversa
    - Hay que tener en cuenta las funciones de validación definidas
por el usuario
    - Que pasa si desarrollo en un framework?

Ahora... los beneficios de una herramienta que haga esto y funcione
bien, son más que impresionantes: El código escrito con esta
combinación de IDE+Plugin, no tendrá vulnerabilidades.

Bah, al menos no va a tener vulnerabilidades clásicas como XSS, SQL
injections, File Inclusions, etc. Eso sí, vulnerabilidades en la
lógica de la aplicación, como por ejemplo que sea posible realizar
transferencias negativas de dinero en un home banking... esas si que
no te las va a salvar!!
