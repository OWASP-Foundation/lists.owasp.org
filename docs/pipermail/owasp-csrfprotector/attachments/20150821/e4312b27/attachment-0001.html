<div dir="ltr">Hi Mithilesh,<br class="Apple-interchange-newline"><div>Thanks for posting this bug! <br><br>I have implemented a model that keeps a queue of tokens created for a user, and keep them until they are consumed. How it works is, whenever a token is created and sent to client, its stored in a list in server side cookies (session variables) and when next request comes, the token is searched for in the list. When a match is found, all tokens issued before this one are removed. This way it solves the case of request sent with a very small time gap. In general case it will work like earlier and the list will always contain two tokens, one issued on last request and one prev to that.</div><div><br></div><div>The new one is implemented in other branch: <a href="https://github.com/mebjas/CSRF-Protector-PHP/tree/simultaneous-tokens">https://github.com/mebjas/CSRF-Protector-PHP/tree/simultaneous-tokens </a></div></div><div class="gmail_extra"><br clear="all"><div><div class="gmail_signature"><div dir="ltr"><div><div dir="ltr"><div><div dir="ltr"><div><div dir="ltr"><div style="font-family:arial"><font size="2" color="#666666">----------------------------------------------------------------------------</font></div><div style="font-family:arial"><font size="2" color="#666666">Kind Regards,</font></div><div><font size="2" color="#666666" style="font-family:arial">Minhaz</font><font size="2" style="font-family:arial" color="#0000ee"> | <a href="http://cistoner.org/projects" target="_blank">My Projects</a></font><font color="#666666" style="font-family:arial"> | <a href="http://cistoner.org/minhaz/" target="_blank">M</a></font><font color="#0000ee" style="font-family:arial"><a href="http://blog.minhazav.me/" target="_blank">y blog</a></font></div></div></div></div></div></div></div></div></div></div>
