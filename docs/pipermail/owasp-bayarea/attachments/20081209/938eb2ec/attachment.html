<p class=MsoNormal><b><i><font size=2 face=Arial><span style='font-size:10.0pt;

<p class=MsoNormal><b><i><font size=2 face=Arial><span style='font-size:10.0pt;
font-family:Arial;font-weight:bold;font-style:italic'>Abstract:&nbsp; <o:p></o:p></span></font></i></b></p>

<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;
font-family:Arial'>Most often the login page or &quot;front door&quot; of a
website is built securely and is difficult for an attacker to break through.&nbsp;
However, web developers often leave serious holes in web applications after the
login, which can allow one malicious user to access another user's personal
information.&nbsp; This presentation will cover several vulnerabilities
frequently found in the &quot;logged-in&quot; portion of web applications, how
to detect these issues, and how to ensure your applications don't have these
security problems.<o:p></o:p></span></font></p>

<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;

<p class=MsoNormal><b><i><font size=2 face=Arial><span style='font-size:10.0pt;
font-family:Arial;font-weight:bold;font-style:italic'>Abstract:&nbsp; <o:p></o:p></span></font></i></b></p>

<p class=MsoNormal style='text-autospace:none'><font size=2 face=Arial><span
style='font-size:10.0pt;font-family:Arial'>Forget Cross-Site Scripting. Forget
SQL Injection. If you want to make some serious cash on the Web, these
aren&#8217;t the tools you need.&nbsp; No expensive toys to license, ninja coding
skillz or all-black wardrobe required.&nbsp; All you need is a Web browser, a sharp
eye, and a couple black hat tricks.&nbsp; In this talk we will be discussing attacks
that are legal by the letter, violating only terms of service (yes that means a
locked account, not a lawsuit)<o:p></o:p></span></font></p>

<p class=MsoNormal style='text-autospace:none'><font size=2 face=Arial><span
style='font-size:10.0pt;font-family:Arial'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; <o:p></o:p></span></font></p>

<p class=MsoNormal style='text-autospace:none'><font size=2 face=Arial><span
style='font-size:10.0pt;font-family:Arial'>Also known as business logic flaws,
often these attacks aren&#8217;t found in books, rarely covered in the news,
and statistics are limits. However, we do know they are out there and the bad
guys have been lining the pockets with them for years. Security vendors
don&#8217;t like talking about business logic flaws because their products
don&#8217;t know how to detect them. Most of the examples we&#8217;ll cover are
only found in online hacker forums and/or stumbled upon by complete accident.&nbsp;
In this session we will open the hood and discuss how the professional hackers make
real cash- the black hat way<o:p></o:p></span></font></p>

<p class=MsoNormal style='text-autospace:none'><font size=2 face=Arial><span
