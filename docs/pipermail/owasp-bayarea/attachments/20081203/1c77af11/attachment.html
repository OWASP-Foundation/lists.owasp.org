<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;
font-family:Arial'>Most often the login page or &quot;front door&quot; of a
website is built securely and is difficult for an attacker to break
through.&nbsp; However, web developers often leave serious holes in web
applications after the login, which can allow one malicious user to access
another user's personal information.&nbsp; This presentation will cover several
vulnerabilities frequently found in the &quot;logged-in&quot; portion of web applications,
how to detect these issues, and how to ensure your applications don't have
these security problems.<o:p></o:p></span></font></p>
