<h3><span class=mw-headline><b><font size=3 face=Arial><span style='font-size:
12.0pt;font-family:Arial'>Development Issues Within <st1:City w:st="on">AJAX</st1:City>
Applications: How to Divert Threats <br>
</span></font></b></span><st1:City w:st="on"><st1:place w:st="on"><span
  class=mw-headline><font size=3 face=Arial><span style='font-size:12.0pt;
  font-family:Arial;font-weight:normal'>AJAX</span></font></span></st1:place></st1:City><span
class=mw-headline><font size=3 face=Arial><span style='font-size:12.0pt;
font-family:Arial;font-weight:normal'> has rapidly emerged as a prominent
enabling technology in the movement to improve the Web as a software platform
fo</span></font></span><font size=3 face=Arial><span style='font-size:12.0pt;
font-family:Arial;font-weight:normal'>r business and consumer applications.
Using <st1:City w:st="on"><st1:place w:st="on">AJAX</st1:place></st1:City>
development techniques provides software developers with a wide-open platform
for creating innovative new Web (2.0) applications. The result is a more
readily responsive Web environment which minimizes the
&#8220;start-stop-start-stop&#8221; nature of Web pages, thus increasing the
speed and user-interactivity of Web-enabled services. <o:p></o:p></span></font></h3>

<p><font size=3 face=Arial><span style='font-size:12.0pt;font-family:Arial'>However,
the open, malleable nature of Web 2.0 also has an often overlooked impact on
application security that is not necessarily initially visible to application
developers, establishing a relatively easy target for malicious behavior to
compromise applications and overall network security. Various security issues
arise from a number of sources, thus increasing the attack surface of AJAX
applications: client side security controls often replace server side data
validation, thus creating a false sense of security; so do calls to
&#8220;hidden&#8221; application functionality and URLs; new XML and JavaScript
data models, such as JSON, also enable new attack vectors, like JavaScript
Hijacking; and the open, easy to use nature of so called Mashups often comes at
the price of various security compromises. <o:p></o:p></span></font></p>

<p><font size=3 face=Arial><span style='font-size:12.0pt;font-family:Arial'>Such
threats, however, can be thwarted with the proper implementation of security
testing. This session will address the development issues of AJAX applications
from a security perspective, looking at how today&#8217;s common web threats
such as SQL injections, Cross Site Scripting, and others are often magnified in
an AJAX environment, and it will also explore new threads, such as JavaScript
Hijacking. Last but not least it also provides Best Practices for <st1:City
w:st="on">AJAX</st1:City> application developers that are designed to help
manage the security complexities inherent to <st1:place w:st="on"><st1:City
 w:st="on">AJAX</st1:City></st1:place> development. <o:p></o:p></span></font></p>

<h3><span class=mw-headline><b><font size=3 face=Arial><span style='font-size:
12.0pt;font-family:Arial'>Mastering Session Management<br>
</span></font></b></span><span class=mw-headline><font size=3 face=Arial><span
style='font-size:12.0pt;font-family:Arial;font-weight:normal'>Almost everyone
is aware of Cross site Scripting and SQL Injection vulnerabilities and their
impact. Every web application implements session management techniques to
maintain context, but application developers do not pay a lot of attention to
session management because they are usually managed by the application server.
Attacks against sessions can result in serious compromises and this
presentation will cover some of the most common session management techniques
and the attacks that can be launched against sessions. It will also discuss some
of the techniques developers can use to protect against session attacks.<o:p></o:p></span></font></span></h3>

<h3><a name="Building_a_Corporate_Application_Securit"></a><span
class=mw-headline><b><font size=3 face=Arial><span style='font-size:12.0pt;
font-family:Arial'>Building a Corporate Application Security Assessment Program
<br>
</span></font></b></span><span class=mw-headline><font size=3 face=Arial><span
style='font-size:12.0pt;font-family:Arial;font-weight:normal'>The</span></font></span><font
size=3 face=Arial><span style='font-size:12.0pt;font-family:Arial;font-weight:
normal'> talk will discuss Intuit's experiences in building a corporate
application security assessment program. Areas of discussion will include
tools, processes, and methodologies utilized to conduct effective security
assessments of applications in a large global software development corporation.
<o:p></o:p></span></font></h3>

<h3><a name="Development_Issues_Within_AJAX_Applicati"></a><a
name="Best_practices_of_combining_scanners_and"></a><a
name="Web_Hacking.2C_Tricks_of_the_Trade"></a><span class=mw-headline><b><font
size=3 face=Arial><span style='font-size:12.0pt;font-family:Arial'>From Rivals
to BFF: WAF &amp; VA Unite<br>
</span></font></b></span><span class=mw-headline><font size=3 face=Arial><span
style='font-size:12.0pt;font-family:Arial;font-weight:normal'>For years there
was a debate in the Web application and data security world about which
approaches are best - black box, white box, SDLC, VA services/software, Web
Application Firewalls (WAF), etc. While it is true that with a limited budget
anything can become competitive &#8211; a new copy machine versus a new coffee
machine, the core value propositions of WAF and VA are distinct and
complementary. This presentation will illustrate how integrating these
solutions can enable more secure Web application development and operations.</span></font><o:p></o:p></span></h3>
