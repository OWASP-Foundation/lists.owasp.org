<p>As some colleagues have pointed out-<br>
</p>
<p>    Trying https:// then going to http:// on failure would not be great either, unless HSTS is enabled. </p>
<p>      With this behavior, anyone on "local network" could easily cause a browser to "downgrade" to port 80 by blocking 443. Once HSTS is prevalent, that might be acceptable... but not today.<br>
</p>
<p><br>
</p>
<p>    (Similar to revocation server attacks, simple ability to block the connection breaks the security of the feature)<br>
</p>
