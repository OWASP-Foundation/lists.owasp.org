I have another question related enforcing browser clients to use HTTPS:
What if you enforced the use of TLS v1.2 on all web servers to ensure that heartbleed, etc. would not be an issue.
Most browsers have updates to support TLS v1.2, but what about those still on TLS v1.1,1.0?
There is no clean error message from the web server for a failed TLS/SSL negotiation.   "Secure Connection Failed" Firefox, blank in some others.
There is no HTTP response either.     How to notify users that they should upgrade their browers?

On Oct 15, 2015, at 9:21 AM, Greg Christopher <gchristopher at vmware.com> wrote:



As some colleagues have pointed out-
    Trying https:// then going to http:// on failure would not be great either, unless HSTS is enabled.       With this behavior, anyone on "local network" could easily cause a browser to "downgrade" to port 80 by blocking 443. Once HSTS is prevalent, that might be acceptable... but not today.

    (Similar to revocation server attacks, simple ability to block the connection breaks the security of the feature)
